## 一、场景大纲定位

场景大纲是小说工厂内容仓库的第四层结构，位于阶段大纲（03_阶段大纲）之下、正文（04_正文）之上，起承上启下的中间层作用。

| 层级 | 文件 | 粒度 | 主要负责 |
|------|------|------|----------|
| 第一层 | 01_全文总体大纲 | 全书 | 顶层设计、核心驱动链 |
| 第二层 | 02_卷大纲 | 每卷 | 卷结构、卷核心事件 |
| 第三层 | 03_阶段大纲 | 每阶段 | 阶段核心情节点（chXXX-chXXX） |
| **第四层** | **04_场景大纲** | **每场景** | **核心事件/情节点展开（200-500字/场景）** |
| 第五层 | 04_正文 | 每章 | 正文写作与扩展 |

---

## 二、场景定义与划分标准

### 2.1 场景定义

场景 = 单一视点人物 + 单一场景目标 + 单一情节点

**判断标准**：
- 视点人物是否唯一（同一场景内不切换视点）
- 场景目标是否单一（不同时追求多个目标）
- 情节是否聚焦（一个核心事件，不混杂多条情节线）

### 2.2 场景与章节的关系

| 情况 | 对应关系 |
|------|----------|
| 简单情节 | 一个场景 = 一章（ch001） |
| 复杂情节 | 一个场景 = 连续多章（ch001-ch003） |
| 多场景章 | 多场景合并为一章（不推荐，仅在节奏需要时使用） |

### 2.3 场景划分检查

划分场景时，检查以下问题：

1. 视点人物是否需要切换？（是 → 开启新场景）
2. 场景地点/时间是否发生重大变化？（是 → 开启新场景）
3. 场景目标是否已经达成或放弃？（是 → 开启新场景）
4. 是否需要引入新的主要冲突？（是 → 开启新场景）

## 三、场景大纲生成流程

### 3.1 前置条件

生成场景大纲前，必须确认以下输入已完成并锁定：

- [ ] 全文总体大纲（01_全文总体大纲）已审核通过
- [ ] 卷大纲（02_卷大纲）已审核通过
- [ ] 阶段大纲（03_阶段大纲）已审核通过

### 3.2 生成步骤

```
STEP 1：阅读阶段大纲
    ↓
STEP 2：拆解阶段核心情节点
    ↓
STEP 3：为每个核心情节点设计场景
    ↓
STEP 4：确定场景序列与视点人物
    ↓
STEP 5：逐场景填写场景大纲
    ↓
STEP 6：审核场景大纲（主编/主控）
    ↓
STEP 7：场景大纲锁定，进入正文写作
```

### 3.3 详细步骤说明

#### STEP 1：阅读阶段大纲

仔细阅读目标阶段的阶段大纲文件，理解：
- 该阶段的核心主题
- 该阶段包含的主要情节点
- 该阶段的章节范围（chXXX-chXXX）
- 该阶段与前后阶段的衔接点

#### STEP 2：拆解阶段核心情节点

将阶段大纲中的描述拆解为可执行的具体情节点。每个情节点对应一个或多个场景。

示例（阶段大纲描述）：
> ch001-005：废土黄昏，林夜父母被暗域生物追杀至死，林夜躲在尸体下逃过一劫。

拆解为场景：
- 场景001：林夜一家在废土上遭遇暗域生物（视点：林夜父亲）
- 场景002：林夜父母抵抗暗域生物，掩护林夜（视点：林夜母亲）
- 场景003：林夜躲在尸体下逃过一劫（视点：林夜）

#### STEP 3：为每个核心情节点设计场景

对于每个场景，确定：
- 视点人物（必须是该场景中最重要的角色）
- 场景类型（战斗章/情感章/日常章/突破章）
- 场景目标（角色想要什么）
- 阻碍来源（什么阻止了他）
- 核心情节点（具体发生了什么）

#### STEP 4：确定场景序列与视点人物

按照时间顺序和逻辑顺序排列场景：
- 确保视点人物切换时有合理过渡
- 确保情绪曲线有起伏
- 确保章末钩子有效连接相邻场景

#### STEP 5：逐场景填写场景大纲

使用「场景大纲_模板.md」逐场景填写。

**注意**：每个场景大纲文件对应一个场景，文件命名格式为：
```
[卷号]_[阶段号]_[场景序号]_[场景名].md
```

#### STEP 6：审核场景大纲

主编/主控对场景大纲进行审核，检查：
- [ ] 场景划分是否合理（无不必要切换）
- [ ] 场景目标是否明确
- [ ] 阻碍来源是否有力
- [ ] 情绪曲线是否连贯
- [ ] 章末钩子是否有效
- [ ] 伏笔是否记录回收位置

#### STEP 7：场景大纲锁定

审核通过后，锁定场景大纲，进入正文写作阶段。锁定后如需修改，必须走变更流程。

## 四、场景大纲审核清单

### 4.1 完整性检查

- [ ] 阶段内所有情节都有对应场景
- [ ] 场景数量与阶段复杂度匹配
- [ ] 场景之间有明确的时间/因果顺序

### 4.2 合理性检查

- [ ] 视点人物选择合理（最重要=最体验）
- [ ] 场景类型与情节匹配
- [ ] 场景目标具体、可实现
- [ ] 阻碍来源有力、具体

### 4.3 连贯性检查

- [ ] 场景之间的情绪传递连续
- [ ] 章末钩子有效连接相邻场景
- [ ] 与前后阶段的衔接点明确

### 4.4 伏笔检查

- [ ] 伏笔有记录
- [ ] 伏笔的回收位置明确
- [ ] 伏笔密度适中（每场景不超过2个新伏笔）

## 五、场景大纲与作家工作室工作流的整合

### 5.1 工作流定位

场景大纲生成位于「作家工作室」工作流的「阶段大纲审核通过」之后、「正文写作」之前：

```
[PHASE_2: 规划期]
  STEP_08：阶段大纲审核 → 通过
      ↓
[新增 STEP_08b：场景大纲生成]  ← 新增步骤
      ↓
[PHASE_4: 写作期]
  STEP_12：正文写作
```

### 5.2 作家工作室工作流整合说明

在现有作家工作室工作流中，在「阶段大纲审核通过」与「正文写作」之间，插入「根据阶段大纲生成场景大纲」步骤：

```
作家工作室工作流（整合后）：

1. 领取阶段写作任务
2. 阅读阶段大纲（03_阶段大纲/卷X_阶段Y/阶段大纲.md）
3. 【新增】根据阶段大纲生成场景大纲
   3a. 拆解阶段核心情节点
   3b. 设计场景序列
   3c. 填写场景大纲（04_场景大纲/卷X_阶段Y_场景Z_场景名.md）
   3d. 自检场景大纲完整性
4. 提交场景大纲审核
5. 审核通过后，领取场景任务
6. 逐场景进行正文写作
7. 完成章节后提交审核
8. 根据审核意见修改
9. 进入下一场景/阶段
```

### 5.3 场景大纲生成的自检清单

完成场景大纲后，作者应自检：

```markdown
## 场景大纲自检

### 日期：[日期]
### 阶段：[卷X 阶段Y]

□ 阶段内所有主要情节点都有对应场景
□ 每个场景有明确的视点人物
□ 每个场景有明确的场景目标
□ 每个场景有具体的阻碍来源
□ 场景序列符合时间/因果逻辑
□ 情绪曲线有起伏，不是单调的
□ 每个场景结尾有有效钩子
□ 伏笔已记录回收位置

### 如有不通过项，说明原因：
-

### 提交审核：是/否
```

## 六、场景大纲变更管理

### 6.1 允许变更的情况

以下情况允许变更场景大纲：
- 阶段审核发现逻辑问题，需要调整场景
- 核心情节发生重大变化
- 视点人物需要更换
- 新增伏笔导致场景顺序调整

### 6.2 变更流程

```
发现变更需求
    ↓
评估影响范围（单场景/多场景/跨阶段）
    ↓
提交变更申请（说明原因和预期效果）
    ↓
主编/主控审批
    ↓
执行变更，更新文件
    ↓
通知相关方（作家/审核员）
```

### 6.3 变更记录

每次变更必须在文件顶部的版本历史中记录：

```markdown
## 版本历史

| 版本 | 日期 | 说明 |
|------|------|------|
| v1.1 | YYYY-MM-DD | 变更原因：[具体描述] |
| v1.0 | YYYY-MM-DD | 初始版本 |
```

## 七、场景大纲质量标准

### 7.1 字数标准

| 项目 | 标准 |
|------|------|
| 单场景大纲字数 | 200-500字（核心情节点部分） |
| 场景数量/阶段 | 5-15个（视阶段复杂度） |
| 伏笔数量/场景 | ≤2个新伏笔 |

### 7.2 质量标准

| 维度 | 标准 |
|------|------|
| 场景目标 | 具体、可衡量、与角色动机一致 |
| 阻碍来源 | 有力、具体、不模糊 |
| 情绪曲线 | 有起伏，结束时情绪余波清晰 |
| 章末钩子 | 有效，触发读者"然后呢"的好奇心 |
| 伏笔 | 有记录，回收位置明确 |

## 八、常见问题处理

### Q1：一个场景可以对应多章正文吗？

A：不推荐。场景是正文写作的最小叙事单元，每个场景大纲应尽量对应单一章节，以保证叙事连贯性和审核粒度。

### Q2：场景之间如何过渡？

A：通过「章末钩子」和「情绪余波」实现过渡。场景A的结尾悬念引出场景B的开始，场景A的情绪余波影响场景B的开始情绪。

### Q3：场景视点人物可以与阶段视点不同吗？

A：可以。阶段可以有主要的视点人物，但具体场景可以根据情节需要切换视点。切换时应确保过渡自然。

### Q4：场景大纲与章节微观结构的关系？

A：场景大纲是写作前的规划工具，章节微观结构（见章节模板）是写作时的执行工具。两者互补：
- 场景大纲：规划阶段填写，回答"这个场景要发生什么"
- 章节微观结构：写作时标注，回答"这个场景如何展开"
