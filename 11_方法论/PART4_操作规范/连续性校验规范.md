# 连续性校验规范

> **版本**：v1.0
> **日期**：2026-05-19
> **目的**：建立类似电影Continuity管理的连续性校验体系，防止人物/道具/场景前后矛盾

---

## 一、连续性校验的定义和重要性

### 什么是连续性（Continuity）

在电影制作中，**Continuity**是指确保镜头与镜头之间、场景与场景之间的一致性。例如：演员的服装、妆容、道具位置、光线方向等在连续拍摄中必须保持一致，否则观众会看到明显的穿帮。

**小说写作中的连续性问题**包括但不限于：
- 人物外观前后不一致（上一章是黑发，下一章变成红发）
- 道具位置凭空改变（昨天还在桌上的钥匙，今天凭空消失）
- 场景描述矛盾（明明是木质地板，踩上去却有瓷砖声）
- 世界观规则自我矛盾（设定中"不能用魔法杀人"，后文却用魔法杀了人）
- 时间线错乱（上文说三年后，下文提到同一事件时变成五年后）

### 连续性校验的重要性

1. **阅读沉浸感**：矛盾会瞬间打破读者的沉浸感，让读者"出戏"
2. **作品可信度**：频繁的矛盾会让读者质疑作者的专业性和作品的质量
3. **工业化生产的基础**：多章节、多卷、长篇作品必须依赖系统化的连续性管理

## 二、审核员在审核正文时的连续性检查要点清单

### 2.1 人物外观检查

> 审核员在阅读正文时，需对照《人物外观档案》检查以下项目：

- [ ] **发型发色**：与档案是否一致
- [ ] **衣着打扮**：服装颜色、款式、配饰是否与档案一致
- [ ] **面容特征**：疤痕、痣、纹身等特殊标记是否与档案一致
- [ ] **表情习惯**：角色在特定情绪下的表情是否符合档案记录
- [ ] **口头禅**：角色是否使用了与档案一致的标志性语言

### 2.2 场景道具检查

> 审核员在阅读正文时，需对照《场景道具档案》检查以下项目：

- [ ] **物品位置**：关键道具的位置是否与档案一致
- [ ] **物品状态**：道具的完好/损坏状态是否与情节进展一致
- [ ] **空间布局**：角色在场景中的移动路线是否合理
- [ ] **光线环境**：场景的光线描述是否与时间段一致
- [ ] **室内/室外**：场景类型是否前后一致

### 2.3 世界观规则检查

> 审核员在阅读正文时，需对照《世界观规则档案》检查以下项目：

- [ ] **规则应用**：情节中规则的运作是否符合档案描述
- [ ] **限制代价**：角色使用规则时是否付出了应有的代价
- [ ] **例外处理**：超出规则的情况是否有过说明
- [ ] **规则冲突**：是否存在本规则与其他规则的矛盾

### 2.4 时间线检查

- [ ] **时间流逝**：昼夜交替、季节变换是否符合现实逻辑
- [ ] **事件顺序**：前后章节的事件因果顺序是否合理
- [ ] **时间标注**：具体时间（几月几号、几年后）是否存在矛盾
- [ ] **角色年龄**：角色的年龄增长是否与时间线一致

### 2.5 名称一致性检查

- [ ] **人名拼写**：同一个人物的姓名是否始终一致
- [ ] **地名拼写**：同一地点的叫法是否始终一致
- [ ] **专有名词**：自创词汇、术语是否始终一致
- [ ] **势力/组织名**：帮派、公司等国家机器名称是否一致

## 三、每章正文提交前的自动校验项目

> 作者在提交章节前，必须完成以下自检项目

### 3.1 人物外观自检

- [ ] 本章涉及的所有角色外观是否与档案一致
- [ ] 角色服装变动是否有合理解释（换衣服、弄脏了、换了新衣）
- [ ] 角色表情是否符合其性格设定
- [ ] 角色口头禅是否使用正确

### 3.2 场景道具自检

- [ ] 本章涉及的所有场景是否已建立档案
- [ ] 关键道具的位置是否有交代
- [ ] 道具状态变化是否有说明
- [ ] 场景光线是否与时间设定匹配

### 3.3 世界观规则自检

- [ ] 本章涉及的世界观规则是否已在档案中登记
- [ ] 规则的运作是否符合档案描述
- [ ] 是否有超越规则限制的情节，如有是否有铺垫
- [ ] 是否引入了新的规则冲突风险

### 3.4 时间线自检

- [ ] 本章时间相对于上一章是否合理推进
- [ ] 时间推进是否有足够的描写支撑
- [ ] 与全局时间线是否存在矛盾
- [ ] 节日、季节、气候是否匹配

### 3.5 名称自检

- [ ] 本章出现的所有人名是否与之前一致
- [ ] 本章出现的地名是否与之前一致
- [ ] 专有名词是否全文统一
- [ ] 新引入的角色/地名/术语是否已记录

## 四、连续性问题的分级与处理

### P0级（致命问题）

- 严重影响情节理解的人物/道具/规则矛盾
- 读者完全无法忽略的重大穿帮
- **处理方式**：必须打回修改，修改完成后重新审核

### P1级（严重问题）

- 对情节有一定影响，但不致命的矛盾
- 细心读者会注意到的不一致
- **处理方式**：建议修改，修改后需审核确认

### P2级（轻微问题）

- 细小的连续性问题
- 大多数读者不会注意到的矛盾
- **处理方式**：记录但不强制修改，必要时可出注

## 五、连续性档案更新流程

### 何时需要更新档案

1. 角色外观发生改变时（换发型、换衣服、受伤留疤等）
2. 场景中物品位置发生改变时
3. 新的世界观规则被引入时
4. 已存在的规则产生新的例外情况时
5. 时间线推进产生新的重要节点时

### 档案更新步骤

1. 在对应的档案中记录变更内容
2. 注明变更的日期和对应的章节
3. 确保档案版本号递增
4. 通知审核员档案已更新
