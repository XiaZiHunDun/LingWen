# 项目复盘操作规范

> 规范小说工厂项目复盘的触发、流程与审核
> 版本：v1.0 | 更新日期：2026-05-19

---

## 一、复盘触发机制

### 自动触发

项目满足以下任一条件时，**自动触发**复盘流程：

| 触发条件 | 触发时间 | 执行动作 |
|---------|---------|---------|
| 小说正式发布 | 发布后 24 小时内 | 系统自动创建复盘任务 |
| 阶段性里程碑完成 | 里程碑达成次日 | 自动生成阶段性复盘 |
| 累积问题超阈值 | 问题数 > 50 条 | 自动生成专项复盘 |

### 手动触发

| 触发场景 | 触发权限 | 执行动作 |
|---------|---------|---------|
| 主公决策 | 主公 | 发布「复盘」指令 |
| 重大问题发现 | 任意 Agent | 通过 Issue 提请复盘 |
| 跨项目重大经验 | 主编 | 提请专项复盘 |

### 触发确认

复盘任务创建后，需由**主编**确认：

```
复盘任务确认
├── 项目名称：[小说名称]
├── 触发类型：[自动/手动]
├── 触发原因：[原因描述]
├── 复盘范围：[全项目/专项/阶段性]
└── 主编确认：[确认/修改/取消]
```

## 二、复盘流程

复盘分为四个阶段：**数据收集 → 问题整理 → 经验沉淀 → 归档**

### 阶段1：数据收集

**执行角色**：汇总部门

**收集内容**：

```markdown
## 数据收集清单

### 1. 项目基本信息
- [ ] 项目名称、周期、参与人数
- [ ] 总字数、总章节数
- [ ] 最终评级（S/A/B级）
- [ ] 各阶段耗时统计

### 2. 问题数据
- [ ] P0/P1/P2 问题清单及修复记录
- [ ] 问题类型分布统计
- [ ] 核心修改点 TOP3

### 3. 流程数据
- [ ] 各阶段实际周期 vs 计划周期
- [ ] 审核迭代次数
- [ ] 关键决策点记录

### 4. 质量数据
- [ ] 各维度评分（S1-S8）
- [ ] 读者反馈摘要
- [ ] 审核意见汇总
```

**输出**：`数据收集清单_填充版.md`

---

### 阶段2：问题整理

**执行角色**：审核部门

**整理内容**：

```markdown
## 问题整理清单

### 1. 大纲阶段问题
- 发现于构思期/规划期/验证期的问题
- 问题的根本原因分析
- 处置方式（修复/接受/延迟）

### 2. 正文阶段问题
- 按章节范围统计
- 按问题类型统计（一致性/逻辑/节奏/情感）
- 反复出现的问题模式

### 3. 审核标准模糊点
- 审核过程中标准不明确的地方
- 不同审核员判定不一致的场景
- 建议补充的审核标准
```

**输出**：`问题整理清单.md`

---

### 阶段3：经验沉淀

**执行角色**：主编（主笔）

**沉淀内容**：

```markdown
## 经验沉淀清单

### 1. 有效的创作习惯
- 在本项目中被验证有效的方法
- 适用场景说明
- 效果量化指标（如有）

### 2. 踩过的坑
- 问题描述
- 影响范围
- 根本原因
- 规避方案（可复用于其他项目）

### 3. 审核标准优化
- 原有标准的问题
- 优化建议
- 新标准草案

### 4. 可复用模式
- 适用于新项目的模式总结
- 模板/工具建议
```

**输出**：`经验沉淀清单.md`

---

### 阶段4：归档

**执行角色**：汇总部门 + 主编

**归档流程**：

1. **报告生成**：汇总前三阶段输出，生成`项目复盘报告_XX项目_YYYYMMDD.md`
2. **主编审核**：主编审核报告内容，填写「归档确认」
3. **主公审批**：S级项目或重大问题项目需主公最终审批
4. **归档入库**：移动至 `07_汇总仓库/项目复盘/` 目录
5. **索引更新**：更新 `目录索引.md` 中的复盘索引

### 审核要点

| 审核节点 | 审核内容 | 通过标准 |
|---------|---------|---------|
| 主编初审 | 数据准确性、问题归类 | 无事实错误 |
| 主编二审 | 经验有效性、改进计划可行性 | 经验可复用、计划可执行 |
| 主公终审 | 重大决策、资源投入 | 符合项目方向 |

### 审核时限

| 审核节点 | 标准时限 | 超时处理 |
|---------|---------|---------|
| 主编初审 | 24 小时 | 自动通过 |
| 主编二审 | 24 小时 | 升级主公 |
| 主公终审 | 48 小时 | 默认通过 |

## 五、复盘索引管理

### 目录索引更新

每次归档复盘报告后，更新 `10_方法论/目录索引.md` 中的复盘索引：

```markdown
## 项目复盘索引

| 项目 | 版本 | 日期 | 复盘报告 |
|------|------|------|---------|
| 星陨纪元 | v1.0 | 2026-05-19 | 项目复盘报告_星陨纪元_优化版_v1.0_20260519.md |
```

### 快速查询

复盘报告按项目分组归档，查询时：

1. 定位 `07_汇总仓库/项目复盘/{项目名称}/`
2. 选择最新版本报告
3. 参考「经验沉淀」和「改进计划」章节

## 六、质量门控

### 复盘报告质量标准

| 检查项 | 标准 | 检查方法 |
|--------|------|---------|
| 数据完整性 | 100% 必填项已填 | 对照收集清单 |
| 问题准确性 | 无事实错误 | 交叉验证 |
| 经验可复用性 | 至少 3 条可复用经验 | 主编判断 |
| 改进计划可行性 | 至少 1 条可执行改进 | SMART 原则 |

### 不合格处理

| 不合格原因 | 处理方式 |
|-----------|---------|
| 数据不完整 | 返回阶段1补充 |
| 经验不可复用 | 返回阶段3重写 |
| 计划不可执行 | 返回阶段3修订 |
