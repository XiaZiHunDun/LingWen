# 评审前置检查

> **版本**：v1.0
> **日期**：2026-05-19
> **目的**：创作完成后进入评审前，作者自检清单

---

## 核心理念

**评审不是找问题的，而是验证创作质量的。**

评审前置检查是为了确保：
1. 创作阶段的问题在进入评审前已经解决
2. 评审资源集中在"需要专业视角"的问题上
3. 减少评审轮次，提高效率

---

## 一、Hard约束自检

Hard约束是**一票否决**项，必须全部通过才能进入评审。

### 1.1 叙事主角唯一性

- [ ] 全书主线是否围绕设定的主角展开
- [ ] 是否出现不合理的"接班人/新主角团/代际叙事夺舍"
- [ ] 有无"传承给新一代当主角"式handoff

### 1.2 修炼/战力里程碑

- [ ] 修炼等级是否与设定体系一致
- [ ] 战力表现是否前后矛盾
- [ ] 关键战斗结果是否有合理铺垫

### 1.3 卷间关键衔接

- [ ] 卷与卷之间是否无缝衔接
- [ ] 章节衔接点（如ch120→121、ch240→241）是否连贯
- [ ] 时间线是否自洽

---

## 二、核心驱动链自检 [新增]

核心驱动链是读者持续阅读的"引擎"，必须在进入评审前确认有效。

### 2.1 驱动链完整性

- [ ] 核心驱动链是否能用一句话描述
- [ ] 每X章的小满足类型是否清晰
- [ ] 每X卷的大满足类型是否清晰

### 2.2 驱动链兑现

- [ ] 驱动链中的每个节点在正文中有兑现
- [ ] 是否存在"承诺的爽点没有兑现"的情况
- [ ] 满足感的递增性是否体现

### 2.3 驱动链频率

| 检查点 | 标准 |
|--------|------|
| 小满足分布 | 每3-5章应有一次小满足 |
| 大满足分布 | 每10-15章应有一次大满足 |
| 章末钩子 | 每章结尾应有"想知道然后呢"的悬念 |

### 2.4 驱动链差异化

- [ ] 与同类型小说的驱动链差异点是否明确
- [ ] 是否有独特的"辨识度"

---

## 三、Soft维度自检（S1-S7）

### S1 剧情完整性

- [ ] 章内目标—障碍—结果是否清楚
- [ ] 卷内因果是否断档
- [ ] 每章是否有明确的存在意义（不是"废章节"）

### S2 逻辑自洽

- [ ] 时间线是否一致
- [ ] 战力系统是否自洽
- [ ] 势力行为是否符合其设定

### S3 文笔与风格

- [ ] 语言、对话、描写是否统一
- [ ] 角色声音是否有区分度
- [ ] 世界观吸引力是否足够

### S4 情感共鸣

- [ ] 情绪曲线是否有效（至少一处读者可代入的波动）
- [ ] 有无"名场面""高光时刻""虐点/爽点"
- [ ] 读者是否能感受到人物的喜怒哀乐

### S5 节奏控制

- [ ] 信息投放是否缓急得当
- [ ] 战斗/文戏比例是否服务于目标
- [ ] 有无长段无聊说教、无冲突日常

### S6 可读性/吸引力

- [ ] 开头是否足够抓人（前3章/前1万字）
- [ ] 读者是否愿意一口气读完
- [ ] 是否期待下一章（章末钩子是否有效）

### S7 主角魅力/认同感

- [ ] 主角行为是否让读者认同
- [ ] 读者是否想看主角成功
- [ ] 主角的成长是否让读者满足

---

## 三、创作质量自检

### 3.1 故事结构

- [ ] 一句话故事是否清晰（主角+目标+阻碍+代价）
- [ ] 三幕式结构是否完整
- [ ] 关键情节点是否到位（诱发事件/中点/高潮）

### 3.2 人物设计

- [ ] 主角目标是否清晰
- [ ] 核心冲突是否存在
- [ ] 人物行为是否符合性格
- [ ] 有无"降智/突然变性格"问题

### 3.3 设定一致性

- [ ] 人名、地名、专有名词是否统一
- [ ] 世界观规则是否前后一致
- [ ] 伏笔是否已记录并可追踪

### 3.4 开篇质量

- [ ] 前300字是否包含动作/悬念/情绪锚点
- [ ] 是否避免了大段背景介绍
- [ ] 主角是否快速出场

### 3.5 章节质量

- [ ] 每章是否有明确目标/阻碍/结果
- [ ] 章末是否有有效钩子
- [ ] 有无"无事发生"式平收

---

## 四、修改质量自检

### 4.1 宏观修改完成

- [ ] 结构性问题是否已解决
- [ ] 情节逻辑漏洞是否已修复
- [ ] 人物动机是否合理

### 4.2 微观修改完成

- [ ] 文字表达是否已打磨
- [ ] 对话是否已优化
- [ ] 视角一致性是否已检查

### 4.3 校对完成

- [ ] 错别字、标点是否已检查
- [ ] 格式是否已统一
- [ ] 时间线/人名硬伤是否已修复

## 六、未通过项处理

如果自检未通过，按照以下优先级处理：

**优先级1（必须修复）**：
- Hard约束问题
- S1、S2低于3分的问题

**优先级2（强烈建议修复）**：
- S6、S7低于3分的问题
- 严重的逻辑漏洞

**优先级3（可以考虑修复）**：
- S3、S4、S5低于3分的问题
- 文笔/节奏优化
