# 阶段6：定稿与发布

> **核心观点**：写完不是结束，但必须学会在某一版停下来。把更多经验留给下一本书。

---

## 一、定稿前检查清单

### 最终校对

| 检查项 | 说明 |
|--------|------|
| 错别字 | 用工具扫描 + 人工朗读检查 |
| 标点 | 统一格式，中文标点无混用 |
| 格式 | 段落分隔、缩进、章节编号 |
| 人名统一 | 同一人名字全程一致 |
| 地名统一 | 同一地名全程一致 |
| 专有名词 | 设定中的术语全程一致 |
| 章节完整 | 无缺失章节、无乱序 |

### 内容一致性

| 检查项 | 说明 |
|--------|------|
| 时间线 | 事件顺序无矛盾 |
| 人物性格 | 无突然"降智"或"变性格" |
| 设定一致 | 能力/规则全程遵守 |
| 伏笔回收 | 所有埋下的伏笔都有回收 |

### 出版格式（参考）

| 类型 | 字数/章 | 说明 |
|------|---------|------|
| 网文 | 2000-5000字/章 | 按平台风格调整 |
| 实体书 | 3000-8000字/章 | 更从容的阅读节奏 |
| 短篇 | 5000-30000字 | 完整故事单本 |

## 二、发布渠道对比

| 渠道 | 适合类型 | 优点 | 缺点 |
|------|----------|------|------|
| **起点/晋江** | 网文长篇 | 流量大，读者成熟，有编辑指导 | 竞争激烈，需稳定更新 |
| **番茄小说** | 免费网文 | 流量大，对新人友好 | 商业压力大 |
| **传统出版** | 实体书 | 有 ISBN，有编辑，有发行渠道 | 门槛高，周期长 |
| **豆瓣阅读** | 文学向/中短篇 | 独立出版，门槛较低 | 流量有限 |
| **Kindle自出版** | 全类型 | 完全自主，门槛低 | 无编辑，需要自己营销 |

## 三、网文平台发布要点

### 更新节奏
| 类型 | 推荐节奏 | 说明 |
|------|----------|------|
| 精品文 | 稳定优先 | 每周2-4更，比爆更后弃更好 |
| 快更文 | 爆更抢榜 | 特定时期冲榜用 |

### 章节要点
- 每章结尾留钩子（让读者想看下一章）
- 控制断章点（在高潮/悬念处断，不要在平静处断）
- 书名+简介要有吸引力（平台推荐的核心）

### 简介写法
> 公式：`背景设定 + 主角 + 核心冲突 + 悬念/看点`
>
> 例：
> "普通上班族林默意外卷入一场杀人游戏，必须在三天内找出真凶。问题是——所有证据都指向他自己。"

## 四、心态

> "作品永远可以再改，但必须学会在某一个版本停下来。"

### 常见心态问题

| 心态 | 应对 |
|------|------|
| 完美主义 | 初稿的使命是存在，不是完美 |
| 恐惧发布 | 没有人第一部作品就是神作 |
| 过度在意反馈 | 区分共性问题 vs 个人偏好 |
| 死磕一本书 | 把经验留给下一本比死磕更健康 |

### 健康创作节奏

```
写完一本书 → 休息一段时间 → 复盘收获 → 开始下一本
     ↑                                              ↓
     ←←←←←←← 经验积累，慢慢进步 ←←←←←←←←←←←←←
```

---

## 五、发布后

- **接受"作品永远可以再改"**
- **收集读者反馈，但保持创作独立性**
- **开始下一本书**（经验比死磕更重要）
- **创作者是练出来的，写够100万字，很多困惑自然就通了**
