# 动态预演工具

> **版本**：v1.0
> **来源**：动画制作流程的Animatic（动态分镜）
> **日期**：2026-05-19

---

## 一、工具说明

动画制作有一个关键环节叫"动态分镜"（Animatic）——把静态的故事板配上临时音轨，"先拍一版电影"来验证节奏。

小说创作可以借鉴这个思路：**在正式写作之前，用简陋的方式"听到"故事的整体节奏**。

> 核心价值：提前发现问题，修改成本远低于写完之后才发现节奏问题。

---

## 二、适用场景

| 适用 | 不适用 |
|---|---|
| 长篇（≥30章） | 短篇（<10章） |
| 新人首个创作项目 | 有多本完本经验的作者（已内化节奏感） |
| 情节复杂、转折多的故事 | 线性叙事、结构简单的故事 |

---

## 三、三种预演方式

### 方式A：语音大纲（TTS）

**操作步骤**：
1. 把章纲（每章一句话描述）整理成一个文档
2. 用文字转语音工具（TTS）逐章朗读
3. 记录每个章节的"阅读时长"
4. 对比预期阅读时长，判断节奏是否合理

**工具推荐**：
- ChatGPT / Claude（文字转语音）
- 浏览器TTS插件
- 讯飞/百度TTS开放平台

**判断标准**：
- 单章语音时长 > 8分钟 → 本章可能太拖沓
- 单章语音时长 < 2分钟 → 本章可能太单薄

### 方式B：剧本预演

**操作步骤**：
1. 选择核心场景（预计10-15个）
2. 把这些场景写成纯对话体（不写描写、不写心理活动）
3. 朗读对话，检验：
   - 对话节奏是否流畅
   - 角色性格是否清晰可辨
   - 信息释放是否自然

**模板**：

```
【场景X：核心场景名称】

角色A：
（动作）

角色B：
（动作）

角色A：
对话内容...

角色B：
对话内容...
```

**判断标准**：
- 朗读时是否觉得角色说话方式有区分度
- 信息是否在对话中自然传递，没有"背课文"感

### 方式C：时间轴预览

**操作步骤**：
1. 用表格列出每个章节的三个信息：
   - 事件（本章发生什么）
   - 情绪（本章的核心情绪色彩）
   - 篇幅（预计字数）
2. 检查：
   - 前10章是否有足够的"翻页冲动"
   - 中段是否出现"无聊区"（连续5章以上无重大转折）
   - 高潮是否足够集中（最后3章是否连续高能）

**时间轴模板**：

| 章 | 事件 | 情绪色彩 | 字数 | 问题 |
|---|---|---|---|---|
| 1 | 穿越+压制 | 深蓝 | 3000 | 太慢？ |
| 2 | 金手指出现 | 暖黄 | 2500 | OK |
| 3 | 第一次战斗 | 橙红 | 4000 | 太长？ |
| ... | ... | ... | ... | ... |

## 四、动态预演检查点

无论选择哪种方式，预演后必须检查以下点：

| 检查项 | 标准 | 未通过处理 |
|---|---|---|
| 总时长 | 是否符合预期（网络小说：30章≈20万字，≈3-5小时阅读量） | 调整章节数量或单章字数 |
| 前10章 | 是否有足够的"翻页冲动"（每3章至少1个钩子） | 增强前10章的冲突密度 |
| 中段节奏 | 是否出现"无聊区"（连续5章以上无重大转折） | 插入支线事件或配角戏份 |
| 高潮分布 | 最后3章是否连续高能 | 重新设计高潮章节的情感色彩 |
| 情绪曲线 | 是否符合色彩曲线规律（见情感色彩曲线工具） | 调整章节顺序或增加过渡章节 |

---

## 五、不通过的处理

预演后如发现问题：

```
发现问题 → 定位问题章节 → 返回规划期修改章纲 → 重新预演
```

**常见问题与解决方案**：

| 问题 | 解决方案 |
|---|---|
| 前10章太慢 | 删除或压缩第2-4章的背景铺垫，提前冲突 |
| 中段无聊 | 插入一个"意外事件"（配角死亡/背叛/奇遇） |
| 高潮太散 | 合并多个小高潮为一个超级高潮 |
| 情绪单调 | 在低谷章节后增加一个温情场景 |
| 节奏太快 | 增加过渡场景，让读者喘息 |

---

## 六、与创作流程的接口

动态预演发生在：

```
锁定门（规划期结束） → 【动态预演】 → 写作期
```

如果预演不通过，返回规划期重新梳理章纲，通过后再进入写作期。
