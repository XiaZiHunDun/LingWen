## 一、核心驱动链设计

### 1.1 一句话驱动链

用一句话描述驱动读者持续阅读的核心模式：

```
[类型]小说的核心驱动链：
[描述读者每X章获得的满足感]
```

**示例（玄幻爽文）**：
> 主角从被轻视到逐渐变强，每3章小爽一次（打脸嘲讽者），每10章大爽一次（跨级战胜强敌），驱动链具有递增性。

### 1.2 驱动链构成

```
驱动链类型：[升级爽/复仇爽/探秘爽/情感甜/悬疑解谜等]

起始状态：[描述主角开始的起点]

核心循环：
[步骤1：触发事件] → [步骤2：遭遇阻碍] → [步骤3：蓄能/准备] → [步骤4：突破/反杀] → [步骤5：获得回报] → [循环回到步骤1]
```

### 1.3 满足感类型定义

| 满足类型 | 定义 | 示例 |
|----------|------|------|
| **小爽** | 3章左右一次的情绪释放点 | 打脸对方、获得小突破、识破谎言 |
| **中爽** | 5-10章一次的中等满足 | 战胜同级别对手、获得重要道具、关系进展 |
| **大爽** | 10+章一次的高潮满足 | 跨级战胜、震惊全场、重大揭秘 |

## 三、驱动链递增性设计

### 3.1 强度递增

| 阶段 | 小满足强度 | 大满足强度 |
|------|------------|------------|
| 第一幕（0-25%） | 1分（轻量打脸） | 5分（初胜） |
| 第二幕前半（25-50%） | 2分（稳定打脸） | 6分（连捷） |
| 第二幕后半（50-75%） | 3分（批量打脸） | 8分（关键胜利） |
| 第三幕（75-100%） | 4分（碾压式打脸） | 10分（最终决战胜利） |

### 3.2 复杂度递增

| 阶段 | 阻碍来源 | 解决难度 |
|------|----------|----------|
| 第一幕（0-25%） | 单一角色 | 1vs1 |
| 第二幕前半（25-50%） | 小势力 | 1vs小团体 |
| 第二幕后半（50-75%） | 大势力 | 1vs组织 |
| 第三幕（75-100%） | 最终boss | 1vs1 + 势力对决 |

## 四、驱动链验证清单

在完成设计后，自检以下问题：

### 4.1 完整性检查

- [ ] 核心驱动链能用一句话描述
- [ ] 小满足的频率和形式已定义
- [ ] 大满足的频率和形式已定义
- [ ] 驱动链具有递增性

### 4.2 差异化检查

- [ ] 与对标作品的驱动链有差异点
- [ ] 有独特的"辨识度"
- [ ] 不是简单复制同类型小说

### 4.3 可执行性检查

- [ ] 驱动链的每个节点在情节中有对应设计
- [ ] 满足感的递增性与三幕式结构匹配
- [ ] 没有"承诺了却无法兑现"的设计

## 五、填写示例：《重生之都市修仙》

### 5.1 项目基本信息

| 字段 | 内容 |
|------|------|
| **小说名称** | 重生之都市修仙 |
| **小说类型** | 都市玄幻·重生修仙 |
| **预计总字数** | 80万字 |
| **预计卷数** | 3卷 |
| **目标读者** | 喜欢都市修仙、装逼打脸的男性读者 |
| **对标作品** | 《都市最强装逼系统》《修仙归来在都市》 |

### 5.2 一句话驱动链

> 主角重生回到都市，凭借上古修仙传承快速崛起，每3章小爽一次（打脸嘲讽者），每10章大爽一次（跨级战胜强敌），从默默无闻到飞升仙界。

### 5.3 驱动链构成

```
驱动链类型：升级爽 + 复仇爽

起始状态：主角是濒死的废材大学生，意外重生回到大学时期

核心循环：
被打压 → 蓄能（修炼/获得传承） → 展示实力 → 打脸对方 → 获得尊重/资源 → 遇到更强对手 → 循环
```

### 5.4 满足感频率

| 小满足 | 每3章 | 被嘲讽 → 当众打脸 |
| 中满足 | 每5章 | 修为突破/获得新能力 |
| 大满足 | 每10章 | 跨级战胜强敌 |

### 5.5 递增性

| 阶段 | 小满足 | 大满足 |
|------|--------|--------|
| 第一卷 | 打脸同学/同事 | 战胜小boss |
| 第二卷 | 打脸富二代/小家族 | 战胜大家族继承人 |
| 第三卷 | 打脸顶级大少/仙二代 | 战胜仙界势力代表 |
