## 四、使用流程

### 4.1 章节写作Prompt生成流程

```
1. 确定章节号 (如 ch025)
       ↓
2. 判断章节类型（开篇/发展/高潮/收尾/过渡）
       ↓
3. 加载项目数据
   - 基础层.yaml (作品类型、风格、人物设定)
   - 深度层.md (世界观、时间线、伏笔、叙事结构)
       ↓
4. 填充模板占位符
   - 共40+个占位符，都有明确的数据来源
       ↓
5. 输出完整Prompt (~2500 tokens)
```

### 4.2 角色与数据来源

| 占位符前缀 | 数据来源 |
|------------|----------|
| `novel_title`, `genre`, `style` | 基础层.yaml |
| `word_count`, `volume`, `time_line` | 深度层.md |
| `character_*` | 基础层.yaml / 深度层.md |
| `world_building_*` | 深度层.md |
| `foreshadowing_*` | 深度层.md |

## 五、使用示例

以 ch025 章节写作为例：

**输入**：
- 章节号：025
- 章节类型：发展
- 前情：林夜独自生存3个月，与苏琳相遇

**生成的Prompt包含**：
1. 作品基础信息（来自基础层.yaml）
2. 章节上下文（定位、前情、目标）
3. 主要角色信息（林夜、苏琳状态）
4. 场景设定（废墟、雪夜、冷峻氛围）
5. 情节要求（必须包含的元素、禁止内容）
6. 写作规范（视角、风格、对话比例）
7. 质量标准（自评检查点）
8. 参考素材（伏笔记录、参考章节）

**输出**：约 2500 tokens 的完整写作指导

## 七、质量保证

所有提示词模板遵循以下标准：

1. **完整性**：包含角色、场景、情节、风格全部要素
2. **可执行性**：指导具体明确，可直接用于LLM调用
3. **质量门控**：内置自评标准和检查点
4. **上下文关联**：与大纲、已有人物设定保持一致
5. **数据溯源**：每个占位符都有明确的数据来源
