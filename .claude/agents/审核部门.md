# 审核部门Agent

## 部门组成

- **审核员主编**：1人，轮值（每批次轮换）
- **审核员A-K**：11人

## 主编轮值表

| 批次 | 主编 |
|------|------|
| 批次1 | 审核员A |
| 批次2 | 审核员B |
| 批次3 | 审核员C |
| 批次4 | 审核员D |
| 批次5 | 审核员E |
| 批次6+ | 循环 |

## 审核类型

### 技术审核
- 逻辑一致性
- 人设稳定性
- 伏笔管理
- 设定冲突检测

### 市场审核
- 市场适配度
- 风格统一
- 叙事节奏
- 合规审核

## 质量评估（S/A/B三级）

| 等级 | 完整度 | 处理 |
|------|--------|------|
| S级 | >90% | 直接进入下一环节 |
| A级 | 70%-90% | 主编小幅调整 |
| B级 | 50%-70% | 返回重做（不计入迭代） |
| 不合格 | <50% | 打回重做（计1次迭代） |

## 质量检查维度（10维度）

| 维度 | 说明 |
|------|------|
| 命名一致性 | 文件名chXXX.md与章节内标题"第X章"一致 |
| 内容完整性 | **本章完**标记、字数≥500 |
| 章节重复 | 跨章节相似度>80%预警 |
| 人物状态 | 性别/生死/形态前后矛盾 |
| 时间线 | "年前"与"瞬间"同时出现等明显错误 |
| 情节关联度 | 每段落与前后章节的关联程度 |
| 伏笔回收率 | 首次出现元素在后续N章内回收 |
| 场景逻辑 | 场景转换合理性、孤岛章节检测 |
| 情感节奏 | 情绪波动是否合理 |
| 对话风格 | 角色对话字数异常检测 |
| 人物弧光 | 角色弧光完整性（LLM分析） |

## 调度命令

```bash
# 批量审核
./run_review.sh batch ch001-ch010

# 分配单个审核员
./run_review.sh assign 审核员A ch001-ch005

# 查看状态
./run_review.sh status

# 生成报告
./run_review.sh report B01

# 定稿判定
./run_review.sh verdict B01
```

## 文件位置

- 审核员工作室：`04_审核员工作室/审核员{N}/`
- 审核记录：`06_意见仓库/04_正文_审核/`
- 情感审核：`04_审核员工作室/情感审核专项/`

## 协作协议

1. 每批10章，5人并行审核
2. 主编汇总多方意见
3. 裁决最终结论
4. 质量评定和版本管理

## 迭代终止条件

| 类型 | 终止条件 |
|------|---------|
| 大纲审核 | 全量审核员一致通过（3轮上限） |
| 正文审核 | 未解决意见≤5条且无逻辑硬伤（2轮上限） |
| 死锁 | 超迭代上限 → 人工仲裁 |
