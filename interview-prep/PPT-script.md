## 幻灯片 5 / 12 · 5 Agent + GoT Router ⭐

### 页面文字

```
5 Agent · 职责分离 · GoT 路由
──────────────────────────────────

[diagrams/png/02-agent-orchestration.png 插入]

为什么 5 个？──── 为什么不是 1 个？
──────────────────────────────────

  1 个 Agent：上下文爆炸 · 测试难 · 失败难定位
  5 个 Agent：每步独立 · 独立测试 · 独立替换

12 SCENARIOS 预定义图 ──── 不用 ReAct
──────────────────────────────────

  ReAct 思考-行动循环 → 3 次跑出 3 个大纲
  GoT 预定义图 → 稳定可复现
```

### 讲法（90 秒 · 配合图讲）

> "5 个专职 Agent 怎么配合？看这张图——**中间是 GoT Router**，把 12 个 SCENARIOS 解耦，每个场景对应一个 handler。每个 Agent 都接收上游的输入和角色卡，调用 LLM 完成自己的专属任务。
>
> 为什么是 5 个不是 1 个？1 个 Agent 上下文会爆炸，测试也难——你不知道哪步出了问题。拆 5 个之后，每步独立可测，失败也好定位。
>
> 为什么用 GoT Router 不用 ReAct？ReAct 是'思考-行动'循环——对网文这种**多步 pipeline** 不稳定，3 次跑出 3 个大纲。我们要的是**预定义图**，不是推理。"

## 幻灯片 6 / 12 · 11 维质量门漏斗 ⭐⭐（最被追问）

### 页面文字

```
11 维质量门 · 漏斗
──────────────────────────────────

[diagrams/png/03-quality-pipeline.png 插入]

数字速记
──────────────────────────────────

  Layer 1 规则 (S1-S8): <500ms · 0 误报
  Layer 2 LLM  (S9-S11): ~3s · 软评分

  8 本样章: 7 本 P0=0
  LLM judge: 7/7 ≥ 4.0 (满分 5)
```

### 讲法（90 秒 · 必讲 Fallback）

> "这是灵文最得意的设计——**11 维质量门漏斗**。上层是 8 维规则硬验证——剧情完整、逻辑自洽、文笔风格、情感共鸣、节奏控制、可读性、主角魅力、人物弧光。速度**不到 500 毫秒**，**0 误报**——确定性、可解释、快。任意 1 维不过 → 返工。
>
> 下层是 3 维 LLM 软评分——角色一致性、逻辑自洽度、伏笔回收率。速度**约 3 秒**，软评分——**不通过不返工**，只警告，标 P1。为什么用 LLM？因为这 3 维是规则挡不住的'软伤'——角色性格漂移、伏笔忘了、逻辑细微漏洞。
>
> **关键设计**：LLM 服务挂了怎么办？**自动 Fallback**——跳过 Layer 2，只跑 Layer 1 规则引擎，**生产不中断**。这就是 7 本零硬错还能稳定跑的工程保障——**单点失败不波全链路**。"

### 页面文字

```
章节状态机 · 3 铁律守门
──────────────────────────────────

[diagrams/png/05-state-machine.png 插入]

3 铁律 = 状态机的护栏，不是 Agent 的 features
──────────────────────────────────

  铁律 1  max_iterations   Agent 单章迭代上限 · 防死循环
  铁律 2  cost cap         单章 $X 熔断 · 超 → 降级 cheap
  铁律 3  checkpoint       每状态转换强制落库 · 崩溃可恢复
```

### 讲法（60 秒 · 选讲）

> "这是章节生命周期的状态机。8 个状态——INIT、OUTLINE、CHARACTER、WRITE、AUDIT、POLISH、DONE，加上一个 LLM 服务挂了的 FB 旁路。
>
> 关键设计是 **3 铁律**——这不是 Agent 的 features，是状态机的护栏。**铁律 1 max_iterations** 防 Agent 死循环；**铁律 2 cost cap** 单章 $X 熔断，超了降级到 cheap tier；**铁律 3 checkpoint** 每状态转换强制落库，崩溃后从最近 checkpoint 重放，**不丢章节**。
>
> 注意：返工和告警是分离的——P0 硬错退回 Layer 1，P1 软伤只警告不返工，避免无限返工循环。"
