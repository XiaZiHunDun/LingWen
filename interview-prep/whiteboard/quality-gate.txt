================================================================================
讲法提示（按漏斗顺序念出来 · 90 秒）
================================================================================

Step 1 · 画漏斗 + Layer 1（30s）
  "这是灵文最得意的设计——11 维质量门。上层是 8 维规则硬验证：剧情完整、
  逻辑自洽、文笔风格、情感共鸣、节奏控制、可读性、主角魅力、人物弧光。
  速度 <500ms，0 误报——确定性、可解释、快。任意 1 维不过 → 返工
  content_writer。"

Step 2 · 画 Layer 2 + 阈值判断（30s）
  "下层是 3 维 LLM 软评分：角色一致性、逻辑自洽度、伏笔回收率。
  速度 ~3s，软评分——不通过不返工，只警告。为什么用 LLM？因为这 3 维
  是规则挡不住的'软伤'——角色性格漂移、伏笔忘记、逻辑细微漏洞。
  关键设计：规则做硬底线，LLM 做软评分。"

Step 3 · 画 Fallback（30s · 必讲）
  "关键设计：LLM 服务挂了怎么办？自动 Fallback——跳过 Layer 2，
  只跑 Layer 1 规则引擎。生产不中断。这就是 7 本 P0=0 还能稳定跑
  的工程保障——单点失败不波全链路。"

================================================================================
数字速记
================================================================================

- S1-S8 规则：8 维 · <500ms · 0 误报
- S9-S11 LLM：3 维 · ~3s · 软评分
- 8 本样章：7 本 P0=0（规则必过）
- LLM judge：7/7 ≥ 4.0（满分 5）
- Fallback 触发：实测 <5% 章节（LLM 服务偶尔抽风）

================================================================================
被追问时
================================================================================

Q · LLM-as-Judge 有偏见怎么办？
A · 3 个偏见：评自己偏高 / 长度偏见 / 风格偏见。解决方案：多 judge 投票
  （MiniMax + Anthropic）+ 反例库（49 个 bad case）+ 规则门做硬底线。
  详见 project/06-reflections.md §2。

Q · LLM 软评分不通过会怎样？
A · 不返工——只警告，标 P1。因为 LLM 判断有不确定性，规则挡不住的软伤
  不会阻塞生产。

Q · 为什么是 8 规则 + 3 LLM 不是别的比例？
A · 业务驱动——S1-S8 是"硬错"（剧情不完整 / 逻辑不自洽 这种返工级），
  S9-S11 是"软伤"（角色漂移 / 伏笔忘 这种加分项）。不是技术比例，
  是业务分类。
