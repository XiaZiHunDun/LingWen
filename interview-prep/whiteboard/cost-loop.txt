讲法提示（按 5 层顺序念出来 · 60 秒）
================================================================================

Step 1 · 画 5 层 + DB（30s）
  "Token 成本怎么控？灵文走 5 层真实跟踪——不是估算。每一层都记录真实
  token：Provider 返回的 usage → Router 按 tier 分流 → AgentBase Mixin
  调 LLM → MasterController 状态机驱动 → got_bridge 12 场景 handler。
  所有 token 写入 cost_tracker.db（SQLite，gitignored）。"

Step 2 · 画 Dashboard + 数字对比（30s）
  "持久化后通过 Dashboard 展示：7d / 30d / all 三档时间窗，per-tier
  multi-series 趋势。数字上——MiniMax 单章 ~$0.063，GPT-4 ~$0.3+，
  便宜 5 倍。这就是为什么选 MiniMax 而不是 GPT-4：中文网文场景
  质量够用，成本可控。"

Step 3 · 治理 3 道关（30s · 选讲）
  "成本治理还有 3 道关：per-tier budget（每个 tier 单独预算）+ cost cap
  （单章 $X 熔断，超了降级到 cheap tier）+ budget calibrate（基于历史
  batch 自动校准，比如 DoD D 3 章 ~$0.19 反推 ~$0.063/章）。"

================================================================================
被追问时
================================================================================

Q · 为什么不直接用 GPT-4？
A · 成本差 5×。MiniMax 在中文网文场景质量够用 + 成本可控。如果用 GPT-4，
  单章 $0.3+，8 本样章 80 章就是 $24，没有商业可持续性。

Q · "真实跟踪"和"估算"差在哪？
A · 估算是按 token 数 × 单价算，但漏了 system prompt 重复消耗、retry
  成本、cache 命中率。真实跟踪是 Provider 返回 usage，**精确到每个
  prompt-completion pair**。8 本样章跑下来，估算和真实差 ~15%。

Q · cost cap 触发会怎样？
A · 单章 cost cap（如 $0.10）触发 → 自动降级到 cheap tier（MiniMax）
  → 重跑。如果 cheap tier 还超 → 标 P0 人工介入。生产可用性 > 单章质量。

Q · 你这套成本系统有什么不做的？
A · 不做：多租户分摊（单机没需求）、云账单对接（CFO 不关心细节）、
  实时流式计费（batch 模式不需要）。这些都是 SaaS 化才需要的，灵文
  Studio 模式不需要。

关键设计决策
================================================================================

  ┌────────────────────────────────────────────────────────────┐
  │ D1 · 真实跟踪 > 估算                                        │
  │   Provider 返回 usage，精确到每个 prompt-completion pair      │
  │                                                              │
  │ D2 · per-tier budget                                        │
  │   cheap / mid / premium 各自预算 + 熔断                      │
  │                                                              │
  │ D3 · cost cap + 降级                                        │
  │   超预算自动降级 cheap tier，生产不中断                      │
  │                                                              │
  │ D4 · budget calibrate                                       │
  │   基于历史 batch 自动校准（如 DoD D 反推 ~$0.063/章）        │
  │                                                              │
  │ D5 · gitignored DB                                          │
  │   本地开发零运维，CI 自动 init                                │
  └────────────────────────────────────────────────────────────┘
