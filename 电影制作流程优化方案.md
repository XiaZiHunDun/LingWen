# 小说工厂优化方案

> ⚠️ **已废弃** - 本文档已归档，最新方案参见 `novel-factory/CLAUDE.md`
>
> 基于电影制作流程方法论 × 小说工厂现状分析

---

## 一、现状对照

| 电影制作阶段 | 小说工厂对应 | 当前差距 |
|------------|------------|---------|
| 开发/立项 | 灵感库 → 全文总体大纲 | ✅ 有对应，但深度不足 |
| 前期筹备 | 内容仓库(大纲层) | ❌ 缺"分场大纲"，缺场景分解 |
| 拍摄制作 | 作家写作正文 | ❌ 缺每日创作任务单/通告单 |
| 后期制作 | 审核 → 修改循环 | ✅ 有迭代，但缺"锁定"机制 |
| 宣发收尾 | 汇总 → 发布 | ✅ 有汇总，缺复盘机制 |

---

## 二、核心问题

1. **大纲与正文的"交接"缺乏锁定机制** — 电影是"定剪后再做声音特效"，小说大纲反复修改导致正文大量返工
2. **创作过程缺乏类似"每日Dailies"的进度审计** — 无法及时发现创作偏离
3. **缺少"场景级"分解** — 阶段大纲直接跳到章节正文，中间缺一层
4. **无连续性校验机制** — 人设/道具/场景容易前后矛盾
5. **项目完成无复盘** — 每部小说的经验无法沉淀到下一部

---

## 三、优化方案（6条）

### 优化1：引入「场景大纲」中间层

**来源**：电影"分镜头脚本"方法

**做法**：
- 在「阶段大纲」和「章节正文」之间，增加「场景大纲」层
- 每个场景 = 一个核心事件/情节点，200-500字
- 场景大纲通过后再展开写章节正文

**目录结构变为**：
```
内容仓库/
├── 01_全文总体大纲/
├── 02_卷大纲/
├── 03_阶段大纲/
├── 04_场景大纲/        # 新增
└── 05_章节正文/        # 原04
```

**触发条件**：阶段大纲审核通过后，自动生成场景大纲

---

### 优化2：引入「大纲锁定 + 变更审批」机制

**来源**：电影的"Locked Script" + 制片人审批制度

**做法**：
- 大纲每层级审核通过后，标记为「已锁定v1.0」
- 锁定后正文创作必须基于锁定版大纲
- 如需修改大纲 → 必须走「变更申请 → 审核员审批 → 版本更新」流程
- 每个变更记录理由，防止"边写边改"无限循环

**版本命名**：
```
全文大纲_v1.0_已锁定.md
全文大纲_v1.1_变更申请通过.md
```

---

### 优化3：引入「每日创作通告单」机制

**来源**：电影的「通告单(Call Sheet)」+ Dailies审片

**做法**：
- 每日生成「创作通告单」：
  - 今日目标：写哪几个场景/章节
  - 截止时间
  - 参考素材：需要的人物设定、已锁定的上文
- 每日收工前自动生成「Dailies报告」：
  - 今日完成字数
  - 遇到的问题
  - 明日计划
- 调度Agent每日汇总Dailies，发现偏离及时预警

---

### 优化4：建立「连续性检查」校验体系

**来源**：电影的「连续性(Continuity)」管理

**做法**：
- 新增「设定库」，包含：
  - `人物外观档案/` — 身高、穿着、表情习惯
  - `场景道具档案/` — 每个场景的物品清单
  - `世界观规则档案/` — 能力上限、时间线、地理设定
- 每章正文提交前，自动校验：
  - 本章人物外貌描写是否与档案矛盾
  - 场景道具是否与已建立场景矛盾
  - 能力设定是否前后一致
- 审核员重点关注「接戏」问题（前后章节逻辑连贯）

---

### 优化5：强化「意见仓库」版本化管理

**来源**：电影的「双备份原则」

**做法**：
- 每次修改生成「版本快照」，结构如下：

```markdown
# 版本记录

| 版本 | 日期 | 修改人 | 审核意见摘要 | 修改内容 |
|------|------|--------|-------------|---------|
| v1.0 | 05-19 | 作家A | 节奏偏慢 | 删减第3段 |
| v1.1 | 05-20 | 作家A | 审核员B | 补充转折 | 新增高潮 |

# v10 → v1.1 Diff
- 删除: "他慢慢走向门口..."
- 新增: "他猛地推开门..."
```

- 意见仓库按「审核类型 × 版本号」归档
- 任何回溯可一键还原到任意版本

---

### 优化6：增加「项目复盘」机制

**来源**：电影的「结算与复盘」

**做法**：
- 每部小说完成后，汇总部门输出「项目复盘报告」：

```markdown
# 《小说名》项目复盘

## 数据统计
- 总字数：
- 创作周期：
- 审核迭代次数：
- 核心修改点：

## 问题清单
- 大纲阶段暴露的问题：
- 正文阶段暴露的问题：
- 审核标准不明确的地方：

## 经验沉淀
- 有效的创作习惯：
- 踩过的坑：
- 下一部改进方向：
```

- 复盘报告存入「经验库」，供下一个项目启动前参考

## 五、落地优先级建议

```
P0（立即落地）:
├─ 优化2「大纲锁定机制」— 解决大纲无限返工的根源问题
└─ 优化4「连续性检查」— 提升小说质量底线

P1（下一版本）:
├─ 优化1「场景大纲中间层」— 让正文创作更有序
└─ 优化5「版本化管理」— 积累可追溯的修改历史

P2（后续迭代）:
├─ 优化3「每日通告单」— 需要调度Agent成熟后再引入
└─ 优化6「项目复盘」— 项目跑通后再总结
```

## 六、总结

电影工业百年沉淀的核心方法论：**前置决策到位 + 过程精细管控 + 锁定后不再反复 + 事后完整复盘**。

小说工厂当前架构已具备工业化的骨架，但缺少两个关键锚点：
1. **锁定机制**（锚定创作基准，防止无限返工）
2. **连续性校验**（锚定质量底线，防止前后矛盾）

建议优先落地这两条，其他作为渐进增强。
