# 二、先补全你断掉的工作流（从18步开始完整收尾）
```
18. 审核部门对最终定稿的各章节正文进行合规/人设/剧情细节终审，输出正文审核意见
19. 作家部门根据正文审核意见微调章节正文
20. 循环18-19，直至所有章节正文审核全部通过
21. 汇总部门拉取内容仓库中已通过审核的所有章节正文
22. 按层级依次生成：阶段汇总 → 卷汇总 → 全文汇总定稿
23. 汇总部门输出成品至汇总仓库，归档锁定版本
24. 意见仓库统一归档本次项目所有：灵感记录、各级审核意见、读者评论、修改日志
25. 全流程闭环：可开启下一部小说新项目，或基于现有作品开启续作/衍生创作流程
```

# 五、补充你原流程缺失的关键规则
1. **迭代阈值规则**
- 大纲类：必须**全量审核员一致通过**才进入下一层级
- 正文读者迭代：你设定剩余意见＜5条停止，可加一条：**核心读者无重大剧情吐槽**即定稿

2. **版本管理**
每一次修改自动生成版本MD，保留：原内容→修改意见→修改后内容，存入意见仓库，方便Claude Code回溯复盘。

3. **SubAgent调度规则**
- 同一层级只能**一个作家Agent主导创作**
- 审核/读者Agent并行批量输出意见，统一汇总后再给作家修改

4. **权限隔离**
审核员、读者只能**读取内容**，不能直接修改；只有作家Agent拥有编辑正文和大纲权限。
