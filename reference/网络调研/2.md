## 一、核心开源项目参考

### 1. 51mazi（我要码字）— 功能最全面的桌面端
- **技术栈**：Electron + Vue 3 + Vite + TipTap富文本编辑器 + Pinia状态管理
- **核心模块**：
  - **多书籍管理系统**：书架密码+书籍密码双重保护
  - **AI辅助创作**：DeepSeek（起名/润色/续写）+ 通义万相（封面/人物图/场景图）
  - **可视化工具**：Canvas地图设计、关系图谱（relation-graph-vue3）、事序图、组织架构图
  - **人物档案**：卡片/表格双视图、AI生成竖版全身人物图、多张人物图管理
  - **词条字典**：树形结构、拖拽排序、与人物档案标签关联
  - **编辑器增强**：人物高亮、禁词提示、段落拖拽、自动保存
- **GitHub**：https://github.com/xiaoshengxianjun/51mazi 

### 2. 91Writing — 纯前端AI创作工具
- **技术栈**：Vue 3 + Element Plus + Pinia + Vite + TypeScript
- **核心模块**：
  - **智能创作引擎**：支持GPT/Claude/Gemini/DeepSeek等多模型切换
  - **世界观构建系统**：复杂世界观模板化管理、AI辅助世界设定生成
  - **数据管理中心**：本地化存储、分类导入导出、完整备份恢复
  - **写作目标系统**：每日/每周/每月字数目标、进度跟踪、成就系统
  - **智能工具库**：细纲生成器、角色生成器、脑洞生成器、爆款书名生成器等10大工具
  - **拆书分析模块**：分析爆款小说结构
- **GitHub**：https://github.com/ponysb/91Writing 

### 3. novelWriter — 结构化纯文本编辑器
- **技术栈**：Python 3 + Qt 5/6
- **核心设计**：
  - **纯文本优先**：核心内容存储为UTF-8编码的`.nwd`文件，确保长期可读性
  - **元数据分离**：项目结构存储在XML格式的`.nwx`文件中，便于版本控制
  - **模块化导出**：导出功能独立为插件系统，支持多种输出格式
  - **树状项目管理**：小说主线（分部>章节>场景）+ 辅助素材 + 废弃内容
  - **标记系统**：`@pov:`视角、`@char:`角色、`@location:`地点等元数据标记
  - **统计面板**：实时字数、写作时长、每日目标、章节字数分布
- **GitHub**：https://github.com/vkbo/novelWriter 

### 4. NovelForge — AI辅助长篇小说创作（新项目）
- **技术栈**：FastAPI + SQLAlchemy + React 19 + TypeScript + Tailwind CSS
- **核心特性**：
  - **Schema驱动的卡片创作**：每种卡片定义结构，AI生成按结构校验
  - **指令流式AI卡片生成**：字段粒度流式填充，更可控
  - **上下文注入+知识图谱一致性**：通过`@DSL`精准引用项目数据
  - **灵感助手（Agent）**：可持续对话、引用卡片、调用工具修改内容
  - **通用审核流程**：草稿预览→确认保存为审核结果卡片
- **GitHub**：https://github.com/RhythmicWave/NovelForge 

### 5. AI Book Generator — 工程化书籍生成流水线
- **技术栈**：纯前端（HTML/CSS/JS），零后端依赖
- **核心设计**：
  - **三阶段流水线**：概念生成→大纲生成→章节撰写
  - **结构化提示链**：每章生成时注入"本书概念+全书大纲+前几章内容+作者风格指南"
  - **多模型策略**：概念/大纲用Claude 3.5 Sonnet，章节撰写用GPT-4o/GPT-4o mini
  - **实时流式输出**：支持Live Streaming体验
- **GitHub**：https://github.com/SimonWaldherr/AI-Book-Generator

CREATE TABLE Project (
    ProjectID INT PRIMARY KEY,
    Title VARCHAR(255),
    AuthorID INT,
    GenreID INT,
    Status ENUM('draft','writing','completed'),
    CreatedAt TIMESTAMP,
    WordCountGoal INT

    ChapterID INT PRIMARY KEY,

    Content TEXT,

    WordCount INT,

    FromID INT,
    ToID INT,
    RelationType VARCHAR(50),
    Description TEXT,
    PRIMARY KEY (FromID, ToID)
);

## 四、AI长文本一致性解决方案

这是小说工作室的核心技术难点，关键方案： 

### 1. 外部记忆管理系统（RAG+向量数据库）
- **知识库构建**：将世界观、人物小传、前文摘要等向量化存储
- **动态检索**：生成新章节时，检索相关背景信息注入上下文
- **实测效果**：5000字段落连贯性保持率可达90%以上

### 2. 分块生成策略
- **层级拆解**：全书→卷→章→节→段落，逐层递进生成
- **记忆更新**：每章生成后自动提取关键事件更新记忆库
- **上下文注入**：生成时携带"全书概念+大纲+前几章摘要+风格指南"

### 3. 智能压缩技术
- **语义骨架提取**：将历史对话分割成语义块，重要性评分后保留高分块完整内容，低分块转化为摘要
- **短期记忆**：当前会话的完整上下文（4K-128K tokens）
- **长期记忆**：用户偏好、历史模式、项目知识库（向量化压缩）

### 4. 多Agent协作模式（参考NovelForge）
- **Story Planner**：负责大纲和设定生成
- **Node Writer**：负责具体章节内容撰写
- **Story Reviewer**：负责一致性检查和审核
- **灵感助手Agent**：持续对话、引用卡片、调用工具修改内容

## 七、优化建议清单

### 立即可参考的优化点

1. **数据安全**：采用novelWriter的"纯文本+元数据分离"方案，核心内容存为UTF-8文本，项目结构存为JSON/XML，便于版本控制和长期兼容

2. **AI工作流**：参考AI Book Generator的三阶段流水线（概念→大纲→章节），确保长文本一致性

3. **提示词系统**：建立分类模板库（大纲/正文/润色/对话/场景），支持变量替换和版本管理

4. **可视化增强**：集成关系图谱（人物关系）、事序图（时间线）、地图设计（世界观）

5. **上下文管理**：构建分层记忆架构（会话级/项目级/知识库级），解决长文本连贯性问题

6. **多模型策略**：支持不同环节调用不同模型（大纲用Claude，续写用GPT，降低成本）

7. **Schema驱动**：参考NovelForge的卡片Schema设计，让AI生成按结构校验，减少混乱输出

8. **导出系统**：模块化导出插件，支持HTML/Markdown/PDF/ePub等多种格式

9. **协作基础**：集成Yjs实现实时协同编辑，为后续多人协作打基础

10. **成就系统**：参考91Writing的目标管理，增加写作动力和数据追踪
