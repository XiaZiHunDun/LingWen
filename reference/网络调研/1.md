## 二、核心模块（逐个集成即可）
### 1）世界观/设定模块（卡片化）
- 世界设定：地理、势力、力量体系、等级、物品、规则
- 角色卡：姓名、年龄、性格、外貌、背景、技能、关系、状态快照
- 关系图谱：角色-角色、角色-物品、角色-地点、势力关系
- 动态输出模型（Pydantic）：强制结构化输出，避免乱写

开源参考：
- NovelForge 卡片编辑器（可直接复刻UI）
- ai-novel-lab AGENTS.md 约束系统

### 2）大纲与剧情生成模块（工业级SOP）
- 选题→粗纲→卷纲→章纲→场景→正文（六步闭环）
- 热点分析：爬番茄/七猫热榜，生成“题材+元素”组合（如“重生+商战+玄学”）
- 三幕结构：铺垫（10%）→冲突（80%）→解决（10%），每章埋2高潮+5伏笔
- 字数控制：每章4000–6000字，不足自动重写，超出智能压缩

提示词模板（直接用）：
```
你是10年网文编辑，请生成[题材]100万字大纲：
- 目标读者：18–35岁男/女
- 风格：爽文/甜宠/悬疑
- 结构：三幕，主线+3支线
- 每卷核心冲突+高潮
```

### 3）长期记忆模块（解决“吃书”，必做）
技术组合：**向量库+事实库+增量更新**
- 向量库（Chroma/Pinecone/FAISS）：
  - 每章生成摘要向量
  - 写作时检索相关前文（角色、地点、伏笔）
  - 注入提示词，保证上下文连贯
- 事实库（JSON/ SQLite）：
  - 角色状态、物品归属、时间线、伏笔清单
  - 每次生成后增量更新（JSON Delta）
- 记忆压缩：长文档分块、摘要化、去重，控制token消耗

开源参考：
- ai-novelgenerator 向量存储+知识管理
- ai-novel-workspace 章节事实摘要+角色快照

### 4）一致性审计模块（质量护城河）
检查维度（33项，核心6项）：
1. 角色记忆连续性（没发生过的事不能回忆）
2. 物品/能力连续性（已损毁不能再用）
3. 时间线合理性（时间流逝、事件顺序）
4. 人设稳定性（性格、行为动机不OOC）
5. 伏笔回收（第三章伏笔后续要呼应）
6. 大纲偏离度（是否脱离卷纲核心）

实现方式：独立Agent/脚本，输出报告+自动修订建议。

### 5）文风统一与润色模块（提升质感）
- 风格提取：输入10–20章样文，提取文风特征（句式、词汇、节奏）
- 风格强制：每次生成时注入文风指南，AI严格遵循
- 去AI痕迹：避免句式重复、过度总结、书面化过重
- 对话自然化：符合角色身份、性格、时代背景

### 6）项目管理与协作模块（工作室必备）
- 多项目管理：每个小说独立空间（设定、大纲、章节、资产）
- 版本控制：章节历史、对比、回滚
- 权限管理：作者/编辑/审核/读者角色
- 协作流程：提交→审核→修改→发布

## 四、关键技术方案（直接抄实现思路）
### 1）长文本上下文突破
- 分层处理：世界观→大纲→章节→场景，逐层细化
- 滑动窗口：每次生成只注入相关上下文（1–3章）
- 摘要链：每章生成100字摘要，构建“摘要→章节→细节”三级检索

### 2）成本优化（工作室盈利关键）
- 模型分级：
  - 大纲/设定：DeepSeek（便宜）
  - 正文：GPT-4o/ Claude（质量）
  - 润色/审查：本地小模型（如Llama 3）
- Prompt优化：精简指令、复用模板、减少冗余描述
- 缓存机制：相同设定/文风模板缓存，避免重复计算

### 3）商业化模式（参考头部平台）
1. **订阅制**：基础免费（每日3章）、专业版39元/月（无限生成+高级设定）
2. **IP孵化**：数据驱动选热点→AI批量生产→人工精修→平台签约/自有渠道分发
3. **定制服务**：为作者提供人设定制、大纲定制、文风训练服务
4. **内容外包**：对接出版社/短剧公司，批量输出小说/剧本

### 4）数据闭环（核心壁垒）
- 爬虫：监控番茄、七猫、起点热榜，每日更新热门题材/元素
- 分析：统计爆款小说的节奏、人设、爽点分布
- 反哺：AI生成时优先采用高热度元素，提升内容市场竞争力

## 五、可直接用的代码片段（Python）
### 1）记忆检索（向量库示例，Chroma）
```python
import chromadb
from sentence_transformers import SentenceTransformer

# 初始化
client = chromadb.PersistentClient("./memory_db")
collection = client.get_or_create_collection("novel_memory")
model = SentenceTransformer("all-MiniLM-L6-v2")

# 存入章节
def add_chapter(chapter_id, content):
    embedding = model.encode(content).tolist()
    collection.add(ids=[chapter_id], embeddings=[embedding], documents=[content])

# 检索相关上下文
def retrieve_context(query, top_k=3):
    embedding = model.encode(query).tolist()
    results = collection.query(query_embeddings=[embedding], n_results=top_k)
    return results["documents"][0]
```

### 2）一致性检查（人设简单校验）
```python
def check_character_consistency(character_profile, chapter_text):
    errors = []
    # 性格一致性
    if character_profile["性格"] == "冷静" and "暴怒" in chapter_text:
        errors.append("人设冲突：冷静性格出现暴怒行为")
    # 物品一致性
    if "上古神剑" in chapter_text and character_profile["物品"]["上古神剑"] == "已损毁":
        errors.append("物品冲突：已损毁的神剑再次出现")
    return errors
```
