### 可复用的模式

```
## 模式1：编号锁定机制

**适用场景**：多章节并行写作
**核心要点**：
1. 阶段开始前，锁定章节编号范围
2. 建立编号分配表（作家A: ch001-036, 作家B: ch037-072...）
3. 变更编号需提交申请，经主编审批
4. 写作完成后，核对编号与内容匹配

**模板参考**：`章节分配表_模板.md`
```

```
## 模式2：人物状态快照

**适用场景**：多视角/长线人物管理
**核心要点**：
1. 每个阶段开始前，生成人物状态快照
2. 快照包含：形态、能力、位置、重要道具
3. 章节写作时，核对当前状态与快照
4. 状态变更需在章节末标注

**模板参考**：`人物状态快照_模板.md`
```

```
## 模式3：伏笔追踪表

**适用场景**：长线伏笔管理
**核心要点**：
1. 大纲阶段建立伏笔清单（VP_001等）
2. 标注：伏笔类型、埋设章节、预期回收章节
3. 写作时，伏笔章节与回收章节需对应
4. 审核时，检查伏笔是否在约定章节回收

**模板参考**：`伏笔追踪表_模板.md`
```

### 流程优化建议

```
### 优化点1：编号锁定前置

**现状问题**：章节编号在写作过程中随意变更，导致大量P0问题
**优化方案**：
1. 阶段大纲审核通过后，立即锁定章节编号范围
2. 建立编号分配表，明确每个作家的编号范围
3. 编号变更需提交变更申请，经主编审批
4. 写作完成后，系统自动核对编号与内容匹配

**预期收益**：预计减少80%的编号相关P0问题
```

```
### 优化点2：人物状态快照自动化

**现状问题**：小九形态等人物状态在360章中出现不一致
**优化方案**：
1. 开发人物状态快照生成工具
2. 每个阶段开始前，自动提取关键人物状态
3. 作家写作时，系统提示当前人物状态
4. 审核时，自动对比章节内人物状态与快照

**预期收益**：预计减少90%的人物状态不一致问题
```
