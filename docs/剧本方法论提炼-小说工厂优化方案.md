## 二、小说工厂现状分析

### 2.1 已有优势

- **创作-评审闭环**：已有完整流程（创作→评审→修改→验证）
- **部门架构清晰**：灵感/作家/审核/读者/汇总五部门分工明确
- **交接标准完善**：`小说工厂部门对接标准.md` 定义了各环节交接检查清单
- **分层审核机制**：总体大纲→卷大纲→阶段大纲→正文，逐层细化

### 2.2 待优化缺口

| 缺口 | 现状 | 剧本可借鉴 |
|------|------|-----------|
| 创作阶段缺乏明确节点模型 | 只有评审维度，无创作流程定义 | 救猫咪节拍表 |
| 人物设计深度不足 | 仅"核心人物"字段，无欲望/缺陷/弧光 | 人物小传深度模型 |
| 大纲节点缺少标准节拍 | 三幕式概念有，无具体节拍位置 | 15节点节拍表 |
| 修改流程缺分层指导 | 一次性修改，缺结构性/细节性区分 | 分层修改法 |
| 场景写作缺微观框架 | 笼统要求，无场景=微型冲突概念 | 场景微观结构 |
| 反馈收集缺标准问题集 | 读者反馈随意，无结构化三问 | 目标读者反馈法 |

## 三、优化方案

### 3.1 新增创作阶段节点模型

**整合剧本救猫咪节拍表 + 小说写小说流程**

```
创作五阶段十五节点：
├── 阶段一：构思期（10%）
│   ├─ 节点1：捕捉核心冲动（一句话锚定）
│   ├─ 节点2：确定类型与读者契约
│   └─ 节点3：提炼一句话梗概 [借鉴剧本Logline公式]
│
├── 阶段二：规划期（20%）
│   ├─ 节点4：人物设定 [增强：欲望/缺陷/秘密/弧光]
│   ├─ 节点5：世界观与规则
│   └─ 节点6：结构设计 [新增：15节点节拍表]
│
├── 阶段三：写作期（40%）
│   ├─ 节点7：初稿策略（禁止回改/从兴奋场景开始）
│   ├─ 节点8：场景微观结构 [新增：目标/阻碍/转折/余波]
│   └─ 节点9：节奏控制（快慢交替/张弛有度）
│
├── 阶段四：修改期（25%）
│   ├─ 节点10：冷却期（至少1-2周）[新增]
│   ├─ 节点11：分层修改 [增强：宏观→中观→微观]
│   └─ 节点12：外部反馈 [增强：目标读者三问]
│
└── 阶段五：完成期（5%）
    ├─ 节点13：标题与开篇优化
    ├─ 节点14：发布渠道适配
    └─ 节点15：持续迭代
```

### 3.2 新增人物小传深度模型

**来源：剧本写作方法论的人物设计**

```
人物档案字段：
├── 外在欲望：主角表面上想要什么（赢得比赛、拯救世界）
├── 内在需求：主角真正需要什么（被认可、战胜阴影、学会爱）
├── 核心缺陷：阻碍他获得成长的性格弱点
├── 前史秘密：他为什么会变成今天这样
├── 人物弧光：故事开始时→结束时发生了什么变化
└── 声纹特征：说话风格关键词（遮名也能辨认）
```

### 3.3 新增十五节点节拍表

**来源：救猫咪节拍表（Save the Cat）**

| 节拍 | 位置 | 说明 |
|------|------|------|
| 1.开场画面 | 0-1% | 展现主角"日常"世界 |
| 2.主题呈现 | 1-5% | 隐晦点题（对话或场景） |
| 3.铺垫 | 5-10% | 介绍主角、缺陷、缺失 |
| 4.催化剂 | 10% | 突发事件打破日常 |
| 5.第二幕衔接点 | 12% | 主角做出不可撤销决定 |
| 6.副线（B故事） | 15-20% | 爱情/友情线，承载主题 |
| 7.游戏时间 | 20-25% | 展示"前提承诺"，冲突升级 |
| 8.中点 | 50% | 重大转折（伪胜利/假失败） |
| 9.反派逼近 | 55-60% | 内外压力同时收紧 |
| 10.一无所有 | 75% | 灵魂黑夜，彻底绝望 |
| 11.第三幕衔接点 | 75-80% | 找到最后希望/新方案 |
| 12.高潮 | 85-90% | 终极对决，人物弧光完成 |
| 13.结局画面 | 90-100% | 展现"新"世界，与开场对比 |
| 14.最后阴影 | 最后 | 低概率威胁残留 |
| 15.终场画面 | 结束 | 与开场呼应，功能性收尾 |

### 3.4 新增场景微观结构

**来源：剧本场景写作方法**

```
每个场景 = 微型冲突单元：
├── 目标：角色进入场景想要什么
├── 阻碍：谁/什么阻止了他
├── 灾难/转折：结果如何（部分胜利但失去更多 / 彻底失败）
└── 余波：情绪/关系发生了什么变化
```

**场景写作法则：晚进早出**
- 不写角色进门、打招呼、倒水等废话
- 直接切入冲突最激烈的那一刻
- 冲突解决或信息传递完，立刻切走

### 3.5 新增分层修改流程

**来源：剧本修改方法论**

```
修改分四层，每层聚焦一类问题：

第一层：宏观结构（动大手术）
├─ 节奏是否太慢？高潮够不够有力？
├─ 有无无关情节？（删掉不影响主线就删）
├─ 第二幕是否塌陷？（最常见结构问题）
└─ 主角是否主动推动剧情？

第二层：中观节奏（动中手术）
├─ 每章结尾是否有"翻页冲动"？
├─ 信息释放节奏是否合理？
└─ 删除冗余场景

第三层：微观语言（动小手术）
├─ 剔除副词（"愤怒地说"→用动作表现）
├─ 精简形容词，多用名词和动词
├─ 检查视角一致性
└─ 大声朗读，标记拗口处

第四层：格式校队（职业素养）
├─ 错别字、标点、格式
└─ 人名、地名、专有名词统一
```

### 3.6 新增目标读者反馈法

**来源：剧本外部反馈方法**

```
反馈收集标准问题集：
1. 哪里让你感到无聊/想跳过？
2. 哪个角色你最喜欢/最不喜欢？为什么？
3. 结尾让你感到满足、意外，还是困惑？

反馈处理原则：
- 3人以上同一问题 → P1级问题
- 个人偏好差异 → 记录为"读者类型差异"
- 意见 vs 问题：观众说"这里无聊"是问题，
  但他们提议的修改方案未必是最好的解决方案
```

### 3.7 新增"评审前置检查"环节

**来源：小说工厂实际痛点**

```
评审前置检查清单：
□ 主角目标是否清晰
□ 核心冲突是否存在
□ 十五节点节拍是否完整
□ 每章是否有明确的目标/阻碍/结果
□ 伏笔是否已记录
□ Hard约束自检（叙事视角唯一性等）
□ 场景微观结构检查（目标/阻碍/转折/余波）

通过后进入评审流程，否则打回创作阶段
```
