## 一、电影制作核心原则提炼

### 1.1 五阶段流程

```
开发(Development) → 前期筹备(Pre-production) → 拍摄制作(Production) → 后期制作(Post-production) → 宣发(Distribution)
```

**与小说类比**：
- 开发 = 构思+规划（节点1-6）
- 前期筹备 = 细纲+模板准备（写作前）
- 拍摄制作 = 写作执行（节点7-9）
- 后期制作 = 修改+评审（节点10-12 + 评审阶段）
- 宣发 = 定稿+发布（节点13-15）

### 1.2 电影制作关键原则

| 原则 | 说明 | 可迁移到小说 |
|------|------|--------------|
| **前置决策** | 后期70%的问题源于前期准备不足 | 大纲阶段决定主线走向 |
| **锁定机制** | 剧本定稿后进入拍摄，变更要走正式流程 | 大纲锁定后进入初稿 |
| **每日审片** | 导演/DP每日回看前日素材，及时发现问题 | 日更review机制 |
| **双备份原则** | 素材现场双备份，避免拍摄事故 | 创作版本备份 |
| **锁门后再做声音** | 剪辑未定稿前，声音/特效投入可能白费 | 修改完成后再润色 |
| **制片人是刹车片** | 导演管艺术，制片人管预算和进度，二者制衡 | 创作vs评审相互制约 |
| **留足应急预算** | 10-15%不可预见费是行业铁律 | 预留修改buffer |

## 四、核心新增：创作阶段锁定规范

### 4.1 三层锁定机制

```
第一锁：大纲锁定（进入初稿前）
├─ 条件：三幕式结构完整、人物设定清晰、核心冲突明确
├─ 变更：需走正式变更流程，评估影响范围
└─ 文档：锁定版本记录

第二锁：初稿锁定（进入修改前）
├─ 条件：全文完成、章节目标达成、伏笔已记录
├─ 变更：仅限结构性问题，不允许大改走向
└─ 文档：锁定版本记录

第三锁：修改锁定（进入润色前）
├─ 条件：S1-S2达标、逻辑漏洞修复、人物动机合理
├─ 变更：仅限字词级修改
└─ 文档：锁定版本记录
```

### 4.2 变更管理流程

```
发现问题 → 评估影响 → 判断是否触发变更 → 实施修改 → 更新版本记录
```

**小改**：直接修改，无需记录
**中改**：标注修改，批量更新
**大改**：触发锁定变更，需重新评估

## 五、核心新增：每日创作Review规范

### 5.1 每日自检清单

```
每日写作后自检：
□ 本章目标是否达成
□ 是否有新问题需要记录
□ 章节是否留有钩子
□ 字数是否符合当日目标
□ 是否有逻辑漏洞需要后续处理
```

### 5.2 每日复盘模板

```markdown
## 每日创作复盘

### 日期：YYYY-MM-DD

**字数**：____ / 目标 ____
**状态**：最佳/正常/低效/疲劳

**完成内容**：
-

**自检发现**：
-

**明日重点**：
-
```

### 5.3 周审机制

```
每周结束前：
1. 回顾本周完成内容
2. 检查周目标达成率
3. 识别连续性问题
4. 调整下周计划
```

## 六、核心新增：版本管理规范

### 6.1 版本命名规则

```
v[大版本].[小版本].[修订次数]

示例：
- v1.0.0：大纲版本
- v1.1.0：初稿进行中
- v1.1.3：初稿第3次修订
- v2.0.0：修改版（大纲锁定后）
- v2.1.0：修改版进行中
- v3.0.0：定稿版本
```

### 6.2 版本记录

```markdown
## 版本记录

| 版本 | 日期 | 说明 | 变更内容 |
|------|------|------|----------|
| v1.0.0 | YYYY-MM-DD | 大纲锁定 | 初始大纲 |
| v2.0.0 | YYYY-MM-DD | 初稿完成 | 进入修改 |
```

### 6.3 备份策略

```
每日创作后：
- 保存当前版本
- 记录当日修改摘要
- 标记需要后续处理的问题

不覆盖历史版本，每次创作后新建版本
```

## 七、关键优化点

### 7.1 解决"修改失控"问题

当前方法论缺少"什么时候该停"的明确标准。

**新增**：三层锁定机制 + 变更管理
- 大纲锁定：进入初稿的门槛
- 初稿锁定：进入修改的门槛
- 修改锁定：进入润色的门槛

### 7.2 解决"质量不稳定"问题

当前方法论缺少每日自检机制。

**新增**：每日创作Review规范
- 每日自检清单
- 每日复盘模板
- 周审机制

### 7.3 解决"版本混乱"问题

当前方法论缺少版本管理意识。

**新增**：版本管理规范
- 版本命名规则
- 版本记录
- 备份策略
