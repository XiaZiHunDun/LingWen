## 三、解决方案

### 方案A：迁移旧脚本到独立目录（推荐）

**思路**: 将 `infra/tools/consistency/` 迁移到项目根目录的 `scripts/` 或 `legacy/` 目录，与新模块彻底分离。

```
# 迁移后的目录结构
scripts/
├── consistency/                   # 旧版一致性检查脚本
│   ├── auto_consistency_checker.py
│   ├── check_battle_density.py
│   ├── ... (其他脚本)
│   └── run_consistency_check.sh
└── ... (其他脚本)

novel-factory/                    # 核心代码（保持干净）
└── infra/
    ├── consistency/              # 新版模块化检测器
    │   ├── checkers/
    │   ├── detectors/
    │   ├── engine/
    │   └── ...
    └── tools/                    # 工具（不包含 consistency 子目录）
```

**优点**:
- 新旧代码完全分离
- 不影响现有导入路径
- 目录结构更清晰

**缺点**:
- 需要修改脚本中的相对导入路径
- 需要更新调用这些脚本的入口点

---

### 方案B：重命名旧目录

**思路**: 将 `infra/tools/consistency/` 重命名为 `infra/tools/consistency_legacy/`

```
# 重命名后的目录结构
infra/tools/consistency_legacy/    # 旧版（重命名后）
├── auto_consistency_checker.py
├── ...
└── run_consistency_check.sh

infra/consistency/                 # 新版（保持不变）
├── checkers/
├── detectors/
├── engine/
└── ...
```

**优点**:
- 改动最小
- 不需要修改脚本内容

**缺点**:
- 目录名不够直观
- `legacy` 标记可能不够明显

---

### 方案C：删除旧目录

**思路**: 如果旧脚本不再使用，直接删除 `infra/tools/consistency/`

**前提条件**:
- 确认 `run_consistency_check.sh` 和其他脚本没有被其他地方调用
- 确认功能已被新版模块覆盖

**优点**:
- 最简单直接
- 消除重复

**缺点**:
- 如果旧脚本仍被使用，会导致功能缺失
- 无法回退

---

## 四、推荐方案

**推荐方案A：迁移旧脚本到独立目录**

理由：
1. 保留旧脚本的可用性（如果将来需要参考）
2. 新旧代码完全分离，结构清晰
3. 不影响现有的模块导入
4. 为未来清理 `infra/tools/` 目录做准备

---

## 五、实施步骤（方案A）

### Step 1: 创建新目录

```bash
mkdir -p scripts/consistency
```

### Step 2: 迁移文件

```bash
# 移动旧脚本（排除文档和配置）
mv infra/tools/consistency/*.py scripts/consistency/
mv infra/tools/consistency/*.sh scripts/consistency/
mv infra/tools/consistency/*.yaml scripts/consistency/

# 保留文档（可选）
mv infra/tools/consistency/*.md scripts/consistency/
```

### Step 3: 更新脚本导入路径

如果脚本中有相对导入，需要调整。例如：
```python
# 旧: from .check_battle_density import ...
# 新: from check_battle_density import ...
```

### Step 4: 删除旧目录

```bash
rm -rf infra/tools/consistency
```

### Step 5: 验证

```bash
# 运行 mypy
python -m mypy infra/consistency/ --ignore-missing-imports

# 运行测试
python -m pytest tests/ -q
```
