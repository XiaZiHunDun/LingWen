## 二、合并流程规范

### 使用工具

```bash
# 全文合并（生成MD+TXT）
python3 tools/merge_chapters.py full

# 卷合并
python3 tools/merge_chapters.py volume 卷1 1 140

# 阶段合并
python3 tools/merge_chapters.py stage 卷1_阶段1 1 20
```

或使用Bash脚本（会调用Python脚本）：

```bash
cd 07_汇总仓库
./run_summary.sh merge full
./run_summary.sh merge volume 卷1
./run_summary.sh merge stage 卷1_阶段1
```

### 合并规则

1. **跳过多余分隔行**: 章节文件结构中的分隔线+文件名+分隔线+空行跳过
2. **只保留标题+正文**: 提取第1行（标题）和第6行及以后（正文）
3. **自动生成双版本**: 同时生成`.md`和`.txt`两个版本
4. **版本号规范**: 使用`v9.10`格式，不再使用`v1.0`

### 章节文件格式

```
Line 1: ==================================================
Line 2: 第xxx.md              ← 跳过
Line 3: ==================================================  ← 跳过
Line 4: (空行)                ← 跳过
Line 5: # 章节标题           ← 保留（标题）
Line 6+: 正文内容             ← 保留（正文）
```

### 汇总输出格式

```
==================================================

# 其一章 废土黄昏（修复版）

正文内容...


==================================================

# 第二章 独自求生

正文内容...

...
```

## 三、文件命名规范

### 版本号格式
- `vX.Y` - 主版本.次版本
- 发布版本使用日期后缀: `{项目名}_v{X.Y}_{YYYYMMDD}/`

### 文件类型后缀
| 后缀 | 格式 | 用途 |
|------|------|------|
