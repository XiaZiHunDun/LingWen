# 工作流模式库

## 概述

本文档定义 9 种常见工作流模式，供各部门在执行任务时参考。

## 模式 1：线性创作流

**适用场景**: 新章节创作
**部门**: 作家部门

```
作家领取任务 → 创作 → 自评 → 提交审核
```

**特点**: 简单直接，适合单个章节

---

## 模式 2：并行审核流

**适用场景**: 批量审核
**部门**: 审核部门

```
任务入队 → 分配给 5 个审核员 → 并行审核 → 汇总意见
```

**特点**: 高效并行，适合大批量

---

## 模式 3：迭代修复流

**适用场景**: 审核发现问题后
**部门**: 作家部门 + 审核部门

```
审核发现问题 → 汇总意见 → 分配作家 → 并行修复 → 重新审核 → 通过/继续迭代
```

**特点**: 闭环反馈，确保质量

---

## 模式 4：分层汇总流

**适用场景**: 阶段/卷/全文汇总
**部门**: 汇总部门

```
收集所有章节 → 主笔整合 → 编辑审核 → 校验检查 → 定稿输出
```

**特点**: 三层把关，保证一致性

---

## 模式 5：跨卷协调流

**适用场景**: 跨卷问题处理
**部门**: 汇总部门 + 审核部门

```
发现问题 → 汇总部门协调 → 相关卷修改 → 重新汇总 → 验证通过
```

**特点**: 跨域协调，维护一致性

---

## 模式 6：紧急修复流

**适用场景**: P0 问题修复
**部门**: 主控 + 作家部门

```
发现 P0 问题 → 主控分配紧急任务 → 专家作家修复 → 快速审核 → 验证通过
```

**特点**: 优先处理，快速闭环

---

## 模式 7：读者反馈流

**适用场景**: 市场验证
**部门**: 读者部门

```
章节发布 → 20 位读者并发阅读 → 收集评论 → 分类整理 → 反馈给作家
```

**特点**: 大规模并行，真实反馈

---

## 模式 8：灵感生成流

**适用场景**: 新项目启动
**部门**: 灵感部门

```
需求确认 → 基础层生成 → 深度层生成 → 主编整合 → 入库
```

**特点**: 层层深入，结构化输出

---

## 模式 9：质量检查流

**适用场景**: 自动化质量检查
**部门**: 技术系统

```
定时触发 → 运行所有检查器 → 生成报告 → 分类问题 → 分配修复
```

**特点**: 自动化、标准化

---

## 模式选择指南

| 场景 | 推荐模式 |
|------|----------|
| 单章创作 | 模式 1 |
| 批量审核 | 模式 2 |
| 问题修复 | 模式 3 |
| 卷汇总 | 模式 4 |
| 跨卷协调 | 模式 5 |
| P0 修复 | 模式 6 |
| 市场反馈 | 模式 7 |
| 新项目 | 模式 8 |
| 日常检查 | 模式 9 |
