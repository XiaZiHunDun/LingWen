## 三、创作阶段核心模型（新增）

### 3.1 五阶段创作模型

```
阶段一：构思期（10%）
├─ 节点1：捕捉核心冲动（一句话锚定）
├─ 节点2：确定类型与读者契约
└─ 节点3：提炼一句话梗概

阶段二：规划期（20%）
├─ 节点4：人物设定（欲望/缺陷/秘密/弧光）
├─ 节点5：世界观与规则
└─ 节点6：结构设计（三幕式 + 节拍表）

阶段三：写作期（40%）
├─ 节点7：初稿策略（定额/禁止回改/从兴奋场景开始）
├─ 节点8：场景微观结构（目标/阻碍/转折/余波）
└─ 节点9：节奏控制（快慢交替/张弛有度）

阶段四：修改期（25%）
├─ 节点10：冷却期（至少1-2周不看）
├─ 节点11：分层修改（宏观结构→中观节奏→微观语言）
└─ 节点12：外部反馈（2-3位目标读者）

阶段五：完成期（5%）
├─ 节点13：标题与开篇优化
├─ 节点14：发布渠道适配
└─ 节点15：持续迭代（下一本书）
```

### 3.2 与评审体系的接口

```
创作阶段完成 → 评审前置检查 → 进入评审流程

评审前置检查清单：
□ 主角目标是否清晰
□ 核心冲突是否存在
□ 三幕式结构是否完整
□ 每章是否有明确的目标/阻碍/结果
□ 伏笔是否已记录
□ Hard约束自检（叙事主角唯一性等）
```

## 四、关键优化点

### 4.1 解决"评审前置"问题

当前小说方法论缺少**创作完成后的自检环节**，
导致评审阶段发现大量"本可在创作时避免"的问题。

**新增**：`评审前置检查.md`（operation/）
- 创作完成进入评审前，作者自检清单
- 对齐 Hard 约束和 S1-S7 维度
- 减少评审轮次，提高效率

### 4.2 解决"创作流程"缺失问题

当前小说方法论只有评审优化工具，
没有指导"怎么写"的方法。

**新增**：`创作流程体系.md`（core/）
- 整合5份文档的精华
- 定义清晰阶段和节点
- 与现有评审维度对齐

### 4.3 解决"工具与流程"脱节问题

当前工具（章节模板等）是独立的，
与创作阶段没有对应关系。

**增强**：
- 章节模板新增"适用阶段"标注
- 爽点公式新增"情绪曲线"映射
- 章末钩子新增"节点适配"说明

## 七、核心原则

1. **只增不改**：现有文档不修改，只新增内容
2. **创作-评审闭环**：创作流程的终点是评审流程的起点
3. **与Hard/S7对齐**：创作阶段的目标与评审标准一致
4. **可执行优先**：每个阶段有可操作的动作和检查点
