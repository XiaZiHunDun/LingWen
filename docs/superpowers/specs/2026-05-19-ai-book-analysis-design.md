## 一、背景与目标

### 1.1 现状问题

当前小说工厂在"学习优秀作品"方面的做法：
- 有 MDA分析模板 和人物弧光设计模板，但都是人工填写模板，不是自动化分析
- 有模板库（玄幻修真/都市职场/悬疑推理），但没有从优秀作品提取模板的机制
- 没有 AI 拆书功能——输入一本小说，无法自动解析叙事结构

### 1.2 优化目标

构建一个 AI 拆书系统：
- **输入**：优秀小说（txt/markdown格式）
- **分析维度**：结构分析（MDA框架）、人物弧光提取、可复用模板生成
- **输出**：完整分析报告 + 可复用模板（存入模板库，供新项目使用）

---

## 二、技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| 文本解析 | Python（正则 + 结构化提取） | 处理 txt/markdown 格式 |
| 结构分析 | LLM 调用（结构化输出） | MDA 框架需要语义理解 |
| 人物弧光提取 | LLM 调用（实体识别 + 弧光追踪） | 需要理解角色发展 |
| 模板生成 | 规则引擎 + LLM | 提取可复用的模板 |
| 部署环境 | Python + Docker | 与现有环境一致 |

## 四、分析维度

### 4.1 维度 1：结构分析

```yaml
structure_analysis:
  input: "优秀小说全文"

  output:
    # 1. 章节结构
    chapter_structure:
      total_chapters: 120
      avg_chapter_length: 3000
      chapter_density: "高/中/低"  # 每章情节点密度

    # 2. 节奏分布
    rhythm_distribution:
      - chapter_range: "ch1-20"
        rhythm_type: "铺设期"
        tension_level: 3/10
      - chapter_range: "ch21-50"
        rhythm_type: "上升期"
        tension_level: 6/10
      - chapter_range: "ch51-80"
        rhythm_type: "高潮期"
        tension_level: 9/10

    # 3. 情节点分布
    plot_points:
      - chapter: 1
        type: "核心事件"
        description: "主角遭遇重大变故"
      - chapter: 30
        type: "中点转折"
        description: "主角进入新世界"
      - chapter: 60
        type: "最低谷"
        description: "主角失去一切"
      - chapter: 90
        type: "最终决战"
        description: "主角vs反派"

    # 4. MDA框架分析
    mda_analysis:
      mechanics: "力量体系/规则设定"
      dynamics: "冲突涌现方式"
      aesthetics: "读者情感体验"
```

## 六、与现有系统集成

### 6.1 与模板库的关系

```
AI拆书 → 分析报告 + 可复用模板
              ↓
         存入模板库
              ↓
    灵感部门创建新项目时使用
```

### 6.2 与灵感部门的关系

```
灵感A/B/C 审阅 AI 拆书报告 → 确认模板质量 → 存入模板库
```

### 6.3 与AI导演的关系

```
AI导演生成大纲时 → 可调用模板库中的模板 → 应用结构/节拍/人物模板
```

### 6.4 与MDA分析模板的关系

```
AI拆书是MDA分析模板的自动化实现
手动填写模板 → 自动解析小说 → 生成报告
```

## 八、实施步骤

### 阶段1（2-3周）：核心分析

1. 实现文本解析器（txt/markdown → 结构化文本）
2. 实现结构分析（MDA框架 → 章节结构/节奏分布）
3. 实现人物弧光提取（角色识别 → 弧光追踪）
4. 验证：分析一本小说，输出结构报告

### 阶段2（2-3周）：模板生成

5. 实现可复用模板提取（结构模板/节拍模板/人物模板）
6. 实现模板存储（存入模板库）
7. 验证：提取的模板可被新项目使用

### 阶段3（1-2周）：集成与优化

8. 与模板库集成
9. 与灵感部门流程集成
10. 与AI导演系统集成
11. 优化分析准确率

## 十、关键设计决策

| 决策 | 说明 |
|------|------|
| 自动化解析 | 输入小说 → 自动解析，不依赖人工填写模板 |
| 模板可复用 | 从分析过的作品中提取模板，存入模板库供新项目使用 |
| 灵感部门审阅 | AI拆书结果由灵感部门审阅确认，保证质量 |
| 与AI导演集成 | AI导演生成大纲时可直接调用模板库中的模板 |
