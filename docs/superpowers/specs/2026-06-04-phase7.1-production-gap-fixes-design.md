## 组件设计

### 1. Fix #1 — `master_controller.py:562`

**Before**:
```python
llm_report = self.auditor.llm_audit(
    chapter_num=chapter_num,
    content=content,
    characters=characters,
    context={"timeline": timeline},
)
```

**After**:
```python
llm_report = self.auditor.audit_chapter(
    chapter_num=chapter_num,
    content=content,
    characters=characters,
    context={"timeline": timeline},
)
```

**为什么不传 `reviewer_id`**: `AuditorTools.audit_chapter` 的 `reviewer_id` 默认 None,内部用 `self._current_reviewer_id` 兜底再 "J"。保持调用方不传 = 行为不变,改动最小化。

**为什么 context 用 `{"timeline": timeline}`**: `master.audit_chapter` 现有调用已经把 timeline 打包到 context,沿用相同 dict shape 避免下游 `_build_audit_prompt` 内部 key 假设变化。

### 2. Fix #3 — `got_bridge.py:79-94` `_handler_chapter_review`

**Before**:
```python
return master.audit_chapter(
    chapter_num=chapter_num,
    content=content,
    characters=_resolve_field(inputs, "characters", []),
    timeline=_resolve_field(inputs, "timeline", []),
    use_llm=bool(_resolve_field(inputs, "use_llm", True)),
)
```

**After**:
```python
report = master.audit_chapter(
    chapter_num=chapter_num,
    content=content,
    characters=_resolve_field(inputs, "characters", []),
    timeline=_resolve_field(inputs, "timeline", []),
    use_llm=bool(_resolve_field(inputs, "use_llm", True)),
)
if isinstance(report, dict):
    report.setdefault("content", content)
return report
```

**为什么用 `setdefault` 而非 `report["content"] = content`**: 防御性 — 如果 `master.audit_chapter` 未来在 report 里加 `content` 字段 (例如把原文嵌进 audit 报告),setdefault 不会覆盖新值。如果下游某个测试故意传 `inputs["content"]` 覆盖,handler 也不应抹掉用户的意图。

**为什么 isinstance 检查**: `master.audit_chapter` 当前返回 dict,但签名没强制。如果未来返回 dataclass (Pydantic 化 / 冻结),`setdefault` 不存在不会爆。`isinstance(report, dict)` 守住向后兼容。

### 4. 新测试 — `tests/agent_system/test_phase7_1_production_fixes.py`

3 个测试,精确覆盖修复点 1+3,避免回归:

```python
"""Phase 7.1 回归测试 — 覆盖修复点 1+3

- Fix #1: master.audit_chapter 内部必须调 auditor.audit_chapter (不是 llm_audit)
- Fix #3: _handler_chapter_review 返回值必须含 content 字段 (供下游 polish 用)
- 韧性:    audit_chapter 抛错不应让 workflow 崩溃 (master.audit_chapter 已有 try/except)
"""
from infra.agent_system.got_bridge import _handler_chapter_review
from infra.agent_system.master_controller import MasterController
from tests.agent_system._e2e_helpers import make_master_with_router, make_stub_router


def test_master_audit_chapter_calls_auditor_audit_chapter(monkeypatch, tmp_path):
    """Fix #1 回归: master.audit_chapter → self.auditor.audit_chapter (不是 llm_audit)"""
    router, providers = make_stub_router()
    master = make_master_with_router(tmp_path, router)

    called = []
    def fake_audit_chapter(chapter_num, content, characters, context, reviewer_id=None):
        called.append({"chapter_num": chapter_num, "content": content, "reviewer_id": reviewer_id})
        return {"issues": [], "scores": {"S1": 8}}

    monkeypatch.setattr(master.auditor, "audit_chapter", fake_audit_chapter)
    master.audit_chapter(chapter_num=1, content="正文", characters=[], timeline=[])

    assert len(called) == 1
    assert called[0]["chapter_num"] == 1
    assert called[0]["content"] == "正文"


def test_review_handler_propagates_content_to_downstream():
    """Fix #3 回归: _handler_chapter_review 返回 dict 必须含 content 字段"""
    from infra.agent_system.master_controller import MasterController
    master = MasterController.__new__(MasterController)  # skip __init__

    # stub master.audit_chapter 返回 audit report (无 content)
    master.audit_chapter = lambda **kwargs: {"issues": [], "scores": {}}

    result = _handler_chapter_review(
        master=master,
        inputs={"chapter_num": 1, "content": "原始正文 100 字", "characters": [], "timeline": []},
    )
    assert result["content"] == "原始正文 100 字"  # 必须透传


def test_audit_chapter_failure_does_not_crash_workflow(tmp_path):
    """韧性: audit_chapter 抛错被 master.audit_chapter try/except 兜住,workflow 仍 COMPLETED"""
    from infra.agent_system.got_bridge import _handler_chapter_review
    from infra.got.data_structures import NodeStatus

    router, providers = make_stub_router_with_responses({
        "anthropic": "ok-then-boom",  # write 1 次成功
    })
    # 第 2 次 LLM 调用 (audit) 时让 anthropic 抛错
    # 用 iter 简单实现:第 2 次 raise
    original_generate = providers["anthropic"].generate
    call_count = [0]
    def flaky_generate(prompt, **kwargs):
        call_count[0] += 1
        if call_count[0] >= 2:
            raise RuntimeError("simulated audit failure")
        return original_generate(prompt, **kwargs)
    providers["anthropic"].generate = flaky_generate

    master = make_master_with_router(tmp_path, router)
    result = master.run_workflow(
        workflow_name="minimal_e2e",
        initial_inputs={
            "chapter_num": 1, "outline": {"chapters": [{"num": 1}]},
            "characters": [], "memory_context": {}, "style_guide": {}, "use_llm": True,
        },
    )

    assert result["summary"].failed == 0
    assert result["summary"].completed == 2  # write + review 都 COMPLETED (audit 失败被吞)
```

**为什么不用 TestClass**: 3 个测试独立,fixture (master/stub) 各自构造,没有共享 setup。Plain functions + 模块级 helpers 即可。

**为什么 test #2 用 `MasterController.__new__`**: `_handler_chapter_review` 只调 `master.audit_chapter`,不依赖 `__init__` 构造完整 controller。`__new__` 跳过 `__init__` 让测试 5x 更快,不污染 state dir。

## 风险 + 缓解

| 风险 | 缓解 |
|------|------|
| `master.audit_chapter` 调 `auditor.audit_chapter` 后下游 `_build_audit_prompt` 内部 key 假设变化 | context dict shape 沿用 `{"timeline": timeline}`,不增 key |
| 修复后 `phase 7 test_at_least_one_llm_call` 实际 LLM 调用数 = 2 但测试 `>= 2` 太松 | 接受 `>= 2` 即可,精确数字断言在 test_phase7_1 单独测 |
| `_handler_chapter_review` 用 `isinstance(report, dict)` 判断,若 `master.audit_chapter` 改返回 Pydantic 模型会失效 | 短期可接受 (生产代码风格一致),长期若改 Pydantic 化时再调整 |
| test #3 (`test_audit_chapter_failure_does_not_crash_workflow`) 第 2 次 LLM 调用预期是 audit,但 `master.write_chapter` 内部可能调 1 次 LLM (chapter_writing prompt) 后才有 audit 调用 | 实际:write_chapter 调 1 次 (chapter_writing),audit_chapter 调 1 次 (chapter_review);第 2 次 raise 正确触发 audit 失败 |
| monkeypatch `master.auditor.audit_chapter` 可能不生效 (属性可能在 __init__ 后才绑定) | 验证:`master.auditor` 在 `MasterController.__init__` 完成时已绑定 (line ~80) |
| `MasterController.__new__` 跳过 `__init__` 后 `master.audit_chapter = lambda` 是否可设置 | 是 — `__new__` 分配对象后 `__dict__` 可写 |
