## 一、部门架构

### 1.1 成员与职责

| 角色 | 编号 | 负责类型 | 核心职责 | 输出维度 |
|------|------|---------|---------|---------|
| 类型专家 | 灵感A | 都市现实、职场商战 | 类型选择、核心冲突、卖点提炼 | 类型分析+冲突设计 |
| 幻想专家 | 灵感B | 玄幻仙侠、科幻奇幻 | 世界观设定、力量体系、升级路线 | 世界观文档 |
| 结构专家 | 灵感C | 悬疑推理、叙事结构 | 时间线设计、多线叙事、伏笔布局 | 叙事结构文档 |
| 灵感主编 | 轮值 | 不固定 | 整合三方输出、版本管理、质量裁定 | 立项包（基础层+深度层）|

### 1.2 主编轮值规则

- 每个项目由 A/B/C 轮流担任主编
- 轮值顺序：项目1→A，项目2→B，项目3→C，项目4→A（循环）
- 主编职责：**整合+裁定**，不参与具体创作
- 主编拥有最终立项包的裁决权

### 1.3 并行执行原则

```
任务：生成立项文件
执行方案：
  灵感A → 类型/冲突/卖点
  灵感B → 世界观/力量体系
  灵感C → 时间线/伏笔/叙事
  ↓ 三方并行产出
  灵感主编 → 整合 → 立项包v1.0
```

## 二、质量评估标准

### 2.1 S/A/B三级评分

| 等级 | 完整度要求 | 特征 | 处理方式 |
|------|-----------|------|---------|
| **S级** | >90% | 创意新颖、有深度洞察、结构完整 | 直接进入整合环节 |
| **A级** | 70%-90% | 结构清晰、可直接使用、局部可优化 | 主编整合时小幅调整 |
| **B级** | 50%-70% | 有基本框架、需较大修改 | 返工重新产出（不计入迭代次数） |
| **不合格** | <50% | 框架缺失、方向偏离 | 打回重做（计1次迭代） |

### 2.2 评分维度

每个灵感输出的评分维度：

| 维度 | 占比 | 评分标准 |
|------|------|---------|
| 类型契合度 | 20% | 是否准确匹配目标类型 |
| 创意独特性 | 20% | 是否有新颖卖点/设定 |
| 结构完整度 | 25% | 世界观/人物/冲突是否完整 |
| 可执行性 | 20% | 作家部门能否直接使用 |
| 市场价值 | 15% | 是否符合目标受众期待 |

### 2.3 评估流程

```
1. 三人并行产出初版
2. 主编读取三方输出，逐项评分
3. 汇总评分，输出「质量报告」
4. 判定等级，决定处理方式：
   - S/A级 → 进入整合
   - B级 → 返回重做（不计入迭代）
   - 不合格 → 打回重做（计1次迭代）
```

## 三、双层对接标准

### 3.1 层级定义

灵感部门的最终交付物为**双层结构**，分别服务于不同部门：

| 层级 | 文件名 | 包含内容 | 服务对象 |
|------|--------|---------|---------|
| **基础层** | `{项目名}_基础层.yaml` | 类型/主题/核心人物/风格禁忌/核心卖点 | 作家部门 |
| **深度层** | `{项目名}_深度层.md` | 世界观体系/时间线/伏笔设计/叙事结构/关键场景 | 审核部门/汇总部门 |

### 3.2 基础层规范（作家用）

```yaml
project_id: starfall_epoch_001
小说名称: 《星陨纪元》
副标题: 废土星火
类型:
  primary: 玄幻修真
  secondary: 废土生存、星际联盟
  tone: 冷峻写实，带废土朋克风
核心主题: "守护的意义，不是征服，而是让所有生命都能在宇宙中安心存在"
核心主角人设:
  - 姓名：林夜
    特征：废土少年，父母死于暗域追杀，性格冷峻内敛，复仇心切但逐渐转向守护
  - 姓名：苏琳
    特征：预言能力者，被星辰会追杀，性格温柔坚定，成为林夜的锚点
核心冲突主线:
  - 复仇vs守护：林夜在仇恨中挣扎，最终选择"为值得保护的人而活"
  - 光明与黑暗的融合：暗域并非纯粹的恶，林夜需找到共存之道
目标受众:
  age_range: 18-35
  appeal: 智斗、反转、成长型主角、废土朋克、星际修真世界观
风格指南:
  language_style: 现代简洁，带古风韵味，废土朋克语境下融入修真术语
  pacing: 快节奏，情节紧凑，不注水，每章有推进
  dialogue_ratio: 30%
  sensory_focus: 心理描写与环境描写并重
  禁忌设定: 不写降智反派、不注水、不圣母主角、无脑升级
  核心卖点:
    - "主角智商全程在线，决策有逻辑支撑"
    - "环环相扣的势力博弈与废土生存"
    - "光明与黑暗的辩证思考，不是非黑即白"
开篇方向: 从林夜父母之死开场，暗域追杀者破门而入
output_version: 1.0
```

## 四、版本迭代流程

### 4.1 迭代规则

| 参数 | 值 | 说明 |
|------|-----|------|
| 最大迭代次数 | 3轮 | 超过由主控仲裁 |
| 质量达标线 | S级或A级 | 才能进入下一环节 |
| 主编裁决权 | 最终版本 | 主编可决定是否需要继续迭代 |

### 4.2 迭代流程图

```
[启动] → 三人并行产出初版
            ↓
        主编评分
            ↓
    ┌─── S/A级 ───┐
    ↓              ↓
进入整合      B/不合格级
    ↓              ↓
主编整合      返回重做
    ↓              ↓
输出v1.0      不计迭代
    ↓              ↓
质量评审        ↓
    ↓         重新产出
否 ──┐    ┌──是
    ↓     ↓
  修改方向  输出v1.0
    ↓     ↓
第二轮迭代 第三轮迭代
    ↓     ↓
...    [主控仲裁]
```

### 4.3 版本命名规范

| 版本 | 命名格式 | 说明 |
|------|---------|------|
| 初版 | `灵感_{项目名}_v1.0.yaml` | 第一轮产出后整合 |
| 第二轮 | `灵感_{项目名}_v2.0.yaml` | 第二轮迭代后整合 |
| 第三轮 | `灵感_{项目名}_v3.0.yaml` | 第三轮迭代后整合 |
| 最终版 | `灵感_{项目名}_final.yaml` | 主编确认的最终版本 |

## 七、协作流程

### 7.1 项目启动流程

```
1. 主控下达任务：「为{项目名}生成灵感」
2. 轮值判定：根据轮值规则确定主编（A/B/C轮流）
3. 主控向主编下达指令：
   - 项目类型：玄幻修真
   - 目标受众：18-35岁
   - 项目规模：360章（三卷）
   - 截止时间：TBD
4. 主编启动并行：
   - 灵感A → 类型/冲突/卖点（产出：类型分析.yaml）
   - 灵感B → 世界观/力量体系（产出：世界观文档.yaml）
   - 灵感C → 时间线/伏笔/叙事（产出：叙事结构.md）
5. 三方产出完成后，提交主编整合
6. 主编评分 → 判定等级 → 决定处理方式
```

### 7.2 主编整合流程

```
1. 读取三方输出
2. 逐项评分（S/A/B三级）
3. 汇总评分，输出「质量报告」
4. 判定：
   - 全部S/A级 → 进入整合
   - 有B/不合格 → 返回重做（标注问题）
5. 整合产出：
   - 合并类型分析+世界观文档 → 基础层.yaml
   - 合并叙事结构 → 深度层.md
6. 输出立项包v1.0
7. 提交主控确认
```

### 7.3 迭代处理流程

```
第N轮迭代：
1. 主控反馈：质量不达标/需优化方向
2. 主编解读修改方向
3. 分配任务：
   - 灵感A → 针对修改方向重做XX维度
   - 灵感B → 针对修改方向重做XX维度
   - 灵感C → 针对修改方向重做XX维度
4. 三方重新产出
5. 主编重新整合 → 输出v(N+1).0
6. 重复直到S/A级或达到3轮上限
```

## 九、模板库使用指南

### 9.1 模板引用流程

```
1. 新项目启动
2. 主编判断项目类型（玄幻/都市/悬疑等）
3. 从模板库选择适用模板：
   - 玄幻项目 → 玄幻修真体系模板.yaml + 废土生存设定模板.yaml（如适用）
   - 都市项目 → 都市职场模板.yaml
   - 悬疑项目 → 悬疑推理模板.yaml
4. 模板作为基础，三人根据项目特点调整
5. 不适用的模板内容可删除或修改
```

### 9.2 模板创建规则

当发现某类设定可复用时，主编可创建新模板：

```
1. 提炼通用设定框架
2. 命名为「{类型}模板.yaml」
3. 放入「01_灵感库/模板库/」
4. 标注适用场景和注意事项
```
