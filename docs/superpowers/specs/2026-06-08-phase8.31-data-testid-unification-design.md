# Phase 8.31 — Dashboard data-testid 统一 设计

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) 或 superpowers:executing-plans 实施本 spec.

## Context

**现状 (Phase 8.30b 后 baseline 2262 passed 27 skipped + 14 vitest spec files / 55 tests)**:
- ✅ Phase 8.13 引入 `data-testid` 约定(23 个现有 testid, kebab-case 镜像 stable class, 96 处 testid 引用在 14 spec 里)
- ❌ **7 component 完全缺 data-testid**: App.vue / StatCard / WorkflowGraph / CoolpointChart / DecisionCard / DecisionsPage / WorkflowsPage
- ❌ **4 处 spec 残留 class selector** (14 spec 中 4 处 = 2.4%):
  - `tests/unit/dashboard.spec.ts:82` — `findAll('.stat-card')` ← StatCard 缺 testid
  - `tests/unit/sidebar-tier-budget.spec.ts:111` — `find('.sidebar-cost-tier-text')` ← 缺 testid (spec 需)
  - `tests/unit/sidebar-tier-budget.spec.ts:137,165` — `find('.progress-bar-fill')` ← **data-testid 已存在**, spec 没用 (反向不一致)
- ❌ **2 page `<h1 class="page-title">` 缺 testid** (DecisionsPage / WorkflowsPage),跟 OverviewPage 已有的 `data-testid="page-title"` 不一致

**触发条件**: 主公已观察到 spec 里 4 处 class selector 是 Phase 8.30b vitest 改造的漏网之鱼(原 ceremonial Playwright spec 走 class, vitest 改造时只 1:1 翻译, 没补 testid), 加新 component 测试时容易混淆用 class 还是 testid。统一约定避免后续每个新 test 都重复讨论。

**目标**: vitest-触达区域 data-testid 100% 统一, 4 处 class selector 归零, 7 个缺-testid component 补齐顶层 testid。

## Goals

- 7 缺-testid component 加 1 顶层 `data-testid` (kebab-case 镜像 stable class, 跟 Phase 8.13 23 个 testid 模式一致)
- 1 spec 需 inner testid: `sidebar-cost-tier-text` 加到 SidebarCostBanner tier-row 内层 span
- 3 spec 文件 4 处 class selector 改 `[data-testid="..."]`
- 0 改 23 个现有 data-testid
- 0 改 component CSS / JSX / props / composables / api
- 0 改 historical spec/plan doc (`:NNN` 行号不动)

## Non-goals

- ❌ **DecisionCard / WorkflowGraph 内部 sub-element testid** (e.g. `decision-card-kind-badge` / `workflow-graph-loading`): 14 vitest spec 不需选 inner element, 未来 E2E 有需要再加, 符合"spec 需才加"原则
- ❌ **App.vue 之外 root element 改** (e.g. `<router-view>` 容器): 1 顶层 testid 足够, 不加嵌套
- ❌ **StatCard 3 个 inner span 加 testid** (`stat-label` / `stat-value` / `stat-trend`): spec 用 `wrapper.findAll('[data-testid="stat-card"]')` 后用 `.find('.stat-value')` 选 inner 走 class 仍可工作, 不影响未来加 testid
- ❌ **kebab-case → camelCase / BEM / 组件前缀名 改命名**: 现有 23 个 testid 统一 kebab-case 镜像 class, 不引入新 prefix, 避免破 14 spec 已有断言
- ❌ **stat-card 1 outer + 多 sub-testid** (e.g. `stat-card-{label}` 唯一化): 当前 spec `findAll('[data-testid="stat-card"]')` 拿数组后用 `[i].text()` 取 label, 唯一化属 YAGNI
- ❌ **cypress / playwright 真 e2e spec 引入** (Phase 8.30b 已用 vitest 替代)
- ❌ **CI / lint 强制 data-testid 存在**: 1 个约定靠 review 维护, lint rule 留 followup

## Design

### 1. Component 改 (8 文件, 8 testid)

所有改遵循**最小变更原则**: 在 root element 加 `data-testid="..."`, 不动 class/structure, 不动其他 template。

#### 1.1 顶层 testid (7 component, 7 testid)

| File | 改法 | 行数 |
|------|------|------|
| `src/App.vue` | root `<div>` 加 `data-testid="app-root"` | +1L |
| `src/components/StatCard.vue` (68L) | root `<div class="stat-card">` 加 `data-testid="stat-card"` | +1L |
| `src/components/WorkflowGraph.vue` (236L) | root `<div class="workflow-graph">` 加 `data-testid="workflow-graph"` | +1L |
| `src/components/CoolpointChart.vue` (154L) | root `<div>` 加 `data-testid="coolpoint-chart"` | +1L |
| `src/components/DecisionCard.vue` (337L) | root `<div class="decision-card">` 加 `data-testid="decision-card"` | +1L |
| `src/pages/DecisionsPage.vue` (306L) | `<h1 class="page-title">` 加 `data-testid="page-title"` | +1L |
| `src/pages/WorkflowsPage.vue` (463L) | `<h1 class="page-title">` 加 `data-testid="page-title"` | +1L |

**示例** (StatCard 模式, 7 处同形):
```vue

<!-- 后 -->

```

#### 1.2 1 spec 需 inner testid (1 file, 1 testid)

```vue
<!-- 前 (L88-91 region) -->

<div data-testid="sidebar-cost-tier-row">
```

(精确 line 跟 `git blame` 时可能微调, 模式: 在 `<span class="sidebar-cost-tier-text">` 末尾加 `data-testid="sidebar-cost-tier-text"`, 不动 class/text/其他 attribute)

## 风险 + 缓解

| 风险 | 缓解 |
|------|------|
| 改坏 14 spec 已有 testid 断言 | 0 改 23 个现有 testid, 8 个新 testid 镜像 stable class, 跟现有命名 0 冲突 |
| `<h1>` 改 testid 影响 SEO / a11y | 0 改 h1 元素, 只加 attribute, semantic HTML 保持 |
| spec 改 selector 后 class 改 CSS 时破测试 | data-testid 是独立 attribute, CSS 改 class 不影响 testid 断言, 反而更稳 |
| 漏改 1 处 class selector | grep 验证 `find\('\.\|findAll\('\." tests/unit/` 0 匹配 (4 处全净化) |
| inner testid 改坏 sidebar layout | 仅加 attribute, 0 改 class/structure/style, CSS layout 0 影响 |
| vitest 跑出新 warning (unused import 等) | 改前跑 baseline 2262, 改后 2262 + 0 new warning |
| `DecisionsPage` 加 `data-testid="page-title"` 跟 `OverviewPage` 重名 | 重名是预期: page-title 在不同 page 重复, spec 用 `wrapper` 隔离, 0 cross-page 断言冲突 |
| 后端 / agent_system 误改 | 0 改 backend (本 spec 全 frontend only), 14 spec 全 frontend |
