## Context

`CostTracker` 自 Phase 2.13 (2026-06-03) 引入 4 月, 含 `CostRecord` (frozen dataclass) + `CostTracker` (in-memory list) + `compute_cost()` 链 `MODEL_TIERS` (HAIKU $1/$5, SONNET $3/$15, OPUS $15/$75 per 1M tokens). 0 接入点 — `git grep "CostTracker\|cost_tracker" infra/` 只在 `cost_tracker.py` 自身引用 + `tiered_router.py:55-66` (Optional 注入) + `llm_compute.py:71-76` (Optional 注入, 但无人调), 生产路径从不写记录, "成本追踪" 实际是空架子.

**当前瓶颈** (per Phase 8.1 / 8.2 / 8.3 实测):
- `agent.chat()` → `router.generate()` → `provider.generate()` 返回 `str`, 用量被丢
- `AgentComputeFn.__call__` (`got_bridge.py:188-243`) 是 perfect 注入点: 知道 `node.prompt_scenario`, handler 返回后能估 token + 写 `cost_tracker.record()`
- `ComputeResult.cost_tokens=0` 硬编码 (`got_bridge.py:219, 243`), GoTScheduler 看到永远是 0
- `SCENARIO_TIER_MAP` (`scenarios.py:108-131`) 已就位 — 12 scenario → 3 tier 静态映射
- `infra/ai_service/model_tiers.compute_cost(in, out, tier)` 已就位 — input/output tokens + tier → USD

## Goal

1 goal — wire `CostTracker` into production GoT compute path (`AgentComputeFn`) + SQLite 持久化, 让成本数据真有用, dashboard 端到端验证 hook 可工作.

## Non-Goals (YAGNI)

- ❌ 改 provider 层返回真实 usage tuple — 12 个 provider 各加返回值, scope 爆炸; 留 Phase 8.6
- ❌ 改 `TieredRouter` 注入点 — 已经有, 0 调用点, 不必重复
- ❌ 改 12 SCENARIOS 路由 — CostTracker 是观察者, 0 改业务
- ❌ 升级 `len()//4` token 估算为真实 usage — 跟 `tiered_router.py:148-149` 公式一致, 4 char/token 经验值 ~85% accuracy
- ❌ Dashboard UI 画 cost by scenario / by tier 柱状图 — 留 Phase 8.7
- ❌ Cost budget alarm (超阈值暂停 workflow) — 留 Phase 8.8+
- ❌ 改 2090 旧测试 — `cost_tracker: Optional[CostTracker] = None` default 兜底, 0 改旧测试
- ❌ 改 18 历史 spec/plan docs 行号 (`:NNN`) — 历史已不准, 改反误导, 永不做 (per Phase 8.4)

## Design

### 注入点: `AgentComputeFn` (got_bridge.py:188-243)

- **选 AgentComputeFn (A)** — perfect: 知道 scenario, handler 返回后能估 token, 0 改 provider
- **不选 TieredRouter (B)** — router 不知道 scenario 也不知道 prompt text, 估不准
- **不选 provider 层 (C)** — 12 个 provider 各自加, 重复代码

### Token 估算: `len() // 4` (跟 `tiered_router.py:148-149` 一致)

- 0 改 provider
- 4 char/token 经验值 (~85% accuracy)
- 跟 `tiered_router.py:148-149` 公式一致
- 可升级: 留 `provider.generate() → tuple[str, dict]` followup (本期不做)

### Tier 来源: `SCENARIO_TIER_MAP[node.prompt_scenario]`

- 已存在, 12 scenario → 3 tier 静态映射
- `.get(scenario, ModelTier.SONNET)` 默认 SONNET (中间档), 不让估算挂

### 成本计算: 复用 `compute_cost(in_tok, out_tok, tier)` (不重新实现)

- 已存在, 含 3 tier 完整费率
- `CostTracker.record()` 内部自动调 `compute_cost` 算 `cost_usd`

### Persistence: SQLite (mirror `ReadingPowerDB`)

- 跟 `ReadingPowerDB` 一致 (单文件, `infra/.state/cost_tracker.db` gitignored)
- 0 引入新依赖, 5ms 写
- 不选 JSONL (10w 章后查询慢), 不选 DuckDB (overkill for 1 table)

### Schema: 单表 `cost_records` (per-scenario 列)

```sql
CREATE TABLE cost_records (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    scenario TEXT NOT NULL,
    tier TEXT NOT NULL,
    input_tokens INTEGER NOT NULL,
    output_tokens INTEGER NOT NULL,
    cost_usd REAL NOT NULL,
    timestamp TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_cost_records_scenario ON cost_records(scenario);
CREATE INDEX idx_cost_records_tier ON cost_records(tier);
```

- 12 scenario 不会同时大量写, 1 表简单
- `cost_by_scenario()` / `cost_by_tier()` 直接 SELECT WHERE / GROUP BY

### `cost_tracker=None` 兜底 (vs 必传)

- `Optional[CostTracker] = None` — backward compat 2090 旧测试零修改
- 旧 master / 旧 e2e tests 走 no-op, `cost_tokens=0` (老行为)

### Dashboard: 加 1 字段 `total_cost_usd: float = 0.0`

- 加 `WorkflowStatusResponse.total_cost_usd` 字段 (Pydantic default 0.0)
- `get_active_workflow_status` 拿 `master.cost_tracker.total_cost()` 填字段 (0.0 if no cost_tracker)
- **0 新端点, 0 新 UI** (留 followup phase)
- 旧 client 读不存在字段 → 0.0 (Pydantic 兼容)

### gitignored: `infra/.state/cost_tracker.db`

- `infra/.state/*.db` 已在 `.gitignore` (per Phase 6.5)
- 跟 `workflow.db` / `decisions.json` 一致
- 主公 setup 时 DB 自动 create (lazy `_init_db`)

### 不动 MEMORY.md

- 201 lines (1 over 200 soft limit) — Phase 8.4 已知 deferred
- 本期只加 1 entry (~1500 chars) → 202-203 lines
- 接受再超 1-2 行, 留 followup phase 统一拆 `phases.md` topic file
