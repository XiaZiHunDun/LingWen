# Phase 8.34 — Store Registry (Decision / Workflow List / Overview) 设计

> **For agentic workers**: 本 spec 是设计文档。实施走 subagent-driven-development (跟 Phase 8.30b/8.31/8.32/8.33 同样 1 phase 多 commit pattern)。

**Goal**: 把 3 page 的 fetch-data state (decisions.all / workflows list / overview+chapters) 抽成 module-level singleton store,镜像 `useWorkflowSocket.js` (Phase 6.4+8.11) + `useCostWindow.js` (Phase 8.16+8.18+8.19) 现成 pattern,page nav 跨 mount 共享 state,避免重复 fetch。

**Architecture**: 3 独立 module-level singleton composable,各管 1 个 domain,跟 useWorkflowSocket 1-domain-per-file 哲学一致。0 new dep,纯 Vue 3 Composition API + module-level ref。Decision store 跟 useWorkflowSocket 边界清晰 — store 只管 all (历史) + mutation,pending 仍走 WS (useWorkflowSocket 仍是权威源 per Phase 6.4)。

**Tech Stack**: Vue 3.4 Composition API + vitest 4.x + @vue/test-utils 2.x (跟 Phase 8.30b 一致)。

## Design

### 1. State ownership (走 store vs 留 page-local)

| Page | 进 store | 留 page-local |
|------|---------|---------------|
| **DecisionsPage** | `all: ref<Decision[]>` (REST 拉历史) | `activeTab`, `expanded` (UI state) — `pending` 走 WS (useWorkflowSocket),不入 store |
| **WorkflowsPage** | `workflows: ref<WorkflowListItem[]>` (REST 拉 list) | `selected`, `initialInputsJson`, `maxBacktracks`, `running`, `showGraph`, `graphData`, `graphLoading`, `active` (WS-driven),`error` (page-specific) |
| **OverviewPage** | `overview: ref<OverviewResponse\|null>`, `chapters: ref<Chapter[]>` | `chartData`, `statCards` (computed, 从 store ref 派生) |

**Rationale**:

- `all/workflows/overview/chapters` 是 **fetch-data state** — 跨 page nav 重用有价值 (per Phase 8.18 useCostWindow singleton 哲学)
- `activeTab/expanded/selected/inputs/showGraph/active` 是 **page-local UI/interaction state** — 切 tab/page 即重置,塞 store 反而丢失 reset 行为
- `pending_decisions` 走 WS (Phase 6.4+8.11 决策),不进 store

### 2. Module-level singleton pattern (跟 useWorkflowSocket 1:1 mirror)

**useDecisionStore.js** (NEW, ~80L):
```js
import { onBeforeUnmount, onMounted, ref } from 'vue';
import { fetchAllDecisions, resolveDecision, deferDecision, cancelDecision } from '../api/index.js';

// ---- Module-level singleton state ----
const all = ref([]);
const loading = ref(false);
const lastError = ref(null);
let mountedCount = 0;

async function refresh() {
  loading.value = true;
  lastError.value = null;
  try {
    all.value = await fetchAllDecisions();
  } catch (e) {
    lastError.value = e?.message || String(e);
  } finally {
    loading.value = false;
  }
}

async function resolve(decisionId, option) {
  try {
    await resolveDecision(decisionId, option);
    await refresh();  // mutation 后立即 refresh
  } catch (e) {
    lastError.value = `解决失败: ${e?.message || e}`;
    throw e;  // 让 caller 知道失败
  }
}

async function defer(decisionId, reason) {
  // mirror resolve pattern
}

async function cancel(decisionId, reason) {
  // mirror resolve pattern
}

export function useDecisionStore() {
  onMounted(() => {
    if (mountedCount === 0) refresh();
    mountedCount += 1;
  });
  onBeforeUnmount(() => {
    mountedCount = Math.max(0, mountedCount - 1);
  });
  return { all, loading, lastError, refresh, resolve, defer, cancel };
}
```

**useOverviewStore.js** (NEW, ~70L):
```js
import { onBeforeUnmount, onMounted, ref } from 'vue';
import { fetchOverview, fetchChapters } from '../api/index.js';

const overview = ref(null);
const chapters = ref([]);
const loading = ref(false);
const lastError = ref(null);
let mountedCount = 0;

async function refresh() {
  loading.value = true;
  lastError.value = null;
  try {
    const [o, c] = await Promise.all([fetchOverview(), fetchChapters('1-30')]);
    overview.value = o;
    chapters.value = c.chapters || [];
  } catch (e) {
    lastError.value = e?.message || String(e);
  } finally {
    loading.value = false;
  }
}

export function useOverviewStore() {
  onMounted(() => {
    if (mountedCount === 0) refresh();
    mountedCount += 1;
  });
  onBeforeUnmount(() => {
    mountedCount = Math.max(0, mountedCount - 1);
  });
  return { overview, chapters, loading, lastError, refresh };
}
```

### 4. Data flow

- **Hydration**: 首个 mount → `refresh()`;后续 mount 共享 state (0 re-fetch)
- **Mutation**: `store.resolve(id, option)` → backend API → `store.refresh()` 重拉 all
- **Error**: `lastError` ref 存 message,page v-if 渲染 banner (跟 useCostWindow lastError 1:1)
- **WS 边界**: decision store 0 接 WS 推送;`useWorkflowSocket.pendingDecisions` 仍是 pending 权威源,DecisionsPage 双 source 合成 pending

### 5. Error handling

- `lastError` ref 存 string message,跟 useCostWindow + useWorkflowSocket 1:1 pattern
- Page template `<div v-if="store.lastError" class="error-banner">{{ store.lastError }}</div>` (跟现状 inline error display 一致)
- Mutation 失败 → `lastError` 写 + `throw e` 让 caller 知道 (caller 0 必 catch,mutation 失败时 store 内已 lastError 显式)
- 0 silent swallow (per common/error-handling rule)
