# Phase 8.8 — Cost Budget Alarm Design

> **For agentic workers:** This is a design spec. After approval, an implementation plan will be created via the writing-plans skill.

**Goal:** 给 novel_factory workflow 装 cost budget alarm,run-time 累计 LLM cost 超过 user 设的阈值就**硬中止** workflow + dashboard 红 banner 提示,防止意外超支。

**Architecture:** 在既有 `cost_tracker.record()` 路径后**同步 inline check** (O(1) 浮点比较,无 hot-path 性能影响),违反 raise `CostBudgetExceeded` 自定义 exception,scheduler 现有 try/except 兜底 mark FAILED。MasterController 加 `_current_budget_usd` 字段由 POST 请求 body 配置,`finally` 块 reset 保 backward compat。

**Tech Stack:** Python 3.13 + FastAPI + Pydantic + Vue 3 + ECharts (沿用 Phase 8.5-8.7 stack)

### 3.2 数据契约

#### CostBudgetExceeded (新增 exception)
```python
# infra/ai_service/cost_tracker.py
class CostBudgetExceeded(Exception):
    """Phase 8.8: raised by CostTracker.check_budget when total_cost() > budget_usd.

    Attributes are exposed for caller introspection (no re-parsing of message).
    """
    def __init__(self, used_usd: float, budget_usd: float, scenario: str | None = None):
        self.used_usd = used_usd
        self.budget_usd = budget_usd
        self.scenario = scenario
        scenario_msg = f" (last scenario: {scenario})" if scenario else ""
        super().__init__(
            f"Cost budget exceeded: ${used_usd:.4f} / ${budget_usd:.4f}{scenario_msg}"
        )
```

#### CostTracker.check_budget (新增方法)
```python
# infra/ai_service/cost_tracker.py
def check_budget(self, budget_usd: Optional[float]) -> None:
    """Phase 8.8: enforce hard cap on cumulative cost. No-op when budget_usd is None.

    Raises CostBudgetExceeded if self.total_cost() > budget_usd.
    Strictly greater than (equal is OK, only over-budget triggers).
    """
    if budget_usd is None:
        return  # unlimited
    used = self.total_cost()
    if used > budget_usd:
        # 提取最后一次 record 的 scenario (best-effort, may be None)
        last_scenario = self._records[-1].scenario if self._records else None
        raise CostBudgetExceeded(used_usd=used, budget_usd=budget_usd, scenario=last_scenario)
```

#### AgentComputeFn (1 行注入,集中调用,双路径均覆盖)
实现策略:tuple + dict 双路径分别调 `record()`,但 `check_budget()` 在 record 后**集中调用 1 次** (用 `if self._cost_tracker is not None:` 守卫包外),避免重复代码。

```python
# infra/agent_system/got_bridge.py:259-322 (伪代码示意,实际写法在 plan)
def __call__(self, node, inputs):
    output = self._handler(self._master, inputs)
    cost_tokens = 0

    if isinstance(output, tuple):
        output, usage = output
        if self._cost_tracker is not None:
            tier = SCENARIO_TIER_MAP.get(scenario, ModelTier.SONNET)
            self._cost_tracker.record(scenario, tier,
                usage.get("input_tokens", 0), usage.get("output_tokens", 0))
            cost_tokens = usage.get("input_tokens", 0) + usage.get("output_tokens", 0)
    else:
        # Backward compat: 旧 path 走 len()//4 估算
        if self._cost_tracker is not None:
            in_tok = len(str(inputs)) // 4
            out_tok = len(str(output)) // 4
            tier = SCENARIO_TIER_MAP.get(scenario, ModelTier.SONNET)
            self._cost_tracker.record(scenario, tier, in_tok, out_tok)
            cost_tokens = in_tok + out_tok

    # === Phase 8.8 NEW: budget enforcement (集中调用,双路径均覆盖) ===
    if self._cost_tracker is not None:
        self._cost_tracker.check_budget(
            getattr(self._master, "_current_budget_usd", None)
        )
    # === END Phase 8.8 NEW ===
    ...
```

#### _extract_budget_status (dashboard helper)
```python
# dashboard/protocols.py
def _extract_budget_status(controller: MasterControllerLike) -> dict[str, Any]:
    """Phase 8.8: extract cost budget state for dashboard display.
    Returns {} if no budget set or controller missing cost_tracker (silent degrade).
    """
    try:
        budget = getattr(controller, "_current_budget_usd", None)
        if budget is None:
            return {}
        cost_tracker = getattr(controller, "cost_tracker", None)
        if cost_tracker is None:
            return {}
        used = cost_tracker.total_cost()
        used_pct = (used / budget * 100) if budget > 0 else 0.0
        status = "exceeded" if used > budget else "ok"
        return {
            "status": status,
            "budget_usd": budget,
            "used_usd": used,
            "used_pct": used_pct,
        }
    except Exception as exc:
        logger.warning("_extract_budget_status 失败: %s", exc)
        return {}
```

### 3.3 触发行为契约

#### 触阈
- `cost_tracker.total_cost() > budget_usd` (严格大于,相等 OK)
- 触发后 `record()` **已发生** (cost 已入账,无法回滚 — 符合"花出去的钱不退回"现实模型)

#### 异常流
- `check_budget()` raise `CostBudgetExceeded` (普通 Exception,无业务基类)
- `AgentComputeFn.__call__` 不做 try/except,**自然向上传播**
- GoTScheduler `node_runner` 现有 try/except (Phase 7.x) 兜底 → node 标 FAILED,ComputeResult.fail=True
- GoTScheduler DAG traversal 遇 FAILED 节点 → 下游 SKIPPED
- workflow 完结:summary 包含 `budget_exceeded=True` flag (由 node 失败信息推断,见 §3.4)

#### 状态机
- 不引入新 NodeStatus 枚举值 (复用 FAILED,加 flag)
- 完结态:workflow 视作 FAILED (1+ node FAILED),但 dashboard banner 用 `_summary.budget_exceeded` 区分

#### 重复抛防护
- 同一 run 内 budget 不变,total_cost 单调递增 → 第 1 个抛的 node 标 FAILED,后续 dependent SKIPPED,无副作用累积
- idempotent (一次 run 内不会"先抛后 OK")

### 3.4 _summary.budget_exceeded 推断
- GoTScheduler 现有 `summary` 不显式存 `budget_exceeded` 字段 (Phase 8.5/8.7 同样 0 改 summary schema)
- 推断触发点:`MasterControllerAdapter.get_active_workflow_status` 在提取 status 时,扫 `executions` 列表里 node.execution.error 字段,匹配 `"Cost budget exceeded"` 子串 → 设置 `budget_exceeded: True` flag
- 推断方法:`if any("Cost budget exceeded" in (e.get("error") or "") for e in executions)` — 简单 string match,无新增 scheduler 字段
- 或更严谨 (后续):`if any(node.execution.metadata.get("budget_exceeded"))` — 需 AgentComputeFn 写 metadata,scope 微增,Phase 8.9+ 再做
- **Phase 8.8 v1**:走 string match in adapter (跟 `_extract_cost_by_scenario` Phase 8.7 同模式:adapter 层 string match,0 改 scheduler)

### 3.5 边界 case

| 场景 | 行为 | 原因 |
|------|------|------|
| `cost_budget_usd=None` (default) | check_budget 立即 return,无限 | unlimited 路径 |
| `cost_budget_usd=0` | 第一次 record 后立刻 raise | "完全不许花" 语义 |
| `cost_budget_usd=-1` | Pydantic 拒绝 (`ge=0`) | 负数无业务意义 |
| `cost_budget_usd=NaN/Inf` | Pydantic 拒绝 (float 验证) | 数值非法 |
| `cost_tracker=None` (test stub) | check_budget 不调用 | `if self._cost_tracker is not None` 守卫 |
| 浮点 5.0 vs 5.000001 | `>` 严格大于触发 | 0.000001 也算超 (跟金融一致) |
| `used_usd` 略低于 `budget_usd` (e.g. 4.9999 / 5.0) | OK,继续 | 严格大于,等号安全 |
| workflow 中途 `_current_budget_usd` 被外部改 | 用最新值 | `getattr` 每次读,不缓存 |

### 3.6 Resume 语义
- `ResumeWorkflowRequest.cost_budget_usd: Optional[float] = None` 默认 None
- 透传 `None` → unlimited 路径,允许 user 清空 budget 重新跑失败 workflow
- 透传新值 → 新 budget,新 run 重新累积 (cost_tracker.total_cost() 持续累计)
- 旧 resume 行为 100% 保留 (字段 optional)

## 5. Data Flow

### 5.1 正常路径 (budget 未超)
```
User click "Run novel_writing" with budget=$5
  → POST /api/workflows/run {workflow_name: "novel_writing", cost_budget_usd: 5.0, ...}
  → RunWorkflowRequest validates (Pydantic ge=0, type=float)
  → master.run_workflow(workflow_name="novel_writing", ..., cost_budget_usd=5.0)
  → self._current_budget_usd = 5.0
  → scheduler.run(start_nodes, initial_inputs)
  → node 1 (chapter_writing) LLM call
      → record("chapter_writing", SONNET, 100, 50) → cost = $0.00X
      → check_budget(5.0) → 0.00X < 5.0 OK (return)
      → ComputeResult {fail: False, ...}
  → node 2 (chapter_review) LLM call → record + check OK
  → ... 5 more nodes ... → all record + check OK
  → workflow completed, summary completed=7, failed=0
  → finally: self._current_budget_usd = None
  → result returned to API handler
  → _workflow_result_to_response(result) returns WorkflowStatusResponse
  → Dashboard polls GET /api/workflows/active
  → status.cost_budget_status = {status: "ok", budget_usd: 5.0, used_usd: 3.45, used_pct: 69.0}
  → Vue 绿色 badge "Budget: $3.45 / $5.00 (69.0%)"
```

### 5.2 触阈路径 (budget 超)
```
... 上面到 node 3 ...
  → node 3 (polish_chapter) LLM call
      → record("polish_chapter", SONNET, 200, 100) → cost = $3.00 → total = $5.50
      → check_budget(5.0) → 5.50 > 5.0 RAISE CostBudgetExceeded(used=5.50, budget=5.0, scenario="polish_chapter")
      → AgentComputeFn.__call__ 异常向上传播
      → GoTScheduler node_runner try/except 兜底
      → node 3 status: FAILED, error message: "Cost budget exceeded: $5.5000 / $5.0000 (last scenario: polish_chapter)"
      → ComputeResult {fail: True, ...}
  → scheduler DAG traversal: downstream nodes (4-7) SKIPPED
  → workflow ends, summary: completed=2, failed=1, skipped=4
  → finally: self._current_budget_usd = None
  → result returned
  → _workflow_result_to_response returns WorkflowStatusResponse
  → status.cost_budget_status = {status: "exceeded", budget_usd: 5.0, used_usd: 5.50, used_pct: 110.0}
  → Vue 红色 banner "⚠️ Budget exceeded: $5.5000 / $5.00 (110.0%) — workflow aborted"
  → User 改 budget 或重跑 (新 run 重新累积 cost,旧 $5.50 仍在 cost_tracker)
```

### 5.3 旧 path 兼容 (无 budget)
```
User click "Run novel_writing" 不填 budget
  → POST body {workflow_name: "novel_writing", cost_budget_usd: null, ...}
  → master.run_workflow(workflow_name="novel_writing", ..., cost_budget_usd=None)  # default
  → self._current_budget_usd = None
  → ... run 跑 ...
  → node X LLM call → record() → check_budget(None) → 立即 return,无 raise
  → workflow 跑完跟 Phase 8.7 完全一致 (unlimited)
  → status.cost_budget_status = {} (空 dict,因为 _current_budget_usd is None)
  → Vue hasBudget = false → 不渲染 banner
```

## 6. Risks + Mitigations

| 风险 | 缓解 |
|------|------|
| 旧 2151 测试破 (默认 None 不该 raise) | `check_budget(None)` 立即 return,AgentComputeFn 用 `getattr(self._master, "_current_budget_usd", None)` 防 test stub 无该属性 |
| Pydantic 拒绝合法 budget (e.g. 0) | `ge=0` 允许 0,默认 None 用 `Optional[float] = None` |
| 浮点精度 (5.0 vs 5.000001) | 严格 `>` 触发,符合金融场景,test 覆盖边界 |
| 重复抛副作用累积 | 同一 run 内 budget 不变,total 单调,idempotent |
| Resume 时 budget 仍存 (上次 run 残留) | finally 块 reset,下个 run 必重新赋值 (即使 None) |
| `_summary.budget_exceeded` 字段缺失 (0 改 schema) | 走 string match in `summary.failed_reasons`,dashboard 端推断,跟 Phase 8.5/8.7 同模式 |
| WebSocket 1-shot 推送 (dashboard UX 延迟) | 现状 client 主动 poll `GET /api/workflows/active`,频率由 client 决定,Phase 8.5 沿用 |
| cost_tracker restart 丢 budget 配置 | in-memory loss 可接受 (per-run config,下个 run 重新设),DB 持久化留 followup |
| 多 workflow 并发 cost_tracker 共享 (race) | 单 instance model (Phase 8.5 已确认,no concurrent runs),无 race |
| 25 旧 e2e tests 破 (无 budget 默认) | 走 default None,check_budget 立即 return,0 改 |
| webnovel_init tests 破 | 走 default None,0 改 |
| CostBarChart 视觉被 banner 遮 | banner 在 cost-section 之上/之下 (magnetic 样式),不重叠 |
| _CostStub test stub 缺 budget 字段 | 复用 Phase 8.7 _CostStub + 加 `_current_budget_usd` 字段 set 即可 |
| MEMORY.md 接受 207-208 lines (1-2 over 软上限) | 跟 Phase 8.5/8.6/8.7 一致,接受再超,Phase 8.8 后必拆 topic |
| `.env` / API key 泄漏 | 0 provider 改,0 .env 改,0 ANTHROPIC_API_KEY 引用,跟 Phase 8.x 一致 |
| `_e2e_helpers.py` 改坏旧 25 tests | 加 `cost_budget_usd: Optional[float] = None` 透传 (default None),旧 test 不传 → unlimited 路径 |
| dashboard 红色 banner 视觉冲突 (跟其他警告) | 复用 Phase 7.6 黄色 `.fallback-warning` 样式 + 改 #f56c6c 红色 border-left,跟黄色明显区分 |
| ComputeResult.cost_tokens 类型变化 (int → int) | 0 改 cost_tokens 计算,raise 不影响该字段 |
| RunWorkflowRequest 缺 cost_budget_usd (旧 client 不传) | Pydantic Optional field,旧 client 走 None=unlimited,0 破 |
| `used_pct` 除以 0 (budget=0 永远超) | guard `used_pct = (used / budget * 100) if budget > 0 else 0.0`,但 budget=0 必 raise,所以 used_pct 实际用不到 |
| TypeError: float(None) | check_budget(None) 立即 return,raise 不发生 |
