## Design

### 1. Frontend: `WorkflowStatus.vue` `hasCost` OR 修复

**File**: `dashboard/frontend/src/components/WorkflowStatus.vue:129-133` (+5L, additive)

```js
// Phase 8.14: hasCost OR check costByTier, 防 tier-only 数据时整 cost section 隐藏
// 跟 Phase 8.13 CostBarChart.hasData 对称 (任一 mode 有正值即 visible)
const costByScenario = computed(() => props.status?.cost_by_scenario || {});
const costByTier = computed(() => props.status?.cost_by_tier || {});
const hasCost = computed(() => {
  const scenarioHas = Object.keys(costByScenario.value).length > 0
    && Object.values(costByScenario.value).some((v) => v > 0);
  const tierHas = Object.keys(costByTier.value).length > 0
    && Object.values(costByTier.value).some((v) => v > 0);
  return scenarioHas || tierHas;
});
```

**关键设计**:
- `costByTier` computed 跟 `costByScenario` 1:1 镜像, `props.status?.cost_by_tier || {}` 兜底 (跟 costByScenario 同 pattern, 保 Phase 8.13 旧 caller 0 改)
- `scenarioHas || tierHas` additive OR, 旧 scenario-only 行为 100% 保留 (tierHas=false → 走旧 scenarioHas)
- 保留 `value > 0` 守卫 (跟 `CostBarChart.hasData` 一致), backend 返 `{chapter_writing: 0}` (zero cost) 不显示空 cost section
- `Object.keys().length > 0 && Object.values().some(v => v > 0)` 双 check pattern 跟原代码完全一致, 无新守卫引入
