## Design

### 1. Architecture

**3 组件**:
1. **ESLint 8.x** (主 linter, 装入 devDeps)
2. **@typescript-eslint/parser 7.x** (parse `.spec.ts` 文件, 装入 devDeps)
3. **eslint-plugin-local-rules 1.x** (community plugin 加载 `eslint-rules/*.js` 本地 rule 不发 npm 包, 装入 devDeps)

**Rule 位置**: `dashboard/frontend/eslint-rules/no-class-selector-in-test.js` (项目内 rule, 走 local-rules plugin 加载)

**触发**: `pnpm lint:testid` 跑 `eslint tests/unit/**/*.spec.ts --no-eslintrc -c .eslintrc.cjs` (限定 spec dir, 0 改 .vue lint, 0 误伤 production)

**Rule 行为**:
- AST visitor pattern, 走 `CallExpression` 节点
- 检测 callee 是 `MemberExpression` 且 property name ∈ {`find`, `findAll`}
- 检测首参是 `Literal` 且 value 是 string 且以 `.` 开头
- Report error, message: `Class selector "{{value}}" forbidden. Use [data-testid="{{value}}"] instead (Phase 8.31+8.32 convention).`

### 4. Rule impl 模板

```js
// eslint-rules/no-class-selector-in-test.js
module.exports = {
  meta: {
    type: 'problem',
    docs: {
      description: 'Forbid class selector (.x) in wrapper.find/findAll. Use [data-testid="x"] instead.',
      category: 'Best Practices',
    },
    schema: [],
    messages: {
      classSelector:
        'Class selector "{{value}}" forbidden. Use [data-testid="{{value}}"] instead (Phase 8.31+8.32 convention).',
    },
  },
  create(context) {
    return {
      CallExpression(node) {
        const callee = node.callee
        if (
          callee.type === 'MemberExpression' &&
          (callee.property.name === 'find' || callee.property.name === 'findAll') &&
          node.arguments.length >= 1
        ) {
          const arg = node.arguments[0]
          if (
            arg.type === 'Literal' &&
            typeof arg.value === 'string' &&
            arg.value.startsWith('.')
          ) {
            context.report({
              node: arg,
              messageId: 'classSelector',
              data: { value: arg.value },
            })
          }
        }
      },
    }
  },
}
```

## 关键设计决策

### 1. ESLint 8.x vs ESLint 9.x

- **选 ESLint 8.x** (A) — LTS, 跟 `@typescript-eslint/parser@7.x` peer dep 100% 兼容, 跟 `eslint-plugin-local-rules@1.x` 100% 兼容
- **不选 ESLint 9.x** (B) — flat config 跟 `eslint-plugin-local-rules` 有 ambient issues, 需手写 rule loader + 多 5-10 行 boilerplate

### 2. Rule 范围: .find / .findAll vs 任意 DOM API

- **选 .find / .findAll only** (A) — 走 `@vue/test-utils` v2 wrapper API, 14 spec 0 用 querySelector / $, 跟 Phase 8.31+8.32 实际用法 1:1
- **不选任意 DOM API** (B) — `document.querySelector('.x')` / `wrapper.$('.x')` / `wrapper.$$('.x')` 0 usage, false positive 风险高 (e.g. console.log 字符串含 '.x')

### 3. Strictness: error vs warn

- **选 error** (A) — Phase 8.31 明确订 testid 约定, 跟 Phase 8.32 spec 加 'additive only' Rule 3, lint 设 error 防 accident regression, 0 妥协
- **不选 warn** (B) — IDE 显示 + 0 CI fail, 防 regression 力度弱

### 4. Trigger: pnpm lint:testid vs husky vs CI

- **选 pnpm lint:testid only** (A) — 手动跑 / 后续可挂 pre-commit/CI 0 改 rule code, scope 最小
- **不选 husky** (B) — 需装 + 0 个独立 phase, 走 phase 后续 0 强
- **不选 CI** (C) — 需 CI 配置, 跟 Phase 8.33 scope 0 关

### 5. Escape hatch: 0 vs standard vs inline

- **选 0 escape** (A) — 跟 Phase 8.31+8.32 'additive only' 原则一致, 任何真需例外 改 rule code release
- **不选 standard `/* eslint-disable */`** (B) — 滥用风险, code review 难控
- **不选 inline `/* eslint-disable-next-line */`** (C) — 例外机制变多, code review 难度升

### 6. Permanent regression fixture: 留 vs 删

- **选 永久留** (A) — 唯一真 e2e 验证 rule 能不能 catch 违规 (走 `pnpm lint:testid` exit 1), 防 rule future regression
- **不选 删** (B) — RuleTester unit test 10 case 已覆盖 90%, 缺真 e2e file-level scan 验证

### 7. Test runner: Node --test vs Mocha vs vitest

- **选 Node --test** (A) — Node 18+ 内置, 0 new devDep, 走 `node --test eslint-rules/*.test.js` 跑 RuleTester
- **不选 Mocha** (B) — 需 +1 devDep (mocha)
- **不选 vitest** (C) — ESLint RuleTester 期望 linter describe/it, vitest 走 OK 但需 +30L integration harness

### 8. devDeps 数量: 3 vs 5

- **选 +3 devDeps** (A) — eslint + @typescript-eslint/parser + eslint-plugin-local-rules, @types/node + jsdom 已装, 0 重装
- **不选 +5** (B) — Design v1 brainstorm 估算多 +2 (typescript + 1 plugin), 实际 0 需 (项目用 JS, 0 tsconfig; @typescript-eslint/parser 自带 ts 解析, 0 需 typescript devDep)
