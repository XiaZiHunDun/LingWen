## Goals

1. **Opt-in 验证 under-budget happy path**: 真实 LLM 跑完整 novel_writing workflow,设 `cost_budget_usd=0.10` (10x 预期 $0.005-0.020 上限),断言 success + cost < 0.10 + finally reset。
2. **Opt-in 验证 over-budget abort path**: 真实 LLM 跑 novel_writing,设 `cost_budget_usd=0.0001` (sub-cent 必超),断言 `CostBudgetExceeded` raise + node FAILED + total < 7 (Phase 7.4 节点数) + finally reset。
3. **CI 集成**: 加 2 tests 到 `.github/workflows/real-llm-tests.yml` `-k` filter,Anthropic only,保持现有 cron + secret 检查 pattern。
4. **模式一致性**: 严格 follow Phase 8/8.1/8.2 opt-in pattern — `@pytest.mark.skipif` + `ANTHROPIC_API_KEY` env var check + `_make_real_router("anthropic")` + `make_master_with_router` helper 复用。

## Non-Goals

- **跨 provider 验证** (Anthropic + OpenAI + MiniMax): budget 行为 provider-agnostic (所有 provider 走 same cost_budget_usd 机制 + same `CostTracker.check_budget()`), 1 provider 足证,避免 over-engineered 跟 Phase 8.2 polish_merge cross-provider 区分。
- **精确 USD 断言**: HAIKU pricing 波动,断言 `< budget` / `> budget` 容忍误差 (Phase 8 测试文档 `~$0.005-0.020` 是估计,非契约)。
- **Dashboard UI 集成**: budget banner (Phase 8.8 T6) 已有,UI 测试用 stub,无需 real LLM 流量。

**Decision**: 复用 `master.run_workflow(cost_budget_usd=budget_usd, max_backtracks=0)` 走完整 yaml workflow 路径,跟 Phase 8.8 T7 stub e2e (commit 9b3216b) 同模式。

### 3. Test 1: under-budget happy path

```python
@_REQUIRES_ANTHROPIC_KEY
def test_run_workflow_under_budget_passes(self, tmp_path: Path) -> None:
    """Phase 8.9: real LLM run with generous budget, 验证 success + cost < budget + finally reset."""
    from infra.ai_service.cost_tracker import CostTracker
    cost_tracker = CostTracker()
    master = make_master_with_router(
        state_dir=tmp_path,
        router=_make_real_router("anthropic"),
        cost_tracker=cost_tracker,
    )
    result = master.run_workflow(
        workflow_name="novel_writing",
        initial_inputs={
            "chapter_num": 1,
            "outline": {"chapters": [{
                "num": 1, "title": "第一章 测试", "events": ["e1"],
                "word_count_target": 800,  # 短内容加速 HAIKU 处理
            }]},
            "characters": [],
            "memory_context": {},
            "style_guide": {},
            "timeline": [],
            "use_llm": True,
        },  # 跟 Phase 8 现有 pattern (L63-74) 完全一致
        cost_budget_usd=0.10,  # 10x 预期 $0.005-0.020
        max_backtracks=0,
    )

    summary = result["summary"]
    # 1. 完整跑通
    assert summary.completed >= 5, f"期望 ≥5 completed (5 LLM 节点), 实际 {summary.completed}"
    assert summary.failed == 0, f"期望 0 failed (under-budget), 实际 {summary.failed}"
    # 2. cost 在预算内
    total_cost = cost_tracker.total_cost()
    assert 0 < total_cost < 0.10, f"期望 0 < cost < 0.10, 实际 {total_cost}"
    # 3. finally reset
    assert master._current_budget_usd is None, f"期望 finally reset to None, 实际 {master._current_budget_usd}"
```

### 4. Test 2: over-budget abort path

```python
@_REQUIRES_ANTHROPIC_KEY
def test_run_workflow_over_budget_raises(self, tmp_path: Path) -> None:
    """Phase 8.9: real LLM run with sub-cent budget, 验证 CostBudgetExceeded + FAILED + finally reset."""
    from infra.ai_service.cost_tracker import CostTracker
    from infra.ai_service.cost_tracker import CostBudgetExceeded
    cost_tracker = CostTracker()
    master = make_master_with_router(
        state_dir=tmp_path,
        router=_make_real_router("anthropic"),
        cost_tracker=cost_tracker,
    )
    result = master.run_workflow(
        workflow_name="novel_writing",
        initial_inputs={
            "chapter_num": 1,
            "outline": {"chapters": [{
                "num": 1, "title": "第一章 测试", "events": ["e1"],
                "word_count_target": 800,
            }]},
            "characters": [],
            "memory_context": {},
            "style_guide": {},
            "timeline": [],
            "use_llm": True,
        },  # 跟 Phase 8 现有 pattern 一致
        cost_budget_usd=0.0001,  # sub-cent, 必超
        max_backtracks=0,
    )

    summary = result["summary"]
    # 1. 至少 1 node FAILED (触阈那个)
    assert summary.failed >= 1, f"期望 ≥1 FAILED, 实际 completed={summary.completed} failed={summary.failed}"
    # 2. workflow 中止, 不全跑 (Phase 7.4 = 7 节点)
    total = summary.completed + summary.failed
    assert total < 7, f"期望 total < 7 (workflow 被中止), 实际 {total}"
    # 3. cost 累加 > budget (触阈发生)
    assert cost_tracker.total_cost() > 0.0001
    # 4. finally reset 在 raise 后仍生效
    assert master._current_budget_usd is None, f"期望 finally reset after raise, 实际 {master._current_budget_usd}"
    # 5. 至少 1 笔 record 写入
    assert len(cost_tracker.records()) >= 1
```

### 5. 关键设计点

1. **Inline inputs 复用 Phase 8 现有 pattern**: 7 字段 dict literal 跟 Phase 8 L63-74 完全一致,新 class 引用相同 shape (无 class-level const, inline 更可读)。
2. **不 assert 精确 USD**: HAIKU pricing 波动, `0 < cost < 0.10` 和 `cost > 0.0001` 容忍误差。
3. **`max_backtracks=0`**: 跟 Phase 8 一致 fail-fast,避免 retry 增加 cost (单次 $0.01 上限可控)。
4. **YAGNI on cross-provider**: budget 行为 provider-agnostic,1 provider 足证;不引入 OpenAI/MiniMax test classes。
5. **复用 helper**: `_make_real_router` + `make_master_with_router` 现有 (Phase 7.1 + 8.2),0 新增 infra。
6. **不 import `_UsageRecordingProvider`**: T7 stub e2e 已用,这里只验 budget 触阈,真 router 走真 usage 更直接。

## Risks + Mitigations

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| HAIKU pricing 变化致 under-budget 触阈 (变贵) | 低 | 测试 fail | 用 0.10 = 10x 预期 $0.01 上限,留 5-10x safety margin; 跟随主公 manual re-tune |
| 0.0001 budget + Python 浮点精度 | 极低 | 测试 fail | `> 0.0001` 严格大于, 任意 positive cost 必触发;cost_tracker 走 Decimal-ish 加法, 无浮点 loss |
| ANTHROPIC_API_KEY leak | 0 | secret 暴露 | 0 .env 改, 0 API key 引用, 跟 Phase 8/8.1/8.2/8.3 一致; CI secret via GitHub UI |
| CI 月度成本涨 (+$1.50) | 确定 | 成本增加 | 接受 (跟 Phase 8 ~$0.75/月 同量级, 总 ~$2.25/月, 主公预算可控) |
| `_current_budget_usd` finally reset 漏测 | 低 | coverage gap | 2 tests 都断言 `is None`, 覆盖 success + exception 双路径 |
| `MINIMAL_INPUTS` 跟 Phase 8 不同步 | 极低 | 测试 fail | 显式 import / inline 跟 Phase 8 现有 const 完全一致, 注释引用 |
| `_make_real_router("anthropic")` 在 CI 不可用 | 极低 | CI fail | CI 已配 ANTHROPIC_API_KEY secret, skipif 默认 SKIP 不跑 |
| New 2 tests 跑超时 (CI timeout) | 低 | CI fail | 跟 Phase 8 同 ~30-60s, 现有 CI timeout 充足; 不引额外 retry |
