---

## 一、背景与目标

### 1.1 现状问题

当前小说工厂的大纲生成环节依赖纯人工：
- 灵感A/B/C 各自生成部分 → 主编整合 → 基础层.yaml + 深度层.md
- 无结构模板辅助，新项目需要从零设计
- 大纲生成后无法动态调整，写作偏离只能事后补救
- 无多方案对比，作者选择有限

### 1.2 优化目标

构建一个 AI 导演体系，覆盖四个核心能力：
- **A) 多方案生成**：从一句话梗概自动生成多套大纲方案
- **B) 方案选择**：提供多套方案供作者选择
- **C) 动态调整**：写作过程中实时调整大纲
- **D) 反馈优化**：根据读者反馈自动优化大纲

同时内置结构模板（三幕式/英雄之旅/雪花写作法），降低新项目启动门槛。

---

## 二、技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| AI 导演引擎 | LLM 调用（结构化输出） | 需要理解创意并生成多套方案 |
| 结构模板 | 规则引擎 + LLM | 模板匹配 + 语义调整 |
| 动态调整 | LLM 判断 + 规则匹配 | 需要理解上下文并提出调整 |
| 反馈优化 | LLM 分析 + 规则执行 | 需要理解读者意图并映射到具体章节 |
| 部署环境 | Python + Docker | 与现有环境一致 |

### 4.3 能力 C：动态大纲调整

```yaml
dynamic_adjustment:
  trigger_conditions:
    - "作家写偏方向（前10章偏离方案）"
    - "读者反馈负面（某章节评价过低）"
    - "情节逻辑冲突（后续章节与前文矛盾）"

  adjustment_types:
    - type: "节奏调整"
      description: "某阶段太慢/太快，调整章节分配"
      example: "阶段3节奏太慢，在ch060-065加入一场战斗"

    - type: "角色弧线修正"
      description: "角色行为偏离预设，调整后续情节"
      example: "林夜在ch040表现得太善良，调整ch050-060使其更冷峻"

    - type: "伏笔补充"
      description: "某伏笔未回收，新增章节补充"
      example: "ch020埋设的伏笔未回收，在ch070新增暗示"

    - type: "结局调整"
      description: "前期走向改变，相应调整结局方案"
      example: "林夜在前30章表现得太黑暗，调整结局为喜结局"
```

### 4.4 能力 D：读者反馈优化

```yaml
feedback_optimization:
  input: "读者反馈（可以是定性描述）"

  workflow:
    1. 主控展示调整建议
    2. 作者（主公）确认/修改/拒绝
    3. 确认后，大纲自动更新
    4. 后续章节按新大纲生成
```

## 六、AI 导演模块设计

### 6.1 核心模块

```yaml
ai_director_modules:
  - name: "PlotGenerator"
    function: "从梗概生成多套方案"
    input: "一句话梗概"
    output: "3-5个备选方案（每个含方向+卷纲+章节目录）"

  - name: "StructureTemplateEngine"
    function: "应用结构模板生成章节骨架"
    input: "选择的方案 + 模板类型"
    output: "章节骨架（节点+转折点+高潮）"

  - name: "OutlineExpander"
    function: "展开为详细大纲"
    input: "章节骨架"
    output: "卷纲 → 阶段大纲 → 章纲"

  - name: "DynamicAdjustmentEngine"
    function: "写作过程中动态调整"
    input: "当前写作状态 + 触发条件"
    output: "调整建议或自动调整"

  - name: "FeedbackOptimizer"
    function: "读者反馈驱动的优化"
    input: "读者反馈"
    output: "调整建议"

  - name: "InspirationIntegrator"
    function: "与灵感部门协作"
    input: "AI生成结果 + 灵感部门反馈"
    output: "整合后的立项包"
```

### 6.2 数据流

```
一句话梗概
     ↓
PlotGenerator → 3-5个方案
     ↓
作者选择方案
     ↓
StructureTemplateEngine → 章节骨架（节点+转折点+高潮）
     ↓
OutlineExpander → 卷纲 → 阶段大纲 → 章纲
     ↓
作家写作（章纲 → 正文）
     ↓
DynamicAdjustmentEngine → 监控写作状态，触发调整
     ↓
FeedbackOptimizer → 分析读者反馈，生成调整建议
     ↓
作者确认 → 大纲更新 → 后续章节按新大纲生成
```

## 七、与灵感部门的集成

### 7.1 协作模式

```
现有：灵感A/B/C → 人工生成大纲 → 主编整合 → 基础层.yaml + 深度层.md

新增：AI导演辅助 → 自动生成方案 → 人工选择/调整 → 确认后展开大纲

协作模式：
  1. AI导演先生成N套方案（如3套）
  2. 灵感A/B/C 分别审阅方案，提供意见
  3. 作者（主公）选择方案或要求调整
  4. 确认后，AI导演展开为详细大纲（卷纲→阶段大纲→章纲）
  5. 灵感部门在此基础上润色/补充/优化
```

### 7.2 职责分工

| 角色 | 职责 |
|------|------|
| AI 导演 | 生成方案、展开大纲、动态调整、反馈分析 |
| 灵感A（类型专家） | 审阅方案的类型/市场定位，给出意见 |
| 灵感B（幻想专家） | 审阅方案的世界观/设定，给出意见 |
| 灵感C（结构专家） | 审阅方案的叙事结构/伏笔，给出意见 |
| 灵感主编 | 汇总意见，整合立项包，版本管理 |
| 作者（主公） | 最终决策，选择方案，确认调整 |

## 十一、关键设计决策

| 决策 | 说明 |
|------|------|
| AI + 人工协作 | AI生成方案，人工审核选择，灵感部门润色 |
| 作者最终决策 | 方案选择、大纲调整均需作者（主公）确认 |
| 结构模板可选 | 作者可选择已有模板或自定义 |
| 动态调整需确认 | 写作过程中的调整建议需作者确认，不能自动执行 |
| 反馈优化阈值 | 可设置"自动批准"阈值，小幅调整自动执行 |
