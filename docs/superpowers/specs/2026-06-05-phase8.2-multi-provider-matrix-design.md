## Feature A: 抽 `_make_real_router(provider_name)` 到 `_e2e_helpers.py`

**当前**: `_make_real_router()` 是 `tests/agent_system/test_novel_writing_real_llm.py:27-44` 模块级函数, 硬编码 anthropic.

**新设计**: 移去 `tests/agent_system/_e2e_helpers.py`, 接受 `provider_name: str` 参数, 派发到对应 provider class:

```python
# tests/agent_system/_e2e_helpers.py (新增)

_PROVIDER_REGISTRY = {
    "anthropic": {
        "class_path": "infra.ai_service.anthropic_provider.AnthropicProvider",
        "env_var": "ANTHROPIC_API_KEY",
        "default_model": "claude-haiku-4-5-20251001",
    },
    "openai": {
        "class_path": "infra.ai_service.openai_provider.OpenAIProvider",
        "env_var": "OPENAI_API_KEY",
        "default_model": "gpt-4o-mini",
    },
    "minimax": {
        "class_path": "infra.ai_service.minimax_provider.MiniMaxProvider",
        "env_var": "MINIMAX_API_KEY",
        "default_model": "MiniMax-M2.7",
    },
}


def _make_real_router(provider_name: str) -> "AIRouter":
    """Phase 8.2: 构造单 provider 的 AIRouter (multi-provider matrix).

    Why: Phase 8 只测 Anthropic, Phase 8.2 加 OpenAI + MiniMax 跨 provider
    验证 Phase 7.5 polish_merge 评分在多 provider 都能走 LLM 路径.
    抽到 _e2e_helpers.py 是因为现在 3 个 provider 共用同一个 helper, 分散在
    多个 test 文件会重复代码.

    Args:
        provider_name: "anthropic" / "openai" / "minimax" (lowercase, 跟
                        @register_provider 注册名一致)

    Returns:
        AIRouter with single provider, primary=provider_name, enable_failover=False

    Raises:
        ValueError: 不支持的 provider_name
        KeyError: 缺少对应 env var (e.g. OPENAI_API_KEY not set)
    """
    if provider_name not in _PROVIDER_REGISTRY:
        raise ValueError(
            f"Unsupported provider_name: {provider_name!r}. "
            f"Supported: {list(_PROVIDER_REGISTRY.keys())}"
        )
    spec = _PROVIDER_REGISTRY[provider_name]
    api_key = os.environ.get(spec["env_var"])
    if not api_key:
        raise KeyError(
            f"Provider {provider_name!r} requires env var {spec['env_var']!r} "
            f"to be set"
        )

    # Lazy import 避免循环 + 允许 provider 缺依赖时 (e.g. openai 包未装) fail gracefully
    import importlib
    module_path, class_name = spec["class_path"].rsplit(".", 1)
    module = importlib.import_module(module_path)
    provider_cls = getattr(module, class_name)

    provider = provider_cls(ProviderConfig(
        api_key=api_key,
        model=spec["default_model"],
        timeout=180,  # Phase 8.2 沿用 180s headroom (1 LLM call 通常 < 30s)
        max_retries=1,  # fail-fast
    ))
    return AIRouter(
        config={provider_name: provider.config},
        primary_provider=provider_name,
        enable_failover=False,  # 单 provider, 失败立即 raise
    )
```

**YAGNI 检查**:
- 不加 `endpoint` 参数支持 MiniMax 自定义 host — `MiniMaxProvider` 内部已读 `MINIMAX_API_HOST` env var
- 不加 `model` override 参数 — 各 provider default model 已 OK
- 不抽到 `infra/ai_service/` — 仍属测试 helpers, 放 `_e2e_helpers.py` 跟 stub helpers 并列

```python
@_REQUIRES_OPENAI_KEY
class TestNovelWritingRealLLMOpenAI:
    """Phase 8.2: OpenAI 真实 LLM E2E 验证 polish_merge 跨 provider 兼容.

    跟 TestPolishMergeSynthesisRealLLM (Anthropic) 同样的 9 步断言.
    Cost: gpt-4o-mini 1 LLM call ≈ $0.001-0.003 (7-8x cheaper than HAIKU).
    """

    def test_polish_merge_synthesis_with_distinct_contents(self, tmp_path: Path) -> None:
        from tests.agent_system._e2e_helpers import make_master_with_router, _make_real_router

        router = _make_real_router("openai")
        master = make_master_with_router(state_dir=tmp_path, router=router)

        content_a = "少年握紧拳头, 眼中燃烧着不甘。寒风刺骨, 但他咬牙前行。" * 20
        content_b = "林夕深吸一口气, 心中暗自立下誓言。纵然前路艰险, 亦不退缩。" * 20

        result = master.polish_merge_synthesis(
            content_a, content_b, labels=("emotional", "motivation"),
        )

        # 9 步断言 (跟 TestPolishMergeSynthesisRealLLM 一致)
        assert result["fallback"] is None, (
            f"polish_merge_synthesis (OpenAI) 应走 LLM 路径, 但 fallback={result['fallback']!r}"
        )
        _assert_s1_s8_score_dict(result["scores_a"], "scores_a")
        _assert_s1_s8_score_dict(result["scores_b"], "scores_b")
        assert result["winner"] in result["_labels"]
        assert 0.0 <= result["scores_total_a"] <= 10.0
        assert 0.0 <= result["scores_total_b"] <= 10.0
        assert abs(
            result["scores_total_a"] - result["scores_total_b"] - result["scores_delta"]
        ) < 1e-6
        assert result["_labels"] == ["emotional", "motivation"]
```

## Risks + 缓解

| 风险 | 缓解 |
|------|------|
| OpenAI gpt-4o-mini 输出 S1-S8 偶尔越界 (e.g. S1=15) | Phase 8.1 `_coerce_score` clamp [0,10], 测试断言 `0 <= score <= 10` 仍过 |
| MiniMax M2.7 返 thinking block 导致 response 文本截断 | `minimax_provider.py:99-103` 已处理 thinking → empty, 测试遇 `polish_merge_synthesis` 返空 content → `identical` fallback (content_a == content_b) 或 `llm_fail`, 都会 fail 当前 `assert fallback is None`. Phase 8.2 接受这是 "MiniMax 不支持 polish_merge" 的信号, 失败暴露即修 |
| gpt-4o-mini 比 HAIKU 慢 (国内代理 / rate limit) | `timeout=180, max_retries=1` fail-fast, 1 LLM call 期望 < 30s |
| MiniMax API host 不稳定 (海外访问慢) | 同样 fail-fast, 主公可选择不跑 MiniMax test (不设 MINIMAX_API_KEY) |
| `_make_real_router` 接受任意 provider_name, 拼错静默 fallback | `ValueError` 显式 raise, 测试不用 try/except |
| lazy import 隐藏依赖缺失 (e.g. `import openai` 失败) | `importlib.import_module` 失败抛 `ImportError`, 测试本身在 `_REQUIRES_XXX_KEY` 跳过前就 fail 清晰 |
| 3 provider 用同 `content_a` / `content_b` 测试数据, LLM 评分可能因 prompt 微变差 | 内容故意保持简单 (400 chars × 20 = 8000 chars), Phase 8.1 truncation 6000 后评分仍合理 |
