## 一、部门架构

### 1.1 成员与职责

| 角色 | 编号 | 核心职责 | 特殊权限 |
|------|------|---------|---------|
| 作家主编 | 从A-J选 | 任务分配、进度监控、跨阶段协调、格式检查、向主控汇报 | 可参与创作，但优先主编职责 |
| 作家 | A/B/C/D/E/F/G/H/I/J | 负责1-2个阶段创作（20-40章），按批次产出，自评+S/A/B分级 | 无 |

### 1.2 任务分配规则

- 按阶段分配：每个作家负责1-2个阶段（20-40章）
- 分配考虑因素：作家能力评分、擅长类型、项目进度
- 调整权限：作家主编提出调整申请，主控审批

### 1.3 主编职责边界

**主编负责：**
- 任务分配与进度监控
- 跨阶段衔接协调（作家间的过渡段落）
- 格式检查（文件命名、章节结构）
- 向主控汇报进度和问题

**主编不负责：**
- 质量评分（由审核部门负责）
- 创作决策（由各作家负责）
- 直接与灵感部门沟通（通过主控协调）

### 1.4 并行执行原则

```
10个作家可同时创作不同章节
每人按自己节奏产出，10章为一批提交
主编统一协调跨阶段衔接
```

## 二、质量评估体系

### 2.1 S/A/B三级评分标准

| 等级 | 完整度要求 | 特征 | 处理方式 |
|------|-----------|------|---------|
| **S级** | >90% | 可直接定稿，无需修改 | 提交审核，标记为高质量 |
| **A级** | 70%-90% | 需小幅修改，方向正确 | 提交审核，附自评说明修改点 |
| **B级** | 50%-70% | 需较大修改，框架可行 | 修改后再自评 |
| **不合格** | <50% | 需重写，方向偏离 | 打回重写，记录1次返工 |

### 2.2 自评机制

**流程：**
1. 作家完成章节后，按S/A/B标准自评
2. 自评时标注：达标/不达标 + 自评意见
3. 提交审核时附带自评结果
4. 审核部门给出最终评分和反馈

**自评与审核评分对比：**
- 记录进作家能力档案
- 差异过大时（审核B级但自评S级）触发复盘
- 帮助识别作家自我认知偏差

### 2.3 评分维度（作家能力档案用）

每个作家的能力评分维度：

| 维度 | 占比 | 评分标准（1-10分） |
|------|------|-----------------|
| 逻辑一致性 | 25% | 时间线、动机、因果链是否自洽 |
| 文笔流畅度 | 25% | 语言表达、节奏把控、可读性 |
| 人物塑造 | 25% | 性格一致、成长有依据、立体感 |
| 伏笔管理 | 25% | 伏笔铺设与回收、前后呼应 |

## 三、与灵感部门对接

### 3.1 对接文件

作家读取灵感部门产出的双层文件：

| 文件 | 用途 | 读取频率 |
|------|------|---------|
| **基础层.yaml** | 主要创作参照：类型/主题/人物/风格禁忌/卖点 | 创作时全程参照 |
| **深度层.md** | 世界观/审核参照：世界观体系/时间线/伏笔设计/叙事结构 | 创作时参考，审核前自查 |

### 3.2 疑问处理流程

```
作家在创作中发现灵感描述不清
    ↓
通过主控提交问题
    ↓
主控协调灵感部门解答
    ↓
灵感部门通过主控回复
    ↓
主控转达给作家
```

**禁止：** 作家直接向灵感部门发起追问

### 3.3 创作启动检查清单

作家开始创作前需确认：
- [ ] 已读取基础层.yaml，理解项目类型/主题/人物/风格
- [ ] 已读取深度层.md，了解世界观和时间线
- [ ] 已确认章节范围和交稿时间节点

## 四、返工优化机制

### 4.1 小批快返制度

| 参数 | 值 | 说明 |
|------|-----|------|
| 批次大小 | 10章 | 每批完成后立即提交审核 |
| 提交节奏 | 完成后即提交 | 不等待整卷完成 |
| 审核节奏 | 10章/批 | 审核部门批量处理 |

**流程：**
```
ch001-ch010 完成
    ↓
自评S/A/B级
    ↓
主编格式检查（1分钟内）
    ↓
提交审核部门
    ↓
审核部门批量审核（10章）
    ↓
发现问题立即修正（不累积）
```

### 4.2 返工记录机制

**记录内容：**
- 返工时间
- 涉及章节
- 返工原因（方向偏差/逻辑错误/文笔问题/其他）
- 修改结果

**触发处理：**

| 触发条件 | 处理方式 |
|---------|---------|
| 单次返工 | 记录进档案，下次创作注意 |
| 连续3次返工 | 主控介入评估，与作家复盘 |
| 返工率>30%（任意20章周期） | 降级处理，重新分配任务 |

### 4.3 返工率计算

```
返工率 = 返工章节数 / 提交章节总数 × 100%
```

例：提交40章，返工6章，返工率=6/40×100%=15%

## 五、作家能力档案

### 5.1 档案结构

```yaml
作家A:
  # 能力评分（1-10分）
  能力评分:
    逻辑一致性: 8
    文笔流畅度: 9
    人物塑造: 7
    伏笔管理: 6

  # 产出统计
  产出统计:
    完成章节: 40
    返工次数: 6
    返工率: 15%
    通过率:
      S级: 20%  # 8章
      A级: 60%  # 24章
      B级: 20%  # 8章

  # 状态
  状态: 正常  # 正常/关注/降级
  最后更新: 2026-05-14
```

### 5.2 档案更新规则

| 数据类型 | 更新时机 | 更新方式 |
|---------|---------|---------|
| 完成章节数 | 每提交一批 | 自动+1 |
| 返工次数 | 每次返工 | 自动+1 |
| 能力评分 | 每5批次复盘 | 主控评估调整 |
| 状态 | 触发条件时 | 主控调整 |

### 5.3 状态流转

```
正常 → 关注（连续3次返工或返工率>20%）
关注 → 降级（连续5次返工或返工率>30%）
降级 → 正常（连续3批次无返工）
```

## 六、与审核部门对接

### 6.1 提交节奏

| 参数 | 值 |
|------|-----|
| 批次大小 | 10章 |
| 提交时机 | 完成后立即提交 |
| 审核节奏 | 10章/批，批量处理 |

### 6.2 主编格式检查清单

主编在转交审核前检查：

- [ ] 文件命名规范：`ch{编号}.md`（如ch001.md）
- [ ] 章节结构完整：开头/中段/结尾
- [ ] 字数符合要求：3000±200字/章
- [ ] 无乱码或格式错误

**主编仅检查格式，不做质量评分。**

### 6.3 转交流程

```
作家完成ch001-ch010
    ↓
作家自评（S/A/B）
    ↓
主编格式检查（1分钟内）
    ↓
格式通过 → 转交审核部门
格式不通过 → 返回作家修正
    ↓
审核部门批量审核（10章）
    ↓
审核结果反馈：S/A/B + 修改意见
    ↓
有修改意见 → 作家返工 → 重新提交
无修改意见 → 定稿，进入下一批次
```

### 6.4 对接文件清单

| 文件 | 路径 | 说明 |
|------|------|------|
| 正文章节 | `03_内容仓库/04_正文/ch{编号}.md` | 作家产出 |
| 自评报告 | `06_意见仓库/04_正文_审核/作家自评_{编号}.md` | 随章节提交 |
| 审核意见 | `06_意见仓库/04_正文_审核/审核意见_{编号}.md` | 审核产出 |

## 七、工作流程总览

### 7.1 单批次流程（10章）

```
T+0: 作家开始创作ch001-ch010
T+3: 完成ch001-ch010，作家自评S/A/B
T+3.1: 主编格式检查
T+3.2: 转交审核部门
T+6: 审核部门完成审核，给出S/A/B+意见
T+6.1: 有意见→作家返工→重新提交
T+6.2: 无意见→定稿
```

### 7.2 多批次并行

```
作家A: [批次1完成] → [批次2进行中]
作家B: [批次1完成] → [批次2进行中]
...
主编: 监控所有作家进度，协调跨阶段衔接
```

### 7.3 跨阶段协调

当章节需要跨阶段衔接时：
1. 作家提交衔接段落给主编
2. 主编协调相邻阶段作家
3. 确保过渡自然，连贯一致

## 九、关键规则汇总

| 规则 | 说明 |
|------|------|
| 任务分配 | 按阶段分配（1-2个阶段/作家，20-40章） |
| 主编职责 | 任务分配+进度监控+格式检查+跨阶段协调 |
| 质量评估 | S/A/B三级，作家自评+审核最终评分 |
| 自评机制 | 完成后自评，对比记入档案 |
| 提交节奏 | 每10章为一批 |
| 返工记录 | 每次记录，连续3次触发主控介入 |
| 档案更新 | 完成章节自动+1，返工自动+1，评分定期复盘 |
| 对接灵感 | 基础层+深度层全套，有疑问通过主控追问 |
| 对接审核 | 批次提交+主编格式检查 |
