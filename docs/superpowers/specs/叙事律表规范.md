# 叙事律表规范（Narrative X-Sheet）

> **来源**：动画制作律表（X-Sheet）概念
> **作用**：以章节为单位的时间轴追踪工具，记录创作状态和进度，便于发现跨章节问题

---

## 一、定义与目的

叙事律表是以章节为单位的时间轴追踪工具，记录每日创作状态、进度和问题。

**与现有"每日创作Review"的区别**：
| 维度 | 每日Review | 叙事律表 |
|------|------------|----------|
| 时间跨度 | 单日 | 跨章节/跨周 |
| 记录内容 | 当日复盘 | 章节进度+连续性追踪 |
| 侧重点 | 自我反省 | 结构性问题发现 |
| 使用者 | 作者自用 | 作者+主控共查看 |

**核心目的**：让跨章节的连续性问题（如连续多天节奏偏慢）无处藏身。

---

## 二、格式模板

### 2.1 基础律表格式

```markdown
## 叙事律表

| 日期 | 章节 | 字数 | 状态 | 节奏 | 自检发现 | 明日重点 |
|------|------|------|------|------|----------|----------|
| 05-19 | Ch15 | 3200 | 最佳 | P2 | 节奏略慢，需加快 | 加快Ch16节奏 |
| 05-20 | Ch16 | 2800 | 正常 | P1 | 冲突够，但过渡少 | 增加过渡描写 |
```

### 2.2 状态评级

| 状态 | 标记 | 说明 |
|------|------|------|
| 最佳 | 😊 | 状态好，思路清晰，效率高 |
| 正常 | 🙂 | 状态一般，按计划推进 |
| 低效 | 😐 | 状态差，卡文或效率低 |
| 疲劳 | 😫 | 需要休息，强制休息日 |

### 2.3 节奏标注

| 节奏 | 标记 | 说明 |
|------|------|------|
| 快节奏 | P1 | 短句、动作、冲突密集 |
| 中节奏 | P2 | 平衡叙述和动作 |
| 慢节奏 | P3 | 描写、内心独白、氛围 |

## 三、关键指标追踪

### 3.1 周进度追踪

```markdown
## 周进度追踪（每周一更新）

### Week 20 (05-13 ~ 05-19)

| 指标 | 计划 | 实际 | 达成率 |
|------|------|------|--------|
| 字数 | 20000 | 18500 | 92.5% |
| 章节 | 8章 | 7章 | 87.5% |
| 状态最佳天数 | 5天 | 4天 | 80% |

**问题识别**：
- 节奏偏慢连续3天（Ch13-15都是P3）
- 周末效率下降明显

**调整方案**：
- 下周节奏目标：P1:P2:P3 = 4:3:1
- 周末减少每日目标字数
```

### 3.2 连续性问题识别

```markdown
## 连续性问题记录

### 问题1：节奏持续偏慢
- 发现时间：05-19
- 影响章节：Ch13, Ch14, Ch15
- 状态：已识别，调整中

### 问题2：打斗场景拖沓
- 发现时间：05-20
- 影响章节：Ch9, Ch11, Ch15
- 状态：待解决
```

## 四、使用规范

### 4.1 记录时机

```
每日创作结束后填写叙事律表
每周一更新周进度追踪
发现连续性问题时立即记录
```

### 4.2 填写要求

1. **字数**：当日完成的实际字数（含修改）
2. **状态**：如实记录，不美化
3. **自检发现**：记录影响后续的问题，不仅仅是当日问题
4. **明日重点**：具体可执行的任务

### 4.3 主控查看

主控在以下时机查看叙事律表：
- 每日调度前
- 周审时
- 发现进度异常时

---

## 五、问题发现机制

### 5.1 自动识别规则

| 规则 | 触发条件 | 处理方式 |
|------|----------|----------|
| 节奏连续偏慢 | 连续3天节奏=P3 | 标记警告，主控介入 |
| 进度落后 | 周达成率<80% | 主控催办或调整计划 |
| 状态持续差 | 连续3天状态=低效/疲劳 | 强制休息日 |
| 字数异常 | 单日字数<1000 | 识别卡文，介入支持 |

### 5.2 主控行动触发

```
警告 → 主控询问原因 → 提供支持 → 跟踪调整
```

## 六、与其他模块的接口

- **动态大纲**：节奏标注来源
- **叙事节奏分级**：状态和节奏评级标准
- **每日创作Review**：数据来源（每日Review汇总到律表）
- **创作阶段锁定**：律表异常可触发锁定评估

## 七、示例

```markdown
## 《修仙逆袭录》叙事律表

### 周追踪

| 日期 | 章节 | 字数 | 状态 | 节奏 | 自检发现 | 明日重点 |
|------|------|------|------|------|----------|----------|
| 05-15 | Ch10 | 3500 | 😊 | P2 | 节奏良好 | 完成Ch11 |
| 05-16 | Ch11 | 3200 | 🙂 | P1 | 打斗紧凑 | 完成Ch12 |
| 05-17 | Ch12 | 2800 | 😐 | P3 | 卡文，日常写不动 | 换P1场景写 |
| 05-18 | Ch12 | 3600 | 😊 | P1 | 换场景后顺了 | 完成Ch13 |
| 05-19 | Ch13 | 3400 | 🙂 | P2 | 正常推进 | 完成Ch14 |

### 周报

**完成**：5章/计划6章 = 83%
**问题**：Ch12卡文一次
**调整**：根据律表发现问题，下周增加P1场景比例
```
