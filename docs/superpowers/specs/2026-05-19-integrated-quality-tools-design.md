## 1. 概述与目标

### 1.1 问题陈述

当前小说工厂缺乏辅助写作工具，存在以下问题：

| 问题 | 说明 |
|------|------|
| 卡文无解 | 写作遇到瓶颈无法突破 |
| 章纲扩展难 | 从大纲到正文需要大量人工工作 |
| 缺乏分析工具 | 无法分析已有作品的学习价值 |
| 文学手法缺失 | 高级写作技法缺乏系统支持 |

### 1.2 解决方案

构建**整合质量工具集**，统一以下原有方案：
- ~~数据闭环~~ → 热梗素材抓取

- ~~卡文突破模块~~

## 2. 工具清单

### 2.1 卡文突破工具

```yaml
writer_block_toolkit:
  功能:
    - 情节分支建议
    - 角色冲突激发
    - 场景转换提示
    - 悬念设置建议

  触发场景:
    - 连续3次生成不通过
    - 作家手动触发

  输出:
    - 5-10个情节分支选项
    - 每个选项的预期效果
```

### 2.2 章纲扩展工具

```yaml
chapter_expansion_toolkit:
  功能:
    - 将大纲扩展为完整章节
    - 保持人设和文风一致
    - 自动注入伏笔和钩子

  输入:
    - chapter_outline
    - character_cards
    - style_guide
    - memory_context

  输出:
    - 完整章节草稿
    - 字数统计
    - 伏笔埋设报告
```

### 2.3 AI拆书工具

```yaml
ai_book_analysis_toolkit:
  功能:
    - 分析书籍结构
    - 提取人物弧光
    - 识别节奏模式
    - 生成学习报告

  输入:
    - 书籍文本/大纲

  输出:
    - 结构分析报告
    - 节奏图谱
    - 可学习的写作技法
```

### 2.4 热梗素材抓取

```yaml
hot_topic_toolkit:
  功能:
    - 抓取当前热门梗
    - 分析读者评论情绪
    - 提供创作素材建议

  数据来源:
    - 社交媒体热榜
    - 读者评论池
    - 竞品分析
```

### 2.5 文学手法分析

```yaml
literary_device_toolkit:
  功能:
    - 识别文本中的文学手法
    - 分析隐喻、象征、伏笔
    - 提供改进建议

  支持手法:
    - 隐喻与象征
    - 伏笔与呼应
    - 悬念与反转
    - 情感渲染
```
