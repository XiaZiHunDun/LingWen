# Phase 8.3 Design: Real LLM Tests CI Integration

> **For agentic workers:** This is a design document. Use superpowers:writing-plans to create an implementation plan from this spec.

**Date**: 2026-06-05
**Phase**: 8.3 (extends Phase 8 / 8.1 / 8.2)
**Doc Reference**: Doc 4 §11 Phase 8.3

## Context

Phase 8 (commit d6e7c25) 加了 2 opt-in 真实 LLM 测试 (Anthropic only, HAIKU), Phase 8.1 (commit 32f088d) 加了 6 静态防御性修复, Phase 8.2 (commit 150be7c) 加了 OpenAI + MiniMax matrix. **总共 4 个真实 LLM 测试**, 全部默认 SKIP, 需主公手动设 `ANTHROPIC_API_KEY` / `OPENAI_API_KEY` / `MINIMAX_API_KEY` 才跑.

**当前问题**: 真实 LLM 路径可能在 production 改动后悄悄回退. 没有 CI 自动化抓 regression.

**目标**: 加 1 个 GitHub Actions workflow, 24h 内发现 production 路径 regression, 主公 opt-in setup.

## Goals

1. 真实 LLM 路径 regression 24h 内发现 (cron daily + 手动 dispatch)
2. 0 production code 改动 (跟 Phase 8/8.1/8.2 一致)
3. 主公 opt-in setup (一次性手动加 GitHub secret)
4. Cost 可控 (~$0.75/月, 单 provider)
5. Fail-loud on missing secret (vs skipif 静默 SKIP)

## Non-Goals (YAGNI)

- ❌ CI 接 Slack webhook (GitHub 默认邮件够用)
- ❌ 自动创 GitHub issue
- ❌ 跑 OpenAI / MiniMax 在 CI (单 Anthropic 优先, cost 节省)
- ❌ Workflow 内 cost cap (主公账号已有 spend limit 假设)
- ❌ Matrix 3 Python version (3.13 only, 节省 2/3 cost)
- ❌ CI badge in README
- ❌ Run on push/PR (cost 担心, 主公明确说"不在 push/PR 跑")

## Design

### File: `.github/workflows/real-llm-tests.yml` (~70L)

**Trigger**:
```yaml
on:
  schedule:
    - cron: '0 2 * * *'  # UTC 02:00 daily = 北京 10:00
  workflow_dispatch:
```

**Job**:
- 1 job (`real-llm-tests`), ubuntu-latest, Python 3.13 only
- 7 steps: checkout → check secret → setup python → install → run tests → ruff (informational) → upload artifact

**Tests selected**:
- `TestNovelWritingRealLLM` (6 LLM calls, production pipeline 端到端)
- `TestPolishMergeSynthesisRealLLM` (1 LLM call, Phase 7.5 核心)
- 不跑 OpenAI / MiniMax (无 secret, 默认 SKIP, 浪费启动时间)

**Cost**:
- 单 run: ~$0.025 (HAIKU 6 + Sonnet 1 LLM calls)
- 月成本: $0.75 (30 daily runs)
- 手动 dispatch × N: 叠加

**Fail-loud on missing secret**:
```yaml
- name: Check ANTHROPIC_API_KEY secret
  run: |
    if [ -z "${{ secrets.ANTHROPIC_API_KEY }}" ]; then
      echo "::error::ANTHROPIC_API_KEY secret is not set in repo Settings → Secrets → Actions"
      exit 1
    fi
```

vs 让 skipif 静默 SKIP (会误以为成功).

**Concurrency**:
```yaml
concurrency:
  group: real-llm-tests-${{ github.ref }}
  cancel-in-progress: true
```

处理 manual dispatch + cron 重叠.

### File: `docs/CI_SETUP.md` (~80L)

主公手动 setup 说明, 含:
- 加 ANTHROPIC_API_KEY secret
- 手动 dispatch 验证
- 失败 troubleshooting
- 禁用 workflow 步骤
- 成本估算 + 安全注意

## Cost Analysis

| Component | Per call | Per run | Per month (30 runs) |
|-----------|----------|---------|---------------------|
| HAIKU 6 calls (polish_emotional_pacing × 2 + polish_ai_trace_removal dialogue + polish_merge synthesis + ...) | $0.001 | $0.006 | $0.18 |
| Sonnet 1 call (write_chapter or audit_chapter, polisher default) | $0.01 | $0.01 | $0.30 |
| 4 skipped tests (OpenAI/MiniMax, no key) | $0 | $0 | $0 |
| **Total** | - | **$0.016** | **$0.48** |

(更保守估算: $0.025/run = $0.75/月, 给真实 LLM 价格波动留 buffer)

## Risks + Mitigations

| Risk | Mitigation |
|------|------------|
| 主公没设 `ANTHROPIC_API_KEY` secret → 测试 SKIP → 误以为成功 | fail-loud check (workflow 第一步), secret 缺失 → job 立刻 fail + error message |
| 真实 LLM 偶尔 flaky (Anthropic timeout) → CI 假阳性 fail | max_retries=1 (Phase 8 helper 已配), 偶发 1 retry 容忍, 持续 fail → 主公修 |
| 跑挂没人看 → regression 累积 | GitHub email 默认通知 (主公账号绑定), workflow run 出现在 Actions tab |
| 成本失控 (spend cap 失效) | 手动 disable workflow (CI_SETUP.md step 6), 或改 cron 注释 |
| ANTHROPIC_API_KEY 泄漏到 git | GitHub secrets 注入, workflow 文件不写明文, `.env` 在 `.gitignore` |
| ANTHROPIC_API_KEY 泄漏到 PR 公开 log | `${{ secrets.ANTHROPIC_API_KEY }}` 自动 redact, GitHub 安全机制 |
| cron 触发但 secrets 被删除 | fail-loud check 立刻 fail, 主公邮件通知 |
| 并发 (manual + cron 同时跑) | `concurrency.cancel-in-progress: true` |
| Python 3.13 单独跑, 3.11/3.12 兼容性 regression | test.yml matrix 仍然跑 3 version, real-llm-tests.yml 跟 test.yml 互补 |
