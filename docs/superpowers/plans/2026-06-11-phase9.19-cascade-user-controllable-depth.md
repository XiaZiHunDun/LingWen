# Phase 9.19 — Cascade BFS User-Controllable Depth Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add `max_depth` override (1..10) to cascade BFS via endpoint `?max_depth=N`, `lingwen.py cascade` CLI, and `RippleStorage.preview_cascade()` — without changing existing source APIs.

**Architecture:** A new `preview_cascade(ripple_id, max_depth)` storage method re-runs the BFS without persisting. Two FastAPI endpoints get an optional `max_depth` query param. A new `lingwen.py cascade` CLI command wraps the same method. Existing `max_depth=3` default stays unchanged for all existing callers.

**Tech Stack:** Python 3.12 + sqlite3 + argparse (lingwen.py) + FastAPI + Pydantic v2 + pytest 8 + ruff.

---
## Context

Phase 9.15 built `/api/cvg/ripples/{id}/cascade` (persisted BFS). Phase 9.16 added `v2_weighted` with `max_depth=3` hardcoded default inside `trigger_cascade()`. No endpoint or CLI exposes the `max_depth` kwarg — callers must accept whatever depth was recorded at `append_ripple` time.

Phase 9.19 adds a live preview path: re-run BFS with caller-specified depth, bounded 1..10, without writing to `ripple_cascade`.

```python
def preview_cascade(self, ripple_id: str, max_depth: int) -> "CascadedRipple":
    """Phase 9.19: re-run BFS without persisting.

    Args:
        ripple_id: ripple to trace
        max_depth: 1..10 BFS depth cap (must be ≥1, caller validates)

    Returns:
        CascadedRipple from trigger_cascade(max_depth=max_depth)

    Raises:
        KeyError: ripple not found
    """
    ripple = self.get_ripple_by_id(ripple_id)
    if ripple is None:
        raise KeyError(f"Ripple {ripple_id} not found")
    return self._graph.trigger_cascade(ripple, max_depth=max_depth, weighted=True)
```

```python
"""cascade command - Phase 9.19: re-run BFS without persisting.

1:1 with infra/cli/commands/ripple_audit.py (Phase 9.14).
Uses RippleStorage.preview_cascade() (Phase 9.19 Task 1).
"""
from pathlib import Path

from infra.cli.options import CascadeOptions
from infra.cross_volume.storage import RippleStorage

from .base import Command

DEFAULT_RIPPLE_DB = Path(".state/ripple.db")


def _get_storage() -> RippleStorage:
    return RippleStorage(db_path=DEFAULT_RIPPLE_DB)


class CascadeCommand(Command):
    """Re-run cascade BFS and print summary."""

    name = "cascade"
    description = "重新运行涟漪扩散 BFS (Phase 9.19)"

    def execute(self, options: CascadeOptions) -> int:
        ripple_id = options.ripple_id
        max_depth = options.max_depth
        if not ripple_id:
            print("Error: ripple_id is required", file=__import__("sys").stderr)
            return 1
        storage = _get_storage()
        try:
            cascaded = storage.preview_cascade(ripple_id, max_depth=max_depth)
        except KeyError:
            print(f"Error: ripple {ripple_id} not found", file=__import__("sys").stderr)
            return 1
        print(f"Cascade for {ripple_id} (max_depth={max_depth}):")
        print(f"  depth_reached: {cascaded.depth_reached}")
        print(f"  nodes: {len(cascaded.cascade_nodes)}")
        print(f"  edges: {len(cascaded.cascade_edges)}")
        print(f"  actions: {len(cascaded.cascade_actions)}")
        print(f"  algorithm: {cascaded.bfs_algorithm_version}")
        return 0
```
