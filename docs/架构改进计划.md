# 架构改进计划 - P0问题优先

## 状态：进行中

## Context

当前项目存在两个关键P0架构问题：
1. **质量系统碎片化**：`Issue` 和 `QualityReport` 在不同模块有不同定义，rule-based checkers 和 LLM analyzers 无法互通
2. **功能重叠**：34个rule-based checkers 和 10个LLM analyzers 检测范围高度重叠，但无协调机制

目标：统一质量系统数据结构，建立 checker/analyzer 协作机制，消除重复检测。

## 技术细节

### UnifiedIssue 结构

```python
@dataclass
class UnifiedIssue:
    id: str
    severity: UnifiedSeverity  # P0/P1/P2/P3
    source: QualitySource        # RULE_BASED / LLM_ANALYZER
    source_name: str             # "CharacterChecker" / "foreshadow_analyzer"
    issue_type: str
    title: str
    description: str
    location: UnifiedLocation   # chapter, paragraph, line
    evidence: str = ""
    suggestion: str = ""
    dimension: Optional[QualityDimension] = None  # S1-S11
    confidence: float = 0.5
    metadata: Dict[str, Any] = field(default_factory=dict)
```

### 去重逻辑

```python
def _deduplicate_issues(self, issues: List[UnifiedIssue]) -> List[UnifiedIssue]:
    """基于 (chapter + paragraph + line + issue_type + source_name) 去重"""
    seen = {}
    for issue in issues:
        key = (issue.location.chapter, issue.location.paragraph,
               issue.location.line, issue.issue_type, issue.source_name)
        if key not in seen or issue.confidence > seen[key].confidence:
            seen[key] = issue
    return list(seen.values())
```
