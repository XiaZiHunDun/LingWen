# Agent 详细定义

## 主控调度 Agent

**定义文件**: `.claude/agents/主控调度.md`

### 职责
- 任务编排：按照工作流步骤依次调度各部门 Agent
- 状态维护：更新 workflow_state.json，记录当前进度
- 大决策确认：大纲审核通过/不通过、卷/全文汇总定稿、死锁仲裁
- 进度汇报：主动向用户汇报当前状态和风险

### 工具权限
- Read, Write, Edit, Bash
- Agent (启动子 Agent)

### 调度约束
- 最大并发 Agent 数：10 个
- 超过时等待已完成后再启动新的
- 每次启动必须记录 task_id

## 作家部门 Agent

**定义文件**: `.claude/agents/作家部门.md`

### 人数
10 人 (作家A - 作家J)

### 职责
- 正文创作
- 每批 10 章产出
- 自评 S/A/B 级提交

### 调度方式
- 主编统筹，按阶段分配（20-40 章/人）
- 每批 10 章分配给不同作家并行

### 输出
- 章节正文 → `03_内容仓库/04_正文/`
- 审核意见处理 → `06_意见仓库/`

## Subagent 调度可靠性（强制流程）

### 调度前检查
1. 确认目标文件存在
2. 确认输出目录存在
3. 检查当前 running 任务数（不超过 10 个并发）

### task_id 追踪机制
启动的每个 agent 必须记录：
```json
"agent_tasks": {
    "ch164_fix": {
        "task_id": "a1234567890",
        "agent": "作家A",
        "status": "running",
        "chapter": "ch164",
        "launched_at": "2026-05-14T10:00:00"
    }
}
```

### 验证步骤（强制）
```
启动Agent → 等待5秒 → TaskOutput验证状态
                    ↓
           status=failed → 重新启动（最多3次）
           status=completed → 继续
           无响应 → 等待5秒再试
```
