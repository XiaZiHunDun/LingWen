# 上下文管理策略

本文档定义小说工厂中的上下文管理规范，防止上下文溢出并保持高效工作。

## 问题背景

CLAUDE.md 当前超过 40KB，包含了所有职责、规则、命令。这导致：
- 上下文加载缓慢
- 容易被截断
- 难以维护

## 解决方案：分层上下文

```
┌─────────────────────────────────────────────────┐
│ CLAUDE.md（主配置，只读引用）                     │
│  - 身份定义、目录结构速查、命令速查              │
│  - 引用 .claude/agents/*.md 获取详细Agent定义    │
└─────────────────────────────────────────────────┘
           ↓ 按需引用
┌─────────────────────────────────────────────────┐
│  .claude/agents/（Agent详细定义）                │
│  - 仅在调度该Agent时加载                         │
└─────────────────────────────────────────────────┘
           ↓ 按需引用
┌─────────────────────────────────────────────────┐
│  .claude/docs/（专题文档）                       │
│  - coordination-rules.md（协调规则）              │
│  - model-tier-guide.md（模型分级）               │
│  - context-management.md（本文档）                │
└─────────────────────────────────────────────────┘
           ↓ 按需引用
┌─────────────────────────────────────────────────┐
│  production/session-state/active.md（会话状态）   │
│  - 当前项目、进度、活跃任务                      │
│  - 每个里程碑后更新                              │
└─────────────────────────────────────────────────┘
```

## 上下文压缩触发条件

| 触发点 | 操作 |
|--------|------|
| 上下文使用 > 60% | 主动提醒用户即将压缩 |
| 上下文使用 > 75% | 立即压缩非必要内容 |
| 完成任务后 | 立即压缩已完成任务的详情 |

## 压缩策略

### 1. 工作记忆压缩

**触发时机**：每完成一个阶段性任务后

**压缩内容**：
- 删除已完成的子任务详情
- 保留最终结果和决策点
- 用摘要替代详细过程

**示例**：
```
压缩前：
- 作家A完成了ch001-ch010的修复（3轮迭代）
- 审核员B审核了ch006-ch008（发现2个P0问题）
- 作家C修复了ch006-ch008的P0问题

压缩后：
- ch001-ch010：已完成定稿（修复P0×2）
```

### 2. 意见仓库压缩

**触发时机**：每10章批次完成后

**压缩内容**：
- 将详细审核意见归档到 `06_意见仓库/`
- 保留意见摘要（优先级+数量）
- 删除正文中的临时意见列表

### 3. 进度追踪压缩

**触发时机**：每卷完成后

**压缩内容**：
- 将卷级进度归档
- 在 workflow_state.json 中标记为已完成
- 保留当前卷的详细状态

## 文件创建规范

### 即时骨架原则

**创建文件时立即创建骨架**：
```markdown
# 新章节标题

## 1. 概述
[待填写]

## 2. 详细内容
[待填写]

## 3. 伏笔/关联
[待填写]
```

**原因**：
1. 避免忘记要写什么
2. 便于断点续写
3. 减少上下文负担

### 逐节讨论原则

**不要一次性讨论整个文件**：
- 每节单独讨论
- 批准后立即写入文件
- 再讨论下一节

**原因**：保持每个上下文的聚焦性

## 上下文加载优先级

| 优先级 | 文件 | 何时加载 |
|--------|------|---------|
| 1 | CLAUDE.md | 始终（基础上下文） |
| 2 | active.md | 始终（会话状态） |
| 3 | 对应Agent定义 | 调度该Agent时 |
| 4 | coordination-rules.md | 跨部门协调时 |
| 5 | model-tier-guide.md | 模型选择时 |

## 会话状态维护

### 每次里程碑后必须更新

```
1. 打开 active.md
2. 更新当前任务
3. 更新进度检查清单
4. 记录关键决策
5. 更新Agent调用记录
6. 添加更新日志
```

### 崩溃恢复流程

```
1. 读取 active.md 获取项目状态
2. 读取 workflow_state.json 获取详细进度
3. 检查 active.md 中的"正在处理的文件"
4. 从断点恢复工作
```

## 目录速查设计

CLAUDE.md 中的 `目录速查` 只列出顶层目录，不列出具体文件：

```markdown
## 目录速查

01_灵感库/          → 灵感部门输出
02_作家工作室/作家{N}/  → 作家画像+记忆
03_内容仓库/         → 四层结构（大纲+正文）
```

**不列出**：
- 具体章节文件（360个，无法列举）
- 详细子目录结构（按需加载）

## 定期清理

### 每卷完成后

1. 归档该卷的审核意见到历史目录
2. 清理临时文件和草稿
3. 更新 active.md

### 全文完成后

1. 归档所有中间产物
2. 保留最终成品和汇总
3. 清理所有临时状态
