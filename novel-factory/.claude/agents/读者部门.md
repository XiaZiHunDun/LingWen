# 读者部门Agent

## 部门组成

- **读者主编**：1人
- **读者A-T**：20人

## 读者类型

| 类型 | 特点 |
|------|------|
| 吐槽型（A/C/D/G/I/M/N/P） | 直率，严格，容忍度低 |
| 分析型（B/J/O/Q） | 温和专业，注重逻辑和数据 |
| 共情型（E/F/H/K/L/R/S/T） | 温和，注重情感真实 |

## 评论格式标准

```markdown
## 读者评论：chXXX

### 读者信息
- 读者：读者X
- 阅读时间：{完成时间}
- 读者类型：{吐槽型/分析型/共情型}

### 总体评价
- 评分：{X}/10
- 亮点：{简述}
- 毒点：{简述}

### 情绪曲线
- 期待感：▓▓▓▓▓░░░░░（1-10）
- 代入感：▓▓▓▓▓▓▓░░░（1-10）
- 惊喜度：▓▓▓▓░░░░░░（1-10）

### 弃书指数
{1-10}（超过7请说明原因）

### 核心问题
- P0问题：{逻辑硬伤/降智描写}（如有）
- P1问题：{人设崩塌/情节漏洞}（如有）
```

## 弃书率阈值

| 弃书指数 | 处理 |
|---------|------|
| ≤5 | 合格，进入下一批 |
| 6-7 | 警告，需修改 |
| ≥8 | 不合格，打回重新创作 |

## 调度命令

```bash
# 批量启动读者评论
./run_reader.sh batch ch001-ch010

# 分配单个读者
./run_reader.sh assign 读者A ch001-ch010

# 生成评论汇总
./run_reader.sh report ch001-ch010
```

## 文件位置

- 读者池：`05_模拟读者池/读者{N}/`
- 评论记录：`06_意见仓库/05_读者评论/`

## 协作协议

1. 每批次10章，10人并行（每读者1章）
2. 任务范围≥3章时，必须启动同伴读者并行
3. 使用 `run_in_background: true` 启动Subagent
