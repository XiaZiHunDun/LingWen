# 汇总部门Agent

## 三角色分工

| 角色 | 职责 | 调度时机 |
|------|------|---------|
| 汇总主笔 | 执行修复、版本管理、质量裁定 | 卷汇总需微调时 |
| 汇总编辑 | 审核修复结果、过渡润色、风格统一 | 修复完成后 |
| 汇总校验 | 最终一致性检查、生成校验报告 | 编辑通过后 |

## 调度触发条件

| 触发条件 | 优先级 | 处理时限 |
|---------|--------|---------|
| 跨卷逻辑矛盾 | P0 | 1小时内 |
| 跨卷时间线不一致 | P1 | 4小时内 |
| 人物状态矛盾 | P1 | 4小时内 |
| 境界/结局需修正 | P2 | 24小时内 |

## 版本迭代规范

| 版本 | 触发条件 | 内容要求 |
|------|---------|---------|
| v1.0 | 初次汇总完成 | 基础内容整合，结构完整 |
| v2.0 | 审核发现需修改 | 修复逻辑矛盾，补充过渡 |
| v3.0 | 二次审核通过 | 细节打磨，风格统一 |
| final | 三审通过+校验完成 | 可发布版本 |

## 质量等级判定

| 等级 | 完整度 | 处理 |
|------|--------|------|
| S级 | >90% | 直接进入下一环节，标记final |
| A级 | 70%-90% | 编辑润色后进入校验 |
| B级 | 50%-70% | 返回主笔重写（不计入迭代） |
| 不合格 | <50% | 打回重做（计1次迭代） |

## 执行流程

```
汇总主笔（修复）→ 汇总编辑（润色）→ 汇总校验（校验）→ 主编裁定
                                              ↓
                                          通过 → 发布
                                          不通过 → 返回主笔修复
```

## 调度命令

```bash
# 阶段汇总
./run_summary.sh stage 阶段1

# 卷汇总
./run_summary.sh volume 卷1

# 全文汇总
./run_summary.sh full

# 验证汇总完整性
./run_summary.sh verify
```

## 协作协议

1. 汇总主笔先修复问题
2. 汇总编辑润色
3. 汇总校验最终检查
4. 主编裁定通过/不通过
