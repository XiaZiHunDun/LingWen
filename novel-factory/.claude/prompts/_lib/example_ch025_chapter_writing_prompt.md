- **章节号**：ch025
- **卷/阶段**：卷1（ch001-ch120）
- **章节类型**：发展（中期章节）
- **前情摘要**：林夜独自在废土生存三个月后，与苏琳相遇
- **本章目标**：林夜与苏琳的初次深入对话，展现两人性格碰撞

## 二、章节上下文

### 章节定位
- **卷/阶段**：`卷1 / 发展阶段`
- **章节范围**：ch001-ch120中的第25章
- **章节类型**：`发展`（非开篇、非高潮，是情节推进期）

### 前情概要
林夜在父母死后独自在废土生存三个月。某个雪天，他在废墟中发现了一个奄奄一息的女孩——苏琳，一个拥有预言能力却被星辰会追杀的少女。这是他们命运的初次交汇点。

### 本章核心目标
1. 展现苏琳的性格：温柔中带着坚韧，不是柔弱花瓶
2. 埋下"预言能力"的伏笔
3. 建立两人的初步羁绊
4. 体现"锚点"概念——苏琳将成为林夜的情感支撑

### 与大纲的对应关系
```
对应大纲节点：卷1早期 - 核心羁绊形成阶段
预期情节走向：废土生存 → 建立羁绊 → 联盟雏形
```

---

## 三、主要角色信息

### 本章出场角色
| 角色 | 出场比重 | 当前状态 |
|------|----------|----------|
| 林夜 | 60% | 独自生存3个月，冷峻坚韧，复仇心切 |
| 苏琳 | 40% | 受伤逃亡，预言能力初显，温柔坚定 |

### 角色当前状态

**林夜**：
- 身体：虚弱但警觉，冻伤未愈
- 心理：冷峻内敛，复仇驱动，但内心有柔软之处
- 处境：独自流浪，生存艰难

**苏琳**：
- 身体：重伤，体力透支
- 心理：恐惧但清醒，知道自己的价值
- 处境：被星辰会追杀，需要帮助

### 角色关系动态
- 本章：陌生人 → 初次相遇
- 苏琳对林夜：好奇（感受到他的不同）
- 林夜对苏琳：警惕 → 恻隐（发现她的伤和恐惧）

---

## 四、场景设定

| 项目 | 内容 |
|------|------|
| **场景类型** | 废墟内部（有遮蔽） |
| **时间** | 冬天的雪夜 |
| **地点** | 林夜的临时庇护所附近 |
| **氛围要求** | 冷峻中带一丝温暖希望 |
| **情绪基调** | 先抑后扬，结尾留希望 |

### 场景特点
- 寒冷、破败、废墟感
- 但有人性的微光在其中
- 雪夜的光影效果（月光、雪光）

---

## 五、情节要求

### 必须包含的情节元素
1. **苏琳的出现**：受伤倒在林夜藏身附近
2. **林夜的内心挣扎**：救还是不救？（他习惯独行）
3. **苏琳的预言初显**：她看到了林夜的某些特质
4. **决定伸出援手**：林夜选择救助这个陌生人
5. **建立初步信任**：非一次性消费，为后续羁绊铺垫

### 禁止出现的内容
- 苏琳一上来就完全信任林夜（不合逻辑）
- 林夜突然变得话多/圣母
- 任何形式的降智描写
- 冗长的背景回忆

### 节奏要求
- 开篇：压抑（雪夜、寒冷、废墟）
- 中段：转机（苏琳出现）
- 后段：希望（建立羁绊的开端）

---

## 六、写作规范

### 视角管理
- **主视角**：林夜（严格控制）
- **视角切换规则**：本章不超过1次视角切换

### 叙事风格
- **语气**：`冷峻但有温度`
- **描写密度**：`适中`（不啰嗦但有细节）
- **对话比例**：30%（符合基础层规范）

### 特殊处理
- **打斗场景**：本章无
- **感情描写**：克制，通过动作和细节展现
- **世界观展示**：通过对话自然带出，不做说明文式讲解

---

## 七、质量标准

| 维度 | 要求 | 自评检查点 |
|------|------|-----------|
| 情节推进 | 有清晰的情节推进线 | [ ] 有 |
| 角色互动 | 至少2处角色互动 | [ ] 有 |
| 情感深度 | 有情感共鸣点 | [ ] 有 |
| 冲突设置 | 有内在冲突（救/不救） | [ ] 有 |
| 伏笔铺设 | 埋设1-2个伏笔 | [ ] 有（预言能力） |
| 悬念设置 | 有钩子吸引继续阅读 | [ ] 有（苏琳为什么被追杀？） |

## 八、输出结构

请按以下结构产出章节：

```
【章节标题】（如：第二章 独自求生 / 隐含标题）

[正文...]

【章节end】
```

### 开篇钩子要求
用一个画面抓住读者——苏琳倒下，或林夜发现异常

### 章节结尾悬念要求
留下一个钩子——苏琳说出的某句话/展现的某种能力/林夜的某种决定

- **苏琳**：预言能力者，被星辰会追杀，性格温柔坚定，成为林夜的锚点

2. **对话要符合两人性格**：林夜话少、直接；苏琳温和但有主见

## 使用总结

通过模板 + 项目数据的结合，我们生成了一个约 **2500 tokens** 的完整写作prompt。相比之前的 ~375 tokens，提升了约 **6-7倍**。

这样的prompt能够：
1. 给LLM提供完整的上下文（不再需要它自己推测）
2. 保持各章节的风格一致性（来自基础层/深度层）
3. 埋设伏笔时有据可查（来自深度层的伏笔设计）
4. 质量标准明确（内置检查点）
