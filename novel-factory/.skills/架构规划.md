# Skills 架构规划

> 明确 .skills/ 和 .workflow_rules/ 的职责边界与协作关系

---

## 一、架构原则

| 原则 | 说明 |
|------|------|
| **.skills/ = 执行层** | Skill 是可执行的能力单元，直接被 Agent 调用 |
| **.workflow_rules/ = 规则层** | 规则是约束和规范，定义"什么时候做什么" |
| **职责互补** | Skill 管"怎么做"，规则管"做到什么标准" |

---

## 二、目录职责

### .workflow_rules/（规则层）
```
.workflow_rules/
├── 工作流规则手册.md      # 工作流总规则
├── AUDIT_ACCEPTANCE_CRITERIA.md  # 验收标准
├── PROGRESS_VALIDATION.md        # 进度验证规则
├── VERIFICATION_RULES.md         # 验证规则
├── character-bible/              # 角色圣经（设定集）
├── lore-registry.md              # 世界观设定注册表
├── faction-design.md            # 势力设计
├── novel-pillars.md              # 小说支柱（核心要素）
├── collaborative-protocol.md    # 协作协议
├── context-management.md        # 上下文管理
└── templates/                    # 模板库
```

**职责**：
- 定义工作流标准和规范
- 维护角色/世界观设定
- 制定协作协议和验证规则
- 提供模板和基准

### .skills/（能力层）
```
.skills/
├── _global/                     # 全局技能
│   ├── workflow-controller/     # 工作流控制
│   └── hooks-controller/        # 钩子控制
│
├── inspiration-dept/            # 灵感部门
│   └── inspiration-generator/   # 灵感生成
│
├── writer-dept/                 # 作家部门
│   ├── outline-drafting/         # 大纲撰写
│   └── review-opinion-synthesizer/  # 审核意见综合
│
├── reviewer-dept/                # 审核部门
│   └── novel-quality-check/      # 小说质量检查
│
├── reader-dept/                 # 读者部门
│   └── reader-feedback-aggregator/  # 读者反馈聚合
│
├── summary-dept/                # 汇总部门
│   └── summary-compiler/         # 汇总编译
│
├── consistency-check/           # 一致性检查
├── quality-report/              # 质量报告
├── publish/                     # 发布流程
└── start/                       # 启动流程
```

**职责**：
- Skill 是可执行的最小能力单元
- 每个 Skill 包含：SKILL.md（定义）+ behavioral-spec.md（行为规范）
- 直接被 Agent 调度执行

## 四、协作流程

```
.workflow_rules/ (规则)
    ↓ 定义标准
.workflow_rules/ (模板/角色圣经/协作协议)
    ↓ 被 Skill 调用
.skills/ (执行)
    ↓ 输出结果
workflow_state.json (状态更新)
```

---

## 五、优化建议

### 1. 当前问题
- 部分 Skill 定义与 workflow_rule 有重叠（如一致性检查）
- Skill 调用链不够清晰

### 2. 优化方向
1. **明确边界**：workflow_rules 管"规则"，skills 管"执行"
2. **建立索引**：每个部门维护自己的 Skill 清单
3. **调用标准化**：所有 Skill 调用都经过 workflow-controller
