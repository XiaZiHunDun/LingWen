# /publish - 发布工作流

## 功能

执行发布工作流，将审核通过的章节/卷/全文标记为已发布。

## 使用方式

```
/publish chapter ch001-ch010
/publish volume 1
/publish full
```

## 发布范围

| 范围 | 说明 |
|------|------|
| chapter | 批量发布章节（指定范围） |
| volume | 发布整卷（指定卷号） |
| full | 发布全文（最终发布） |

## 执行流程

### 章节发布 (chapter)

1. **验证待发布章节**
   - 确认全部审核通过
   - 确认无 P0/P1 问题

2. **移动到发布目录**
   ```
   03_内容仓库/04_正文/ch001.md
   → 08_已发布/卷X/ch001.md
   ```

3. **更新状态**
   - 更新 workflow_state.json
   - 记录发布时间

### 卷发布 (volume)

1. **验证整卷状态**
   - 所有章节已发布
   - 卷大纲定稿

2. **创建卷打包**
   ```
   08_已发布/卷1/卷1_星陨纪元_完整版.md
   ```

3. **更新发布记录**

### 全文发布 (full)

1. **最终检查**
   - 所有卷已发布
   - 全文一致性验证

2. **生成发布包**
   ```
   08_已发布/星陨纪元_v5.0_完整版.zip
   ```

3. **更新 release记录**

## 发布检查清单

发布前自动运行：
- [ ] 一致性检查通过
- [ ] 情感质量达标
- [ ] 无遗留问题
