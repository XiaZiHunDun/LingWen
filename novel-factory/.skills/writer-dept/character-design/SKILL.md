# 人物设计 Skill

## 功能

为小说项目创建系统化的人物档案，包括外在需求、内在需求、核心缺陷、前史、人物弧光。

## 核心原则

**"所有故事的发动机，都是人物的欲望与困境"**
**"没有变化的人物是死的"**

---

## 方法论嵌入

使用 `小说方法论/tools/人物小传模板.md` 作为执行标准。

---

## 人物档案结构

### 1. 外在需求（External Goal）

主角表面上想要什么？这是故事表层推进的驱动力。

| 项目 | 说明 | 示例 |
|------|------|------|
| 身份 | 他是谁？社会角色？ | 落魄的剑修 |
| 目标 | 他具体要什么？ | 赢得宗门大比、为家族复仇 |
| 障碍 | 谁/什么阻止他？ | 强大的对手、内心的恐惧 |
| 行动 | 他为此做什么？ | 艰苦修炼、寻找机缘 |

### 2. 内在需求（Internal Need）

主角内心深处真正缺什么？这是人物弧光的根基。

| 项目 | 说明 | 示例 |
|------|------|------|
| 缺失 | 他真正需要什么？ | 被认可、找到归属感、学会信任 |
| 恐惧 | 他最怕什么？ | 被遗弃、一无所有 |
| 伤口 | 是什么让他缺这个？ | 童年被遗弃的经历 |
| 不自知 | 他自己知道吗？ | 通常不知道，需要故事来揭示 |

**关键**：内在需求是主角不自知的，需要通过情节来揭示。

### 3. 核心缺陷（Core Flaw）

阻碍他获得成长的性格弱点。

| 项目 | 说明 | 示例 |
|------|------|------|
| 性格缺陷 | 他性格上有什么弱点？ | 自负、懦弱、冷漠、冲动 |
| 表现 | 这个缺陷如何体现？ | 关键时刻总是逃避、不愿信任他人 |
| 代价 | 这个缺陷让他付出什么？ | 失去重要的人、错失机缘 |

**原则**：完美的主角是无聊的。他的缺陷是读者代入和同情的原因，也是他成长弧光的起点。

### 4. 前史（Backstory）

他为什么会变成今天这样？

| 项目 | 说明 |
|------|------|
| 关键事件 | 塑造他的最重要的一件事 |
| 重要人物 | 影响他最深的人 |
| 转折点 | 他如何成为现在的自己 |

### 5. 人物弧光（Character Arc）

主角在故事开始和结束时的状态必须有本质变化。

| 阶段 | 状态 | 变化描述 |
|------|------|---------|
| 开始 | 他是什么样的人？ | 懦弱、不敢担当 |
| 过程 | 他经历了什么？ | 被逼到绝境、失去一切 |
| 结尾 | 他变成什么样？ | 勇敢、承担责任 |

## 人物关系网络

### 核心关系类型

| 关系 | 主角视角 | 说明 |
|------|---------|------|
| **对手/反派** | 阻碍主角实现目标 | 动机合理，不纯粹坏 |
| **帮手/导师** | 帮助主角成长 | 给予智慧、资源或情感支持 |
| **爱人/挚友** | 承载情感主题 | B故事线，辅助人物弧光 |
| **配角** | 丰富世界观 | 展示社会环境、风土人情 |

## 实操检查清单

创建人物小传后，逐项检查：

- [ ] 主角的外在目标和内在需求是否一致又矛盾？
- [ ] 核心缺陷是否足够明显、足够影响决策？
- [ ] 前史是否能解释当前的缺陷，但不解释太多？
- [ ] 人物弧光的起点和终点是否足够清晰？
- [ ] 每个关键配角是否有存在的理由（推动剧情/揭示主题/丰富世界）？
- [ ] 反派的动机是否合理？
