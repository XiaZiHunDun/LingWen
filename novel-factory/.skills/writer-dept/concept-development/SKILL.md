# 概念开发 Skill

## 功能

将模糊创意锤炼为清晰、有力的故事概念（Logline）。

## 核心原则

**"概念不成立，后面花再多工夫都可能是徒劳。这是成本最低的试错阶段。"**

---

## 方法论嵌入

使用 `小说方法论/tools/概念开发流程.md` 作为执行标准。

---

## 概念开发流程

### 1. 一句话梗概（Logline）

#### 公式

```
一个【性格缺陷】的【主角】，为了【目标】，必须克服【阻碍】，否则就会【代价】
```

#### 要素解析

| 要素 | 说明 | 示例 |
|------|------|------|
| **主角** | 一个有鲜明特质和缺陷的人 | 患有严重口吃的国王 |
| **目标** | 他具体要做什么 | 在二战爆发前发表鼓舞全国的演讲 |
| **阻碍** | 什么阻止他实现目标 | 不仅是口吃，还有宫廷内外的政治压力 |
| **代价** | 如果失败会怎样 | 国家将陷入绝望 |
| **缺陷** | 让主角有血有肉的性格弱点 | 口吃 + 内心恐惧权威 |

#### 检验标准

讲给别人听，看对方眼睛会不会一亮，是否立刻追问"然后呢？"

---

### 2. 主题定位

#### 定义

主题是故事探讨的中心思想，是剧本的指南针。当写到一半不知道该怎么选时，看主题。

#### 原则

**主题是通过人物的行动和选择，在冲突中自然"论证"出来的，不是通过台词说教喊出来的。**

#### 常见主题类型

| 类型 | 示例 |
|------|------|
| 成长 | "真正的勇气是在恐惧中依然前行" |
| 救赎 | "每个人都有重新开始的机会" |
| 爱情 | "爱能战胜偏见与阶级" |
| 复仇 | "贪婪会毁灭人性" |
| 自由 | "自由的代价是什么" |
| 身份 | "我是谁不由出身决定" |

---

### 3. 类型选择

#### 确定类型

| 类型 | 受众特征 | 风格特点 |
|------|---------|---------|
| 甜宠 | 年轻女性为主 | 轻松、浪漫、HE |
| 悬疑 | 广泛受众 | 紧张、反转、悬念 |
| 现实 | 成熟受众 | 真实、深刻、反思 |
| 古风 | 广泛受众 | 史诗、权谋、江湖 |
| 搞笑 | 广泛受众 | 轻松、幽默、反转 |
| 恐怖 | 青年男性为主 | 惊悚、压抑、逃生 |

#### 配套要素

| 要素 | 说明 |
|------|------|
| **受众定位** | 决定台词风格和情节复杂度 |
| **类型规则** | 每个类型有约定俗成的节奏（如悬疑前3章必须有钩子） |
| **对标作品** | 参考同类成功作品的结构 |

---

## 概念验证清单

起草概念后，逐项检查：

- [ ] 一句话概括能否在30秒内讲清楚？
- [ ] 主角是否有鲜明的性格缺陷？
- [ ] 目标是否足够具体、可量化？
- [ ] 阻碍是否足够大、足够合理？
- [ ] 代价是否足够严重、足够紧迫？
- [ ] 是否有强烈的戏剧反讽或张力？
- [ ] 讲给别人听时，对方是否追问"然后呢？"

## 输出格式

### 概念文档

```markdown
# {项目名} 概念文档

## 一句话梗概
（完整的Logline）

## 主题定位
- 核心主题：（一句话）
- 主题类型：（成长/救赎/爱情/复仇/自由/身份/其他）
- 主题论证：（通过什么情节来"论证"这个主题）

## 类型定位
- 类型：（如：玄幻/悬疑/现实）
- 受众：（如：16-25岁男性）
- 对标作品：（参考的同类成功作品）

## 概念要素表
| 要素 | 内容 |
|------|------|
| 主角 | |
| 性格缺陷 | |
| 目标 | |
| 阻碍 | |
| 代价 | |

## 概念验证
- [ ] 通过眼神测试
- [ ] 有戏剧反讽
- [ ] 代价足够严重
```

## 触发时机

| 时机 | 说明 |
|------|------|
| 新项目启动 | 必须先完成概念开发，才能进入大纲阶段 |
| 现有概念模糊 | 需要先清晰化再继续 |
| 概念验证失败 | 需要重新锤炼概念 |
