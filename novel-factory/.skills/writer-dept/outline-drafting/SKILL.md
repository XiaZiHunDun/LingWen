# 大纲起草 Skill

## 功能

根据灵感输入，生成结构化的大纲文档。

## 大纲类型

| 类型 | 适用步骤 | 说明 |
|------|---------|------|
| 全文大纲 | STEP_02 | 项目级整体规划 |
| 卷大纲 | STEP_06 | 单卷范围（3卷） |
| 阶段大纲 | STEP_10 | 阶段内详细情节 |

---

## 方法论嵌入（重要）

每次起草大纲前，必须已完成以下前置环节：

1. **概念开发**（使用 `/concept`）→ 获得清晰的一句话梗概
2. **人物设计**（使用 `/character-design`）→ 获得完整人物小传
3. **结构设计**（使用 `/structure-design`）→ 获得节拍表

大纲起草时，参照 `小说方法论/tools/节拍表模板.md` 的检查清单：`10_方法论/05_大纲结构设计.md` 的检查清单：

### 三幕式结构节点

| 幕 | 占比 | 节点 |
|----|------|------|
| 第一幕 | 1-20% | 日常→突发意外→第一扇门（主角踏入冲突） |
| 第二幕 | 20%-70% | 尝试失败→帮手/反派→中点转折→最低谷→决心反击 |
| 第三幕 | 70%-100% | 终极对决→关键选择→结局收尾 |

### 8节点大纲速查

```
节点1：主角日常
节点2：突发意外，打破日常
节点3：第一次尝试失败
节点4：遇帮手，反派加压
节点5：跌入最低谷
节点6：醒悟，决心反击
节点7：终极对决
节点8：结局收尾，人物蜕变
```

### 章纲写法（每章1-2句话）

每章必须包含：
- **本章目标**：这章要推进什么？
- **出场人物**：谁出现？
- **剧情走向**：发生什么？
- **情绪落点**：读者读这章的感受？
- **留钩子**：章末如何让读者想翻下一页？

## 输出规范

```markdown
# {卷/阶段}大纲

## 基本信息
- 章节范围：chXXX-chYYY
- 方法论阶段：阶段3_大纲结构设计
- 结构模板：三幕式/8节点

## 大纲节点检查
- [ ] 开端（1-20%）：日常被打破，主角被迫上路
- [ ] 发展（20%-70%）：闯关+成长+阻碍升级+接近低谷
- [ ] 高潮（70%-90%）：终极对决，直面最大阻碍
- [ ] 结局（90%-100%）：伏笔回收，人物蜕变

## 情节曲线
...

## 伏笔埋设
...
```
