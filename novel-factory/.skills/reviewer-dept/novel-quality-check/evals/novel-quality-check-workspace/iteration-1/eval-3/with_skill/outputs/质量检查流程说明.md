# 每批次10章创作完成后的质量检查流程

## 检查时机

- 每批次10章创作完成后，进入审核前
- 全书创作完成后，进入汇总前
- 人工审核发现疑似问题后需自动化验证

## 检查维度

| 维度 | 类型 | 说明 | 阈值 |
|------|------|------|------|
| 命名一致性 | 规则 | 文件名chXXX.md与章节内标题"第X章"一致 | 100%匹配 |
| 内容完整性 | 规则 | **本章完**标记、字数≥500 | 字数下限500 |
| 章节重复 | 规则 | 跨章节相似度>80%预警 | 相似度<80% |
| 人物状态 | 规则 | 性别/生死/形态前后矛盾 | 无矛盾 |
| 时间线 | 规则 | "年前"与"瞬间"同时出现等明显错误 | 无矛盾 |
| 情节关联度 | 规则 | 每段落与前后章节的关联程度 | 自适应 |
| 伏笔回收率 | 规则 | 首次出现元素在后续50章内回收 | 窗口50章 |
| 场景逻辑 | 规则 | 场景转换合理性、孤岛章节检测 | 相似度>15% |
| 情感节奏 | 规则 | 情绪波动是否合理 | 方差>0.01 |
| 对话风格 | 规则 | 角色对话字数异常检测 | 2倍典型范围 |
| 人物弧光 | LLM | 角色弧光完整性（通过Agent调用） | 完整 |

## 调度策略

1. **规则类检查**：可并行执行，互不依赖
2. **LLM类检查**：需按角色顺序执行，建议每批≤50章
3. **建议流程**：
   - 创作阶段：规则类检查即可
   - 审核阶段：加入LLM类检查（人物弧光）
   - 汇总阶段：完整检查
