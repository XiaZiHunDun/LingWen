# 情节骨架验证

> **版本**：v1.0
> **来源**：借鉴游戏制作"灰盒原型（Greyboxing）"方法论
> **日期**：2026-05-19
> **状态**：新增

---

## 核心理念

**情节骨架**是用最简陋的文字（bullet point）写出核心情节走向，验证"故事是否成立"。

> **核心原则**：如果骨架不好玩，加上再好的文笔内核依然是空洞的。

> **写作规范**：骨架验证阶段**禁止精细化描写、禁止润色文笔**。

---

## 一、概念定义

### 1.1 什么是情节骨架

**游戏中的灰盒原型**：
> 用最简陋的几何体（Cube/Sphere）搭建核心玩法，验证机制是否有趣。**此时绝不投入美术资源**。

**小说中的情节骨架**：
> 用最简陋的文字（bullet point）写出核心情节走向，验证"故事是否成立"。

### 1.2 情节骨架 vs 详细大纲

| 维度 | 情节骨架 | 详细大纲 |
|------|----------|----------|
| 粒度 | 每个情节点1-3句话 | 每章详细展开 |
| 文笔 | 无（bullet point） | 有（完整段落） |
| 目的 | 验证"故事是否成立" | 指导"如何写" |
| 阶段 | 规划期验证环节 | 规划期产出 |
| 修改成本 | 极低 | 中等 |

## 二、写作规范

### 2.1 基本规则

| 规则 | 说明 |
|------|------|
| **用第三人称叙述** | 不使用"我"、"你" |
| **只写"发生了什么"** | 不写"怎么发生的" |
| **不写心理描写** | 不写内心独白、情绪描写 |
| **不写环境描写** | 不写场景细节、氛围渲染 |
| **不写具体对话** | 只写"[对话发生]"或"双方发生争执" |
| **每个情节点1-3句话** | 简洁明了 |

### 2.2 模板格式

```
# [小说名] 情节骨架

## 第一幕（开头~25%）
- [场景1]：主角状态 + 日常
- [场景2]：诱发事件发生
- [场景3]：主角做出选择，第一扇门开启

## 第二幕（25%~75%）
### 上半部分（25%~50%）
- [场景4]：主角尝试解决问题
- [场景5]：第一次小胜利/小失败
- [场景6]：中点转折

### 下半部分（50%~75%）
- [场景7]：虚假胜利或局势恶化
- [场景8]：一切崩溃
- [场景9]：第二扇门，绝境

## 第三幕（75%~100%）
- [场景10]：灵魂黑夜
- [场景11]：核心高潮
- [场景12]：新平衡

## 核心驱动链验证
- [ ] 骨架好玩吗？
- [ ] 冲突是否层层升级？
- [ ] 人物动机是否成立？
```

## 三、验证检查点

### 3.1 结构验证

| 检查点 | 标准 |
|--------|------|
| **诱发事件** | 发生在全书的10%左右位置 |
| **第一扇门** | 主角主动做出选择，25%左右 |
| **中点转折** | 信息/局势重大转变，50%左右 |
| **第二扇门** | 主角被逼入绝境，75%左右 |
| **高潮** | 核心冲突爆发，90%左右 |

### 3.2 冲突验证

| 检查点 | 标准 |
|--------|------|
| **冲突是否存在** | 每个情节点是否有明确的阻碍 |
| **冲突是否升级** | 随着故事发展，冲突是否越来越严重 |
| **冲突是否多样** | 是否有不同类型的冲突（外部/内部） |

### 3.3 动机验证

| 检查点 | 标准 |
|--------|------|
| **主角动机是否成立** | 主角的选择是否合乎逻辑 |
| **反派动机是否成立** | 反派的阻碍是否有合理动机 |
| **选择是否有代价** | 主角的选择是否付出代价 |

### 3.4 驱动链验证

| 检查点 | 标准 |
|--------|------|
| **驱动链是否体现** | 核心驱动链的每个节点是否可识别 |
| **满足感分布** | 读者的"爽点"是否合理分布 |
| **递增性** | 满足感是否递增 |

## 四、常见问题与解决

### 4.1 骨架无聊

**症状**：用bullet point叙述后，故事显得平庸无聊

**诊断**：核心冲突不够强，或驱动链设计有问题

**解决**：
- 加强主角面临的阻碍
- 增加冲突的代价
- 重新审视核心驱动链设计

### 4.2 动机不成立

**症状**：主角的选择显得不合理

**诊断**：前因后果交代不足，或性格设定有问题

**解决**：
- 补充主角的背景动机
- 调整性格设定使其更合理
- 增加必要的铺垫

### 4.3 节奏问题

**症状**：前紧后松，或前松后紧

**诊断**：冲突分布不均匀

**解决**：
- 重新分布冲突点
- 检查每个阶段的冲突密度

### 4.4 高潮不够高

**症状**：高潮显得平淡

**诊断**：前面积累不足

**解决**：
- 加强前面的冲突和代价
- 确保高潮是全书最强的"满足点"

## 五、与创作流程的接口

### 5.1 位置

情节骨架验证位于**规划期末尾、写作期开头**。

### 5.2 前置条件

- [ ] 核心驱动链设计已完成
- [ ] 三幕式结构大纲已完成
- [ ] 人物设定已完成

### 5.3 验收标准

- [ ] 结构完整（三幕式十二个节点齐全）
- [ ] 冲突成立（每个情节点有明确阻碍）
- [ ] 动机成立（主角/反派选择合理）
- [ ] 节奏合理（冲突分布均匀）
- [ ] 驱动链体现（核心驱动链可识别）
- [ ] 骨架"好玩"（自问：如果这是游戏，好玩吗？）

### 5.4 输出物

情节骨架验证通过后，输出：
- `情节骨架_v1.0.md` — 验证通过版本
- `问题记录.md` — 验证过程中发现的问题及处理方案

### 5.5 与核心样章验证的关联

```
情节骨架验证 → 核心样章验证 → 目标读者测试 → 写作期正式启动
     ↓
  如未通过，打回规划期重新设计
```

## 六、示例

### 6.1 示例：《重生之都市修仙》情节骨架

```
# 《重生之都市修仙》情节骨架

## 第一幕（0-25%）
- 主角林凡在渡劫飞升时陨落，魂穿到同名大学生身上
- 发现原主是个被欺辱的废物，被校花女友抛弃
- 获得上古传承，决心重新修炼
- 第一次小试牛刀，当众打脸前女友现任男友

## 第二幕上（25-50%）
- 进入都市修炼圈，发现灵气稀薄
- 依靠传承的功法快速提升
- 结识商界大佬，获得资源支持
- 中点：发现前世陨落并非意外，背后有更大阴谋

## 第二幕下（50-75%）
- 修为渐强，开始调查前世死因
- 与隐藏的修仙世家产生冲突
- 虚假胜利：一度压制敌人
- 一切崩溃：敌人背后是更强大的势力
- 绝境：挚爱之人被绑架

## 第三幕（75-100%）
- 灵魂黑夜：在绝境中领悟新功法
- 核心高潮：当面击败幕后黑手
- 新平衡：飞升仙界，前世今生的恩怨终结

## 核心驱动链验证
- [x] 打脸爽点：4处打脸点明确
- [x] 升级快感：修为提升路径清晰
- [x] 冲突升级：敌对势力逐步增强
- [x] 动机成立：复仇动机合理

## 自检问题
- 初期打脸略显频繁，考虑减少到3处
- 中点转折力度足够
- 高潮部分需要加强情感铺垫
```
