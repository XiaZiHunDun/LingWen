# 世界观构建指南

> **版本**：v1.0
> **来源**：整合写小说流程5份文档
> **日期**：2026-05-19

---

## 核心理念

**设定是为故事服务的，不要为了展示设定而拖慢情节。**

只做"够用"的设定，精简够用即可，别堆废设定。

---

## 一、设定精简原则

### 1.1 极简设定法

新手最容易犯的错误：一上来写几十页设定表，很容易写崩。

建议用"极简设定"：
- 只记关键信息
- 不追求完美详尽
- 先有故事再补细节

### 1.2 设定层级

| 层级 | 内容 | 详细程度 |
|------|------|----------|
| 核心设定 | 影响主线剧情的规则 | 必须清晰 |
| 外围设定 | 丰富世界的细节 | 按需补充 |
| 隐藏设定 | 自己知道但不出现在书中 | 大量储备 |

**原则**：核心设定要服务于故事，外围设定不要喧宾夺主。

## 二、世界/背景设计

### 2.1 基础要素

| 要素 | 选项 |
|------|------|
| 时代 | 现代 / 古代 / 未来 / 架空 |
| 地域 | 单场景 / 多场景 / 跨世界 |
| 科技/魔法 | 无 / 科技 / 魔法 / 异能 / 混合 |
| 社会结构 | 平等 / 阶级 / 宗门 / 帮派 / 其他 |

### 2.2 只记关键信息

- 时代背景：什么时期，对故事有什么影响
- 核心规则：故事主要发生地的规则（1-2个重点场景）
- 特殊限制：魔法/科技/异能的边界在哪里

### 2.3 世界观自检

- [ ] 这个世界有什么独特之处
- [ ] 这个设定如何推动主线剧情
- [ ] 读者需要知道多少才能理解故事

---

## 三、规则体系设计

### 3.1 规则类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 力量体系 | 修炼/能力/等级 | 筑基→金丹→元婴 |
| 社会规则 | 制度/阶层/潜规则 | 宗门规矩、江湖法则 |
| 物理规则 | 世界运行规律 | 灵力循环、时空法则 |
| 信息规则 | 信息如何传递 | 传音、千里镜、密信 |

### 3.2 规则设计原则

1. **自洽性**：规则内部不能矛盾
2. **边界清晰**：限制在哪里，越界会怎样
3. **服务剧情**：规则要有存在的意义

### 3.3 规则展示技巧

不要大段说明文，而是：
- 通过角色行动展示
- 通过冲突暴露规则
- 通过对话解释规则

## 四、场景设计

### 4.1 重点场景

故事主要发生地只需要1-2个重点场景：
- 每个场景有明确功能
- 每个场景有独特视觉/氛围
- 每个场景能承载特定情节

### 4.2 场景要素

| 要素 | 说明 |
|------|------|
| 视觉 | 颜色、形状、光线 |
| 声音 | 环境音、特定声音 |
| 感觉 | 温度、湿度、触感 |
| 气味 | 环境味、特定气味 |
| 空间 | 大小、结构、进出路线 |

### 4.3 场景自检

- [ ] 场景是否服务于当前情节
- [ ] 场景是否有独特氛围
- [ ] 场景是否能给读者留下深刻印象

## 五、势力设计

### 5.1 势力类型

| 类型 | 示例 |
|------|------|
| 主角阵营 | 主角所属势力 |
| 对手阵营 | 直接对抗主角的势力 |
| 中立势力 | 第三方势力，可能被利用 |
| 隐藏势力 | 幕后势力，最后才揭示 |

### 5.2 势力设计要素

```
势力 = 目标 + 资源 + 规则 + 限制
```

| 要素 | 说明 |
|------|------|
| 目标 | 势力想要什么 |
| 资源 | 势力有什么（人、财、物、信息） |
| 规则 | 势力内部如何运作 |
| 限制 | 势力的边界在哪里 |

### 5.3 势力关系

势力之间不是静态的，而是动态的：
- 盟友可能变成敌人
- 中立可能变成敌对
- 每一次变化要有合理契机

## 六、细节设计

### 6.1 生活细节

让世界更真实：
- 货币体系
- 饮食习惯
- 服饰风格
- 交通工具
- 节日习俗

### 6.2 文化细节

让世界更有深度：
- 语言特点
- 宗教信仰
- 艺术风格
- 价值观念

### 6.3 细节使用原则

- 按需出现，不堆砌
- 服务于当前情节
- 体现势力/角色特点
