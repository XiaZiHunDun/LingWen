# 创作阶段锁定规范

> **版本**：v1.0
> **来源**：整合电影制作流程文档的"锁定"概念
> **日期**：2026-05-19

---

## 核心理念

**锁定不是限制，而是保护。**

在关键节点进行"锁定"，确保每个阶段的工作成果不会被轻易推翻，从而让下一阶段能够安心开展。

---

## 一、三层锁定机制

```
第一锁：大纲锁定 → 进入初稿
第二锁：初稿锁定 → 进入修改
第三锁：修改锁定 → 进入润色
```

### 1.1 第一锁：大纲锁定

**触发时机**：完成三幕式大纲后，准备进入初稿写作前

**锁定条件**（必须全部满足）：
- [ ] 三幕式结构完整（开端/发展/高潮/结局）
- [ ] 人物设定确定（主角欲望/缺陷/秘密/弧光）
- [ ] 核心冲突明确（主要矛盾不模糊）
- [ ] 世界观核心规则已定义
- [ ] 每卷核心事件已确定

**变更管理**：
- 锁定后变更需要走正式流程
- 小改（局部调整）：可直接修改，需记录
- 中改（影响后续章节）：标注影响范围，评估后再实施
- 大改（影响主线）：触发重新评审，评估是否重新锁定

**锁定记录**：
```markdown
## 大纲锁定记录

日期：YYYY-MM-DD
版本：v1.0.0
状态：已锁定

锁定内容：
-
-
变更记录：（后续变更在此记录）
```

---

### 1.2 第二锁：初稿锁定

**触发时机**：完成全文初稿后，准备进入修改阶段前

**锁定条件**（必须全部满足）：
- [ ] 全书初稿完成（不求完美，但求完整）
- [ ] 每章有明确的目标/阻碍/结果
- [ ] 伏笔已记录（可追踪）
- [ ] 人物结局已确定（不临时改死）
- [ ] 章节字数符合预期（无严重偏离）

**变更管理**：
- 仅限结构性问题
- 不允许大改故事走向
- 人物命运调整需要评审通过

**锁定记录**：
```markdown
## 初稿锁定记录

日期：YYYY-MM-DD
版本：v2.0.0
状态：已锁定

锁定内容：
-
-
变更记录：（后续变更在此记录）
```

---

### 1.3 第三锁：修改锁定

**触发时机**：完成宏观修改（结构/逻辑/人物），准备进入润色阶段前

**锁定条件**（必须全部满足）：
- [ ] S1（剧情完整性）≥ 3分
- [ ] S2（逻辑自洽）≥ 3分
- [ ] 人物动机合理性已验证
- [ ] 情节漏洞已修复
- [ ] 无影响阅读的重大问题

**变更管理**：
- 仅限字词级修改
- 不允许调整情节
- 润色不能改变叙事节奏

**锁定记录**：
```markdown
## 修改锁定记录

日期：YYYY-MM-DD
版本：v3.0.0
状态：已锁定

锁定内容：
-
-
变更记录：（后续变更在此记录）
```

## 二、变更管理流程

```
发现问题 → 评估影响范围 → 判断变更类型 → 实施修改 → 更新记录
```

### 2.1 变更类型判断

| 类型 | 影响范围 | 处理方式 |
|------|----------|----------|
| 小改 | 单章节内 | 直接修改，记录摘要 |
| 中改 | 跨章节 | 标注影响，评估后实施 |
| 大改 | 主线/人物命运 | 重新评审，必要时重新锁定 |

### 2.2 变更评估清单

```markdown
## 变更评估

变更描述：
-

影响范围：
□ 单章节
□ 跨章节（影响____章）
□ 主线/人物命运

变更类型：□ 小改 □ 中改 □ 大改

是否需要重新锁定：□ 是 □ 否

实施记录：
-
```

## 三、锁定与评审的关系

```
大纲锁定 → 初稿写作 → 初稿锁定 → 修改 → 修改锁定 → 润色
    ↓                      ↓                      ↓
  评审前置              评审前置              评审前置
```

**关键原则**：
- 每个锁定点前都需要通过"评审前置检查"
- 评审是"门"，不是"补"
- 通过评审才能锁定，锁定后才能进入下一阶段

---

## 四、"锁门"概念详解

借鉴电影制作的说法：

> **"剪辑未定稿前，任何声音和特效投入都可能白费"**

对应小说：
> **"大纲未定稿前，任何初稿投入都可能因大改而浪费"**

### 4.1 常见错误

| 错误 | 问题 | 正确做法 |
|------|------|----------|
| 大纲没定就写初稿 | 写到一半推倒重来 | 先锁定大纲再写初稿 |
| 初稿没定就润色 | 润色完发现要重写结构 | 先锁定初稿再修改 |
| 修改没定就校对 | 校对完发现核心问题 | 先锁定修改再润色 |

### 4.2 判断是否需要"锁门"

```
如果这个问题需要修改前面的章节才能解决，
说明还没到"锁门"的时机。
```

## 五、锁定检查清单

### 5.1 大纲锁定前检查

- [ ] 一句话故事是否清晰
- [ ] 三幕式结构是否完整
- [ ] 人物设定是否确定
- [ ] 核心冲突是否明确
- [ ] 世界观规则是否定义
- [ ] 每卷核心事件是否确定

### 5.2 初稿锁定前检查

- [ ] 全书初稿是否完成
- [ ] 每章是否有目标/阻碍/结果
- [ ] 伏笔是否已记录
- [ ] 人物结局是否确定
- [ ] 章节字数是否正常

### 5.3 修改锁定前检查

- [ ] S1≥3分
- [ ] S2≥3分
- [ ] 人物动机是否合理
- [ ] 情节漏洞是否修复
- [ ] 无重大问题
