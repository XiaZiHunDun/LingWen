# 调度Agent_通告单生成逻辑

> **版本**：v1.0
> **日期**：2026-05-19
> **用途**：说明调度Agent如何生成每日创作通告单

---

## 核心理念

**前一天晚上生成第二天的通告单，让作家一早就能开始工作。**

调度Agent的核心职责是：将长期计划分解为每日任务，并确保任务可执行、有衔接、能追踪。

---

## 一、触发时机

### 1.1 固定调度时间

| 时间 | 动作 |
|------|------|
| 每日 20:00 | 检查今日Dailies报告是否已提交 |
| 每日 20:05 | 生成次日通告单（如Dailies已提交） |

### 1.2 异常触发

| 场景 | 触发条件 | 处理方式 |
|------|----------|----------|
| Dailies未提交 | 20:00前未收到报告 | 提醒作家，延迟生成 |
| 任务未完成 | Dailies显示任务未完成 | 分析原因，调整明日任务量 |
| 问题升级 | Dailies报告H/S级问题 | 立即通知主控 |

## 二、输入数据

### 2.1 必需输入

| 数据 | 来源 | 用途 |
|------|------|------|
| 今日Dailies报告 | `07_意见仓库/dailies/Dailies_YYYY-MM-DD.md` | 了解今日完成情况 |
| 章节计划表 | `03_内容仓库/卷X/章节计划.md` | 确定剩余任务 |
| 作家画像 | `02_作家工作室/作家X/profile.md` | 匹配任务难度 |
| 人物设定库 | `02_作家工作室/作家X/characters/*.md` | 提供素材引用 |
| 已锁定上文 | `03_内容仓库/卷X/*_上文.md` | 提供续写起点 |

### 2.2 可选输入

| 数据 | 来源 | 用途 |
|------|------|------|
| 主控备注 | 用户直接输入 | 特殊指令 |
| 审核意见 | `06_意见仓库/review/*.md` | 需要修复的问题 |

## 三、生成流程

### 3.1 步骤概览

```
Step 1: 读取Dailies报告 → 分析今日完成情况
Step 2: 读取章节计划表 → 确定明日可执行任务
Step 3: 匹配作家画像 → 确定任务难度和数量
Step 4: 关联素材 → 自动挂载参考文件
Step 5: 生成通告单 → 输出标准化文档
Step 6: 通知作家 → 确认收到
```

### 3.2 Step 1：读取Dailies报告

```
输入：今日Dailies报告
输出：今日完成情况摘要

处理逻辑：
1. 解析「今日完成情况」表格
2. 计算目标字数 vs 实际字数
3. 提取「遇到的问题」列表
4. 判断是否有H/S级问题
5. 提取「遗留问题」状态
```

### 3.3 Step 2：确定明日任务

```
输入：章节计划表 + 今日完成情况
输出：明日任务列表

处理逻辑：
1. 找到「未完成」或「待写」章节
2. 按优先级排序（P0 > P1 > P2）
3. 考虑章节之间的依赖关系
4. 选取1-2个场景作为明日任务
5. 避免选取有依赖未解决的章节
```

### 3.4 Step 3：匹配作家画像

```
输入：作家画像 + 明日任务
输出：调整后的任务

处理逻辑：
1. 读取作家日均产能（如：2000字/天）
2. 读取作家擅长类型（如：动作场面）
3. 匹配任务与作家：
   - 作家擅长 → 正常任务量
   - 作家不擅长 → 降低任务量
   - 新手 → 选取简单场景，降低字数要求
4. 调整任务优先级
```

### 3.5 Step 4：关联素材

```
输入：明日任务列表
输出：素材关联表

处理逻辑：
1. 解析每个任务涉及的角色
2. 查找对应的人物设定文件
3. 查找对应章节的「已锁定上文」
4. 生成素材引用列表
5. 如有缺失素材，标注警告
```

### 3.6 Step 5：生成通告单

```
输入：整理好的所有数据
输出：标准化通告单文档

处理逻辑：
1. 按模板格式组装
2. 生成「今日任务」表格
3. 生成「参考素材」表格
4. 填写「质量要求」
5. 添加「特殊备注」（如有遗留问题）
6. 写入文件：`07_意见仓库/dailies/通告单_YYYY-MM-DD.md`
```

### 3.7 Step 6：通知作家

```
动作：
1. 生成次日通告单后，通知作家
2. 通知内容：任务概要 + 文件位置
3. 要求作家确认收到
```

## 四、决策规则

### 4.1 任务量决策

| 情况 | 规则 |
|------|------|
| 今日任务100%完成 | 明日任务量维持不变 |
| 今日任务>80%完成 | 明日任务量+10% |
| 今日任务<50%完成 | 明日任务量-20%，排查原因 |
| 连续3天<80%完成 | 通知主控，启动干预流程 |

### 4.2 任务选择决策

| 规则 | 说明 |
|------|------|
| 优先选择驱动链关键场景 | 确保主线推进 |
| 避免选择有前置依赖的场景 | 除非前置已完成 |
| 优先选择作家擅长的场景类型 | 提高完成率 |
| 同一卷内连续场景优先 | 保持连贯性 |

### 4.3 问题升级决策

| 触发条件 | 动作 |
|----------|------|
| Dailies报告H级问题 | 立即停止，通知主控 |
| 连续3天同类问题 | 生成专项修复任务 |
| 作家报告卡文>3天 | 主控介入决策 |
| 偏离大纲>1章 | 通知主控，确认是否修正 |

## 五、输出规范

### 5.1 通告单文件命名

```
通告单_YYYY-MM-DD.md
示例：通告单_2026-05-20.md
```

### 5.2 存储路径

```
07_意见仓库/dailies/
├── 通告单_2026-05-19.md    # 今日通告单
├── Dailies_2026-05-19.md   # 今日Dailies报告
├── 通告单_2026-05-20.md    # 明日通告单（预生成）
└── ...
```

### 5.3 通知格式

```markdown
## 明日通告单已生成

**日期**：2026-05-20
**执行作家**：[作家名]
**任务数**：2个场景

### 任务概要
1. 第X章-X节「场景名称」- XXXX字
2. 第X章-X节「场景名称」- XXXX字

**参考素材**：已挂载
**截止时间**：22:00

文件位置：`07_意见仓库/dailies/通告单_2026-05-20.md`

请确认收到。
```

## 六、人机协同模式

### 6.1 Agent自动执行

以下场景Agent自动处理，无需人工介入：
- 正常每日通告单生成
- 任务量小幅调整（±10%）
- 素材关联
- 通知发送

### 6.2 需要主控决策

以下场景Agent生成报告，提交主控决策：

| 场景 | 说明 |
|------|------|
| 任务量大幅调整 | 超过20%的调整 |
| 连续3天未完成任务 | 需要排查原因 |
| 重大偏离大纲 | 影响主线走向 |
| 新角色引入 | 超出原设定 |
| 作家更换 | 需要主控批准 |

### 6.3 主控干预场景

```
主控干预触发条件：
1. 连续3天任务完成率<50%
2. Dailies报告H级问题
3. 作家申请更换任务
4. 大纲重大偏离
5. 作家产能长期不达标
```

## 七、异常处理

### 7.1 Dailies报告缺失

```
20:00 未收到 Dailies 报告：
    ↓
Agent发送提醒给作家
    ↓
等待至 21:00
    ↓
仍未收到 → 生成「空通告单」，仅含基础任务
    ↓
通知主控备案
```

### 7.2 素材缺失

```
发现素材缺失（如人物设定未完成）：
    ↓
在通告单中标注「素材待补充」
    ↓
提供替代方案或降低任务难度
    ↓
通知主控协调
```

### 7.3 作家状态异常

```
作家连续3天报告「疲劳状态」：
    ↓
在通告单中自动降低任务量50%
    ↓
通知主控关注
    ↓
主控决策是否调整长期计划
```

## 八、调度Agent执行检查清单

### 每日执行检查（20:00）

```markdown
□ 读取今日Dailies报告
□ 分析完成情况
□ 更新章节计划表进度
□ 检查是否有H/S级问题
□ 生成次日通告单
□ 写入文件
□ 通知作家
□ 记录执行日志
```

### 异常上报检查

```markdown
□ H级问题 → 立即通知主控
□ 连续3天未完成 → 通知主控
□ 素材缺失 → 通知主控
□ 作家状态异常 → 通知主控
```
