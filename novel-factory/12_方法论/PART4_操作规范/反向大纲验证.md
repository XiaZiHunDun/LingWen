# 反向大纲验证流程

> **版本**：v1.0
> **日期**：2026-05-19
> **目的**：通过生成每章一句话总结，与原大纲对比，检测正文跑偏程度
> **来源**：参考写小说流程文档"反向大纲"方法论

---

## 核心理念

**反向大纲**：写完后，每章/每场景自己总结"这章在干嘛"，再和原大纲对比，看跑偏多少。

---

## 一、使用场景

| 场景 | 是否适用 | 执行时机 |
|------|----------|----------|
| 阶段大纲审核 | ✅ 必须执行 | 阶段大纲审核前 |
| 卷大纲审核 | ✅ 必须执行 | 卷大纲审核前 |
| 正文Block修改期 | ✅ 推荐执行 | Block阶段第一轮 |
| 全书汇总前 | ✅ 必须执行 | STEP_23全文汇总前 |

---

## 二、执行流程

### STEP 1：提取原大纲摘要

```
读取原大纲文件（阶段/卷/全文）
提取每个章节预计发生的关键事件
生成大纲事件清单
```

### STEP 2：生成正文反向大纲

```
读取每个章节正文
每章生成一句话总结，格式：
[章节号] [视点人物] [发生了什么事] [结果如何]

示例：
ch045 林夜 前往联盟议事厅请求帮助 被拒绝但获得长老接见
ch046 铁蛋 演武场挑战星徒 获胜但引发嫉妒
```

### STEP 3：大纲对比分析

```
对照检查：
├─ 原大纲事件是否都发生了？
├─ 发生的事件顺序是否一致？
├─ 是否有原大纲没有的新事件？（需要判断是否必要）
└─ 原大纲预计的事件是否缺失？

生成跑偏报告
```

### STEP 4：跑偏分类

| 跑偏类型 | 定义 | 处理方式 |
|----------|------|----------|
| 偏离主线 | 添加了与核心驱动链无关的支线 | 删除或收紧 |
| 伏笔断裂 | 埋下的伏笔未回收 | 补充回收或在适当时机说明 |
| 节奏拖沓 | 无意义的过渡章节过多 | 压缩或合并 |
| 顺序错位 | 事件发生顺序与大纲不符 | 调整顺序 |
| 信息泄露 | 过早透露应后期揭示的信息 | 修改为更隐晦的表达 |
