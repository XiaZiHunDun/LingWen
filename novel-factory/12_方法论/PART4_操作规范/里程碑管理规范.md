# 里程碑管理规范

> **版本**：v1.0
> **来源**：借鉴游戏制作里程碑体系
> **日期**：2026-05-19
> **状态**：新增

---

## 核心理念

**里程碑**是创作过程中的关键交付节点，每个里程碑有明确的交付物和验收标准。

> **核心原则**：只有通过当前里程碑验收，才能进入下一阶段。

---

## 一、里程碑定义

### 1.1 里程碑总览

| 里程碑 | 阶段 | 交付物 | 验收标准 |
|--------|------|--------|----------|
| **M1: 核心驱动链稿** | 构思期 | 核心驱动链设计 + 一句话梗概 | 驱动链清晰，读者收益点明确 |
| **M2: 情节骨架稿** | 规划期 | 三幕式情节骨架（bullet point） | 冲突成立，动机成立，结构完整 |
| **M3: 核心样章稿** | 规划期 | 5000-10000字核心样章 | 风格立得住，节奏达标 |
| **M4: 初稿（Alpha）** | 写作期 | 全部章节初稿完成 | 所有章节完成，情节完整 |
| **M5: 评审稿（Beta）** | 修改期 | 经过评审标注的稿件 | H/S级缺陷已标注 |
| **M6: 定稿（GM）** | 完成期 | 零H级缺陷的终稿 | 可发布状态 |

### 1.2 里程碑对照游戏开发

| 游戏里程碑 | 小说里程碑 | 核心含义 |
|------------|------------|----------|
| First Playable | M2 情节骨架稿 | 核心循环成立 |
| Alpha | M4 初稿 | 功能完整 |
| Beta | M5 评审稿 | 内容完整，待修改 |
| Gold Master | M6 定稿 | 可发布 |

---

## 二、阶段交付流程

```
构思期完成 ─── M1验收 ──→ 规划期开始
                              ↓
                         M2 + M3验收
                              ↓
                         验证期开始
                              ↓
                    情节骨架验证 + 核心样章验证
                              ↓
                         通过验证
                              ↓
                         写作期开始
                              ↓
                         M4 初稿完成
                              ↓
                         修改期开始
                              ↓
                         M5 评审稿完成
                              ↓
                         完成期开始
                              ↓
                         M6 定稿完成
                              ↓
                         发布
```

## 三、M1: 核心驱动链稿

### 3.1 交付物

- 核心驱动链设计文档
- 一句话梗概

### 3.2 验收标准

| 标准 | 要求 |
|------|------|
| **类型定位** | 小说类型明确 |
| **核心驱动链** | 能用一句话描述 |
| **小满足频率** | 每X章的小满足类型清晰 |
| **大满足频率** | 每X卷的大满足类型清晰 |
| **递增性** | 驱动链具有递增性 |
| **差异化** | 与同类型小说有差异点 |

### 3.3 验收方式

- 提交核心驱动链设计文档
- 自我检查清单
- 无需外部评审

---

## 四、M2: 情节骨架稿

### 4.1 交付物

- 三幕式情节骨架（bullet point格式）

### 4.2 验收标准

| 标准 | 要求 |
|------|------|
| **结构完整** | 三幕式十二节点齐全 |
| **冲突成立** | 每个情节点有明确阻碍 |
| **动机成立** | 主角/反派选择合理 |
| **节奏合理** | 冲突分布均匀 |
| **驱动链体现** | 核心驱动链可识别 |
| **骨架好玩** | 自问：如果这是游戏，好玩吗？ |

### 4.3 验收方式

- 提交情节骨架文档
- 自我检查清单
- 如有需要，可找目标读者试读骨架

---

## 五、M3: 核心样章稿

### 5.1 交付物

- 5000-10000字核心样章
- 读者测试记录（如有）

### 5.2 验收标准

| 标准 | 要求 |
|------|------|
| **字数** | 5000-10000字 |
| **开篇钩子** | 前300字有悬念 |
| **中段冲突** | 包含完整冲突 |
| **叙事声音** | 风格统一，有辨识度 |
| **情感锚点** | 有情绪波动点 |
| **结尾满足** | 有明确的小满足点 |
| **自检通过** | 全部自检项通过 |

### 5.3 验收方式

- 提交核心样章
- 完成自检清单
- 如有条件，进行目标读者测试

---

## 六、M4: 初稿（Alpha）

### 6.1 交付物

- 全部章节初稿

### 6.2 验收标准

| 标准 | 要求 |
|------|------|
| **字数** | 达到预计总字数的80%以上 |
| **情节完整** | 三幕式结构完整 |
| **章节完成** | 所有章节都有内容 |
| **无重大逻辑漏洞** | 重大逻辑问题已记录 |
| **人物完整** | 主要角色都有交代 |

### 6.3 验收方式

- 提交初稿文档
- 完成自我检查清单
- 如有重大问题，记录待解决

---

## 七、M5: 评审稿（Beta）

### 7.1 交付物

- 经过系统化评审的稿件
- 缺陷标注清单

### 7.2 验收标准

| 标准 | 要求 |
|------|------|
| **评审完成** | 按S1-S7维度完成评审 |
| **缺陷标注** | 所有H/S级缺陷已标注位置和描述 |
| **缺陷分级** | 每个缺陷已分级（H/S/M/L） |
| **修改优先级** | 已排定修改优先级 |

### 7.3 验收方式

- 提交评审稿
- 提交缺陷标注清单
- 按评审前置检查完成自检

---

## 八、M6: 定稿（GM）

### 8.1 交付物

- 零H级缺陷的终稿
- 定稿说明文档

### 8.2 验收标准

| 标准 | 要求 |
|------|------|
| **H级缺陷** | 零H级缺陷 |
| **S级缺陷** | 已评估并处理 |
| **结构完整** | 三幕式结构完整 |
| **逻辑自洽** | 时间线、势力行为合理 |
| **文笔统一** | 语言风格统一 |
| **可读性** | 读者愿意一口气读完 |

### 8.3 验收方式

- 提交终稿
- 提交定稿说明（包含处理的缺陷列表）
- 完成最终自我检查

## 九、里程碑管理原则

### 9.1 通过原则

- 每个里程碑必须通过才能进入下一阶段
- 如果里程碑未通过，返回上一阶段修改
- 记录未通过原因，用于持续改进

### 9.2 变更原则

- 里程碑顺序可以根据项目实际情况调整
- 但M1必须在M2前，M2必须在M3前
- M4-M6必须按顺序完成

### 9.3 加速原则

- 对于有经验的作者，可以并行完成部分里程碑
- 但核心验证环节不能跳过
- 核心样章验证是可选的，但强烈建议执行
