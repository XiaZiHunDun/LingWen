# 每日创作通告单规范

> **版本**：v1.0
> **来源**：借鉴电影制作「通告单」和「Dailies」制度
> **日期**：2026-05-19

---

## 核心理念

**任务清晰、执行简单、当日闭环。**

类比电影制作的「通告单」（Call Sheet）和「每日样片」（Dailies），建立每日创作的任务下达和成果汇报机制。

> **核心原则**：每天有明确任务，每天有结果反馈，形成快速闭环。

---

## 一、通告单定义

### 1.1 什么是通告单

**通告单**是每日创作任务的「下达单」，由调度 Agent 前一天生成，包含：
- 明日要完成的场景/章节
- 所需的参考素材（人物设定、上文锁定内容）
- 质量要求和截止时间

### 1.2 通告单的作用

| 作用 | 说明 |
|------|------|
| **任务明确** | 作家知道明天写什么，不用临场思考 |
| **素材就位** | 参考资料提前备好，减少查找时间 |
| **预期对齐** | 主控和作家对任务范围达成共识 |
| **进度可控** | 按单执行，便于追踪和管理 |

---

## 二、Dailies报告定义

### 2.1 什么是Dailies报告

**Dailies报告**是每日创作的「完成报告」，由作家当日填写，包含：
- 今日完成情况（字数、章节）
- 遇到的问题
- 偏离情况
- 明日计划

### 2.2 Dailies报告的作用

| 作用 | 说明 |
|------|------|
| **成果可见** | 主控即时了解创作进度 |
| **问题暴露** | 及时发现卡点，避免积压 |
| **偏差纠正** | 偏离大纲时能及时发现 |
| **衔接顺畅** | 明日任务建立在今日基础上 |

---

## 三、调度Agent生成通告单的逻辑

### 3.1 生成时机

- **每天固定时间生成**：例如每日 20:00
- **触发条件**：前一天的 Dailies 报告已提交

### 3.2 生成流程

```
T-1日 20:00
    ↓
调度Agent读取今日Dailies报告
    ↓
分析完成情况 + 剩余任务
    ↓
生成明日通告单（含任务、素材、要求）
    ↓
写入通告单仓库（`07_意见仓库/dailies/`）
    ↓
通知作家确认
```

### 3.3 生成规则

| 规则 | 说明 |
|------|------|
| **任务量** | 根据作家日均产能设定，通常为1-2个场景 |
| **难度匹配** | 任务难度与作家画像匹配（新手→简单场景） |
| **素材预置** | 自动关联需要的人物设定和上文 |
| **优先级** | 优先安排驱动链关键场景 |

### 3.4 生成输入

调度Agent生成通告单时需要读取：

```markdown
1. 当前阶段状态（PHASE_X）
2. 今日Dailies报告（完成情况、问题记录）
3. 章节计划表（剩余未完成章节）
4. 作家画像（日产能、擅长类型）
5. 人物设定库（相关角色）
6. 已锁定上文（续写起点）
```

### 3.5 生成输出

```markdown
## 明日通告单

**日期**：YYYY-MM-DD（次日）
**执行作家**：[作家名]
**项目**：[项目名称]

### 今日任务
1. 场景X：第X章-X节「场景名称」
2. （可选）场景Y：第Y章-Y节「场景名称」

### 参考素材
- 人物设定：[角色名] → `02_作家工作室/作家X/characters/角色名.md`
- 已锁定上文：`03_内容仓库/卷X/第X章_上文.md`

### 质量要求
- 目标字数：2000-3000字
- 截止时间：当日 22:00
- 特殊要求：[如有]

### 备注
[其他注意事项]
```

## 四、作家填写Dailies报告的流程

### 4.1 填写时机

- **每日创作结束后**：例如每日 22:00 前
- **触发条件**：当日通告单任务已完成（或放弃）

### 4.2 填写流程

```
每日创作结束
    ↓
作家填写Dailies报告
    ↓
写入通告单仓库（`07_意见仓库/dailies/`）
    ↓
调度Agent读取 → 生成次日通告单
```

### 4.3 填写检查项

作家填写Dailies报告时检查：

```markdown
### 今日完成情况
□ 场景X：完成 / 部分完成 / 未完成
□ 场景Y：完成 / 部分完成 / 未完成

### 字数统计
- 目标字数：____
- 实际字数：____
- 达成率：____%

### 遇到的问题
- [问题描述]

### 偏离情况
□ 无偏离
□ 有偏离：[偏离描述]

### 明日计划
- [明日的创作计划]
```

## 五、通告单与Dailies的衔接

### 5.1 正常流程

```
通告单（T日） → 作家执行 → Dailies报告（T日） → 通告单（T+1日）
```

### 5.2 问题升级

如果Dailies报告显示连续3天未完成任务：

```
连续3天未完成
    ↓
触发预警
    ↓
调度Agent报告主控
    ↓
主控决策：
  - 调整任务量（降低目标）
  - 更换作家
  - 暂停项目
```

### 5.3 状态可视化

```
每日创作状态：

[今日通告单执行] ██████████ 100% ✓
[今日Dailies] █████████░ 90% (字数达成)
[问题数] 1个（已记录）
[连续未完成] 0天

本周进度：
Mon ████
Tue ████
Wed ████
Thu ███░
Fri ░░░░
```

## 六、文件存储规范

### 6.1 目录结构

```
07_意见仓库/
└── dailies/
    ├── 通告单_YYYY-MM-DD.md      # 每日通告单
    └── Dailies_YYYY-MM-DD.md     # 每日Dailies报告
```

### 6.2 命名规范

| 类型 | 格式 | 示例 |
|------|------|------|
| 通告单 | `通告单_YYYY-MM-DD.md` | `通告单_2026-05-20.md` |
| Dailies报告 | `Dailies_YYYY-MM-DD.md` | `Dailies_2026-05-19.md` |

## 七、与现有机制的衔接

### 7.1 与「每日创作Review」的关系

```
通告单 → 执行创作 → Dailies报告
                    ↓
              每日创作Review（自我复盘）
                    ↓
              发现问题 → 纳入下一通告单
```

- **通告单**侧重「任务下达」
- **每日Review**侧重「自我复盘」
- 两者互补，共享同一存储位置

### 7.2 与里程碑管理的关系

```
M4初稿阶段：
  里程碑目标：30天完成第一卷
       ↓
  分解为每日任务 → 通告单机制
       ↓
  每日Dailies追踪进度
       ↓
  里程碑验收
```
