## 一、表结构说明

每个角色一个状态表，记录关键状态及最后确认章节。

| 字段 | 说明 |
|------|------|
| 状态类型 | 生死/形态/位置/关系/情绪等 |
| 状态值 | 具体状态描述 |
| 最后确认 | 最后确认此状态的章节号 |
| 状态来源 | 自动检测/手动确认 |

## 三、使用说明

### 3.1 状态检查时机

- **创作前**: 作家在写本章前确认相关角色状态
- **审核中**: 审核员检查本章角色状态是否与表一致
- **修改后**: 修改完成后更新表中的确认章节

### 3.2 状态冲突检测

当发现本章角色状态与表中不符时：
1. 标记为 P1 问题
2. 根据情节合理性判定：
   - 如本章正确：更新表中状态值和确认章节
   - 如本章错误：返回作家修改

### 3.3 新角色登记

新角色首次出现时，在本表添加记录：
```markdown
### {角色名}

| 状态类型 | 状态值 | 最后确认章节 | 状态来源 |
|----------|--------|--------------|----------|
| 生死 | | chXXX | 首次登场 |
| 形态 | | chXXX | 首次登场 |
| 位置 | | chXXX | 首次登场 |
```

## 四、状态类型定义

| 类型 | 说明 | 关注点 |
|------|------|--------|
| 生死 | 角色是否存活 | 特别注意死亡与复活 |
| 形态 | 外貌形态（人类/变身/灵体等） | 多次变化的角色 |
| 位置 | 当前所在地点 | 跨场景切换 |
| 关系 | 与其他角色的关系 | 情感变化 |
| 情绪 | 当前情绪状态 | 重大情绪变化 |
| 能力 | 当前能力状态 | 升级/封印/丧失 |
| 身份 | 社会身份/地位 | 职位变化 |

---

## 五、状态一致性规则

1. **单一真相**: 同一时间同一角色同一状态只能有一个值
2. **可追溯**: 状态变化必须有对应章节记录
3. **可验证**: 状态变化需有情节支撑
4. **禁止漂移**: 角色状态不得在无情节支撑的情况下变化
