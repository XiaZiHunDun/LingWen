# 情节嵌套循环模板 v1.0

> 从game-designer.md的嵌套循环模型迁移，专门适配小说叙事

## 嵌套循环模型

### 微循环（30秒~5分钟 → 单场景）

```markdown
场景目标：角色想达成什么？
场景障碍：什么阻止他们？
行动：角色如何尝试？
结果：行动带来什么新情况？
```

示例：

```
场景目标：主角想逃离监狱
场景障碍：门锁需要钥匙，狱卒在巡逻
行动：主角贿赂狱卒获取钥匙
结果：钥匙到手但被其他囚犯发现
```

### 中循环（15~30分钟 → 单章节）

```markdown
章节悬念：本章结尾读者最想知道什么？
积累阶段：如何逐步加深悬念？
高潮点：何时揭晓/反转？
释放点：高潮后如何过渡到下章钩子？
```

### 宏循环（全书级）

```markdown
卷1：建立世界规则 + 主角困境 + 第一次重大选择
卷2：扩展冲突 + 主角成长 + 揭示更深层真相
卷3：最终决战 + 主角决策 + 结局收束

回归理由：为什么读者会继续读？
- 悬念钩子（核心谜题未解）
- 情感钩子（角色命运牵挂）
- 成长钩子（主角潜力未完全释放）
```

## 设计工具

### 锯齿模式

```
张力峰值 → 略降基线 → 更高张力峰值 → ...
    ↓           ↓           ↓
 危机事件    喘息的间    更大危机
```

### 反馈设计

- 微观反馈（0.5秒内）：角色反应、表情变化
- 中观反馈（5-15分钟）：本章结局的满足感/悬念感
- 宏观反馈（章节后）：跨章节的伏笔回收、新谜题

## 使用说明

1. 为每卷设计宏观循环（起承转合）
2. 为每阶段设计章间的中循环（悬念-高潮-释放）
3. 为每场景设计微循环（目标-障碍-行动-结果）
4. 检查嵌套是否连贯：微循环如何服务中循环？中循环如何服务宏循环？
