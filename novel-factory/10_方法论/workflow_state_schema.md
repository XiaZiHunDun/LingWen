# workflow_state.json Schema 说明

> 本文档说明 `workflow_state.json` 的结构，用于新项目初始化时参考。
> 方法论嵌入：在每个步骤中增加 `methodology_markers` 字段，使工作流与方法论对齐。

---

## 核心概念

### 方法论标记（methodology_markers）

在每个步骤的 `output` 或 `note` 字段中，增加方法论溯源：

```yaml
methodology_markers:
  stage: "阶段X: 阶段名称"     # 对应方法论文档
  chapter_scope: "chXXX-chYYY"  # 该步骤影响的章节范围
  structure: "三幕式/8节点"     # 使用的结构模板
  check_completed: true
  check_items:
    - "[x] 检查项1"
    - "[x] 检查项2"
```

## 新项目初始化时的方法论检查清单

初始化 `run_init.sh` 时，确保：

- [ ] `01_灵感库/模板库/` 存在（含所有模板文件）
- [ ] `10_方法论/` 存在（含所有方法论文档）
- [ ] `workflow_state.json` 的每个步骤可追溯到方法论文档
- [ ] 审核意见模板包含 `methodology_dimension` 字段
