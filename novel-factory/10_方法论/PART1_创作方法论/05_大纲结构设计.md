# 阶段3：大纲与结构设计

> **核心观点**：大纲是地图，能防止你迷路。先搭骨架，再添肉。

---

## 一、三幕式结构（新手推荐）

| 幕 | 占比 | 内容 |
|----|------|------|
| 第一幕 | 1-20% | 日常被打破，主角被迫上路 |
| 第二幕 | 20%-70% | 闯关+成长+阻碍升级+接近低谷 |
| 第三幕 | 70%-100% | 终极对决+收尾+人物蜕变 |

### 详细节点

```
第一幕（建置）
├── 日常：主角正常生活 → 展现性格+处境
├── 诱发事件：意外卷入 → 打破日常
└── 第一扇门：主角做出选择，踏入冲突（无法回头）

第二幕（对抗）
├── 尝试：主角第一次尝试解决问题 → 失败
├── 帮手+反派：遇到帮手 / 反派施压
├── 中点：重大转折（信息揭露/局势逆转）
├── 最低谷：一切崩溃，主角陷入绝境
└── 决心：主角触底反弹，决心反击

第三幕（解决）
├── 终极对决：主角直面最大阻碍
├── 关键选择：做出最重要的决定/牺牲
└── 结局：目标达成/遗憾，伏笔回收，蜕变闭环
```

## 三、15节拍表（进阶版）

来自经典三幕+节拍设计：

| 节拍 | 位置 | 内容 |
|------|------|------|
| 诱发事件 | ~10% | 打破日常平衡的事件 |
| 第一情节点 | ~25% | 主角主动踏入冲突 |
| 第一次答辩 | 25%-~ | 盟友帮助，应对第一考验 |

| 结局 | ~95% | 新平衡，变化后的人物 |

## 四、章纲写法

**每章一句话**（1-2句），只写"这章发生什么事"

| 要素 | 内容 |
|------|------|
| 本章目标 | 这章要推进什么？ |
| 出场人物 | 谁出现？ |
| 剧情走向 | 发生什么？ |
| 情绪落点 | 读者读这章的感受？ |
| 留钩子 | 章末如何让读者想翻下一页？ |

### 示例
```
第3章：本章目标=建立主角的初始能力；出场人物=主角+导师；
剧情走向=主角意外触发能力，被导师发现；情绪落点=兴奋+期待；
留钩子=导师说出"你和我见过的人都不一样"。
```

## 五、动态大纲（预览机制）

> **来源**：动画制作动态分镜（Animatic）概念
> **作用**：在细纲完成后，用可视化方式预览全篇节奏，验证结构是否流畅

### 5.1 什么是动态大纲

动态大纲是在细纲完成之后、正式写作之前，用"章节摘要+字数预估+情绪标签"的方式拼成可视化预览文档。

**核心目的**：让节奏问题在写作前暴露，而不是写完才发现。

### 5.2 动态大纲格式

```markdown
## 动态大纲预览表

| 章节 | 核心事件 | 字数预估 | 情绪基调 | 节奏类型 | 备注 |
|------|----------|----------|----------|----------|------|
| Ch1 | 主角出场+困境 | 3000 | 低沉→希望 | P3 | 开篇需抓人 |
| Ch2 | 金手指出现 | 2500 | 兴奋 | P1 | 第一个爽点 |
| Ch3 | 第一次战斗 | 4000 | 紧张 | P1 | 打斗场景 |
| Ch4 | 战后休整 | 2000 | 平稳 | P3 | 呼吸空间 |
| ... | ... | ... | ... | ... | ... |

节奏类型说明：
- P1（快节奏）：短句、动作、冲突密集 → 打斗、高潮、危机
- P2（中节奏）：平衡叙述和动作 → 日常冲突、情节推进
- P3（慢节奏）：描写、内心独白、氛围 → 日常、感情、过渡
```

### 5.3 动态大纲检查清单

- [ ] 每章有明确的核心事件（不是流水账）
- [ ] 节奏类型分配合理（快/慢交替）
- [ ] 连续快节奏不超过3章，需插入1章慢节奏"呼吸"
- [ ] 每卷至少有2个快节奏高潮
- [ ] 情绪基调变化有逻辑
- [ ] 字数分布符合目标读者预期

### 5.4 使用场景

| 场景 | 时机 | 目的 |
|------|------|------|
| 细纲完成后 | 进入写作前 | 验证整体节奏是否流畅 |
| 第一幕写完后 | 写作中途 | 对照检查实际节奏是否符合预期 |
| 修改阶段 | 修改前 | 评估修改对整体节奏的影响 |

### 5.5 与章纲的区别

| 维度 | 章纲 | 动态大纲 |
|------|------|----------|
| 视角 | 局部（单章） | 整体（全篇） |
| 重点 | 剧情内容 | 节奏分布 |
| 格式 | 文字描述 | 表格+标签 |
| 使用时机 | 写作前+写作中 | 写作前+写作中+修改前 |

### 5.6 示例

```markdown
## 《修仙逆袭录》动态大纲

| 章节 | 核心事件 | 字数 | 情绪 | 节奏 |
|------|----------|------|------|------|
| Ch1 | 废物主角被退婚 | 3000 | 屈辱→愤怒 | P3 |
| Ch2 | 偶获上古传承 | 2500 | 震惊→狂喜 | P1 |
| Ch3 | 第一次修炼突破 | 3500 | 专注→兴奋 | P2 |
| Ch4 | 家族大比碾压对手 | 4000 | 紧张→爽 | P1 |
| Ch5 | 大比后的嘲讽与轻视 | 2000 | 压抑 | P3 |
| Ch6 | 导师指点+境界提升 | 3000 | 平静→希望 | P2 |
| Ch7 | 强敌来袭+危机 | 4000 | 紧张→危机 | P1 |
```

## 六、大纲检查清单

- [ ] 第一幕有没有快速建立主角+困境？（前3章决定读者去留）
- [ ] 开篇是否有抓人的钩子？（悬念/危机/反常）
- [ ] 主角的选择是主动的还是被动的？（主动更有张力）
- [ ] 每章结尾是否有"翻页钩子"？
- [ ] 伏笔有没有在后续回收？（不要埋而不收）
- [ ] 高潮是否足够"高"？（所有冲突在此爆发）
- [ ] 结局是否收拢所有主要线索？（不烂尾，不留坑）
- [ ] 动态大纲节奏分布是否合理？（P1:P2:P3 ≈ 4:4:2）
