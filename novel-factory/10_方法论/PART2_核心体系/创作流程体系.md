# 创作流程体系

> **版本**：v4.0

---

## 核心理念

**"先完成，再完美"** —— 初稿的使命是"存在"，而不是"完美"。

### 修改成本系数

动画制作有一句格言："改一张草图只要几分钟，而改一个渲染好的3D模型可能要几天"。小说创作同样适用这个原则——越往后改，成本越高。

| 阶段 | 修改成本系数 | 说明 |
|---|---|---|
| 构思期 | 1x | 几乎无成本，纯思路整理 |
| 规划期（章纲阶段） | 5x | 文字修改，影响面有限 |
| 初稿阶段（1-50%） | 20x | 修改影响多个章节，但仍可重构 |
| 初稿阶段（50-100%） | 50x | 修改牵一发动全身，代价高昂 |
| 修改期 | 100x+ | 越往后越贵，每字改动都影响周围语境 |

**使用原则**：如果在当前阶段发现前面的问题，且成本系数 ≤ 20x，立即回去修正；成本系数 > 20x，用 `//TODO: 标记` 先搁置，等下一版处理。

创作是一个从"灵感碎片"到"完整作品"的系统工程，遵循科学流程能大幅减少烂尾和卡文。

---

## 六阶段创作模型

```
阶段一：构思期（10%）
└─ 把"想写"变成"值得写"的故事核心

阶段二：规划期（20%）
└─ 搭建骨架，设定人物与世界
└─ 核心驱动链设计 [新增]
└─ 范围边界确定 [新增]

阶段三：验证期（10%）[新增]
└─ 情节骨架验证（灰盒原型）
└─ 核心样章验证（垂直切片）
└─ 目标读者早期测试

阶段四：写作期（40%）
└─ 完成初稿，不回头修改

阶段五：修改期（25%）
└─ 分层修改，精雕细琢

阶段六：完成期（5%）
└─ 定稿与发布
```

### 阶段重心说明

| 阶段 | 核心理念 | 禁止事项 |
|------|----------|----------|
| 构思期 | 找到核心冲动 | 禁止开始写正文 |
| 规划期 | 搭好骨架 | 禁止投入文笔资源 |
| **验证期** | **验证"故事是否成立"** | **禁止跳过验证** |
| 写作期 | 完成初稿 | 禁止回头修改 |
| 修改期 | 分层修改 | 禁止大改结构 |
| 完成期 | 定稿发布 | — |

## 节点详解

### 阶段一：构思期

#### 节点1：捕捉核心冲动

先问自己：**"我最想写的是一个什么'感觉'？"**

- 一个画面（深夜末班车上，邻座乘客没有脸）
- 一种情绪（被背叛后重生复仇的爽感）
- 一个"如果"（如果人类可以贩卖寿命）
- 一段关系（两个杀手相爱，任务却是杀死对方）

**不要急着写故事，先把这个"核心冲动"用一句话锚定。**

#### 节点2：确定类型与读者契约

类型决定叙事规则和读者预期：

| 类型 | 核心契约 |
|------|----------|
| 悬疑 | 必须给足线索，结尾反转要公平 |
| 言情 | 情感张力优先，人物动机要细腻 |
| 科幻 | 设定自洽，逻辑推演要硬核 |
| 玄幻 | 升级体系清晰，爽点密集 |
| 现实主义 | 细节真实，社会肌理要准确 |

**新手建议**：先写"类型明确"的小说，降低市场认知成本。

#### 节点3：提炼"一句话梗概"

用模板压缩：**"谁，想要什么，但遇到什么阻碍，如果失败会怎样"**

> 示例：一个失忆的刑警（谁）必须在72小时内找回记忆并阻止连环杀手（想要什么），但他发现所有证据都指向自己就是凶手（阻碍），如果失败，下一个死者将是他的女儿（代价）。

#### 节点3.5：核心驱动链设计 [新增]

定义读者每X章获得的"满足感"类型和频率：

| 问题 | 回答 |
|------|------|
| 类型定位 | 你的小说是哪种类型？ |
| 核心期待 | 读者最想获得什么？（爽感/悲伤感/好奇感/甜蜜感） |
| 小满足频率 | 每几章给读者一次小满足？ |
| 大满足频率 | 每几卷给读者一次大满足？ |
| 递增性 | 随着故事发展，满足感是否越来越强？ |

详见：`core/核心驱动链设计.md`

#### 节点3.6：范围边界确定 [新增]

在规划期必须明确回答以下问题：

| 问题 | 说明 | 建议 |
|------|------|------|
| 字数边界 | 预计总字数？ | 新手控制在30-50万字内 |
| 主线数量 | 单线还是多线？ | 新手建议单线 |
| 世界观边界 | 设定深度？ | 够用就好 |
| 核心角色数 | 主要角色几个？ | 不超过10个 |
| 情感线数量 | 多条情感线？ | 建议一条主线 |

**原则**：所有新增需求记录到"需求池"，等本书完成后再考虑。

---

### 阶段二：规划期

#### 节点4：人物设定（比情节更重要）

每个主要角色需要回答：
- **表面欲望**（想要什么）vs **深层需求**（真正需要什么）
- **致命弱点**（性格缺陷或心理创伤）
- **秘密**（不愿被人知道的过去）
- **弧线**（故事结束时他/她改变了什么）

> 好的人物不是"完美的人"，而是"有缺陷的人做出了艰难的选择"。

**核心公式**：
```
有缺陷的主角 + 清晰目标 + 层层阻碍 = 故事骨架
```

#### 节点5：世界观与规则

哪怕写现实题材，也要建立"故事内的微观规则"：
- 时间跨度（三天？三十年？）
- 空间边界（一座小镇？一艘飞船？）
- 特殊规则（魔法体系？社会潜规则？）
- 信息差（读者知道但角色不知道？还是反过来？）

**设定原则**：设定是为故事服务的，不要为了展示设定而拖慢情节。只做"够用"的设定，精简够用即可，别堆废设定。

#### 节点6：结构设计（三幕式 + 节拍表）

```
第一幕（设定，25%）：日常 → 诱发事件 → 第一扇门
第二幕（对抗，50%）： midpoint虚假胜利/失败 → 一切崩溃 → 第二扇门
第三幕（解决，25%）：灵魂黑夜 → 高潮 → 新平衡
```

**关键节点检查**：
- **诱发事件**（Inciting Incident）发生在全书约10%处
- **第一情节点**（Plot Point 1）约25%，主角主动踏入冲突
- **中点**（Midpoint）约50%，信息/局势发生重大转变
- **第二情节点**（Plot Point 2）约75%，主角被逼入绝境
- **高潮**（Climax）约90%，核心冲突爆发

**章纲写法**：每卷定**卷名+核心事件**，每章写：
- 本章目标
- 出场人物
- 剧情走向
- 情绪落点
- 留钩子

#### 节点6.5：锁定门（Lock Gate）

在进入写作期之前，必须通过以下检查清单：

| 检查项 | 标准 |
|---|---|
| 一句话梗概 | 已确定，不超过50字 |
| 核心角色 | 主要角色（≤5个）设定完成，包含欲望/阻碍/代价 |
| 三幕式大纲 | 关键节点标记完毕（诱发事件/第一扇门/中点/第二扇门/高潮） |
| 世界观核心规则 | 已文字化，不超过500字 |
| 第一卷章纲 | 已完成，每章有明确的目标/情绪落点/翻页钩子 |

**通过条件**：全部通过 → 进入写作期；任一项未通过 → 返回对应节点重新梳理。

> 锁定门是动画制作"前期锁定"原则的核心体现。小说创作同样适用：规划期的充分准备能大幅降低写作期的返工成本。

### 阶段三：验证期 [新增]

> **核心理念**：如果骨架不好玩，加上再好的文笔内核依然是空洞的。

#### 节点3.7：情节骨架验证（灰盒原型）

用最简陋的文字（bullet point）写出核心情节走向：

**写作规范**：
- 用第三人称叙述，不写具体对话
- 只写"发生了什么"，不写"怎么发生的"
- 不写心理描写、环境描写
- 每个情节点用1-3句话概括

**验证检查点**：
- [ ] 结构完整（三幕式十二节点齐全）
- [ ] 冲突成立（每个情节点有明确阻碍）
- [ ] 动机成立（主角/反派选择合理）
- [ ] 节奏合理（冲突分布均匀）
- [ ] 驱动链体现（核心驱动链可识别）

详见：`core/情节骨架验证.md`

#### 节点3.8：核心样章验证（垂直切片）

写一个完整段落（5000-10000字），代表全书最高水平：

**验证标准**：
- 前300字是否有悬念
- 是否包含完整冲突
- 叙事声音是否统一
- 是否有情感锚点
- 结尾是否有满足感

**适用范围**：自我验证、编辑投稿、读者测试

详见：`core/核心样章验证.md`

#### 节点3.9：目标读者早期测试

在核心样章验证通过后，进行目标读者早期测试：

**测试规模**：3-5名目标读者

**测试方式**：
- 给出核心样章，让读者自己阅读
- 观察读者行为（不要现场提问）
- 读后问三个问题

**反馈处理原则**：
- 行为数据优先于意见
- 沉默的大多数：如果多数人行为一致，重视
- 聚焦改进点：每轮只改最关键的1-2个问题

### 阶段四：写作期

#### 节点10：初稿策略

- **设定每日定额**：新手建议500-1500字/天，固定时间写作
- **关闭自我审查**：初稿阶段禁止回头修改，用 `//TODO: 这里要改//` 标记跳过
- **从最容易的场景开始**：不必按顺序写，先写你最兴奋的那场戏

#### 节点11：场景的微观结构

每个场景都是一次"微型冲突"：
- **目标**：角色进入场景想要什么？
- **阻碍**：谁/什么阻止了他？
- **灾难/转折**：结果如何？
- **余波**：情绪/关系发生了什么变化？

#### 节点12：节奏控制

- **快节奏**：短句、动作、对话、章节结尾留钩子
- **慢节奏**：描写、内心独白、氛围渲染、人物关系深化
- **交替原则**：紧张场景后接"呼吸空间"，避免读者疲劳

**实用节奏**：
- 3章一小爽，10章一大转折
- 张弛有度：打斗/冲突→日常/铺垫→再爆发
- 全程埋伏笔，前后呼应

---

### 阶段五：修改期

#### 节点13：冷却期

初稿完成后，**至少放1-2周不看**。你需要用"陌生读者"的眼光重新审视。

#### 节点14：分层修改（Block → Polish 两阶段）

动画制作有两个阶段：Blocking（摆关键姿势）和 Polishing（精修补间）。小说修改借鉴这个思路，分为两个完全分离的阶段。

### 阶段一：Block（粗修改）—— 解决"姿势对不对"

**目标**：验证结构问题是否解决。在这个阶段，只改结构，不改文字。

**检查清单**：

| 检查项 | 标准 |
|---|---|
| 人物弧线 | 主角在结局是否发生了可感知的变化 |
| 三幕结构 | 第一幕是否钩人，第二幕是否撑住，第三幕是否有力 |
| 节拍清晰度 | 每章是否有明确的目标、阻碍、结果 |
| 信息释放节奏 | 读者是否在正确的时间获得正确的信息 |
| 势力行为逻辑 | 配角/反派的行为是否与他们的设定一致 |

**禁止操作**：本阶段禁止修改具体措辞、句式、形容词。如发现文字问题，用 `//TODO: 文笔-章节X-内容` 标记，搁置到Polish阶段。

### 阶段二：Polish（精修改）—— 解决"动作流不流畅"

**目标**：解决文字质量问题。在这个阶段，不碰结构，只改文字。

**检查清单**：

| 检查项 | 标准 |
|---|---|
| 句式变化 | 是否有长有短，有动有静 |
| 描写密度 | 关键场景是否有足够的感官细节，非关键场景是否足够简洁 |
| 对话性格 | 每个角色是否有独立的说话方式（通过词汇/句式/语气区分） |
| 视角一致性 | 是否避免了视角跳跃（尤其是第三人称） |
| 可读性 | 朗读一遍，标记读起来拗口的地方 |

**禁止操作**：本阶段禁止修改任何结构、情节、人物设定。只改文字。

### 为什么必须分离

如果在Block阶段一边改结构一边改文字，会陷入"改了结构后发现文字不对，改了文字后发现结构要调整"的无尽循环。两阶段分离能大幅提高效率。

#### 节点15：外部反馈

找 **2-3位目标读者**（不要找"会无条件夸你的人"）

问他们三个问题：
1. 哪里让你感到无聊/想跳过？
2. 哪个角色你最喜欢/最不喜欢？为什么？
3. 结尾让你感到满足、意外，还是困惑？

---

### 阶段六：完成期

#### 节点16：标题与开篇

- 标题要么**有悬念**（《消失的爱人》），要么**有情绪**（《追风筝的人》），要么**有概念**（《三体》）
- 开篇300字必须包含：**一个动作、一个悬念、一个情绪锚点**

#### 节点17：发布渠道适配

| 渠道 | 特点 | 要点 |
|------|------|------|
| 网络平台（起点/晋江） | 连载，考验更新频率和故事噱头 | 按平台风格拆章、控制每章字数与断章点 |
| 传统出版 | 完整书稿，审稿周期长 | 准备好完整书稿、故事梗概、作者简介 |
| 自出版（Kindle/豆瓣） | 完全自主权 | 排版、封面、简介、样章 |

#### 节点18：持续迭代

把更多经验留给下一本书，比在一本上死磕更健康。

## 六、并行管线（进阶·可选）

对于有1-2本完本经验的作者，可以采用多卷并行创作来提高效率。

### 6.1 基本结构

```
卷1：写作期（进行中）  ← 当前主力
卷2：规划期（已锁定，等待）  ← 准备下一卷
卷3：构思期（进行中）  ← 储备概念

轮转：卷1完成后 →
      卷2进入写作期 →
      卷1进入修改期 →
      卷3进入规划期
```

### 6.2 限制条件

| 限制 | 说明 |
|---|---|
| 并行卷数 ≤ 3 | 超过3卷会导致精力分散和质量下降 |
| 只允许相邻阶段并行 | 写作期只能和规划期/构思期并行，不能和修改期并行 |
| 前一个锁定门未通过，不能启动下一个 | 并行不是跳步，是提前准备 |
| 初稿阶段不并行 | 写初稿时必须专注，不能同时写两个初稿 |

### 6.3 不适用场景

- 新手首个创作项目
- 总字数 < 20万的作品
- 小说结构高度线性、前后强关联的作品

> 并行管线的灵感来自动画制作的并行制作管线："第1集进入上色时，第3集可能还在分镜"。但小说创作不如动画工业化程度高，此机制为可选进阶，非必须。

## 与评审体系的接口

```
创作阶段完成 → 评审前置检查 → 进入评审流程
```

### 评审前置检查清单

进入评审前，必须自检：

**Hard约束自检**：
- [ ] 叙事主角唯一性是否保持
- [ ] 修炼/战力里程碑是否对齐设定
- [ ] 卷间关键衔接是否连贯

**Soft维度自检**：
- [ ] S1 剧情完整性：章内目标—障碍—结果是否清楚
- [ ] S2 逻辑自洽：时间线、战力、势力行为是否合理
- [ ] S3 文笔风格：语言、对话、描写是否统一
- [ ] S4 情感共鸣：情绪曲线是否有效
- [ ] S5 节奏控制：信息投放缓急是否服务于目标
- [ ] S6 可读性：读者是否愿意一口气读完
- [ ] S7 主角魅力：读者是否想看主角成功

**创作质量自检**：
- [ ] 主角目标是否清晰
- [ ] 核心冲突是否存在
- [ ] 三幕式结构是否完整
- [ ] 每章是否有明确的目标/阻碍/结果
- [ ] 伏笔是否已记录

## 极简启动路径（7天）

| 天数 | 任务 |
|------|------|
| Day 1 | 写出"核心冲动" + 一句话梗概 |
| Day 2 | 设定主角（欲望、缺陷、秘密）+ 反派/对手 |
| Day 3 | 画出三幕式大纲（标记关键节点） |
| Day 4 | 写开篇2000字（不要回头改） |
| Day 5-7 | 每天写1000字，完成第一幕 |
