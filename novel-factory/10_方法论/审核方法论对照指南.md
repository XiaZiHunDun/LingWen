## 审核清单（按方法论阶段）

### 灵感阶段审核（对应 01_灵感捕捉与筛选.md）

| 检查项 | 标准 |
|--------|------|
| 灵感是否满足筛选标准 | 冲突/反差/情绪价值，至少满足一个 |
| 一句话故事是否清晰 | 主角+目标+阻碍+代价，四要素完整 |
| 核心三要素是否明确 | 主角/目标/阻碍三要素齐全 |

---

### 大纲阶段审核（对应 05_大纲结构设计.md）

#### 三幕式结构检查

| 节点 | 位置 | 检查 |
|------|------|------|
| 日常打破 | 第一幕开头 | 主角是否快速出场，困境是否明确 |
| 第一扇门 | 约25% | 主角是否主动踏入冲突 |
| 中点转折 | 约50% | 信息/局势是否重大转变 |
| 最低谷 | 约75% | 一切是否崩溃，主角是否被逼入绝境 |
| 高潮 | 约90% | 核心冲突是否爆发 |
| 结局 | 90%-100% | 伏笔是否回收，人物是否蜕变 |

#### 8节点检查

```
节点1：主角日常 —— [ ] 有
节点2：突发意外 —— [ ] 有
节点3：第一次尝试失败 —— [ ] 有
节点4：遇帮手，反派加压 —— [ ] 有
节点5：跌入最低谷 —— [ ] 有
节点6：醒悟，决心反击 —— [ ] 有
节点7：终极对决 —— [ ] 有
节点8：结局收尾，人物蜕变 —— [ ] 有
```

#### 章纲检查

| 检查项 | 标准 |
|--------|------|
| 每章目标是否明确 | 1-2句话能说明这章干什么 |
| 每章结尾是否有钩子 | 让读者想翻下一页 |
| 伏笔是否埋下 | 前期小细节，中后期回收 |

### 初稿阶段审核（对应 06_初稿写作.md）

#### 开篇三原则（前3章）

| 原则 | 检查 |
|------|------|
| 快速出场主角 | 前3段让人知道谁是主角 |
| 立刻抛出困境/危机 | 让读者知道主角面临什么 |
| 直接给希望/转机 | 让读者知道主角有机会 |

#### Show Don't Tell 检查

| ❌ 讲述（扣分） | ✅ 展示（加分） |
|---------------|----------------|
| "他很紧张" | 他反复解开又扣上袖扣，咖啡洒了也没察觉 |
| "她恨他" | 她把他送的围巾剪碎，却把碎布留在抽屉里 |

#### 场景节拍检查

每个场景是否有：
- **目标**：角色想要什么？
- **阻碍**：谁/什么阻止他？
- **转折/灾难**：结局更好还是更糟？
- **余波**：情绪/关系发生了什么变化？

### 修改阶段审核（对应 07_修改打磨.md）

#### 第一轮：宏观结构

| 检查维度 | 问题表现 |
|---------|---------|
| 节奏 | 某段拖沓超过3章无实质进展 |
| 高潮 | 高潮不够有力 |
| 废章节 | 删掉某章不影响主线 |
| 人物降智 | 人物行为突然不符合性格 |
| 逻辑漏洞 | 时间线矛盾/设定矛盾 |

#### 第二轮：中观节奏

| 检查维度 | 问题表现 |
|---------|---------|
| 翻页钩子 | 章末没有让读者想问"然后呢" |
| 冗余场景 | 删掉某场景不影响主线 |
| 信息节奏 | 读者困惑太久或过早剧透 |
| 情绪曲线 | 全程一个情绪，无起伏 |

#### 第三轮：微观语言

| 检查维度 | 问题表现 |
|---------|---------|
| 副词滥用 | "X地说"出现多次 |
| 形容词堆砌 | 形容词过多，名词/动词过少 |
| 拗口 | 朗读出来别扭 |
| 对话书面语 | 对话不像人说话 |
| 心理直白 | 过多"我觉得..." |

## P0/P1/P2 问题与方法论映射

| 优先级 | 典型问题 | 对应方法论阶段 |
|--------|---------|---------------|
| **P0 阻塞** | 章节重复/内容丢失/逻辑矛盾 | 宏观结构（第一轮） |
| **P1 影响** | 节奏拖沓/伏笔断裂/人物状态矛盾 | 中观节奏（第二轮） |
| **P2 不阻塞** | 文笔粗糙/对话生硬/描写冗余 | 微观语言（第三轮） |

## 审核报告模板（方法论字段）

```yaml
review_report:
  chapter_range: "chXXX-chYYY"
  review_phase: "宏观/中观/微观"

  methodology_dimension: "07_修改打磨.md"  # 属于哪个方法论文档
  methodology_stage: "第一轮：宏观结构"     # 具体哪个阶段
  methodology_rule: "节奏检查"              # 具体触犯的规则

  issues:
    - severity: "P0/P1/P2"
      location: "chXXX"
      description: "..."
      methodology_reference: "05_大纲结构设计.md #章纲写法"
```

## 使用说明

1. 审核员在进行质量检查时，参照本指南的方法论对齐
2. 记录审核意见时，填写 `methodology_dimension` 字段
3. 定期汇总各方法论阶段的问题分布，用于改进方法论本身
