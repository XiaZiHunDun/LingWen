# 小说工厂方法论落地方案

> 分析日期：2026-05-19
> 状态：草稿

---

## 一、现状分析

### 1.1 现有架构

小说工厂已具备 7 阶段 25 步的成熟工作流，覆盖：
- 灵感 → 大纲 → 卷大纲 → 阶段大纲 → 正文 → 汇总 → 定稿发布

### 1.2 发现的主要GAP

| GAP | 现状 | 方法论要求 | 差距 |
|-----|------|----------|------|
| **灵感筛选** | 有模板库，输出灵感概念 | 需要"筛选-评估-一句话故事"流程 | 缺少结构化筛选机制 |
| **核心三要素** | 无专门记录，隐含在大纲中 | 需要在立项时明确主角/目标/阻碍 | 未独立文档化 |
| **世界观设定** | 作家自由发挥，无模板 | 极简设定表（时代/规则/场景/跨度） | 缺少约束，容易堆砌 |
| **人物弧光设计** | 审核员事后检查 | 事前设计+事后验证 | 缺少前置设计环节 |
| **大纲节点检查** | 有审核，但缺方法论标准 | 三幕式8节点/15节拍对照检查 | 检查项和方法论脱节 |
| **初稿写作指导** | 无实时指导文档 | Show Don't Tell / 场景节拍 / 开篇原则 | 作家靠自己领悟 |
| **修改轮次** | 有质量检查脚本 | 宏观→中观→微观三轮修改法 | 质量检查脚本较全，但缺"先大后小"的操作规程 |
| **方法论可访问性** | 文档在10_方法论/目录 | 方法论要嵌入工作流各环节 | 方法论和workflow_state.json 无关联 |

### 1.3 根本问题

> **方法论文档已建立，但未与工作流集成。**

作家在写大纲、初稿时，无法直接获取对应的方法论指导。审核员在审核时，缺少与方法论对齐的检查标准。方法论成了"参考书"而非"操作手册"。

---

## 二、目标

1. **方法论嵌入工作流**：每个阶段开始时，Agent 能获取对应的方法论要点
2. **关键产出模板化**：灵感→三要素→设定→大纲→正文，各有可操作的模板
3. **审核标准对齐**：审核员的检查维度，逐一对应方法论各阶段的质量要求
4. **经验闭环**：从实际生产中沉淀经验，反哺方法论文档

### 3.2 具体改进

#### 改进1：灵感库增加筛选阶段

**现状**：`01_灵感库/{项目}/` 有基础层.yaml和深度层.md

**新增**：
- `10_方法论/模板库/灵感筛选流程.md` —— 灵感筛选标准操作程序
- `01_灵感库/{项目}/立项/` 中增加 `00_核心三要素.yaml`

**流程**：
```
灵感生成 → 筛选评估(对照5条标准) → 一句话故事(模板) → 核心三要素.yaml → 进入立项
```

#### 改进2：作家工作室增加方法论调用

**现状**：作家 Agent 有画像，但创作时无实时方法论指导

**新增**：
- `10_方法论/作家操作手册.md` —— 各阶段方法论要点速查（供作家实时参考）
- 每个作家目录下增加 `创作参照.md`（可从模板生成）

**关键**：作家在以下节点必须参照方法论：
- 写大纲前 → 读取 05_大纲结构设计.md 的检查清单
- 写初稿前 → 读取 06_初稿写作.md 的开篇+Show Don't Tell 要点
- 修改时 → 读取 07_修改打磨.md 对应轮次的检查清单

#### 改进3：内容仓库各层大纲增加方法论字段

**现状**：大纲文件无方法论溯源字段

**新增**：`{项目}/03_内容仓库/01_全文总体大纲/` 等各目录下的 index.json 增加：

```yaml
methodology_markers:
  stage: "阶段3:大纲结构设计"     # 对应方法论阶段
  structure: "三幕式/8节点"       # 使用的结构模板
  check_completed: true
  check_items:
    - "[x] 开端勾人"
    - "[x] 三幕比例合理"
    - "[x] 高潮足够高"
```

#### 改进4：审核员工作室——审核标准与方法论对齐

**现状**：有 10 维质量检查，但部分检查项与创作方法论不对应

**改进**：
- 对齐 `quality-dimensions.md` 与 `07_修改打磨.md` 的三轮修改框架
- 新增"方法论专项检查"：
  - 灵感筛选检查（评估灵感是否满足5条标准）
  - 三要素完整性检查（主角/目标/阻碍是否明确）
  - 世界观克制检查（是否堆砌设定）
  - 人物弧光设计检查（是否有人物弧光设计）
  - 开篇三原则检查（前3章是否满足方法论开篇要求）

#### 改进5：意见仓库——增加方法论问题分类

**现状**：P0/P1/P2 分级，但没有方法论维度

**新增**：审核意见模板中增加字段：

```yaml
methodology_dimension: "06_初稿写作"  # 这个问题属于哪个方法论文档
methodology_rule: "开篇三原则"         # 具体触犯了哪条规则
```

这样可以统计哪类方法论问题最常出现，持续改进方法论本身。

#### 改进6：新建"创作指导快速入口"

**新增**：`novel-factory/10_方法论/作家快速入口.md`

一篇文档，包含所有创作阶段的方法论速查要点，作家不需要读多个文件：

```markdown
# 作家速查手册

## 灵感阶段
[→ 01_灵感捕捉与筛选.md]

## 构思阶段
[→ 02_核心三要素.md]

## 设定阶段
[→ 03_世界观与设定.md] + [→ 04_人物塑造体系.md]

## 大纲阶段
[→ 05_大纲结构设计.md]
三幕节点速查：[8节点大纲模板]

## 初稿阶段
[→ 06_初稿写作.md]
开篇三原则 / Show Don't Tell速查 / 场景节拍模板

## 修改阶段
[→ 07_修改打磨.md]
第一轮(宏观)检查清单 / 第二轮(节奏)检查清单 / 第三轮(语言)检查清单
```

## 四、实施计划

### Phase A：基础设施（优先级最高）

1. ✅ `作家快速入口.md` —— 立即可用，作家最先受益
2. ✅ 各阶段方法论检查清单标准化（嵌入现有工作流各节点）
3. ✅ 更新 `workflow_state.json` 增加 `methodology_markers` 字段说明

### Phase B：产出标准化（优先级中）

4. 新增 `01_灵感库/模板库/灵感筛选流程.md`
5. 新增 `01_灵感库/{项目}/立项/00_核心三要素.yaml` 模板
6. 更新 `01_灵感库/模板库/基础层.yaml` 加入方法论字段
7. 新增 `03_内容仓库/各层大纲/index.json` 的 methodology_markers 模板

### Phase C：审核对齐（优先级中）

8. 对齐 `quality-dimensions.md` 与三轮修改框架
9. 新增方法论专项审核检查单（按方法论阶段）
10. 更新 `06_意见仓库/审核模板` 增加 methodology_dimension 字段

### Phase D：经验闭环（优先级低）

11. 在 `10_方法论/` 增加 `实践反馈.md` —— 记录方法论与实际生产的差距
12. 定期（每项目后）复盘，更新方法论文档
