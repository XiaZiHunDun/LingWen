# 小说工厂详细流程文档

> 本文档详细描述小说工厂的完整工作流程，供主控调度和各部门执行参考
> 版本: 1.0 | 更新: 2026-05-19

---

## 一、工作流总览

```
PHASE_0_SETUP     → 项目初始化
PHASE_1_LAUNCH    → 立项（灵感+大纲）
PHASE_2_OUTLINE   → 全文大纲迭代
PHASE_3_VOLUME    → 卷大纲迭代
PHASE_4_STAGE     → 阶段大纲迭代
PHASE_5_BODY      → 正文创作与双轨反馈
PHASE_6_SUMMARY   → 分层汇总与终审
PHASE_7_CLOSE     → 归档闭环
```

---

## 二、PHASE_0 项目初始化

### STEP_00 项目初始化

| 项目 | 内容 |
|------|------|
| 执行者 | 主控调度 |
| 输入 | 项目名称、类型、基础设定 |
| 输出 | workflow_state.json 初始化 |
| 触发 | 新项目启动 |

**执行步骤**:
1. 创建项目目录结构
2. 初始化 workflow_state.json
3. 建立角色状态表
4. 建立伏笔追踪表
5. 通知各部门准备

---

## 三、PHASE_1 立项阶段

### STEP_01 灵感生成

| 项目 | 内容 |
|------|------|
| 执行者 | 灵感部门（灵感A/B/C） |
| 输入 | 项目名称、类型方向 |
| 输出 | 基础层.yaml + 深度层.md |
| 并行 | 3个灵感Agent并行 |

**执行步骤**:
1. 灵感A：生成类型分析、卖点提炼
2. 灵感B：生成世界观、时间线、伏笔布局
3. 灵感C：生成叙事结构、角色弧光设计
4. 合并输出到 `01_灵感库/{项目名}/`
5. 主控验证输出完整性

**输出规范**:
```
01_灵感库/{项目名}/
├── 灵感A_类型分析.md
├── 灵感B_世界观.md
├── 灵感C_叙事结构.md
└── 基础层.yaml
```

### STEP_02 全文大纲初稿

| 项目 | 内容 |
|------|------|
| 执行者 | 作家主编 + 汇总部门 |
| 输入 | 灵感输出 |
| 输出 | {项目名}_优化版大纲.md |
| 依赖 | STEP_01完成 |

**执行步骤**:
1. 作家主编根据灵感整合大纲初稿
2. 汇总主笔整理全局结构
3. 生成章节清单（预计章节数）
4. 主控初审

**输出规范**:
```
03_内容仓库/01_全文总体大纲/
├── index.json
└── {项目名}_优化版大纲.md
```

## 四、PHASE_2 全文大纲迭代

### STEP_03 全文大纲审核

| 项目 | 内容 |
|------|------|
| 执行者 | 审核部门（6人） |
| 输入 | 全文大纲 |
| 输出 | 大纲审核报告 |
| 依赖 | STEP_02完成 |

**执行步骤**:
1. 主控分配6名审核员（每卷2人）
2. 审核员独立审核
3. 汇总审核意见
4. 判定：P0问题数量

**审核维度**:
- 类型定位清晰度
- 核心冲突强度
- 角色弧光完整性
- 世界观自洽性
- 伏笔布局合理性
- 与受众匹配度

### STEP_04 全文大纲修改

| 项目 | 内容 |
|------|------|
| 执行者 | 作家主编 |
| 输入 | 审核意见 |
| 输出 | 修改记录 |
| 依赖 | STEP_03完成 |

**执行步骤**:
1. 主控汇总审核意见
2. 判定修改范围和优先级
3. 作家主编执行修改
4. 主控验证修改完成

**修改分类**:
- P0：必须修复（逻辑矛盾、关键角色OOC）
- P1：应该修复（叙事断裂、细节不一致）
- P2：建议修复（文笔优化）

### STEP_05 全文大纲终审

| 项目 | 内容 |
|------|------|
| 执行者 | 审核主编 |
| 输入 | 修改后大纲 + 修改记录 |
| 输出 | 终审意见 |
| 依赖 | STEP_04完成 |

**判定标准**:
- P0=0：通过
- P0≤2：复审
- P0>2：不通过，打回重做

## 五、PHASE_3 卷大纲迭代

### STEP_06 卷大纲生成

| 项目 | 内容 |
|------|------|
| 执行者 | 作家部门（10人并行） |
| 输入 | 全文大纲 |
| 输出 | 卷1/2/3大纲 |
| 依赖 | STEP_05通过 |

**执行步骤**:
1. 主控分配：作家A-D→卷1，作家E-G→卷2，作家H-J→卷3
2. 每卷内部按阶段分工
3. 格式统一校验
4. 主控合并三卷大纲

**输出规范**:
```
03_内容仓库/02_卷大纲/
├── 卷1_废土星火_大纲.md
├── 卷2_星际争锋_大纲.md
└── 卷3_宇宙守护_大纲.md
```

### STEP_07 卷大纲审核

| 项目 | 内容 |
|------|------|
| 执行者 | 审核部门（抽样+全量） |
| 输入 | 三卷大纲 |
| 输出 | 卷大纲审核报告 |
| 依赖 | STEP_06完成 |

**审核策略**:
- 全量审核：核心章节
- 抽样审核：辅助章节（每卷抽30%）

### STEP_08 卷大纲修改

| 项目 | 内容 |
|------|------|
| 执行者 | 原作家团队 |
| 输入 | 审核意见 |
| 输出 | 修改记录 |
| 依赖 | STEP_07完成 |

### STEP_09 卷大纲终审

| 项目 | 内容 |
|------|------|
| 执行者 | 审核主编 |
| 输入 | 修改后卷大纲 |
| 输出 | 终审意见 |
| 依赖 | STEP_08完成 |

**验证要点**:
- 三卷范围正确
- 卷之间衔接连贯
- 无逻辑矛盾

## 六、PHASE_4 阶段大纲迭代

### STEP_10 阶段大纲生成

| 项目 | 内容 |
|------|------|
| 执行者 | 作家部门 |
| 输入 | 卷大纲 |
| 输出 | 各卷阶段大纲 |
| 依赖 | STEP_09通过 |

**阶段划分原则**:
- 每个阶段约20-30章
- 阶段有独立起承转合
- 阶段结尾留悬念/转折

**输出规范**:
```
03_内容仓库/03_阶段大纲/
├── 卷1_阶段1.md
├── 卷1_阶段2.md
├── ...
└── 卷3_阶段5.md
```

### STEP_11 阶段大纲审核

| 项目 | 内容 |
|------|------|
| 执行者 | 审核部门（抽样4个阶段） |
| 输入 | 阶段大纲 |
| 输出 | 审核报告 |
| 依赖 | STEP_10完成 |

**抽样规则**:
- 卷1：抽阶段1、4、7
- 卷2：抽阶段1、3、5
- 卷3：抽阶段2、5

### STEP_12 阶段大纲修改

| 项目 | 内容 |
|------|------|
| 执行者 | 原作家团队 |
| 输入 | 审核意见 |
| 输出 | 修改记录 |
| 依赖 | STEP_11完成 |

### STEP_13 阶段大纲终审

| 项目 | 内容 |
|------|------|
| 执行者 | 审核主编 |
| 输入 | 修改后阶段大纲 |
| 输出 | 终审意见 |
| 依赖 | STEP_12完成 |

## 七、PHASE_5 正文创作与双轨反馈

### STEP_14 正文创作

| 项目 | 内容 |
|------|------|
| 执行者 | 作家部门（10人并行） |
| 输入 | 阶段大纲 |
| 输出 | ch001-ch360正文 |
| 依赖 | STEP_13通过 |

**批次规则**:
- 每批10章
- 每批分配给10个作家（每人一章）
- 批次内并行创作
- 批次间串行（等上一批审核）

**输出规范**:
```
03_内容仓库/04_正文/
├── ch001.md
├── ch002.md
├── ...
├── ch360.md
└── index.json
```

### STEP_15 读者评论

| 项目 | 内容 |
|------|------|
| 执行者 | 读者部门（20人） |
| 输入 | 正文批次 |
| 输出 | 读者评论报告 |
| 并行 | 20读者并行 |

**读者分工**:
- 吐槽型(A/C/D/G/I/M/N/P)：关注逻辑硬伤
- 分析型(B/J/O/Q)：关注情节合理
- 共情型(E/F/H/K/L/R/S/T)：关注情感共鸣

### STEP_16 审核部门技术审核

| 项目 | 内容 |
|------|------|
| 执行者 | 审核部门（5人/批） |
| 输入 | 正文批次 + 读者评论 |
| 输出 | 多维审核报告 |
| 并行 | 5审核员并行 |

**10+1维审核**:
1. 命名一致性
2. 内容完整性
3. 章节重复
4. 人物状态
5. 时间线
6. 情节关联度
7. 伏笔回收率
8. 场景逻辑
9. 情感节奏
10. 对话风格
11. 人物弧光（LLM）

### STEP_17 作家修改

| 项目 | 内容 |
|------|------|
| 执行者 | 原作家团队 |
| 输入 | 审核报告 |
| 输出 | 修改后正文 |
| 依赖 | STEP_16完成 |

**修改流程**:
1. 主控汇总审核意见
2. 判定修改范围
3. 分配给原作家
4. 并行修改
5. 主控验证

### STEP_18 章节定稿判定

| 项目 | 内容 |
|------|------|
| 执行者 | 审核主编 |
| 输入 | 修改后正文 + 审核报告 |
| 输出 | 定稿判定 |
| 依赖 | STEP_17完成 |

**判定标准**:
- P0=0：通过
- P0>0：打回重写

| 项目 | 内容 |
|------|------|

| 输出 | 发布决策 |
| 依赖 | STEP_23完成 |

**发布等级**:
- S级：直接发布
- A级：小幅调整后发布
- B级：修改后复审
- 不合格：打回重做
