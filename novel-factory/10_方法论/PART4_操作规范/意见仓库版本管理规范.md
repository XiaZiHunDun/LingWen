# 意见仓库版本管理规范

> 本规范定义了意见仓库的版本化管理流程，确保每次修改都有完整的历史记录可追溯。

---

## 1. 版本快照记录格式

每次修改意见后，必须在对应章节的版本目录下创建快照记录。使用以下表格格式：

```markdown
# 版本快照记录

| 版本 | 日期 | 修改人 | 审核意见摘要 | 修改内容 |
|------|------|--------|-------------|---------|
| v1.0 | 05-19 | 作家A | 节奏偏慢 | 删减第3段 |
| v1.1 | 05-20 | 作家A | 补充细节 | 新增场景描写 |
```

---

## 2. 版本Diff自动生成规则

每次修改意见后，系统自动生成Diff记录，格式如下：

```markdown
# v1.0 → v1.1 Diff

## 删除内容
- 原文: "......"
- 位置: 第3段

## 新增内容
- 改文: "......"
- 位置: 第3段

## 修改原因
- 原因: 节奏偏慢，删减冗余描写
```

### Diff生成触发条件

| 操作 | 是否生成Diff |
|------|-------------|
| 新增意见 | 是 (v1.0初始快照) |
| 修改意见 | 是 (增量Diff) |
| 删除意见 | 是 (记录删除内容) |
| 审核通过 | 是 (终版快照) |

---

## 3. 回溯还原流程

### 3.1 回溯触发条件

- 作者对修改有异议
- 需要对比不同版本的意见差异
- 需要还原到特定历史版本

### 3.2 回溯操作流程

```
1. 确定目标版本
   └── 查看 版本快照记录表 确定目标版本号

2. 读取目标版本内容
   └── 进入 对应章节ID/v{版本号}/ 目录

3. 对比差异
   └── 查看 版本A → 版本B Diff 文件

4. 执行还原
   └── 复制目标版本文件覆盖当前版本

5. 记录还原操作
   └── 在版本快照记录表中新增一行
```

### 3.3 回溯权限

| 角色 | 权限 |
|------|------|
| 审核员 | 可回溯到自己创建的审核意见 |
| 主笔作家 | 可回溯到所有审核意见 |
| 项目管理员 | 可回溯所有意见并强制锁定 |

## 5. 版本号命名规则

| 类型 | 格式 | 示例 |
|------|------|------|
| 主版本 | v{X} | v1, v2, v3 |
| 正式版本 | v{X}.0 | v1.0, v2.0 |
| 修订版本 | v{X}.{Y} | v1.1, v1.2 |

### 版本递增规则

- 新增意见: 从 v1.0 开始
- 修改意见: 次版本号递增 (v1.0 → v1.1)
- 重大修订: 主版本号递增 (v1.X → v2.0)

---

## 6. 自动化脚本支持

版本记录设计为自动化友好格式，便于脚本处理：

```bash
# 提取版本列表
grep "^| v" 版本快照记录.md

# 提取最新版本号
grep "^| v" 版本快照记录.md | tail -1 | cut -d'|' -f2

# 生成Diff报告
./run_opinion.sh --diff v1.0 v1.1 --chapter ch001
```

---

## 7. 归档时机

| 意见类型 | 归档时机 |
|----------|----------|
| 大纲审核意见 | 每轮审核完成后归档 |
| 正文审核意见 | 章节审核通过后归档 |
| 读者评论意见 | 每50章汇总后归档 |

---

## 8. 版本锁定规则

- **草稿状态**: 可自由修改，无版本记录
- **提交审核**: 自动生成 v1.0 快照
- **审核通过**: 版本锁定，不可直接修改
- **解锁修改**: 需新建修订版本 (vX.Y)，保留原版本
