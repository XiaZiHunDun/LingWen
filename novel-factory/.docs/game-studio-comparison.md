# 游戏工作室 vs 小说工厂 — 对比分析与改进建议

## 一、核心架构对比

### 游戏工作室架构

```
.claude/
├── agents/          # 代理定义（48个专业化代理）
│   ├── creative-director.md
│   ├── game-designer.md
│   ├── producer.md
│   └── ...（每个领域专属）
├── skills/          # 技能（80+个专业技能）
│   ├── sprint-plan/
│   ├── design-review/
│   ├── architecture-decision/
│   └── ...
├── docs/            # 文档
│   ├── coordination-rules.md
│   ├── coding-standards.md
│   ├── context-management.md
│   └── ...
└── hooks/           # 钩子（如 session-start）

production/          # 生产管理
├── session-state/   # 会话状态（active.md）
└── session-logs/    # 会话审计日志
```

### 当前小说工厂架构

```
novel-factory/
├── CLAUDE.md        # 主配置（40KB+，混合了所有职责）
├── workflow_state.json  # 工作流状态机
├── tools/           # 工具脚本（分散）
├── .skills/         # Skill管理（6个骨架）
├── .docs/           # 架构文档
└── （无代理定义、无会话状态管理）
```

## 二、关键改进领域

### 1. Agent 定义格式（优先级：P0）

**现状**：我们的代理（如审核员A、作家A）没有正式定义文件，纯靠 CLAUDE.md 中的文本描述。

**改进**：为每个角色创建正式的 agent 定义文件。

**参考格式**：
```markdown
---
name: 审核员-技术审核
description: "技术审核员负责逻辑一致性、人设稳定性、伏笔管理。使用此代理进行正文审核、问题判定。"
tools: Read, Glob, Grep, Write, Edit
model: sonnet
maxTurns: 20
skills: [novel-quality-check]
memory: project
---

[角色详细描述、协作协议、工作流程]
```

**文件位置**：
```
.claude/agents/
├── 灵感A.md
├── 灵感B.md
├── 灵感C.md
├── 作家主编.md
├── 作家A.md ~ 作家J.md
├── 审核员主编.md
├── 审核员A.md ~ 审核员K.md
├── 读者主编.md
├── 读者A.md ~ 读者T.md
├── 汇总主笔.md
├── 汇总编辑.md
└── 汇总校验.md
```

### 2. 会话状态管理（优先级：P0）

**现状**：无会话状态管理，依赖 CLAUDE.md 的长文本。

**改进**：创建 `production/session-state/active.md` 作为活跃检查点。

**参考游戏工作室的 active.md 格式**：
```markdown
# 当前会话状态

## 当前任务
[任务描述]

## 进度检查清单
- [x] 步骤1完成
- [x] 步骤2完成
- [ ] 步骤3进行中
- [ ] 步骤4未开始

## 关键决策
- 决策1：2026-05-18，原因是XXX
- 决策2：...

## 正在处理的文件
- `03_内容仓库/04_正文/ch050.md`

## 待解答问题
- 问题1：关于XXX，用户尚未回复

## 代理调用记录
- 2026-05-18 15:00: 审核员A 完成 ch001-ch010 审核
- 2026-05-18 15:30: 作家B 开始修复 ch051-ch060

<!-- STATUS -->
部门: 审核部门
当前批次: ch001-ch010
状态: 审核完成，待作家修改
<!-- /STATUS -->
```

**实现**：
1. 创建 `production/` 目录
2. 创建 `production/session-state/active.md`
3. 添加 session-start 钩子恢复状态

### 3. 协调规则（优先级：P1）

**现状**：CLAUDE.md 中有简单的调度规则，但没有跨部门协调机制。

**改进**：创建 `.claude/docs/coordination-rules.md`。

**参考内容**：
```markdown
# 协调规则

## 垂直委派
领导层代理委派给部门主管，部门主管再委派给专家。
对于复杂决策，不得跳过层级。

## 横向协商
同一层级的代理可以相互协商，但不得在其领域之外做出约束性决策。

## 冲突解决
当两个代理意见不一致时，上报至共同的上级。
如果没有共同上级：
- 设计冲突 → 汇总主编
- 技术冲突 → 技术总监（待设）

## 变更传播
当设计变更影响多个领域时，由相应的部门主管协调传播。

## 禁止单方面跨域变更
代理未经明确委派，不得修改其指定目录之外的文件。
```

### 4. 模型分级（优先级：P1）

**现状**：所有任务使用同一模型（MiniMax-M2.7）。

**改进**：为不同复杂度任务分配不同模型。

**参考**：
| 任务类型 | 模型 | 说明 |
|---------|------|------|
| 状态检查、格式化 | Haiku | 无需创意判断 |
| 实现、设计、分析 | Sonnet | 默认 |
| 多文档综合、高风险判定 | Opus | 跨系统审查 |

---

### 5. 设计文档标准（优先级：P2）

**现状**：我们有 `09_规范文档/` 但没有强制性的章节结构。

**改进**：为每个设计文档定义8个必填章节。

**参考格式**：
```markdown
# [设计文档标题]

## 1. 概述
一段式总结

## 2. 玩家幻想（小说中的"读者体验"）
期望的感受与体验

## 3. 详细规则
无歧义的机制描述

## 4. 公式
所有数学定义及变量说明

## 5. 边界情况
已处理的异常情况

## 6. 依赖关系
其他系统列表（双向）

## 7. 调优旋钮
可配置数值及其安全范围

## 8. 验收标准
可测试的成功条件
```

---

### 6. 上下文管理（优先级：P2）

**现状**：CLAUDE.md 作为主上下文，容易超出限制。

**改进**：参考游戏工作室的增量文件写入和主动压缩策略。

**关键实践**：
1. 创建文件时立即创建骨架（所有章节标题，空正文）
2. 讨论和起草逐节进行
3. 每节批准后立即写入文件
4. 在 60-70% 上下文使用时主动压缩
5. 自然压缩点：完成章节后、提交后、完成任务后

---

### 7. 架构决策记录（ADR）（优先级：P3）

**现状**：重要决策记录在 workflow_state.json 的 issues_found 中，混乱。

**改进**：创建正式的 ADR 文档。

**格式**：
```markdown
# ADR-001: [决策标题]

## 背景
[导致此决策的情况]

## 决策
[具体决策内容]

## 备选方案
- 方案A：[描述] → 拒绝原因
- 方案B：[描述] → 采纳

## 后果
- 正面：...
- 负面：...

## 状态
已接受 | 已拒绝 | 已修改
```

**位置**：
```
09_规范文档/架构决策/
├── ADR-001_伏笔回收率检测策略.md
├── ADR-002_角色弧光分析方式.md
└── ...
```

## 三、改进优先级总结

| 优先级 | 改进项 | 工作量 | 价值 |
|--------|-------|--------|------|
| P0 | Agent 定义文件 | 高 | 明确职责边界 |
| P0 | 会话状态管理 | 中 | 崩溃恢复、状态追踪 |
| P1 | 协调规则文档 | 低 | 减少冲突 |
| P1 | 模型分级 | 低 | 成本优化 |
| P2 | 设计文档标准 | 中 | 质量保证 |
| P2 | 上下文管理策略 | 低 | 避免上下文溢出 |
| P3 | ADR 文档系统 | 中 | 决策追溯 |

---

## 四、建议的实施路径

### Phase 1: 基础建设（1-2天）
1. 创建 `.claude/` 目录结构
2. 为主要角色创建 agent 定义文件
3. 创建 `production/session-state/active.md`
4. 编写协调规则文档

### Phase 2: 流程优化（3-5天）
1. 实现 session-start 钩子
2. 更新 CLAUDE.md，将长描述分散到各 agent 文件
3. 创建 ADR 目录和模板
4. 定义设计文档标准

### Phase 3: 持续改进（进行中）
1. 根据使用反馈迭代 agent 定义
2. 优化协调规则
3. 添加新的专业化代理
