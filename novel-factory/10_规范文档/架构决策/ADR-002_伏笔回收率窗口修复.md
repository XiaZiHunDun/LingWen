# ADR-002: 伏笔回收率检测窗口条件修复

## 背景

在质量检查工具 `check_plot_device_tracking.py` 中，伏笔回收率始终显示为 0%，即使实际存在伏笔回收。这导致质量检查工具失效。

**问题代码**：
```python
if first_ch > end_ch - self.window:
    # 跳过（窗口不足）
```

当 `window=50`, `end_ch=30` 时：
- `end_ch - self.window = 30 - 50 = -20`
- `first_ch > -20` 永远为真（因为章节号总是 >= 1）
- 导致所有伏笔都被跳过

## 决策

修复窗口判断条件：

**修复前**：
```python
if first_ch > end_ch - self.window:
    continue  # 窗口不足，跳过
```

**修复后**：
```python
if first_ch + self.window > end_ch:
    continue  # 窗口不足，跳过
```

当 `first_ch=10`, `end_ch=30`, `window=50` 时：
- 修复前：`10 > 30 - 50 = 10 > -20 = True`（跳过，无法检测）
- 修复后：`10 + 50 = 60 > 30 = True`（正确，检测到伏笔）

## 备选方案

- **方案A（修复判断条件）**：[采纳]
  - 简单直接，只改一行
  - 修复后伏笔回收率从 0% 变为 83.3%

- **方案B（重写检测逻辑）**：
  - 完全重写伏笔检测逻辑
  - 工作量大，且当前逻辑已经正确，只是判断条件错误

## 后果

### 正面
- 伏笔回收率正确显示（83.3%）
- 质量检查工具恢复正常
- 修复简单，风险低

### 负面
- 无

已接受
