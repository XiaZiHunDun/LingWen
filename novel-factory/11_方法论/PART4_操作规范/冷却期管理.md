# 冷却期管理规范

> **版本**：v1.0
> **日期**：2026-05-19
> **目的**：制度化"冷却期"，确保创作者能以客观眼光审视作品
> **来源**：参考写小说流程文档"初稿完成后，请先把它放一放，冷静1-2周"

---

## 核心理念

**冷却期**让创作者抽离出来，用"陌生读者"的眼光重新审视作品。

---

## 一、冷却期规则

### 强制冷却时机

| 阶段 | 冷却时长 | 豁免条件 |
|------|----------|----------|
| 初稿完成 → Block修改 | ≥24小时 | P0问题除外 |
| 卷汇总完成 → 卷审核 | ≥4小时 | 主公特批 |
| 阶段汇总完成 → 审核 | ≥4小时 | 主公特批 |
| 全书汇总完成 → 终审 | ≥24小时 | 主公特批 |

### 冷却期计算规则

1. 冷却期从"阶段完成时间戳"开始计算
2. 工作日（周一至周五）正常冷却
3. 周末/节假日顺延
4. 主控可按项目紧急程度豁免，但需在workflow_state中记录

### 豁免申请流程

```
作家/主控申请豁免 → 说明原因 → 主公审批 → 记录到workflow_state
```

豁免记录格式：
```json
{
  "cooldown_waiver": {
    "step": "STEP_XX",
    "reason": "原因",
    "approver": "主公",
    "date": "YYYY-MM-DD"
  }
}
```

---

## 二、冷却期禁止事项

在冷却期内，禁止：
- ❌ 回头修改刚完成的章节
- ❌ 查看上一轮的审核意见
- ❌ 与其他作家/审核员讨论刚完成的内容
- ❌ 对外分享未审核的内容获取反馈

冷却期应该做：
- ✅ 创作其他部分的初稿
- ✅ 休息调整状态
- ✅ 阅读其他作品获取灵感
- ✅ 准备下一阶段的创作素材

---

## 三、冷却期后检查清单

冷却期结束后，重新开始工作前：

- [ ] 能否清楚说出本章/本卷/本书的核心事件？
- [ ] 能否用一句话概括主角的处境？
- [ ] 是否有"陌生感"——看到自己写的内容像在看别人写的？
- [ ] 是否发现了之前没注意到的问题？（如：节奏问题、逻辑漏洞）
- [ ] 审核意见中指出的问题，现在看起来是否明显？

如果以上检查发现问题≥3个，说明冷却期**不足**，建议延长。

---

## 四、特殊情况

### 项目紧急情况

当项目有硬性deadline时：
1. 主公可批准缩短冷却期
2. 必须在workflow_state中记录
3. 必须安排后续"二次审核"补救

### 长篇连载特殊情况

对于网文连载（日更/周更）：
- 冷却期可缩短至4-12小时
- 但必须保证每10万字有一次≥24小时的"大冷却"
- 大冷却期用于阶段性复盘和调整
