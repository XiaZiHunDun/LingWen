> **版本**：v1.0
> **来源**：借鉴游戏制作"垂直切片（Vertical Slice）"方法论
> **日期**：2026-05-19
> **状态**：新增

---

## 核心理念

**核心样章**是写一个完整段落（5000-10000字），代表全书最高水平，用于验证风格+节奏+叙事声音是否立得住。

> **核心目的**：用一小段完整、可读的文本，验证"这样写是否有效"。

> **适用范围**：内部评估、编辑投稿、读者测试。

---

## 一、概念定义

### 1.1 什么是核心样章

**游戏中的垂直切片**：
> 用1-2个月时间制作一个**可玩的、接近最终品质的5-15分钟Demo**，用于内部评估或向投资人展示。

**小说中的核心样章**：
> 写一个完整段落（5000-10000字），代表全书**最高水平**，用于验证风格+节奏+叙事声音。

### 1.2 核心样章 vs 普通章节

| 维度 | 核心样章 | 普通章节 |
|------|----------|----------|
| **目的** | 验证风格方向 | 填充内容 |
| **质量** | 全书最高水平 | 正常完成水平 |
| **修改次数** | 多次打磨 | 一次完成 |
| **读者可见性** | 通常不直接公开 | 公开连载 |
| **时机** | 写作前验证 | 写作中产出 |

## 二、写作标准

### 2.1 长度标准

| 类型 | 推荐字数 | 说明 |
|------|----------|------|
| 短篇样章 | 5000字 | 一个完整的小高潮 |
| 中篇样章 | 8000字 | 包含一个小完整的冲突 |
| 长篇样章 | 10000字 | 代表全书最高水平 |

### 2.2 内容标准

核心样章必须包含：

| 维度 | 要求 | 说明 |
|------|------|------|
| **开篇钩子** | 前300字 | 让读者想知道"然后呢" |
| **中段冲突** | 包含一个明确冲突 | 有阻碍、有挣扎、有结果 |
| **叙事声音** | 主角视角统一 | 读者能产生代入感 |
| **情感锚点** | 有情绪波动 | 让读者笑/哭/紧张/兴奋 |
| **文笔风格** | 语言统一 | 有辨识度的叙事风格 |
| **结局** | 有明确结局 | 一个小的"满足点" |

### 2.3 质量标准

核心样章必须达到：

- [ ] **可读性**：读者愿意一口气读完
- [ ] **辨识度**：有独特的叙事声音或风格
- [ ] **情感共鸣**：至少有一个让读者情绪波动的点
- [ ] **节奏感**：张弛有度，不是平铺直叙
- [ ] **代入感**：读者关心主角遭遇

## 三、验证流程

### 3.1 自检清单

写完核心样章后，先自检：

**开篇检查**
- [ ] 前300字是否有悬念？
- [ ] 前300字是否交代了主角/情境？
- [ ] 前300字是否让读者想读下去？

**结构检查**
- [ ] 是否有明确的开端-发展-结局？
- [ ] 每个段落是否有推进作用？
- [ ] 是否有"翻页冲动"点？

**风格检查**
- [ ] 叙事声音是否独特？
- [ ] 语言风格是否统一？
- [ ] 是否有辨识度？

**情感检查**
- [ ] 读者是否关心主角？
- [ ] 是否有情绪波动点？
- [ ] 结尾是否有满足感？

### 3.2 读者测试

自检通过后，进行目标读者测试。

**测试规模**：3-5名目标读者

**测试方式**：
1. 给出核心样章，让读者自己阅读
2. 观察读者行为（不要现场提问）
3. 读后问三个问题

**行为观察要点**：
| 行为 | 可能含义 |
|------|----------|
| 读得慢 | 可能在欣赏，或可能觉得难懂 |
| 读得快 | 可能觉得无聊或不重要 |
| 表情变化 | 情绪波动点 |
| 抬头看你 | 可能困惑或不认同 |
| 继续往后翻 | 想知道"然后呢" |

**读后三问**：
1. 哪里让你想跳过/无聊？
2. 哪个角色你最感兴趣？
3. 读完后你想知道"然后呢"吗？

### 3.3 反馈处理原则

| 反馈类型 | 处理方式 |
|----------|----------|
| **行为显示跳读** | 优先关注，说明那段无效 |
| **沉默的大多数** | 如果多数人行为一致，重视 |
| **意见与行为矛盾** | 以行为为准 |
| **改进点过多** | 每轮只改最关键的1-2个 |

## 四、常见问题

### 4.1 风格不统一

**症状**：前半段一个风格，后半段另一个风格

**解决**：
- 明确自己的叙事声音
- 统一语言风格
- 多朗读检查

### 4.2 开篇不够钩人

**症状**：读者前300字就放弃

**解决**：
- 前300字必须有一个悬念
- 避免冗长的环境/背景介绍
- 从动作或冲突开场

### 4.3 节奏拖沓

**症状**：样章读起来无聊

**解决**：
- 删除冗余的描写
- 每段必须有推进作用
- 增加冲突密度

### 4.4 情感空洞

**症状**：读者看完无感

**解决**：
- 增加情感锚点
- 让主角有明确的欲望/恐惧
- 增加冲突的代价

## 六、与创作流程的接口

### 6.1 位置

核心样章验证位于**情节骨架验证之后、写作期正式开始前**。

### 6.2 前置条件

- [ ] 情节骨架验证通过
- [ ] 核心驱动链设计完成
- [ ] 人物设定完成

### 6.3 验收标准

- [ ] 字数符合要求（5000-10000字）
- [ ] 包含开篇钩子
- [ ] 包含完整冲突
- [ ] 叙事声音统一
- [ ] 有情感锚点
- [ ] 结尾有满足感
- [ ] 自检全部通过
- [ ] 读者测试通过（行为数据显示有效）

### 6.4 输出物

核心样章验证通过后，输出：
- `核心样章_v1.0.md` — 验证通过版本
- `读者测试记录.md` — 读者测试反馈
- `写作风格指南.md` — 从样章中提炼的写作风格要点

### 6.5 与后续写作的关系

```
核心样章验证 → 写作期正式启动
       ↓
  核心样章 = 全书写作标杆
       ↓
  按核心样章的质量和风格写其他章节
```

## 七、核心样章的意义

### 7.1 对作者的意义

- **风格确认**：确定自己的叙事声音
- **质量锚点**：知道"好的写作"是什么样子
- **自信来源**：有了标杆，不再迷失方向

### 7.2 对读者的意义

- **预期管理**：让读者知道这本书的风格
- **信任建立**：样章质量 = 全书最低质量保证

### 7.3 对编辑/平台的意义

- **评估依据**：判断作者水平的标准
- **决策参考**：决定是否签约/推荐
