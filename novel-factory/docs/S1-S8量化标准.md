# S1-S8 审核量化标准

> 版本：v1.0
> 日期：2026-05-20
> 依据：docs/优化方案-v8.1.md Phase 3.1
> 状态：用于审核维度的自动化判定

---

## 概述

本文档定义8个审核维度（S1-S8）的量化标准，每个维度包含：
- **可量化指标**：可直接计算的具体数值
- **判定规则**：通过/警告/不通过的阈值
- **验证方法**：自动化脚本对应的检测逻辑

### 判定等级

| 等级 | 说明 | 处理方式 |
|------|------|----------|
| ✅ 通过 | 满足所有量化指标 | 进入下一审核项 |
| ⚠️ 警告 | 部分指标不满足 | 主编判断是否接受 |
| ❌ 不通过 | 核心指标不满足 | 打回修改 |

## S1 剧情完整性

### 定义
章节内的目标-障碍-结果链条是否完整有效。

### 量化指标

| 指标 | 阈值 | 说明 |
|------|------|------|
| 目标明确性 | ✅/❌ | 章内存在明确的主角目标（获取/逃避/调查/保护） |
| 障碍相关性 | ✅/❌ | 障碍与目标直接相关，非随机事件 |
| 结果推导性 | ✅/❌ | 结果由角色行动推导，非机械降神 |

### 判定规则

```
通过条件：目标明确 ✅ AND 障碍相关 ✅ AND 结果推导 ✅
不通过条件：目标不明确 OR 障碍与目标无关 OR 结果无推导
警告条件：目标模糊但基本可辨
```

### 验证方法

```python
# 检查点1：目标明确性
# - 搜索关键词：想要、必须、为了、决定、寻找、追求
# - 或检测到明确的句式：主角+目标动词+目标对象

# 检查点2：障碍相关性
# - 障碍必须直接阻碍目标达成
# - 排除：无关的天气描写、无因果的NPC行为

# 检查点3：结果推导性
# - 结果前必须有主角行动或决定
# - 排除：机械降神（突然出现帮助/敌人突然死亡）
```

## S2 逻辑自洽

### 定义
时间线、空间位置、角色能力在章节内外是否保持一致。

### 量化指标

| 指标 | 阈值 | 说明 |
|------|------|------|
| 时间线矛盾检测 | 通过/失败 | "年前"与"瞬间"不同时出现于同一因果链 |
| 空间位置一致 | 通过/失败 | 角色移动距离与时间匹配 |
| 角色能力边界 | 通过/失败 | 角色行为不超出已设定的能力范围 |

### 判定规则

```
通过条件：时间线 ✅ AND 空间一致 ✅ AND 能力边界 ✅
不通过条件：任一检测器报告矛盾
警告条件：存在轻微不一致但不影响阅读
```

### 验证方法

```python
# 检查点1：时间线矛盾
# - 提取时间表达：年前、月前、日前、瞬间、片刻、忽然
# - 检测因果链中的矛盾：同一事件"十年前发生"+"瞬间想起"同时成立
# 阈值：0处矛盾 = 通过，1-2处 = 警告，>2处 = 失败

# 检查点2：空间位置
# - 记录角色位置变化：场景A → 场景B
# - 检查移动时间是否合理：同一章节内不能从京城瞬间到江南
# 阈值：位置矛盾 ≤ 1 = 通过

# 检查点3：能力边界
# - 根据角色能力档案检测行为是否越界
# - 例如：未设定飞行的角色不能"腾空而起"
# 阈值：能力越界事件 = 0 = 通过
```

## S3 文笔风格

### 定义
语言表达的多样性、句式变化、描写丰富程度。

### 量化指标

| 指标 | 阈值 | 说明 |
|------|------|------|
| Shannon多样性指数 | ≥ 2.5 | 词汇使用的多样性 |
| 句式种类 | ≥ 6种 | 包含的句式类型数量 |
| 单一句式占比 | < 40% | 同一种句式在全文中的比例上限 |

### 句式种类定义

| 句式类型 | 识别特征 |
|----------|----------|
| 主谓句 | SVO基本结构 |
| 被动句 | 被+动词 |
| 把字句 | 把+O+动词 |
| 疑问句 | 疑问词结尾 |
| 感叹句 | 感叹号结尾 |
| 短句 | ≤10字的独立句子 |
| 复合句 | 从句+主句 |
| 省略句 | 省略主语或宾语 |
| 倒装句 | 谓语在主语前 |

### 判定规则

```
通过条件：Shannon ≥ 2.5 AND 句式种类 ≥ 6 AND 单一占比 < 40%
警告条件：Shannon 2.0-2.5 OR 句式4-5种 OR 单一占比40-50%
不通过条件：Shannon < 2.0 OR 句式 < 4种 OR 单一占比 ≥ 50%
```

### 验证方法

```python
# 检查点1：Shannon多样性指数
# 使用公式：H = -Σ(pi * log2(pi))，pi为词频
# 实现：text_stats['shannon_diversity']

# 检查点2：句式种类统计
# 统计上述8种句式类型出现种类数
# 阈值：种类 ≥ 6

# 检查点3：单一占比
# 找出使用最多的句式类型及其占比
# 阈值：< 40%
```

## S4 情感共鸣

### 定义
章节的情绪曲线是否有效波动，能否引起读者情感共鸣。

### 量化指标

| 指标 | 阈值 | 说明 |
|------|------|------|
| 情绪转折点 | ≥ 1次/章 | 情绪方向发生明显变化的节点 |
| 情绪波动方差 | ≥ 0.05 | 章节内情绪强度的离散程度 |
| 高潮场景情绪强度 | ≥ 7/10 | 章节高潮点的情绪强度评分 |

### 情绪强度量化

| 情绪词 | 强度值 |
|--------|--------|
| 狂喜、激动、愤怒、悲痛 | 9-10 |
| 高兴、紧张、悲伤、恐惧 | 7-8 |
| 平静、轻松、担忧、失落 | 5-6 |
| 平淡、无聊、漠然 | 1-4 |

### 判定规则

```
通过条件：转折点 ≥ 1 AND 方差 ≥ 0.05 AND 高潮强度 ≥ 7
警告条件：转折点 = 1 但方差 < 0.05 OR 高潮强度 6-7
不通过条件：转折点 = 0 OR 高潮强度 < 6
```

### 验证方法

```python
# 检查点1：情绪转折点检测
# - 情绪词序列：[平静, 紧张, 恐惧, 释然] = 3个转折点
# - 平移均值法：连续3个点偏离均值 > 阈值算一个转折
# 阈值：≥ 1

# 检查点2：情绪波动方差
# - 计算章节内各段落情绪强度的方差
# - Var = Σ(xi - mean)² / n
# 阈值：≥ 0.05

# 检查点3：高潮强度检测
# - 找出章节中情绪强度最高的段落/场景
# - 阈值：≥ 7
```

## S5 节奏控制

### 定义
信息投放的密度、冲突频率、章末悬念设置是否合理。

### 量化指标

| 指标 | 阈值 | 说明 |
|------|------|------|
| 信息投放密度 | 均匀/过密/过疏 | 每千字信息承载量 |
| 冲突频率 | ≥ 3次/千字 | 含实质对抗的事件次数 |
| 章末悬念 | 有/无 | 章节结尾是否有钩子 |

### 冲突类型定义

| 冲突类型 | 识别关键词 |
|----------|-----------|
| 人vs人 | 争吵、威胁、攻击、对抗、竞争 |
| 人vs自然 | 困境、绝境、障碍、自然灾害 |
| 人vs内心 | 犹豫、抉择、挣扎、痛苦 |
| 人vs命运 | 预言、诅咒、命运、无奈 |

### 判定规则

```
通过条件：密度均匀 OR 过密但有效 AND 冲突频率 ≥ 3/千字 AND 悬念 = 有
警告条件：密度过疏 OR 冲突 2-3/千字 OR 悬念模糊
不通过条件：密度严重过疏 OR 冲突 < 2/千字 OR 无悬念且无完整结局
```

### 验证方法

```python
# 检查点1：信息投放密度
# - 统计章节总字数
# - 统计有效信息句数量（推动剧情的句子）
# - 计算密度 = 信息句数 / (字数 / 1000)
# 阈值：均匀 1.5-3句/千字，过密 > 3，过疏 < 1.5

# 检查点2：冲突频率
# - 搜索冲突关键词出现次数
# - 每出现一次冲突算一个事件
# - 计算频率 = 冲突事件数 / (字数 / 1000)
# 阈值：≥ 3次/千字

# 检查点3：章末悬念
# - 检测最后200字是否包含：
#   - 疑问句（？结尾）
#   - 未完成的动作
#   - 新信息的引入
#   - 冲突未解决
# 阈值：有（满足任一条件）
```

## S6 可读性

### 定义
文字是否易于阅读，句子长度、词汇难度、段落结构是否合理。

### 量化指标

| 指标 | 阈值 | 说明 |
|------|------|------|
| 句子平均长度 | ≤ 30字 | 章节所有句子的平均字数 |
| 生僻词密度 | < 5% | 生僻词/总词数的比例 |
| 段落平均长度 | 3-8句 | 每段落包含的句子数 |

### 生僻词定义

| 类型 | 示例 | 说明 |
|------|------|------|
| 成语/典故 | 画龙点睛、缘木求鱼 | 需文化背景知识 |
| 专业术语 | 灵力、经脉、筑基 | 仅特定题材使用 |
| 生造词 | 频繁自创的怪词 | 无语境可推断 |

### 判定规则

```
通过条件：平均长度 ≤ 30 AND 生僻词 < 5% AND 段落 3-8句
警告条件：平均长度 30-40 OR 生僻词 5-8% OR 段落 2或9-10句
不通过条件：平均长度 > 40 OR 生僻词 ≥ 8% OR 段落 > 10句
```

### 验证方法

```python
# 检查点1：句子平均长度
# - 按句号/感叹号/问号分句
# - 计算每句字数（不含标点）
# - 求平均值
# 阈值：≤ 30字

# 检查点2：生僻词密度
# - 使用词典匹配检测成语/术语
# - 统计超出常见词汇表的词
# - 计算密度 = 生僻词数 / 总词数
# 阈值：< 5%

# 检查点3：段落平均长度
# - 按空行分段落
# - 统计每段落句子数
# - 求平均值
# 阈值：3-8句
```

## S7 主角魅力

### 定义
主角在章节中的存在感、决策影响力、目标清晰度。

### 量化指标

| 指标 | 阈值 | 说明 |
|------|------|------|
| 主角出场覆盖率 | ≥ 80% | 主角出现的段落/总段落数的比例 |
| 主角决策影响力 | ≥ 60% | 主角决策推动的剧情节点/总节点数 |
| 目标清晰度 | 明确/模糊 | 读者能否明确说出主角的目标 |

### 判定规则

```
通过条件：覆盖率 ≥ 80% AND 决策影响 ≥ 60% AND 目标明确
警告条件：覆盖率 70-80% OR 决策影响 50-60% OR 目标较模糊
不通过条件：覆盖率 < 70% OR 决策影响 < 50% OR 目标完全模糊
```

### 验证方法

```python
# 检查点1：主角出场覆盖率
# - 统计以主角名/代词/绰号开头的句子数
# - 计算覆盖率 = 主角句数 / 总句数
# 阈值：≥ 80%

# 检查点2：主角决策影响力
# - 标记所有剧情转折点（冲突发生、高潮、变化）
# - 判断每个转折是否由主角决策引发
# - 计算影响力 = 主角决策转折数 / 总转折数
# 阈值：≥ 60%

# 检查点3：目标清晰度
# - 检测章节内是否存在明确的目标表达
# - 关键词：想要、必须、为了、决定、寻找
# 阈值：明确（有明确目标句）
```

## S8 人物弧光

### 定义
角色在故事中是否发生可感知的变化，变化是否有因可循。

### 量化指标

| 指标 | 阈值 | 说明 |
|------|------|------|
| 角色变化标记数 | ≥ 2次 | 角色状态/心理/行为发生变化的节点 |
| 变化有因可循 | ✅/❌ | 变化前有充分的铺垫/原因 |
| 伏笔回收率 | ≥ 80% | 本章埋设的伏笔在后续章节回收的比例 |

### 变化标记类型

| 类型 | 示例 |
|------|------|
| 心理变化 | 犹豫→坚定、恐惧→勇敢、冷漠→关心 |
| 能力变化 | 获得新能力、突破瓶颈 |
| 关系变化 | 从敌对到合作、从陌生到信任 |
| 认知变化 | 误解→理解、盲目→清醒 |
| 立场变化 | 背叛、效忠、转变 |

### 判定规则

```
通过条件：变化标记 ≥ 2 AND 变化有因 ✅ AND 伏笔回收 ≥ 80%
警告条件：变化标记 = 1 OR 有因但不够充分 OR 伏笔回收 60-80%
不通过条件：变化标记 = 0 OR 变化无因 OR 伏笔回收 < 60%
```

### 验证方法

```python
# 检查点1：角色变化标记
# - 使用LLM检测角色心理/行为状态转移
# - 统计检测到的变化事件数
# 阈值：≥ 2次

# 检查点2：变化有因可循
# - 检测变化前是否有铺垫（300字内）
# - 铺垫类型：回忆、解释、冲突、决定
# - 阈值：有铺垫存在

# 检查点3：伏笔回收率（见伏笔追踪系统）
# - 统计本章埋设的伏笔总数
# - 统计在50章窗口内回收的数量
# - 计算回收率 = 已回收数 / 埋设总数
# 阈值：≥ 80%
```

## 附录：综合评分计算

### 加权总分

```
总分 = S1×0.15 + S2×0.20 + S3×0.10 + S4×0.10 + S5×0.15 + S6×0.10 + S7×0.10 + S8×0.10
```

### 最终等级

| 等级 | 分值区间 | 说明 |
|------|----------|------|
| S级 | 90-100 | 全部维度通过，可直接发布 |
| A级 | 75-89 | 1-2个维度警告，可小幅调整 |
| B级 | 60-74 | 存在1-2个不通过，需修改 |
| 不合格 | < 60 | 存在3+个不通过，需重大修改 |
