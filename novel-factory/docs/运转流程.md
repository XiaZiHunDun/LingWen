## 一、项目结构

```
novel-factory/
├── 03_内容仓库/              # 内容存储
│   ├── 01_全文总体大纲/       # 全文大纲（1份）
│   ├── 02_卷大纲/            # 卷大纲（3份）
│   ├── 03_阶段大纲/          # 阶段大纲（15+份）
│   └── 04_正文/              # 正文章节
│       ├── ch001.md          # 正文章节
│       ├── ch001_大纲.md     # 章节大纲
│       └── ...
├── 06_意见仓库/              # 审核意见
├── 07_汇总仓库/              # 分层汇总
│   ├── 全文正文/             # 全本汇总
│   ├── 卷正文/              # 卷汇总
│   └── 阶段正文/            # 阶段汇总
└── tools/                   # 工具脚本
    ├── generate_chapter_outlines.py  # 章节大纲生成
    ├── verify_quality.py              # 质量验证
    ├── batch_repair.py                # 批量修复
    ├── llm_quality_deep_check.py     # 正文LLM深度质检
    ├── llm_outline_quality_check.py  # 大纲LLM质检（新增）
    ├── fix_worldview.py              # 世界观统一
    ├── fix_ai_traces.py              # AI痕迹消除
    └── rules/                        # 规则配置
        ├── worldview_rules.yaml
        └── ai_trace_rules.yaml
```

## 二、大纲层次结构

### PHASE_5_LLM_QUALITY 详细流程

```
┌─────────────────────────────────────────────────────────────┐
│              PHASE_5_LLM_QUALITY - LLM深度质检               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  STEP_18a: 角色一致性深度检测                               │
│  - 使用CharacterConsistencyLLMChecker                       │
│  - 分析角色行为、对话、决策一致性                            │
│  - 360章分12批，每批30章并行处理                            │
│  - 预估LLM调用：360-720次                                   │
│                      ↓                                      │
│  STEP_18b: 逻辑矛盾全面扫描                                 │
│  - 使用LogicContradictionLLMChecker                         │
│  - 检测时间线、因果链、设定冲突                             │
│  - 需要前后各5章上下文（共10章上下文）                       │
│  - 预估LLM调用：约720次                                      │
│                      ↓                                      │
│  STEP_18c: 伏笔回收完整性验证                               │
│  - 使用ForeshadowRecoveryChecker                            │
│  - 对照章节大纲检查伏笔铺设与回收                            │
│  - 预估LLM调用：约360次                                      │
│                      ↓                                      │
│  STEP_18d: 情感节奏诊断                                     │
│  - 使用EmotionalRhythmChecker                               │
│  - 分析高潮分布、情感曲线、爽点密度                          │
│  - 预估LLM调用：约360次                                      │
│                      ↓                                      │
│  STEP_18e: 质量问题修复                                     │
│  - 使用CharacterRepairer/LogicRepairer/ForeshadowRepairer    │
│  - P0问题必须修复，P1问题应该修复                           │
│  - 预估LLM调用：200-500次                                   │
│                      ↓                                      │
│         门控：P0/P1修复率100% → 进入PHASE_6_SUMMARY          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 四、质检工具使用流程

### 4.1 章节大纲生成

```bash
# 生成指定范围章节大纲
python tools/generate_chapter_outlines.py --chapters 1-30

# 预览模式（不写入）
python tools/generate_chapter_outlines.py --chapters 1-30 --dry-run

# 生成全部章节大纲
python tools/generate_chapter_outlines.py --chapters 1-360
```

### 4.2 质量验证

```bash
# 验证指定章节质量
python tools/verify_quality.py --chapters 1-30

# 随机抽样验证
python tools/verify_quality.py --sample --sample-size 30

# 输出JSON报告
python tools/verify_quality.py --chapters 1-360 --output logs/quality_report.json
```

### 4.3 批量修复

```bash
# 世界观统一（科幻→修真）
python tools/batch_repair.py --chapters 1-120 --track worldview

# AI痕迹消除
python tools/batch_repair.py --chapters 1-120 --track ai_trace

# 全量修复
python tools/batch_repair.py --chapters 1-360 --track all

# 预览模式
python tools/batch_repair.py --chapters 1-30 --track all --dry-run
```

### 4.4 LLM深度质检（新增 v9.2）

```bash
# 角色一致性深度检测
python tools/llm_quality_deep_check.py --check character --chapters 1-30

# 逻辑矛盾全面扫描
python tools/llm_quality_deep_check.py --check logic --chapters 1-360 --batch 30

# 伏笔回收完整性验证
python tools/llm_quality_deep_check.py --check foreshadow --chapters 1-60

# 情感节奏诊断
python tools/llm_quality_deep_check.py --check rhythm --chapters 1-60

# 全量检测+修复
python tools/llm_quality_deep_check.py --check all --repair

# 预览模式（不写入）
python tools/llm_quality_deep_check.py --check all --dry-run

# 输出报告
python tools/llm_quality_deep_check.py --check all --output logs/llm_quality_report.json
```

### 4.6 大纲LLM质检（新增 v9.2）

```bash
# 全文大纲质检
python tools/llm_outline_quality_check.py --check full

# 卷大纲质检（可指定卷号）
python tools/llm_outline_quality_check.py --check volume --volume 1
python tools/llm_outline_quality_check.py --check volume --volume all

# 阶段大纲质检
python tools/llm_outline_quality_check.py --check stage --volume all

# 章节大纲质检（可指定章节范围）
python tools/llm_outline_quality_check.py --check chapter --chapters 1-30
python tools/llm_outline_quality_check.py --check chapter --chapters 1,5,10

# 全量质检（所有层级）
python tools/llm_outline_quality_check.py --check all

# 输出报告
python tools/llm_outline_quality_check.py --check all --output logs/outline_quality_report.json

# 修复问题
python tools/llm_outline_quality_check.py --check all --repair
```

### 4.7 直接修复工具

```bash
# 世界观统一
python tools/fix_worldview.py --chapters 1-120

# AI痕迹消除
python tools/fix_ai_traces.py --chapters 1-360
```

## 五、质检维度与标准

| 维度 | 优先级 | 检测方式 | 修复工具 |
|------|--------|----------|----------|
| 世界观一致性 | P0 | 自动规则检测 | `batch_repair.py --track worldview` |
| AI痕迹 | P1 | 自动规则检测 | `batch_repair.py --track ai_trace` |
| 角色一致性 | P2 | LLM深度分析 | `llm_quality_deep_check.py --check character` |
| 逻辑矛盾 | P0 | LLM深度分析 | `llm_quality_deep_check.py --check logic` |
| 伏笔回收 | P1 | LLM分析+上下文 | `llm_quality_deep_check.py --check foreshadow` |
| 情感节奏 | P2 | LLM分析 | `llm_quality_deep_check.py --check rhythm` |

### 5.1 S1-S8质量维度（S级标准）

| 维度 | 说明 | 检测器 |
|------|------|--------|
| S1 | 剧情完整性 | PlotIntegrityChecker |
| S2 | 逻辑自洽 | LogicConsistencyChecker |
| S3 | 文笔风格（句式多样性） | SentenceDiversityChecker |
| S4 | 情感共鸣 | EmotionalResonanceChecker |
| S5 | 节奏控制 | PacingChecker |
| S6 | 可读性 | ReadabilityChecker |
| S7 | 主角魅力 | ProtagonistCharmChecker |
| S8 | 人物弧光 | CharacterArcChecker |

### 5.2 LLM深度质检新增维度

| 维度 | 说明 | 检测器 |
|------|------|--------|
| S9 | 角色一致性 | CharacterConsistencyLLMChecker |
| S10 | 逻辑自洽度 | LogicContradictionLLMChecker |
| S11 | 伏笔回收率 | ForeshadowRecoveryChecker |

### 5.3 大纲LLM预检维度

| 层级 | 检测重点 | 工具 |
|------|----------|------|
| 全文大纲 | 360章全貌一致性、主线逻辑链、伏笔全局布局 | `llm_outline_quality_check.py --check full` |
| 卷大纲 | 卷内阶段衔接、卷主题一致性、跨卷角色状态 | `llm_outline_quality_check.py --check volume` |
| 阶段大纲 | 阶段内逻辑自洽、人物弧光连贯性、与卷大纲对齐 | `llm_outline_quality_check.py --check stage` |
| 章节大纲 | 大纲与正文一致性、核心事件提取准确性、伏笔铺设质量 | `llm_outline_quality_check.py --check chapter` |

### 5.4 质检阈值

| 等级 | 分数 | 说明 |
|------|------|------|
| S级 | ≥90% | 优秀，可直接发布 |
| A级 | 75-89% | 良好，小幅调整后发布 |
| B级 | 60-74% | 一般，需要修改后复审 |
| C级 | <60% | 不合格，重大问题需返工 |
