# 部门协调协议

## 概述

本文档定义小说工厂中各部门之间的协调规则、交接流程和冲突升级路径。

## 部门职责回顾

| 部门 | 人数 | 核心职责 | 输出 |
|------|------|----------|------|
| 灵感部门 | 3 | 生成立项文件 | 01_灵感库/ |
| 作家部门 | 10 | 正文创作 | 03_内容仓库/04_正文/ |
| 审核部门 | 11 | 质量审核 | 06_意见仓库/ |
| 读者部门 | 20 | 评论收集 | 06_意见仓库/读者评论/ |
| 汇总部门 | 3 | 分层汇总 | 07_汇总仓库/ |

## 部门间交接流程

### 1. 灵感部门 → 作家部门

**触发条件**: 灵感部门完成立项文件

**交接内容**:
- 基础层.yaml（世界观、人物设定）
- 深度层.md（详细大纲）

**交接检查**:
- [ ] 文件齐全
- [ ] 大纲完整
- [ ] 人物卡齐全

**流程**:
```
灵感部门完成 → 通知作家部门主编 → 主编分配任务 → 作家开始创作
```

### 2. 作家部门 → 审核部门

**触发条件**: 作家完成章节（每批10章）

**交接内容**:
- 章节正文（chXXX.md）
- 阶段大纲（参考）

**交接检查**:
- [ ] 自评等级（ S/A/B）
- [ ] 字数达标
- [ ] 格式正确

**流程**:
```
作家完成 → 提交审核队列 → 审核部门领取 → 并行审核 → 汇总意见
```

### 3. 审核部门 → 作家部门

**触发条件**: 审核发现需修改问题

**交接内容**:
- 审核报告
- 具体修改意见

**交接检查**:
- [ ] 意见明确
- [ ] 优先级标记
- [ ] 无遗漏

**流程**:
```
审核完成 → 汇总意见 → 判定需修改 → 启动作家修复 → TaskOutput验证 → 完成
```

### 4. 作家部门/审核部门 → 汇总部门

**触发条件**: 需要分层汇总时

**触发场景**:
- 阶段完成，需要阶段汇总
- 卷完成，需要卷汇总
- 全文完成，需要全文汇总

**交接内容**:
- 所有章节正文
- 阶段/卷/全文大纲
- 审核记录

**流程**:
```
阶段完成 → 汇总部门主笔整合 → 编辑审核 → 校验检查 → 定稿
```

## 冲突升级路径

### 设计冲突（创意/剧情）

**升级路径**: 作家 → 主编 → 汇总部门主笔 → 人工仲裁

**处理时限**: 24小时内

### 技术冲突（一致性问题）

**升级路径**: 审核部门 → 汇总部门 → 主控调度

**处理时限**: 即时

### 资源冲突（人员/时间）

**升级路径**: 部门内部 → 主控调度

**处理时限**: 4小时

## 协作禁止事项

| 禁止项 | 说明 | 后果 |
|--------|------|------|
| 跳过审核 | 作家不得直接跳过审核环节 | 作品无效，打回重做 |
| 跨部门修改 | 未经授权不得修改其他部门输出 | 记录违规，扣分 |
| 隐瞒问题 | 发现问题不报告 | 加重处罚 |
| 绕过主控 | 单独联系人工仲裁 | 警告 |

## 质量门槛

各部门产出必须达到以下最低标准：

| 部门 | 最低标准 |
|------|----------|
| 灵感部门 | 完整度 90% |
| 作家部门 | 自评 B 级以上 |
| 审核部门 | 发现率 95% |
| 读者部门 | 反馈覆盖率 100% |
| 汇总部门 | 一致性 100% |
