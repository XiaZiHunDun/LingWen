# 版权与原创性声明

> 版本：v1.0 | 日期：2026-05-31
> 用途：指导AI辅助创作的版权主张和原创性保护

---

## 一、法律背景

### 1.1 各國版权法对AI生成内容的态度

| 国家/地区 | 立场 | 要件 |
|---------|------|------|
| **美国** | 需人类创造性参与 | 人类作者身份是版权保护的前提 |
| **欧盟** | 正在建立AI版权框架 | 目前倾向于保护人类创作部分 |
| **中国** | 正在讨论 | 《生成式AI服务管理暂行办法》要求标注AI生成内容 |
| **日本** | 灵活处理 | 允许AI辅助创作，人类贡献部分是版权客体 |

### 1.2 关键判例（截至2026年）

- **美国**：USPTO要求人类必须对AI生成内容有足够控制才能获得版权
- **欧盟**：提议要求AI生成内容必须标注，但允许"AI辅助作品"获得保护
- **中国**：鼓励在AI生成内容中保留人类创造性贡献的证据

## 二、原创性判定标准

### 2.1 可主张版权的要素

以下要素如果由人类创作/决策，可以增加版权主张的成功率：

| 要素 | 说明 | 证据要求 |
|------|------|---------|
| **核心设定** | 世界观、力量体系、社会结构 | 大纲文档、版本历史 |
| **人物设计** | 角色性格、成长弧光、关系设计 | 角色档案、审核记录 |
| **情节设计** | 关键转折点、伏笔铺设、高潮设计 | 大纲时间线、审核意见 |
| **世界观约束** | 禁用列表、风格指南 | 规则文档、修改记录 |
| **人工润色** | 显著修改AI生成内容 | 版本对比、修改痕迹 |
| **关键台词** | 核心对白、独创性表达 | 正文版本记录 |

### 2.2 无法主张版权的要素

- 完全由AI生成的纯文字内容
- 模板化的情节走向
- 通用设定（如"升级流"基础框架）
- 纯功能性的文字描述

## 三、项目保护策略

### 3.1 人力创造性投入标准

为确保版权主张，应保证以下人力投入：

| 阶段 | 投入要求 | 记录方式 |
|------|---------|---------|
| **立项决策** | 人类完成核心设定方向 | 立项文档+审批记录 |
| **大纲设计** | 人类完成关键情节点设计 | 大纲版本+修改意见 |
| **正文写作** | AI生成+人类审核修改≥30% | 审核报告+修改记录 |
| **润色定稿** | 人类完成最终审核 | 定稿报告+审核意见 |

### 3.2 证据链保存

**关键证据类型**：

1. **创作过程文档**
   - 灵感库立项文档（01_灵感库/{项目}/）
   - 大纲版本历史（03_内容仓库/大纲/）
   - 审核意见（06_意见仓库/）

2. **修改记录**
   - AI生成版本 vs 人工修改版本对比
   - 修改原因说明（为何修改）
   - 审核员意见（审核维度评估）

3. **时间戳证明**
   - 文档创建时间
   - 修改时间线
   - git提交记录

### 3.3 版权声明模板

**文档头部版权声明**：

```markdown
---
版权声明：
本文档为《{项目名}》{版本号}版本
创作方式：AI辅助创作
人类创造性贡献：设定(100%) + 大纲(100%) + 正文修改({修改百分比}%)
创作日期：{YYYY-MM-DD}
最后修改：{YYYY-MM-DD}
---

## 人力投入说明

本项目采用"人类设定+AI生成+人工审核"模式：
- **设定阶段**：完全由人类完成（100%人力）
- **大纲阶段**：人类设计关键节点，AI辅助细化
- **正文阶段**：AI生成初稿，人工审核修改比例≥{修改百分比}%
- **定稿阶段**：人类最终审核确认
```

## 四、质量门槛与版权保护关联

### 4.1 S/A级质量与版权主张

| 质量等级 | 人工投入要求 | 版权主张建议 |
|---------|-------------|-------------|
| **S级 (≥90%)** | 修改≥50% | 可主张完整版权 |
| **A级 (75-89%)** | 修改≥30% | 可主张AI辅助创作版权 |
| **B级 (60-74%)** | 修改≥20% | 部分版权保护 |
| **不合格 (<60%)** | 修改<20% | 难以主张版权 |

### 4.2 审核报告模板

```yaml
版权保护评估:
  章节: ch{编号}
  版本: v{版本号}
  评估日期: {YYYY-MM-DD}

  人力投入评估:
    原创设定: {百分比}%
    大纲设计: {百分比}%
    正文修改: {百分比}%
    最终润色: {百分比}%
    综合人力贡献: {百分比}%

  版权主张等级: {S/A/B/不合格}
  建议:
    - {如需增加人力投入的具体建议}
    - {保存证据的具体操作}
```

## 五、操作流程

### 5.1 项目启动时

1. 创建项目版权档案（`{项目}/版权声明.md`）
2. 记录设定阶段的人力投入
3. 确定人工审核比例门槛（建议≥30%）

### 5.2 写作阶段

1. 保存AI生成版本（`chXXX_raw.md`）
2. 记录人工修改内容（审核意见）
3. 保留版本对比证据

### 5.3 定稿阶段

1. 生成版权保护评估报告
2. 确认综合人力贡献≥30%
3. 在文档头部添加版权声明

### 5.4 存档要求

| 证据类型 | 保留时间 | 存储位置 |
|---------|---------|---------|
| 立项文档 | 永久 | 01_灵感库/{项目}/ |
| 大纲版本 | 永久 | 03_内容仓库/大纲/ |
| AI生成原始版本 | 项目周期+2年 | 08_已发布/AI原始/ |
| 审核修改记录 | 项目周期+2年 | 06_意见仓库/ |
| 定稿报告 | 永久 | 08_已发布/ |

## 六、免责声明

本项目不对以下情况提供版权保护：
- 完全由AI生成的纯文字内容
- 模板化/套路化情节
- 通用设定和框架
- 未保留创作过程证据的文档
