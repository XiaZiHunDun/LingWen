#### 2.1 三条铁律代码强制（P0-3）

**铁律1：禁止跳过**
```python
# infra/state/workflow_validator.py（新增）
def validate_transition(current_step: str, target_step: str) -> bool:
    """校验状态转换是否合法"""
    VALID_TRANSITIONS = {
        'STEP_14': ['STEP_15'],
        'STEP_15': ['STEP_16'],
        'STEP_16': ['STEP_17', 'STEP_16'],  # 允许重审
        'STEP_17': ['STEP_18'],
        'STEP_18': ['STEP_19', 'STEP_16'],  # 验证失败可退回
    }
    return target_step in VALID_TRANSITIONS.get(current_step, [])
```

**铁律3：禁止自改**
```python
# run_workflow.sh cmd_verify 函数
function cmd_verify() {
    local task_id=$1
    # 检查调用者是否为原任务的执行Agent
    local original_agent=$(jq_get ".agent_tasks.$task_id.agent")
    if [ "$CURRENT_USER" = "$original_agent" ]; then
        log_error "禁止自改：执行Agent不能验证自己的任务"
        exit 1
    fi
}
```

### Phase 3：S1-S8量化标准 + Agent补全（Week 3）

#### 3.1 S1-S8量化阈值定义（P1-1）

```markdown
## S1 剧情完整性 - 量化标准
- 章内目标明确: ✅/❌
- 障碍与目标相关: ✅/❌
- 结果由行动推导: ✅/❌
- 缺失任一项 = 不通过

## S2 逻辑自洽 - 量化标准
- 时间线矛盾检测: 通过/失败
- 空间位置一致: 通过/失败
- 角色能力边界: 通过/失败

## S3 文笔风格 - 量化标准
- Shannon多样性指数 ≥ 2.5
- 句式种类 ≥ 6种
- 单一句式占比 < 40%

## S4 情感共鸣 - 量化标准
- 每章情绪转折点 ≥ 1
- 情绪波动方差 ≥ 0.05
- 高潮场景情绪强度 ≥ 7/10

## S5 节奏控制 - 量化标准
- 信息投放密度: 均匀/过密/过疏
- 冲突频率: 次/千字
- 章末悬念设置: 有/无

## S6 可读性 - 量化标准
- 句子平均长度 ≤ 30字
- 生僻词密度 < 5%
- 段落平均长度: 3-8句

## S7 主角魅力 - 量化标准
- 主角出场覆盖率 ≥ 80%
- 主角决策影响力 ≥ 60%
- 目标清晰度: 明确/模糊

## S8 人物弧光 - 量化标准
- 角色变化标记数 ≥ 2
- 变化有因可循: ✅/❌
- 伏笔回收率 ≥ 80%
```
