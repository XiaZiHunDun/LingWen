> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Expose `cost_tracker.cost_by_tier()` to dashboard API and add a tier-mode toggle in `CostBarChart.vue` so operators can switch between scenario-level and tier-level (haiku/sonnet/opus) cost breakdowns.

**Architecture:** Silent-degrade helper `_extract_cost_by_tier` in `dashboard/protocols.py` mirrors the existing `_extract_cost_by_scenario` pattern. Pydantic `WorkflowStatusResponse.cost_by_tier: dict[str, float]` exposed via FastAPI. Frontend `CostBarChart.vue` gains a `mode` prop (`'scenario'|'tier'`) and tab toggle, parent `WorkflowStatus.vue` keeps `v-model:mode`. Default `mode='scenario'` ensures zero breaking change for existing callers.

**Tech Stack:** Python 3.13 / FastAPI / Pydantic v2 / pytest 9.0.2 / Vue 3 Composition API / ECharts / TypeScript.

```python
# dashboard/protocols.py — top of file (after existing imports)
from infra.ai_service.model_tiers import ModelTier


# dashboard/protocols.py — after _extract_cost_by_scenario (around L316)
def _extract_cost_by_tier(controller: Any) -> dict[str, float]:
    """Phase 8.13: Extract per-tier cost breakdown as {tier_name: usd_float}.

    Mirrors _extract_cost_by_scenario silent-degrade pattern: returns empty
    dict when controller has no cost_tracker or when cost_tracker.cost_by_tier()
    raises. ModelTier enum keys are serialized via .value (e.g. HAIKU → "haiku").
    """
    cost_tracker = getattr(controller, "cost_tracker", None)
    if cost_tracker is None:
        return {}
    try:
        return {tier.value: float(amt) for tier, amt in cost_tracker.cost_by_tier().items()}
    except Exception as exc:  # noqa: BLE001 — silent degrade by design
        logger.warning("cost_tracker.cost_by_tier() failed: %s", exc)
        return {}
```
