## 关键设计决策

### 1. 任务分解: 2 tasks (按 provider),不按 test 类型

- **选 2 tasks (A)** — Task 1 OpenAI class + Task 2 MiniMax class,跟 brainstorming provider 拆分一致
- **不选 1 task (B)** — 4 new tests 一次性加,失去 incremental verification (test count 23→25→27)
- **不选 2 tasks by test type (C)** — 2 under + 2 over 拆分,跨 2 provider 共用测试代码难 (因 inline dict 不抽 helper)

### 2. 验证方式: collect-only + pytest -q (无 API key 走 SKIP)

- **选 pytest --collect-only + pytest -q** — 静态验证 test class 存在 + skipped 数量对,无 API key 也能跑 (跟 Phase 8.9 同样模式)
- **不选 跑 opt-in test** — 需主公手动 opt-in (主公跑, 本 plan 不强制)

### 3. Inline 7-key dict 不抽 helper (跟 Phase 8.9 一致)

- **不抽** `_make_budget_test_initial_inputs` — 4 inline dicts + 2 (Phase 8.9) = 6 inline dicts,跟 Phase 8.2 9 步断言同样 boilerplate 可接受 pattern
- **抽 helper** 会让 Phase 8.10 跟 Phase 8.9 pattern 不一致,违反 backward compat

### 4. `_ = CostBudgetExceeded` keep-alive 沿用 Phase 8.9

- **沿用** — ruff F401 sanctioned,test 2 local import 模式
- **不抽 fixture** — 4 tests 都用同样 import,inline 简单

### 5. budget 阈值 $0.10 under / $0.0001 over (跟 Phase 8.9 一致)

- **gpt-4o-mini 7-8x cheaper** — 5 calls $0.001-0.010,$0.10 留 10x safety
- **M2.7 <$0.01/call** — 5 calls <$0.05,$0.10 留 2x safety
- **不差异化** — 跟 Phase 8.9 一致简化,失败主公 1 行放宽到 $0.20

### 6. CI workflow 0 改 (跟 Phase 8.2 一致)

- **0 改** `.github/workflows/real-llm-tests.yml` L58 `-k` filter
- **0 改** L4/L5 header comment
