### 痛点
1. **新加 SQLite 表 → 复制 `_get_connection`/`_init_db` 样板** — 现状每个新类都重写。
2. **测试无法换 in-memory DB 单点** — 必须 monkeypatch `app_module._default_storage` 单独一处；infra/storage/cost/budget 端测试每个重新 `_connect_to_test_db`。
3. **connection pool 缺失** — 同一进程同 path 反复 `sqlite3.connect()`，5 backend × N call × request = 多余 fd。
4. **DDL 散落** — schema 升级时必须 grep 全部 8 文件找 `CREATE TABLE`，无注册中心。

## 范围

### T2.1 — `infra/persistence/registry.py` (新文件)
**目标**: 提供统一 singleton registry + 连接缓存。

**API**:
```python
# 注册 (启动期 1 次, 静态声明)
register("ripple", RippleStorage, db_path="infra/.state/ripple.db")
register("cost", CostTrackerDB, db_path="infra/.state/cost_tracker.db")
register("workflow", WorkflowDB, db_path="novel-factory/.state/workflow.db")
register("reading", ReadingPowerDB, db_path="novel-factory/.state/reading_power.db")
register("relationship", RelationshipTracker, db_path="infra/.state/social_engine/relationship_network.db")
register("budget", BudgetPersistence, db_path="infra/.state/cost_tracker.db")  # 共用 cost db

# 访问 (lazy init, thread-safe)
storage = get("ripple")             # → RippleStorage singleton
storage = get("reading", db_path=":memory:")  # override for tests

# Test reset
reset_all()                         # 清理全部 singleton (测试间隔离)
reset("ripple")                     # 单个 reset
```

**实现要点**:
- `_registry: dict[str, RegisteredStorage]` 模块级
- `get(name, db_path=None)` 用 `threading.Lock` 守护 double-checked init
- `reset_*()` 删除 instance，下个 `get()` 重新构造
- `register(name, cls, db_path, **kwargs)` 接收 class 引用，存到 registry 即可，不立即构造

### T2.2 — 共享 `get_connection(path)` 工厂
**目标**: 提取 `_get_connection` 样板。

```python
# infra/persistence/connection.py
def get_connection(db_path: str | Path, *, timeout: float = 5.0) -> sqlite3.Connection:
    conn = sqlite3.connect(str(db_path), timeout=timeout)
    conn.row_factory = sqlite3.Row
    return conn
```

**保留各存储类的 `_get_connection` 方法 (不删)**，但内部分别改为 `return get_connection(self.db_path)`。
这样测试如需替换原方法，签名兼容。

### T2.3 — Schema DDL 注册中心
**目标**: 8 类的 `CREATE TABLE`/`CREATE INDEX` 集中管理。

```python
# infra/persistence/schemas.py
SCHEMAS: dict[str, list[str]] = {
    "ripple": [...],      # 现状 storage.py 的 _apply_schema_migrations 输出
    "cost": [...],        # cost_persistence._init_db
    "workflow": [...],    # state/database._init_db + state_manager
    "reading": [...],     # reading_power/db._init_db
    "relationship": [...],# social_engine/relationship_tracker
    "budget": [...],      # budget_persistence (sub-table of cost)
}

def apply_schema(name: str, conn: sqlite3.Connection) -> None:
    """Run registered DDL for `name` against the connection."""
    for stmt in SCHEMAS[name]:
        conn.execute(stmt)
```

**关键**：DDL 内容 0 改（搬迁 + 集中），storage.py 的 `_apply_schema_migrations` 保留原逻辑（additive migration），只是初始 DDL 走 registry。
