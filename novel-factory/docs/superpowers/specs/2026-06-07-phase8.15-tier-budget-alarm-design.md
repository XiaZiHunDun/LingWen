## 1. Context

Phase 8.5–8.13 已经建好 cost tracking 全栈:
- **CostTracker** (in-memory) + **CostTrackerDB** (SQLite) — 记录每次 LLM 调用 (scenario, tier, tokens, cost_usd)
- **CostTracker.cost_by_tier()** + **CostTrackerDB.cost_by_tier()** — 按 tier 聚合 (Phase 8.5/8.7)
- **Phase 8.13** — dashboard 暴露 `cost_by_tier` 到 `WorkflowStatusResponse` (read-only 视图)

**Phase 8.12** 加 run/day/week **total** budget enforcement (跟 `cost_budget_usd` 累加, 1 个 cap 涵盖所有 tier):
- `BudgetService.check_all_scopes(total_cost_usd, run_id)` — 超 raise `CostBudgetExceeded(scope='run'|'day'|'week')`
- AgentComputeFn 集中 check (跟 Phase 8.8 `check_budget` 同位置)
- `SidebarCostBanner` priority cascade (1 active bar, exceeded 优先)

**Phase 8.15 gap**: tier 是 cost 维度, 但 budget enforcement 仍是 total. 用户可能想设 "opus 单价贵, opus 预算 $1.00; sonnet 便宜, 预算 $5.00" — per-tier 隔离, 跟 run/day/week orthogonal. 本期收尾:

> **Add per-tier hard cap (haiku/sonnet/opus 各自 budget), 跟 Phase 8.12 run/day/week 完全 orthogonal**

## 2. Goals

1. **Backend**: per-tier budget 持久化 (append-only `budgets_by_tier` 表) + AgentComputeFn per-tier check (硬中止, 跟 Phase 8.12 同模式) + CostBudgetExceeded `.tier` 字段扩展
2. **Dashboard API**: `GET /api/budgets/by-tier` (3 tier current) + `PUT /api/budgets/by-tier/{tier}` (set) + `WorkflowStatusResponse.budget_by_tier` field (3 tier dict)
3. **UI**: `SidebarCostBanner` 加 3 tier 进度条 (haiku/sonnet/opus) 在 activeBudget 进度条**下方**, 未设跳过
4. **Tests**: 22 backend unit + integration + 1 ceremonial e2e (Phase 8.11/8.12/8.13 同 convention)
5. **0 改 production code 行为破**: 旧 2202 tests 0 改, 5 层 backward compat (exception 字段 / service 类 / compute fn / MC / response model)

## 3. Non-goals

- 软警告 / 三态 (80% warn + 100% hard) — YAGNI, 跟 Phase 8.12 硬中止一致
- 跟 run/day/week **互斥** (e.g. 设了 tier 就 disable run) — orthogonal, 叠加 OK
- `polish_merge_synthesis` / `generate_outline` / 3 utility LLM 暴露 — YAGNI, 跟 Phase 8.13 同
- `ComputeResult` schema 改 — YAGNI
- `CostTrackerDB.cost_records` schema 改 — 已含 `tier` 列, 0 改
- 改 `BudgetService` run/day/week methods — 新增 tier methods 平行, 旧 0 改
- 跨 process / 跨 phase tier budget 兼容 — DB schema CREATE IF NOT EXISTS 幂等
- Tier budget per-run 隔离 — tier budget 是 GLOBAL (1 进程 1 instance), 不需 run_id 维度
- 软警告 UI (黄/红两色) — 跟 Phase 8.12 ok/exceeded 二态一致

## 4. Design

### 4.1 Architecture (跟 Phase 8.12 完全正交)

```text
LLM Call (AgentComputeFn.__call__)
  ↓
record(scenario, tier, in_tok, out_tok) → CostTracker._records  ← cost_by_tier[tier] 累加
  ↓
check_budget(_current_budget_usd)         ← Phase 8.8 总 cap (未变)
  ↓
check_all_scopes(total_cost, run_id)      ← Phase 8.12 run/day/week (未变)
  ↓
check_all_tiers(cost_by_tier)             ← Phase 8.15 NEW per-tier
  ├─ tier='haiku'  budget=$0.10 → cost=$0.05 → ok
  ├─ tier='sonnet' budget=$0.50 → cost=$0.60 → raise CostBudgetExceeded(scope='tier', tier=SONNET)
  └─ tier='opus'   budget=$1.00 → cost=$0.20 → ok
  ↓ (raise 时)
scheduler 兜底 → node FAILED + total_cost_usd 不变 (record 已在 raise 前)
```

**Key invariants**:
- Tier budget 跟 run/day/week orthogonal (4 维 budget 可同时设, 任一超 raise)
- 检查顺序: total → run/day/week → tier (跟 Phase 8.12 同栈, raise 早于 scheduler 兜底)
- Tier check 顺序: haiku → sonnet → opus (enum 顺序, 第 1 个超阈 raise, deterministic)

class CostBudgetExceeded(Exception):
    def __init__(
        self,
        used_usd: float,
        budget_usd: float,
        scenario: str | None = None,
        scope: str = "run",  # 'run'|'day'|'week'|'tier'  (Phase 8.15 +1 value)

        self.used_usd = used_usd
        self.budget_usd = budget_usd
        self.scenario = scenario
        self.scope = scope
        self.tier = tier  # NEW
        scenario_msg = f" (last scenario: {scenario})" if scenario else ""
        scope_msg = f" [scope={scope}]" if scope != "run" else ""
        tier_msg = f" [tier={tier.value}]" if scope == "tier" and tier else ""  # NEW

            f"Cost budget exceeded: ${used_usd:.4f} / ${budget_usd:.4f}{scope_msg}{tier_msg}{scenario_msg}"

# dashboard/protocols.py
def _extract_budget_by_tier(controller: Any) -> dict[str, dict[str, Any]]:
    """Mirror _extract_budget_per_window silent-degrade pattern.
    
    Returns:
        dict[tier_name, {status, budget_usd, used_usd, used_pct}] (3 tier always)
        未设 / 异常 / 无 service → 3 tier 都空 dict
    """
    service = getattr(controller, "budget_service_by_tier", None)
    if service is None:
        return {tier.value: {} for tier in ModelTier}
    try:
        result = {}
        for tier in ModelTier:
            entry = service.get_by_tier(tier)
            if entry is None or entry.usd <= 0:
                result[tier.value] = {}
                continue
            cost_tracker = getattr(controller, "cost_tracker", None)
            used = float(cost_tracker.cost_by_tier().get(tier, 0.0)) if cost_tracker else 0.0
            used_pct = (used / entry.usd * 100) if entry.usd > 0 else 0.0
            status = "exceeded" if used > entry.usd else "ok"
            result[tier.value] = {
                "status": status, "budget_usd": entry.usd,
                "used_usd": used, "used_pct": used_pct,
            }
        return result
    except Exception as exc:
        logger.warning("_extract_budget_by_tier failed: %s", exc)
        return {tier.value: {} for tier in ModelTier}

@app.put("/api/budgets/by-tier/{tier}")
async def set_budget_by_tier(tier: str, req: BudgetTierSetRequest) -> dict[str, Any]:
    if tier not in ("haiku", "sonnet", "opus"):

    except ValueError:
        raise HTTPException(400, f"invalid tier: {tier!r}")

## 5. Risks + mitigations

| 风险 | 缓解 |
|------|------|
| 旧 2202 tests 破 (tier budget 误注入 / exception 字段冲突) | 5 层 backward compat (exception `.tier=None` default / service 类不破坏 / compute fn kwarg None / MC kwarg None / response field default_factory / banner 加 row 不动 cascade) |
