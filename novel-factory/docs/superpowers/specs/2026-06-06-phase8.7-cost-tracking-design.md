## 4. Design

### 4.1 架构 (5 层 additive)

```
[Layer 1: Polisher agent (inherited AgentBase.chat_with_usage — Phase 8.6)]
    self.polisher.chat_with_usage(prompt, system, T=0.2, max_tokens=2000)
    → (response_str, {"input_tokens": int, "output_tokens": int})
    → 真实 usage (5 provider override 支持)

[Layer 2: MasterController (NEW polish_merge_synthesis_with_usage)]
    polish_merge_synthesis(content_a, content_b, *, labels) → dict
        → _impl(record_usage=False) → 旧 path (Phase 7.5 baseline)
    polish_merge_synthesis_with_usage(content_a, content_b, *, labels)
        → tuple[dict, dict] → _impl(record_usage=True)
            - 4 fallback path (empty/identical/json_parse_failed/llm_fail)
              返 normal dict + {"input_tokens": 0, "output_tokens": 0}
            - LLM success path
              返 scored dict + 真实 usage
            - 韧性契约: LLM fail 返 normal dict (无 _error, 跟 8.6.2 _impl_audit_chapter 同)

[Layer 3: GoT Bridge]
    _handler_polish_merge (改) 返 tuple[dict, dict]
    → master.polish_merge_synthesis_with_usage(content_a, content_b, labels)
    → AgentComputeFn.__call__ isinstance(tuple) True
    → 真实 usage 喂 cost_tracker.record("polish_merge", HAIKU, in, out)

[Layer 4: Dashboard data API (扩展 Phase 8.5 字段)]
    WorkflowStatusResponse + cost_by_scenario: dict[str, float]
    MasterControllerAdapter.get_active_workflow_status:
        - _extract_total_cost (Phase 8.5, 0 改)
        - _extract_cost_by_scenario (NEW) → cost_tracker.cost_by_scenario()
    _workflow_result_to_response (修 Phase 8.5 留的 gap):
        也加 total_cost_usd + cost_by_scenario (POST /run + /resume 端点)
    WebSocket /api/ws/workflows 推送: 跟 score_data 同位置加 cost_by_scenario

[Layer 5: Dashboard UI (NEW CostBarChart.vue)]
    ECharts 5.5 bar, Press Start 2P 像素风 (跟 CoolpointChart 同模式)
    x: 12 SCENARIOS (sorted by cost desc), y: cost_usd
    WorkflowStatus.vue 底部 <CostBarChart v-if="hasCost"> 集成
    api/index.js @typedef JSDoc 注释 (无 TS, 跟现状一致)
```

### 4.3 数据流 (3 流向)

#### Flow 1: Backend → cost_tracker (polish_merge_synthesis_with_usage)

```
master.polish_merge_synthesis_with_usage(content_a, content_b, labels)
    → _impl_polish_merge_synthesis(content_a, content_b, labels, record_usage=True)
        ├─ empty_content? → _merge_synthesis_len_fallback(...) → dict
        │       return (dict, {"input_tokens": 0, "output_tokens": 0})
        ├─ identical? → fallback dict
        │       return (dict, zero_usage)
        ├─ LLM path:
        │   try:
        │       response, usage = self.polisher.chat_with_usage(
        │           prompt=build_merge_synthesis_prompt(...),
        │           system=get_merge_synthesis_system_prompt(),
        │           temperature=0.2, max_tokens=2000,
        │       )
        │       parsed = self._parse_merge_response(response)
        │       ...compute winner/scores...
        │       return (scored_dict, usage)
        │   except _MergeParseError:
        │       return (fallback_dict, zero_usage)
        │   except Exception:
        │       return (fallback_dict, zero_usage)

→ _handler_polish_merge 收 (scored_dict, usage) tuple
    return result.setdefault(...), usage

→ AgentComputeFn.__call__ isinstance(output, tuple) True
    output, usage = tuple
    if output._error: fail=True
    else: cost_tracker.record("polish_merge", HAIKU, 100, 50)
```

#### Flow 2: Backend → WorkflowStatusResponse (cost_by_scenario)

```
GET /api/workflows/active
    → master_controller.get_active_workflow_status()
        → _extract_total_cost(self._controller)  # Phase 8.5
            = float(cost_tracker.total_cost())
        → _extract_cost_by_scenario(self._controller)  # NEW
            = dict(cost_tracker.cost_by_scenario())
    → dict["cost_by_scenario"] = {"chapter_writing": 0.012, ...}
    → WorkflowStatusResponse(**dict)

POST /api/workflows/run + /resume
    → _workflow_result_to_response(...)
        → 也加 cost_by_scenario (修 Phase 8.5 留的 gap)

WebSocket /api/ws/workflows
    → 推送 dict 加 cost_by_scenario 字段
    → Vue useWorkflowSocket composable 自动 rerender
```

#### Flow 3: Frontend → ECharts bar (cost display)

```
api/index.js fetchActiveWorkflow()
    → Promise<{ cost_by_scenario: Record<string, number>, ... }>
    → JSDoc @typedef

WorkflowStatus.vue
    hasCost = Object.keys(props.status?.cost_by_scenario || {}).length > 0
    <div v-if="hasCost" class="pixel-card">
        <h3>💰 Token 成本 (累计 USD)</h3>
        <p>总计: ${{ status.total_cost_usd.toFixed(4) }}</p>
        <CostBarChart :cost-by-scenario="status.cost_by_scenario" />
    </div>

CostBarChart.vue (~140L, 跟 CoolpointChart 同模式)
    import * as echarts from 'echarts'
    props: { costByScenario: { type: Object, required: true } }
    chartRef = ref(null), let chartInstance = null
    onMounted: echarts.init(chartRef.value) + render()
    onUnmounted: chartInstance.dispose()
    watch(costByScenario, render, { deep: true })
    render():
        scenarios = Object.keys(costByScenario).sort(by cost desc)
        chartInstance.setOption({
            grid: { top: 30, right: 20, bottom: 60, left: 60 },
            xAxis: { type: 'category', data: scenarios,
                     axisLabel: { rotate: 30, fontSize: 8, fontFamily: 'Press Start 2P' } },
            yAxis: { type: 'value', name: 'USD',
                     nameTextStyle: { fontFamily: 'Press Start 2P', fontSize: 8 } },
            series: [{
                type: 'bar',
                data: scenarios.map(s => costByScenario[s]),
                itemStyle: { color: '#ff6b6b', borderColor: '#2a220f', borderWidth: 2 },
                barWidth: '60%',
                label: { show: true, position: 'top',
                         fontFamily: 'Press Start 2P', fontSize: 8,
                         formatter: p => `$${p.value.toFixed(4)}` },
            }],
            tooltip: { trigger: 'axis',
                       textStyle: { fontFamily: 'Press Start 2P', fontSize: 8 } },
        })
```

### 4.4 错误处理 (5 错误场景, 全保持韧性契约)

| # | 场景 | 触发条件 | 处理 | 韧性契约 | 失败 mode |
|---|------|---------|------|---------|----------|
| 1 | LLM 抛错 (network/timeout) | `chat_with_usage` raise | `_impl` try/except → `_merge_synthesis_len_fallback(reason="llm_fail")` | 返 normal dict, `usage=zero` | workflow 不中断 |
| 2 | JSON parse 失败 | `polisher.parse_response` 返 `{raw}` | `_impl` try/except → `_merge_synthesis_len_fallback(reason="json_parse_failed")` | 同上 | workflow 不中断 |
| 3 | Empty content | `not content_a or not content_b` | `_merge_synthesis_len_fallback(reason="empty_content")` | 返 normal dict, `usage=zero` | 不调 LLM |
| 4 | Identical content | `content_a == content_b` | inline dict `fallback="identical"` | 返 normal dict, `usage=zero` | 不调 LLM |
| 5 | `cost_tracker=None` | MasterController 注入 None | `_extract_cost_by_scenario` getattr 兜底 | 返 `{}` empty dict | UI `hasCost=false` |

**关键**:
- **`_impl_polish_merge_synthesis` 不设 `_error`** (跟 8.6.2 `_impl_audit_chapter` 同 fix, 避免 AgentComputeFn `fail=True`)
- **Frontend 兜底**: `props.status?.cost_by_scenario` 默认 `{}`, `v-if` 不渲染避免 ECharts init 失败
- **WebSocket 断开**: 复用 Phase 7.x `useWorkflowSocket` 1s auto-reconnect

## 5. Risks + Mitigations

| 风险 | 缓解 |
|------|------|
| 旧 5 TestPolishMergeSynthesis tests 破 | `_impl` 内部 LLM path 改 `chat_with_usage`, 4 fallback path 行为跟 Phase 7.5 baseline 100% 一致, 旧 method 走 `record_usage=False` 保原 `chat()` 路径 |
| AgentComputeFn `_error` 误设触发 fail=True | 显式不设 `_error` (跟 8.6.2 `_impl_audit_chapter` 同 fix) |
| WebSocket 字段缺 cost_by_scenario | 跟 score_data 同位置加, Phase 7.6 模式已验证 |
| `cost_tracker=None` 抛错 | `_extract_cost_by_scenario` 走 `getattr(self._controller, "cost_tracker", None)` 兜底, 返 `{}` |
| `cost_tracker.db` 污染 git status | `.gitignore` 加 `infra/.state/cost_tracker.db` (跟 `social_engine/relationship_network.db*` 模式一致) |
| Vue ECharts init 失败 (无数据) | `hasCost = false` 时不渲染 `<CostBarChart>`, `chartRef` 不会 ref 到 null DOM |
| Phase 8.5 run/resume gap 漏修 | 显式在 `_workflow_result_to_response` 加 cost_by_scenario + total_cost_usd, 4 tests 覆盖 |
| ECharts 重渲染性能 | 复用 CoolpointChart `dispose + re-init` 模式, 单图 12 柱 < 16ms |
