# Phase 9.20 Spec — Cascade Persistence

> **Goal:** Add persistent cascade run history — every `--persist` cascade saves 1 row to `cascade_runs` table; dashboard / CLI can list & inspect historical runs.
> **Architecture:** New `cascade_runs` table (additive, keep Phase 9.15 `ripple_cascade` untouched) + `RippleStorage.record_cascade_run()` + `GET /api/ripples/cascade/{id}?persist=true` + `lingwen.py cascade --persist`.
> **Tech Stack:** Python 3.12 + sqlite3 + argparse (lingwen.py 1:1) + FastAPI + Pydantic (0 改) + pytest 8 + ruff.

---

## Context

Phase 9.15 → 9.19 演进:
- **Phase 9.15**: `ripple_cascade` 表存 **每 ripple 最新 1 cascade** (JSON 内嵌 nodes/edges/actions), 1 row per ripple
- **Phase 9.16**: weighted BFS + cascade real-time WS push (CascadeGraph hover + CascadeUpdatePayload)
- **Phase 9.17**: CascadeUpdatePayload Pydantic schema + handler guard
- **Phase 9.18**: ripple-reset CLI (test idempotency)
- **Phase 9.19**: `RippleStorage.preview_cascade(ripple_id, max_depth)` — **不持久化**, 纯 BFS 模拟 + `lingwen.py cascade` CLI + endpoint `?max_depth=` query param (13 new tests)

**问题**: Phase 9.19 `preview_cascade` 每次调用是瞬时的, 跑完就丢. 用户无法:
1. 看历史 cascade 跑了哪些 (`last run 30 min ago depth=3, last run 5 min ago depth=5`)
2. 比对多次 cascade 结果 (e.g. depth=3 vs depth=5 哪个 nodes 多)
3. 复盘 cascade 决策 (为什么 apply 那次 cascade depth=5 不是 depth=3)

**目标**: 加 `cascade_runs` 表存**全部历史 cascade run**, `record_cascade_run()` method + history list endpoint + CLI `--persist` flag, dashboard 可查询历史 (Phase 9.22 UI 留 followup).

## Goals

1. **新表 `cascade_runs`** — additive, 1 row per `--persist` cascade run, 4 status (running/completed/cancelled/failed)
2. **`RippleStorage.record_cascade_run(ripple_id, cascaded, max_depth)`** — INSERT 1 row, return cascade_run_id
3. **`RippleStorage.get_cascade_runs(ripple_id, limit=50, offset=0, status=None)`** — list history, ORDER BY id DESC (latest first), filter by status (Phase 9.21 留 cancelled filter)
4. **`RippleStorage.get_cascade_run_by_id(run_id)`** — fetch single run
5. **`GET /api/ripples/cascade/{ripple_id}?persist=true`** — execute cascade + record + return run row JSON
6. **`GET /api/ripples/cascade/{ripple_id}/runs?limit=50&offset=0`** — list history (新 endpoint, 不影响既有)
7. **`lingwen.py cascade <ripple_id> --max-depth N --persist`** — 新 flag, default off 保 Phase 9.19 backward compat
8. **12 new tests** (4 storage + 5 endpoint + 3 CLI)

## Design

### 1. Schema — 新表 `cascade_runs`

```sql
CREATE TABLE IF NOT EXISTS cascade_runs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    ripple_id TEXT NOT NULL,
    max_depth INTEGER NOT NULL,
    depth_reached INTEGER NOT NULL,
    algorithm TEXT NOT NULL,  -- 'v2_weighted' (Phase 9.16)
    started_at TEXT NOT NULL,
    completed_at TEXT NOT NULL,
    status TEXT NOT NULL CHECK(status IN ('running', 'completed', 'cancelled', 'failed')),
    nodes_json TEXT NOT NULL,  -- JSON array of {id, dimension, volume, chapter, ...}
    edges_json TEXT NOT NULL,  -- JSON array of {id, from_node_id, to_node_id, ...}
    actions_json TEXT NOT NULL,  -- JSON array of action strings
    FOREIGN KEY (ripple_id) REFERENCES reference_ripples(id) ON DELETE CASCADE
);
CREATE INDEX IF NOT EXISTS idx_cascade_runs_ripple ON cascade_runs(ripple_id, id DESC);
CREATE INDEX IF NOT EXISTS idx_cascade_runs_status ON cascade_runs(status) WHERE status != 'completed';
```

**对比 Phase 9.15 `ripple_cascade` 表** (保留, 0 改):
- `ripple_cascade` = 1 latest per ripple (dashboard 既 path), `id DESC LIMIT 1` pattern
- `cascade_runs` = 全部历史 (Phase 9.20 新 path), `id DESC LIMIT N` pattern
- 两表无数据冗余 (cascade_runs 是 ripple_cascade 的 superset)

### 2. `RippleStorage.record_cascade_run()`

```python
def record_cascade_run(
    self,
    ripple_id: str,
    cascaded: "CascadedRipple",
    max_depth: int,
    status: str = "completed",  # 'running'/'completed'/'cancelled'/'failed'
) -> int:
    """Phase 9.20: append 1 row to cascade_runs (independent commit).

    跟 record_audit / record_cascade 1:1 pattern: 主 write path 失败不传播到这里.

    Returns:
        cascade_run_id (autoincrement PK)
    """
    now_iso = datetime.now(timezone.utc).isoformat()
    with self._connect() as conn:
        cur = conn.execute(
            "INSERT INTO cascade_runs (ripple_id, max_depth, depth_reached, algorithm,"
            " started_at, completed_at, status, nodes_json, edges_json, actions_json)"
            " VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?)",
            (
                ripple_id,
                max_depth,
                cascaded.depth_reached,
                cascaded.bfs_algorithm_version,
                now_iso,  # started_at = now (cascade 0.5s, started=completed)
                now_iso,
                status,
                json.dumps([self._node_to_dict(n) for n in cascaded.cascade_nodes], ensure_ascii=False),
                json.dumps([self._edge_to_dict(e) for e in cascaded.cascade_edges], ensure_ascii=False),
                json.dumps(list(cascaded.cascade_actions), ensure_ascii=False),
            ),
        )
        conn.commit()
        return cur.lastrowid
```

### 3. `RippleStorage.get_cascade_runs()`

```python
def get_cascade_runs(
    self,
    ripple_id: str,
    limit: int = 50,
    offset: int = 0,
    status: str | None = None,  # Phase 9.21+ filter
) -> list["CascadeRun"]:
    """Phase 9.20: list cascade runs for ripple_id (latest first)."""
    clauses, params = ["ripple_id = ?"], [ripple_id]
    if status is not None:
        clauses.append("status = ?")
        params.append(status)
    where = f"WHERE {' AND '.join(clauses)}"
    sql = (f"SELECT * FROM cascade_runs {where}"
           f" ORDER BY id DESC LIMIT ? OFFSET ?")
    params.extend([limit, offset])
    with self._connect() as conn:
        return [self._row_to_cascade_run(row) for row in conn.execute(sql, params)]
```

### 4. `RippleStorage.get_cascade_run_by_id()`

```python
def get_cascade_run_by_id(self, run_id: int) -> "CascadeRun | None":
    """Phase 9.20: fetch single cascade run by PK, or None."""
    with self._connect() as conn:
        row = conn.execute(
            "SELECT * FROM cascade_runs WHERE id = ?", (run_id,)
        ).fetchone()
        return self._row_to_cascade_run(row) if row else None
```

### 6. Endpoint `GET /api/ripples/cascade/{ripple_id}?persist=true`

```python
# dashboard/app.py: 既 get_ripple_cascade 加 persist query param
async def get_ripple_cascade(
    ripple_id: str,
    max_depth: int | None = Query(default=None),  # Phase 9.19
    persist: bool = Query(default=False),  # Phase 9.20 NEW
) -> dict:
    """Phase 9.19 + 9.20: 既 preview cascade + optional persist.

    persist=False (default): 走 Phase 9.19 既 path — get_cascade_by_ripple_id
                             (latest persisted cascade from ripple_cascade table)
    persist=True: 调 preview_cascade + record_cascade_run + return CascadeRun row

    Raises:
        404 if ripple not found
        400 if max_depth invalid (Phase 9.19 _validate_max_depth)
    """
    # 既 path (persist=False)
    if not persist:
        # 调 Phase 9.19 既 logic — 0 改
        cascaded = storage.get_cascade_by_ripple_id(ripple_id)
        if cascaded is None:
            raise HTTPException(404, ...)
        return CascadedRippleResponse(...).model_dump()

    # Phase 9.20 新 path (persist=True)
    try:
        cascaded = storage.preview_cascade(ripple_id, max_depth=_resolve_max_depth(max_depth))
    except KeyError:
        raise HTTPException(404, ...)
    run_id = storage.record_cascade_run(ripple_id, cascaded, max_depth=_resolve_max_depth(max_depth))
    run = storage.get_cascade_run_by_id(run_id)
    return CascadeRunResponse.from_dataclass(run).model_dump()
```

```python
def execute(self, options):
    storage = _get_storage()

        cascaded = storage.preview_cascade(options.ripple_id, max_depth=options.max_depth)
    except KeyError:
        return 1  # Phase 9.19 既 error path

    print(f"edges: {len(cascaded.cascade_edges)}")
    print(f"actions: {len(cascaded.cascade_actions)}")
    print(f"algorithm: {cascaded.bfs_algorithm_version}")
    return 0
```
