## 4. Design

### 4.1 Phase 8.45.1 — DecisionCard 3 status testid

**Files**:
- Modify: `dashboard/frontend/src/components/DecisionCard.vue` (L13-18)
- Create: `dashboard/frontend/tests/unit/decision-card-status.spec.ts` (NEW, ~60L, 4 tests)

**Change**:
```vue
<!-- L13-18, before -->
<span
  v-if="statusBadge"
  class="status-badge"
  data-testid="status-badge"
  :class="statusBadge.cls"
>{{ statusBadge.label }}</span>

<!-- after, 加 dynamic specific testid -->
<span
  v-if="statusBadge"
  class="status-badge"
  data-testid="status-badge"
  :data-testid="`status-${statusKey}`"
  :class="statusBadge.cls"
>{{ statusBadge.label }}</span>
```

`statusKey` computed:
```js
const statusKey = computed(() => {
  // map decision.status → kebab-case key
  return (props.decision.status || '').replace(/_/g, '-');
});
```

(实际 decision.status 已是 resolved/deferred/cancelled, 0 改 `_` chars, 但加 kebab-case 转换跟 Phase 8.38 `kindClass` 模式一致防御未来 `in_progress` 之类多 token status)

**Test pattern**:
```ts

    const wrapper = mount(DecisionCard, {
      props: { decision: { kind: 'outline_judgment', status: 'resolved', prompt: '...', options: [] } }
    })
    expect(wrapper.find(byTestid('status-resolved')).exists()).toBe(true)
    expect(wrapper.find(byTestid('status-deferred')).exists()).toBe(false)
  })
  // 3 more tests: deferred / cancelled / pending (no badge)

```
