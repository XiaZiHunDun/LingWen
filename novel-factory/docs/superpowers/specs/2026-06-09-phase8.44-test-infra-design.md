### 3.3 关键 design decision rationale (跟 §3 design sessions 1:1)

#### 3.3.1 Coverage threshold 70/60

| 维度 | 值 | Rationale |
|------|------|-----------|
| lines | 70% | 16 spec 95 test 估实测 75-85%, 70% 是 realistic minimum |
| branches | 60% | vue v-if/v-else 链路多分支, 70% 假阴性高 |
| functions | 70% | 跟 lines 对齐 |
| statements | 70% | 跟 lines 对齐 |

跟 backend `--cov-fail-under=30` 不对齐 (backend 30% 是历史债大, 后期慢慢提, 留 informational).

#### 3.3.2 Codecov 2 flags + after_n_builds: 2

- **2 flags 隔离**: 1 PR 同时改 frontend + backend 时, Codecov 等 2 个 build 跑完再 comment
- **`wait_for_ci: true`**: 防 Codecov 在 CI fail 时误报
- **`require_ci_to_pass: true`**: Codecov 不会在自己 fail 时报
- **`require_head: true` + `require_base: false`**: fork PR 也能报 (0 base data 兜底)
- **`fail_ci_if_error: false`**: token 缺失 / service outage 0 阻塞 CI

#### 3.3.3 lint-staged 双 runner (eslint + vitest related)

- **eslint --fix 在前**: lint 错误导致 test 假阴性, 顺序不可调 (lint-staged serial 跑)
- **vitest --related 在后**: vitest 3.0+ --related flag, 按 git diff 算 import graph, 0 跑全量
- **常态 <2s, worst case 5s** (vs 全量 5-10s)

#### 3.3.4 Pre-commit pytest smoke (-x + 30s timeout)

- **`--ignore` 跟 test.yml 同步**: e2e + legacy 跳过, 0 hang
- **`-x`**: fail-fast, 1 fail 立即 abort
- **`--timeout=30`**: 单 test 超 30s fail
- **估 <30s** (vs 全跑 45-60s)
- **0 跑 2262 全量**: smoke focus 改的部分, CI 兜底

#### 3.3.5 Codecov 跟 gitignored `.coverage` / `coverage/` 关系

- `coverage/` (frontend): vitest 产出, 加进 `.gitignore` (8.44.1 在 `dashboard/frontend/.gitignore` 加, 或 root .gitignore 加 `dashboard/frontend/coverage/`)
- `coverage.xml` (backend): pytest 产出, CI 临时文件, 0 提交, 0 gitignore (CI only artifact)
- `.coverage` (backend SQLite): pytest-cov 产出, 跟现状一致 gitignored, 0 改
