# Phase 8.11 — Persistent Cost Banner (Sidebar)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Move cost + budget status from inside `WorkflowStatus.vue` (only visible when viewing active workflow) to a persistent Sidebar bottom banner visible across all 4 dashboard pages (Overview / Decisions / Workflows / placeholders).

**Architecture:** Backend 0 改 (Phase 8.7/8.8 路径已就位, ws.py 1s 轮询 + protocols.py 提取 cost_by_scenario + total_cost_usd + cost_budget_status 全 OK)。Front-end 加 1 focused `.vue` component (SidebarCostBanner, ~100L), App.vue 引入并绑 useWorkflowSocket().status (5L), WorkflowStatus.vue 删 budget banner 重复块 (DRY, -25L, 保留 CostBarChart cost section 因为 by-scenario 详情 ≠ total 信息密度)。

**Tech Stack:** Vue 3 Composition API + Vite + Playwright (e2e), 跟 Phase 7.6/8.7/8.8 一致。

## 4. Design

### 4.1 Component tree (改动后)

```
App.vue
├── <aside class="sidebar">          ← sidebar 200px 宽
│   ├── <div class="sidebar-header"> 标题
│   ├── <nav class="nav-menu"> 导航
│   └── <SidebarCostBanner :status="status" />  ← NEW
└── <main>
    └── <WorkflowsPage>
        └── <WorkflowStatus :status="status">  ← 删 budget banner (L86-109)
                                                ← 保留 CostBarChart (L78-84)
```

### 4.2 Data flow (0 backend 改)

```
MasterControllerAdapter.get_active_workflow_status()
  → protocols.py:283-286 提取 cost_by_scenario + total_cost_usd + cost_budget_status
  → ws.py:_broadcast_loop 1s 轮询 + _events_equal 变化 push
  → useWorkflowSocket.js status.value = payload
  → App.vue 解构 status, <SidebarCostBanner :status="status" /> props
  → SidebarCostBanner computed 派生 totalBudget / hasCost / usedPct
  → Vue reactivity 自动 re-render 3 行
```

│ └────────────────────┘ │

### 4.4 Styling 决策 (跟 Phase 8.8 + sidebar nav 一致)

- Press Start 2P 8px (跟 nav-item 字体一致)
- 容器: `border-top: 2px solid var(--border-color)` + `padding: var(--space-sm)` (跟 nav-item 间距对齐)
- 进度条: `<div>` 内嵌 `<div :style="{ width: pct + '%' }">` 4px high, 绿色 `var(--color-success)` 当 ok, 红色 `#f56c6c` 当 exceeded
- 颜色: 复用 Phase 8.8 (exceeded 红 `#f56c6c` / exceeded bg `#fef0f0` / ok 绿 `var(--color-success)` / text-dim 灰)
- emoji icon: 💰 (Apple Color Emoji fallback, 跟 Phase 8.8 banner icon 同)

### 4.5 Empty / no-budget 行为

- **Outer v-if**: `Object.keys(costByScenario).length > 0` (有数据才显示整个 banner) — **主公决策**
- **Inner budget row**: 条件渲染 — 有 `budget_usd` 时显 budget line + progress bar; 无时整组隐藏, 只剩 Row 1 total + (无 progress bar, Row 3 隐藏); 不浪费 vertical space
- 等价于: `hasCost` true → 显示容器, 内部 3 行按需 conditional

### 4.6 WorkflowStatus.vue 删 banner 块 (DRY)

- **删 L86-109** `<div v-if="hasBudget" class="cost-budget-section">` (banner)
- **保留 L78-84** `<div v-if="hasCost" class="cost-section">` (含 CostBarChart)
- 理由: CostBarChart 是 by-scenario 详细 bar chart,sidebar 只有 total; 信息密度互补,不重复; WorkflowsPage 重点用户仍能从工作流 context 看到 by-scenario
- 跟 Phase 8.8 banner 落地逻辑一致 (banner 全局化, chart 留 context 详情)
- 删 budget section 后,WorkflowStatus 内部 -1 import + -1 computed,清理 budgetStatusClass / ariaRole / ariaLive 3 computed (cost_budget_status 不再被引用)
