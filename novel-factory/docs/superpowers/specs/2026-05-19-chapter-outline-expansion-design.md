> 日期：2026-05-19

---

## 一、背景与目标

### 1.1 现状问题

当前小说工厂的大纲层级：

| 层级 | 内容 | 粒度 |
|------|------|------|
| 全局大纲 | 书概述、人物、主题 | 无章节级 |
| 卷大纲 | 6个阶段，ch001-120 | 20章/阶段 |
| 阶段大纲 | 详细情节点，ch001-ch005 | 5章/块 |

**缺失**：没有章纲（ch001 单独的大纲），作家写每章时缺乏具体指导。

### 1.2 优化目标

构建一个从**阶段大纲**到**章纲**再到**正文**的自动化生成系统：
- **章纲生成器**：从阶段大纲自动拆分生成每章的章纲（约200字）
- **正文扩展器**：将章纲自动扩展为3000字正文初稿
- **两种模式**：全自动生成 或 作家编辑后再扩展

---

## 二、技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| 章纲生成 | LLM 调用（结构化输出） | 需要理解阶段大纲上下文 |
| 正文扩展 | LLM 调用（Writer Persona） | 扩展需要语义理解和风格注入 |
| 章纲存储 | YAML 文件 | 结构化、可编辑、人类可读 |
| 部署环境 | Python + Docker | 与现有环境一致 |

## 四、章纲结构

### 4.1 章纲数据格式

每章章纲包含：

```yaml
chapter_outline:
  chapter_id: "ch051"
  version: "v1.0"
  generated_at: "2026-05-19T10:00:00Z"

  # 节纲信息（从阶段大纲继承）
  stage_overview: "林夜和苏琳的感情在相处中加深"
  stage_id: "卷1_阶段3"

  # 本章核心要素
  chapter_goal: "林夜带苏琳去废土高地看星空"  # 章目标
  chapter_conflict: "苏琳担心预言会成真，不敢靠近"  # 冲突点
  chapter_climax: "林夜握住苏琳的手，说'那颗星会爆炸，但那就够了'"  # 高潮点

  # 角色状态（从RAG获取）
  protagonist_state: "复仇心渐淡，开始思考守护的意义"
  deuteragonist_state: "预言让她恐惧，但她选择相信林夜"

  # 关键情节点（3-5个）
  plot_points:
    - "林夜发现废土高地的星空很美"
    - "苏琳开始讲述她的预言"
    - "两人发生小争执（关于未来）"
    - "和好后并肩看星空"
    - "林夜说出守护的理由"

  # 本章伏笔埋设
  foreshadowing:
    - "苏琳提到'你可能走向的道路'"
    - "小九在远处观察，表情复杂"

  # 本章收尾钩子
  chapter_hook: "苏琳的预言画面中出现了一个她从未见过的场景"

  # 目标字数
  target_word_count: 3000

  # 风格指导
  style_notes: "细腻描写型，强调情感铺垫，对话占比30%"

  # 扩展后的正文路径
  expanded_body_path: null
  expanded_status: "pending"
```

## 五、章纲生成流程

### 5.1 流程图

```
阶段大纲（ch001-ch005 的详细指导）
       │
       ▼
┌──────────────────────────────────────────┐
│           章纲生成器                      │
│                                          │
│  输入：阶段大纲 + 前几章已写内容           │
│  处理：                                   │
│    1. 拆分阶段大纲为每章节纲               │
│    2. 补充角色状态（从RAG系统获取）         │
│    3. 检查伏笔埋设机会                      │
│    4. 生成章节钩子                         │
│  输出：每章 200 字的章纲                   │
└──────────────────────────────────────────┘
       │
       ▼
   章纲（ch001-ch005 各一份）
       │
       ▼
┌──────────────────────────────────────────┐
│           正文扩展器                       │
│                                          │
│  输入：章纲（约200字）                    │
│  处理：                                   │
│    1. 分析章纲的关键情节点                 │
│    2. 调用 Writer Persona（默认：叙事经典型）│
│    3. 生成 3000 字正文                    │
│    4. 自检：情节完整性、角色一致性          │
│  输出：3000字正文初稿                     │
└──────────────────────────────────────────┘
```

### 5.2 章纲生成规则

| 规则 | 说明 |
|------|------|
| 从阶段大纲继承节概 | 直接复用阶段大纲中的"ch001-005"描述 |
| 从RAG获取角色状态 | 查询角色当前状态（位置/形态/情绪） |
| 检查伏笔时机 | 如果阶段大纲中提到某伏笔应在chN埋设，章纲中体现 |
| 生成章节钩子 | 章纲结尾必须留悬念，与下章开头呼应 |
| 200字限制 | 章纲必须简洁，作家可在生成后手动调整 |

### 5.3 章纲生成提示词模板

```yaml
chapter_outline_generation_prompt: |
  你是一位小说大纲专家，正在从阶段大纲生成每章节纲。

  阶段大纲：
  {stage_outline}

  当前要生成的章节：ch{chapter_num}

  请生成章纲，要求：
  1. 约200字，简洁有力
  2. 包含：章节目标、冲突点、高潮点、3-5个关键情节点
  3. 结尾留悬念钩子
  4. 角色状态从RAG系统获取

  输出格式：YAML
```

## 六、正文扩展器

### 6.1 扩展策略

```yaml
expansion_strategy:
  # 基础扩展（200字 → 3000字，约15倍）

  # 1. 情节细化
  每个plot_point:
    - 扩展为 500-600 字
    - 增加场景描写（环境/氛围）
    - 增加角色心理描写
    - 增加对话细节

  # 2. 章节结构
  - 开头（10%）：场景建立 + 角色出场
  - 发展（50%）：3-4个情节点逐步推进
  - 高潮（20%）：核心冲突爆发
  - 结尾（20%）：钩子 + 下章铺垫

  # 3. 风格注入
  使用 Writer Persona（默认：叙事经典型）
  - 动作/对话/描写比例：4:3:3
  - 融入修真术语
  - 章节结尾留情绪钩子
```

### 6.2 扩展示例

**输入（章纲）**：
```yaml
chapter_goal: "林夜带苏琳去废土高地看星空"
chapter_conflict: "苏琳担心预言会成真，不敢靠近"
chapter_climax: "林夜握住苏琳的手，说'那颗星会爆炸，但那就够了'"
plot_points:
  - "林夜发现废土高地的星空很美"
  - "苏琳开始讲述她的预言"
  - "两人发生小争执（关于未来）"
  - "和好后并肩看星空"
  - "林夜说出守护的理由"
chapter_hook: "苏琳的预言画面中出现了一个她从未见过的场景"
```

**输出（扩展后正文，约500字）**：
```
废土高地的夜风很冷，但林夜觉得值得。

他第一次带苏琳来这里，没有别的原因，只是觉得她应该看看这片星空。

"那颗。"苏琳指向远处，"那颗星会在明年爆炸。"

林夜没有说话。他抬头看着那片光点，突然觉得它们很像苏琳的眼睛——那么亮，那么远，又那么近。

"你觉得吗？"苏琳轻声问。

"什么？"

"如果我预知了一切，结局还是不会改变呢？"

林夜沉默了很久。风把他的头发吹乱了，他没有去理。

"那就够了。"

苏琳转过头看他。他没有看她，只是盯着那颗即将爆炸的星。

"值得吗？"她问。

"值得。"他说，"因为那是你。"

风停了。苏琳没有再问。
```

## 七、工作模式

### 7.1 三种模式

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| **模式A（全自动）** | 章纲生成 → 正文扩展 → 作家审阅修改 | 快速生成初稿 |
| **模式B（半自动）** | 章纲生成 → 作家编辑章纲 → 正文扩展 → 作家修改 | 作家需要控制方向 |
| **模式C（手动）** | 作家直接写，不使用章纲扩展 | 作家风格强烈 |

### 7.2 默认模式

默认使用**模式B（半自动）**，作家可以：
- 审阅生成的章纲，调整方向
- 确认章纲后触发扩展
- 扩展后继续编辑修改

## 八、与现有系统集成

### 8.1 与作家工作室的关系

```
现有：作家写 ch051 → 直接输出
新增：作家写 ch051 → 先看章纲 → 有章纲则扩展，无章纲则先生成
```

### 8.2 与RAG系统的关系

```
章纲生成时 → 调用RAG获取：
  - 前几章的情节摘要
  - 当前角色状态
  - 待回收伏笔
  - 时间线信息

正文扩展时 → 调用RAG获取：
  - 角色语音档案
  - 设定约束（如修真境界体系）
  - 前文关键细节
```

### 8.3 与质量门禁系统的关系

```
章纲扩展后的正文 → 进入质量门禁系统（方向2）

  - 变体生成（3个风格）
  - 硬性验证器
  - 软性评分器
  - 质量分级（Bronze/Silver/Gold/Platinum）
```

### 8.4 与阶段大纲的关系

```
阶段大纲（ch001-ch005）→ 章纲生成器 → 章纲（ch001, ch002, ch003, ch004, ch005）

阶段大纲变更 → 章纲重新生成（需确认是否覆盖已有正文）
```

## 十、实施步骤

### 阶段1（1-2周）：章纲生成器

1. 定义章纲数据结构（YAML格式）
2. 实现从阶段大纲拆分为章纲的逻辑
3. 调用RAG获取角色状态
4. 实现伏笔检查（确保章纲中有伏笔埋设点）
5. 实现章节钩子生成
6. 验证：章纲能正确反映阶段大纲的意图

### 阶段2（2-3周）：正文扩展器

7. 实现正文扩展引擎（200字 → 3000字）
8. 实现 Writer Persona 集成（默认：叙事经典型）
9. 实现章节结构检测（开头/发展/高潮/结尾）
10. 实现自检：情节完整性、角色一致性
11. 验证：扩展后的正文符合章纲要求

### 阶段3（2-3周）：集成与优化

12. 与作家工作室流程集成
13. 与RAG系统集成（RAG记忆系统方向1）
14. 与质量门禁系统集成（方向2）
15. 提供手动编辑章纲的界面
16. 提供三种模式切换（A/B/C）
17. 性能优化：扩展等待时间 < 60秒

### 阶段4（持续）：优化

- 根据作家反馈调整扩展策略
- 丰富 Writer Persona 库
- 优化伏笔埋设逻辑
- 支持更多风格变体

## 十二、关键设计决策

| 决策 | 说明 |
|------|------|
| 章纲格式为YAML | 结构化、可编辑、人类可读 |
| 默认半自动模式 | 作家审阅章纲后再扩展，平衡速度和质量 |
| 扩展使用叙事经典型 | 适合玄幻修真类型，动作/对话/描写均衡 |
| 章纲可手动编辑 | 作家可以调整章纲方向后再扩展 |
| 阶段大纲变更需确认 | 避免覆盖已有正文，作家需确认是否重写 |
| 与RAG系统深度集成 | 章纲生成时获取角色状态，正文扩展时获取设定约束 |
