## Context

Phase 7 (commit 671db78) 引入 `tests/agent_system/test_master_controller_e2e_real_llm.py` (540L, 14 tests) 名字含 "real_llm" 但实际用 `StubProvider` 注入 mock router 跑生产路径 (`MasterController.run_workflow` → `AgentComputeFn` → `Router` → master 方法) 而非真实 LLM.

Phase 8 (commit d6e7c25) 引入真 LLM 测试 `test_novel_writing_real_llm.py` 后, 命名冲突明显:

- 旧 `test_master_controller_e2e_real_llm.py` = StubProvider 测生产路径
- 新 `test_novel_writing_real_llm.py` = 真实 Anthropic API

旧 "real_llm" 指 StubProvider 实现 LLM 形接口, 新 "real_llm" 指真 Anthropic API. 名字一眼看不出"哪个是 stub 哪个是真".

**目标**: 把旧文件重命名为 `test_master_controller_stub_router_e2e.py`, 一眼明确"用 stub router 跑 E2E". 跟 `_e2e_helpers.make_stub_router()` 函数名直接呼应.

## Goals

1. 消除命名混淆, 1 个 git mv + N 个 sed 解决
2. 保留 `git log --follow` history (用 `git mv` 不用 `rm + add`)

## Non-Goals (YAGNI)

- ❌ 重构 14 tests 内容
- ❌ 改任何断言 / 任何 production code
- ❌ 拆 MEMORY.md 到 phases.md topic file (已知问题, 留待后续)
- ❌ 改 spec/plan doc 行号引用 (历史 docs 行号已不准, 不在本期范围)
- ❌ 重命名 `_e2e_helpers.py` 或 `test_novel_writing_real_llm.py` (无命名冲突)

## Design

### File rename: `git mv` (1 文件)

```bash
git mv tests/agent_system/test_master_controller_e2e_real_llm.py \
       tests/agent_system/test_master_controller_stub_router_e2e.py
```

**为什么保留 `e2e` 后缀 + pytest `test_` 前缀**: pytest 约定. 改后缀破坏发现.
**为什么用 `stub_router` 而非 `stub_llm`**: 文件是测 `MasterController` + `AIRouter` 集成, router 是注入点, router 是关键词.

### Active code docstrings (3 文件, 4 行)

| File | Line | Reason |
|------|------|--------|
| `tests/agent_system/_e2e_helpers.py` | 4 | docstring 引用同目录旧文件名 |
| `tests/agent_system/test_novel_writing_real_llm.py` | 9 | docstring 区分两个文件 |
| `tests/agent_system/test_novel_writing_real_llm.py` | 10 | docstring 区分两个文件 |
| `infra/got/workflows/minimal_e2e.yaml` | 8 | 注释引用旧 docstring 位置 |

### Historical spec/plan docs (18 文件, sed 机械化)

`docs/superpowers/{specs,plans}/2026-06-0{4,5}-phase{7,8}*.md` 共 18 文件提及旧名. 全部用 `sed -i 's|test_master_controller_e2e_real_llm|test_master_controller_stub_router_e2e|g'` 改.

**不改行号引用** (`:NNN`): 历史 docs 行号已不准, 改了反而误导.

### MEMORY.md (本机文件, 不在 git commit)

`/home/ailearn/.claude/projects/.../memory/MEMORY.md` 8 entries 提及旧名. `sed -i` 改.

## Risks + Mitigations

| Risk | Mitigation |
|------|------------|
| 误改测试代码 → 测试 fail | 0 测试代码改动是硬约束, `git mv` + `sed` 仅改文件名 |
| 历史 docs 行号过时 → 改行号反而误导 | 本期**不**改 `:NNN` 行号引用 |
| MEMORY.md 8 entries 拆分仍是已知问题 | 本期仅改文件名, 拆分留待 phases.md topic file |
| `git rm + add` 丢失 `git log --follow` | 用 `git mv` 保留完整 file history |
| Pre-commit hook 拦截 | 不加 `--no-verify`, 改的是文件名应 hook 不会拦 |
