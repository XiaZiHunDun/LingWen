## 4. Design

### 4.1 MC variant pattern (5 variants, 5 `_impl_*(record_usage)` 私有)

跟 Phase 8.6.1 agent pattern 一致: public method 委托给 `_impl(record_usage=True)`, 旧 method 走 `_impl(record_usage=False)`.

```python
# master_controller.py:533-588 (write_chapter region)

def write_chapter(
    self, chapter_num, outline, characters, memory_context, style_guide, use_llm=True,
) -> Dict:
    """旧 method 签名 0 改 — 内部调 _impl(record_usage=False)."""
    return self._impl_write_chapter(
        chapter_num, outline, characters, memory_context, style_guide, use_llm,
        record_usage=False,
    )

def write_chapter_with_usage(
    self, chapter_num, outline, characters, memory_context, style_guide, use_llm=True,
) -> tuple[Dict, Dict[str, int]]:
    """Phase 8.6.2: 真实 usage variant — 调 content_writer.generate_chapter_with_usage()."""
    return self._impl_write_chapter(
        chapter_num, outline, characters, memory_context, style_guide, use_llm,
        record_usage=True,
    )

def _impl_write_chapter(
    self, chapter_num, outline, characters, memory_context, style_guide, use_llm, record_usage,
):
    chapter_outline = self.outline_master.schema.get_chapter_outline(outline, chapter_num)
    context = self.context_builder.build_writing_context(...)
    suggestions = self.writing_suggestion.generate_suggestions(...)
    if use_llm:
        if record_usage:
            result, usage = self.content_writer.generate_chapter_with_usage(chapter_num, context)
            return {
                "content": result["content"],
                "word_count": result.get("word_count", len(result["content"])),
                "suggestions": suggestions,
                "context": context,
            }, usage
        result = self.content_writer.generate_chapter(chapter_num, context)
        return {
            "content": result["content"],
            "word_count": result.get("word_count", len(result["content"])),
            "suggestions": suggestions,
            "context": context,
        }
    else:
        # 调试 path 仍返 prompt dict, 无 LLM 调用
        prompt = self.content_writer.build_writing_prompt(context)
        return {"prompt": prompt, "suggestions": suggestions, "context": context}
```

**5 variants signatures** (跟旧 method 签名 1:1 对齐, 加 `_with_usage` 后缀):
| 旧 method | 新 variant | Tuple return |
|-----------|------------|--------------|
| `write_chapter(chapter_num, outline, characters, memory_context, style_guide, use_llm=True)` | `write_chapter_with_usage(...)` | `(Dict, Dict)` |
| `audit_chapter(chapter_num, content, characters, timeline, use_llm=True)` | `audit_chapter_with_usage(...)` | `(Dict, Dict)` |
| `polish_chapter(content)` | `polish_chapter_with_usage(chapter_num, content, style_guide=None)` | `(str, Dict)` (注: polish_chapter 返回 str 而非 dict, variant 也返 str) |
| `polish_emotional_pacing(content)` | `polish_emotional_pacing_with_usage(chapter_num, content)` | `(str, Dict)` (注: 2 LLM sum) |
| `polish_ai_trace_removal(content)` | `polish_ai_trace_removal_with_usage(chapter_num, content)` | `(str, Dict)` (注: 1 LLM + 1 rule) |

**Polish 签名注**: 旧 `polish_chapter(content)` / `polish_emotional_pacing(content)` / `polish_ai_trace_removal(content)` 都只接 `content`; 新 variants 加 `chapter_num=0` 参数 (handler 工厂传 0, 跟 Phase 7.4 polish 节点语义一致 — polish 是"内容改写"非"章节特定"). Variant 签名扩展而非完全保持, 旧 method 签名 0 改.

### 4.2 Handler tuple return (4 handlers 改)

```python
# got_bridge.py:59-78 (_handler_chapter_writing)

def _handler_chapter_writing(
    master: MasterController, inputs: dict[str, Any],
) -> tuple[dict[str, Any], dict[str, int]]:
    """Phase 8.6.2: 调 master.write_chapter_with_usage 返 tuple."""
    chapter_num = int(_resolve_field(inputs, "chapter_num", 0))
    if not chapter_num:
        return {"_error": "chapter_num is required for chapter_writing"}, {"input_tokens": 0, "output_tokens": 0}
    result, usage = master.write_chapter_with_usage(
        chapter_num=chapter_num,
        outline=_resolve_field(inputs, "outline", {}),
        characters=_resolve_field(inputs, "characters", []),
        memory_context=_resolve_field(inputs, "memory_context", {}),
        style_guide=_resolve_field(inputs, "style_guide", {}),
        use_llm=bool(_resolve_field(inputs, "use_llm", True)),
    )
    if isinstance(result, dict):
        result.setdefault("chapter_num", chapter_num)
    return result, usage
```

**`_make_polish_handler` 工厂** 改调 variants (line 103-117):
```python
def _make_polish_handler(method_name: str) -> HandlerFn:
    """Phase 7.4 factory: 通过 method_name 路由到 MC method.
    Phase 8.6.2: 优先调 *_with_usage variant, return (output, usage) tuple.
    Backward compat: 旧 MC 无 variant → 调原 method wrap tuple.
    """
    def handler(master: MasterController, inputs: dict[str, Any]) -> tuple[dict, dict]:
        content = _resolve_field(inputs, "content", "")
        if not content:
            return {"_error": "content is required for polish scenarios"}, {"input_tokens": 0, "output_tokens": 0}
        # 优先调 *_with_usage variant
        with_usage_method = f"{method_name}_with_usage"
        method = getattr(master, with_usage_method, None)
        if method is not None:
            return method(chapter_num=0, content=content)
        # Backward compat: 旧 MC 无 variant, 调原 method
        original = getattr(master, method_name)
        result = original(content)
        if isinstance(result, dict):
            return result, {"input_tokens": 0, "output_tokens": 0}
        return {"content": result}, {"input_tokens": 0, "output_tokens": 0}
    return handler
```

**`_handler_polish_merge`** 0 改 (out of scope, polish_merge_synthesis 留 Phase 8.7).

### 4.3 AgentComputeFn tuple detection (双路径)

```python
# got_bridge.py:223-266 (__call__ cost hot path)

# 替换原 line 255-264
if isinstance(output, tuple):
    output, usage = output
    cost_tokens = 0
    if self._cost_tracker is not None:
        tier = SCENARIO_TIER_MAP.get(scenario, ModelTier.SONNET)
        in_tok = usage.get("input_tokens", 0)
        out_tok = usage.get("output_tokens", 0)
        self._cost_tracker.record(scenario, tier, in_tok, out_tok)
        cost_tokens = in_tok + out_tok
else:
    # Backward compat: dict return → 走 len()//4 估算
    cost_tokens = 0
    if self._cost_tracker is not None:
        in_tok = len(str(inputs)) // 4
        out_tok = len(str(output)) // 4
        tier = SCENARIO_TIER_MAP.get(scenario, ModelTier.SONNET)
        self._cost_tracker.record(scenario, tier, in_tok, out_tok)
        cost_tokens = in_tok + out_tok
```

**关键安全**: `isinstance(output, tuple)` 严格检测, length 不强制 (因 `(str, dict)` for polish 跟 `(dict, dict)` for write/audit 都是 2-tuple). 万一 handler 返 1-tuple 或 3-tuple, 抛 ValueError, 但保留 dict path 作为兜底.

## 5. Risks + mitigations

| 风险 | 缓解 |
|------|------|
| 旧 2129 测试破 (MC variants 必传 / tuple 必返) | `record_usage` kwarg default False 兜底, 双路径 (tuple 走真实, dict 走估算) 保 backward compat |
| type alias Union mypy/pyright 错 | 显式 `isinstance(output, tuple)` 检测 + `Union[dict, tuple[dict, dict]]` type hint |
| e2e stub 不支持 record_usage | `_RecordingRouter` 复用 Phase 8.6.1, `_e2e_helpers.make_master_with_router` 注入 |
| `_StubMaster` 漏 `*_with_usage` method | 显式加 5 methods (max len fallback), 保 test_got_bridge 旧 tests 通过 |
| AgentComputeFn tuple 检测漏分支 (e.g. list 返) | 严格 `isinstance(output, tuple)` + `.get(key, 0)`, 防 KeyError |
| cost_tracker.record() 拿到 0 token 误报 | usage dict 拿 `.get(key, 0)`, 防 KeyError |
| ComputeResult.cost_tokens 类型变化 (int → int) | int sum 永远 int, 0 type break, GoTScheduler 不依赖具体值 |
| `_RecordingRouter` 不替换 agent._router | `_e2e_helpers.make_master_with_router` 显式赋值 3 处 (content_writer/auditor/polisher) |
| `master_controller._impl_*` method 漏 `record_usage` kwarg | 测试覆盖 5 个 _impl_* method 通过 *_with_usage 间接验证 |
| Phase 8.6.2 entry 让 MEMORY.md 超 205 行 | 接受再超 1-5 行 (205-210), 留 followup phase 拆 phases.md topic |
| API key 泄漏 | 0 provider 改, 0 .env 改, 0 ANTHROPIC_API_KEY 引用, 跟 Phase 8.5/8.6/8.6.1 一致 |
| 1 旧 handler 漏调 MC variant | 4 handlers 显式 list (chapter_writing/chapter_review/2 polish), `_make_polish_handler` 工厂统一 |
| `_make_polish_handler` 工厂拼 method_name 出错 | `getattr(master, with_usage_method, None)` + None check + 原 method fallback, 防 AttributeError |
| polish method 签名变化 (加 chapter_num) | 旧 `polish_X(content)` 签名 0 改, 新 `polish_X_with_usage(chapter_num, content)` 加 chapter_num 跟 Phase 7.4 handler 工厂语义一致 |
