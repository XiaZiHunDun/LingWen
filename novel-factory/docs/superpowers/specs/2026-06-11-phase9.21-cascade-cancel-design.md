> **Tech Stack:** Python 3.12 + sqlite3 + argparse (subparser group) + FastAPI + Pydantic (0 改既有 schema) + pytest 8 + ruff.

## Design

### 1. Storage: `cancel_cascade_run`

```python
# infra/cross_volume/storage.py: 加在 record_cascade_run 后 (跟 record_audit 1:1 风格)

def cancel_cascade_run(self, run_id: int, reason: str = "") -> bool:
    """Phase 9.21: mark cascade run as cancelled.

    Idempotent: 已 cancelled 返 False (no-op). completed/cancelled/failed
    → cancelled 都允许 (permissive transition, YAGNI 状态机校验).

    Reuses completed_at for cancel timestamp (0 加列, 跟 Phase 9.20 YAGNI 决策一致).
    reason 仅 INFO log, 不持久化 (跟 dashboard/cascade_notifier cascade.cancel WS
    payload 透传 reason 给 dashboard UI 显示).

    Args:
        run_id: cascade_runs.id PK
        reason: optional cancel reason (logged + WS payload, not persisted)

    Returns:
        True if status flipped (was running/completed/failed → cancelled)
        False if already cancelled (idempotent no-op)
    """
    now_iso = datetime.now(timezone.utc).isoformat()
    with self._connect() as conn:
        row = conn.execute(
            "SELECT status FROM cascade_runs WHERE id = ?", (run_id,)
        ).fetchone()
        if row is None:
            raise KeyError(f"Cascade run {run_id} not found")
        if row["status"] == "cancelled":
            return False  # idempotent no-op
        conn.execute(
            "UPDATE cascade_runs SET status = 'cancelled', completed_at = ? WHERE id = ?",
            (now_iso, run_id),
        )
        conn.commit()
        return True
```

**Status transition matrix** (permissive, YAGNI):
- `running` → `cancelled`: ✅ return True
- `completed` → `cancelled`: ✅ return True (permissive, 用户可事后 mark 误判)
- `failed` → `cancelled`: ✅ return True (permissive)
- `cancelled` → `cancelled`: ❌ return False (idempotent no-op)
- 不存在 (run_id): raise KeyError → 404 (endpoint / CLI 各自处理)

### 3. Endpoint: `POST /api/ripples/cascade/{ripple_id}/runs/{run_id}/cancel`

```python
# dashboard/app.py: 加 endpoint (跟 get_ripple_cascade_runs 1:1 风格)

class CascadeCancelRequest(BaseModel):
    """Phase 9.21: cancel request body (optional reason)."""
    reason: str = ""


@app.post(
    "/api/ripples/cascade/{ripple_id}/runs/{run_id}/cancel",
    response_model=CascadeRunResponse,
)
def post_ripple_cascade_run_cancel(
    ripple_id: str,
    run_id: int,
    body: CascadeCancelRequest = CascadeCancelRequest(),
) -> CascadeRunResponse:
    """Phase 9.21: cancel a persisted cascade run.

    Returns:
        updated CascadeRunResponse (status='cancelled', completed_at = cancel time)

    Raises:
        404: cascade run not found
        200: cancelled (status flipped) OR already cancelled (idempotent, no-op)
    """
    storage = _default_storage()
    try:
        flipped = storage.cancel_cascade_run(run_id, reason=body.reason)
    except KeyError:
        raise HTTPException(404, f"Cascade run {run_id} not found")
    run = storage.get_cascade_run_by_id(run_id)
    # Notify WS (best-effort, 跟 notify_cascade_update 1:1)
    if flipped:
        notify_cascade_cancel(CascadeCancelPayload(
            run_id=run_id,
            ripple_id=ripple_id,
            reason=body.reason,
        ))
    return CascadeRunResponse.from_dataclass(run)
```

**行为契约**:
- `run_id` 存在 → 翻 status (or no-op) → 200 + updated CascadeRunResponse + WS push (if flipped)
- `run_id` 不存在 → 404
- 已 cancelled → 200 + CascadeRunResponse (status='cancelled', completed_at=初次 cancel 时间) + **0 WS push** (avoid duplicate event)
