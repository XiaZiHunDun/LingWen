## 4. Design

### 4.1 Layer 1: Provider (`base.py` + 3 overrides)

**base.py 新增** (concrete default impl, 非 abstract — 让旧 subclass 不强制实现):
```python
def generate_with_usage(
    self, prompt: str, **kwargs
) -> tuple[str, dict[str, int]]:
    """生成文本 + 返回 usage dict

    Default impl: 调 self.generate() + len()//4 估算 (跟 Phase 8.5 一致)
    Subclass 可重写以返回 SDK 原生 usage (real input_tokens / output_tokens)

    Args:
        prompt: 输入提示
        **kwargs: 传给 generate() 的额外参数

    Returns:
        (text, usage) 元组, usage 含 "input_tokens" / "output_tokens" keys

    Raises:
        AIProviderError: 当 generate() 失败
    """
    text = self.generate(prompt, **kwargs)
    return text, {
        "input_tokens": len(prompt) // 4,
        "output_tokens": len(text) // 4,
    }
```

**anthropic_provider.py override** (4-5 行, 跟 generate() 同 retry 模式):
```python
def generate_with_usage(self, prompt, **kwargs):
    model = kwargs.pop("model", self._model)
    system = kwargs.pop("system", None)
    temperature = kwargs.pop("temperature", 0.7)
    max_tokens = kwargs.pop("max_tokens", 4096)
    messages = []
    if system:
        messages.append({"role": "assistant", "content": system})
    messages.append({"role": "user", "content": prompt})

    last_error = None
    for attempt in range(self.config.max_retries):
        try:
            response = self._client.messages.create(
                model=model, system=system, messages=messages,
                temperature=temperature, max_tokens=max_tokens, **kwargs
            )
            for block in response.content:
                if hasattr(block, 'text') and block.text:
                    text = block.text
                    break
            else:
                text = (response.content[0].text
                        if hasattr(response.content[0], 'text')
                        else str(response.content[0]))
            usage = {
                "input_tokens": response.usage.input_tokens,
                "output_tokens": response.usage.output_tokens,
            }
            return text, usage
        except anthropic.APITimeoutError:
            last_error = TimeoutError(...)
            # ... 同 generate() retry 模式
    raise last_error or AIProviderError("Max retries exceeded")
```

**openai_provider.py override** (类似, `prompt_tokens` / `completion_tokens`):
```python

    "input_tokens": response.usage.prompt_tokens,
    "output_tokens": response.usage.completion_tokens,

```

**minimax_provider.py override** (M2.7 走 Anthropic 兼容, `response.usage.input_tokens/output_tokens`):

### 4.2 Layer 2: Router (`tiered_router.py` 新增)

```python
def generate_with_usage(
    self, scenario: str, prompt: str, **kwargs
) -> tuple[str, dict[str, int]]:
    """按 scenario 路由 + 失败时降级 + 返回 real usage

    跟 generate() 区别:
    - 返回 tuple[str, dict] (real usage)
    - **不调 _notify_tracker** (AgentComputeFn 在上层记录, 防双计数)
    """
    primary_tier, _ = self.route(scenario)
    downgrade_chain = self._build_downgrade_chain(primary_tier)
    last_error: Optional[Exception] = None

    for tier in downgrade_chain:
        provider = self._providers[tier]
        try:
            text, usage = provider.generate_with_usage(prompt, **kwargs)
            return text, usage
        except Exception as exc:
            last_error = exc
            if self.disable_downgrade:
                break
            continue

    raise TieredRouterError(
        f"all tiers failed for scenario {scenario!r}: {last_error}"
    )
```

**注意**: 旧 `generate() -> str` 保持不变 (含 `_notify_tracker`); 新 `generate_with_usage() -> tuple[str, dict]` 跳过 notify (AgentComputeFn 负责).

### 4.3 Layer 3: AgentBase (`agents/base.py` 新增)

```python
def chat_with_usage(
    self,
    prompt: str,
    system: Optional[str] = None,
    model: Optional[str] = None,
    temperature: Optional[float] = None,
    max_tokens: Optional[int] = None,
    **kwargs,
) -> tuple[str, dict[str, int]]:
    """chat() + usage 透传

    限制: 需 router 具备 generate_with_usage() 方法 (TieredRouter ✓, AIRouter ✗)
    fallback: router 无新方法 → 调 self.chat() + len()//4 估算 (跟 Phase 8.5 一致)

    Args:
        (同 chat())

    Returns:
        (text, usage) 元组, usage dict 含 "input_tokens"/"output_tokens" keys
    """
    if self._fallback_mode:
        text = self._fallback_response(prompt, **kwargs)
        return text, {
            "input_tokens": len(prompt) // 4,
            "output_tokens": len(text) // 4,
        }

    if not hasattr(self._router, "generate_with_usage"):
        # 兼容旧 AIRouter (无新方法) → 走 chat() + 估算
        text = self.chat(prompt, system=system, model=model,
                        temperature=temperature, max_tokens=max_tokens, **kwargs)
        return text, {
            "input_tokens": len(prompt) // 4,
            "output_tokens": len(text) // 4,
        }

    return self._router.generate_with_usage(
        prompt=prompt, system=system,
        model=model or self.default_model,
        temperature=temperature or self.default_temperature,
        max_tokens=max_tokens or self.default_max_tokens,
        **kwargs,
    )
```

**设计点**: `hasattr` 防御性 fallback 是关键 — 让 AgentBase.chat_with_usage 对 TieredRouter AND AIRouter 都工作, 不破旧 path.

    跟 TieredRouter.generate_with_usage 区别: 接受任意 router (TieredRouter
    或 AIRouter), 若 router 无新方法 → 走 len()//4 估算 fallback.

### 4.7 Error 契约

- Provider 抛 `AIProviderError` → 透传到 TieredRouter / AgentBase / Master (无 try/except 包装)
- TieredRouter 抛 `TieredRouterError` (所有 tier 失败) → 透传
- AgentBase 调 `_fallback_response` 在 fallback mode (router is None) → 走 len()//4 估算, 不抛
- Master.chat_with_usage 走 hasattr fallback (router 无新方法) → 走 len()//4 估算, 不抛

**韧性契约** (跟 Phase 7.2/8.1 一致):
- 任何 layer 失败 → 上层 try/except 兜底 (调用方决定)
- `generate_with_usage` 不静默吞错, 失败透传给调用方

## 6. 关键设计决策

### 1. 加 default impl vs abstract method
- **选 default impl** (concrete base class method) — 旧 StubProvider / FakeProvider 不强制实现, 走 len()//4 fallback
- **不选 abstract** — 强制所有 subclass 实现, 12+ existing test stubs 全破
- **不选 mixin** — 过度设计

### 2. dict vs dataclass (UsageRecord)
- **选 dict[str, int]** (用户决策) — 跟 Anthropic/OpenAI SDK usage field 形态一致, 灵活扩展
- **不选 dataclass** — 字段固定, 加 cache_creation 要改 dataclass
- **不选 NamedTuple** — 不可变, 不能 default

### 3. TieredRouter.generate_with_usage 是否调 _notify_tracker
- **不调** — 避免双计数 (AgentComputeFn 在上层记录)
- **理由**: Phase 8.5 CostTracker 注入 AgentComputeFn, 不注入 tiered_router (per Phase 8.5 "0 改 tiered_router"); 若双注入, 同一 cost_tracker 写 2 次 → double cost

### 4. AgentBase.chat_with_usage fallback 策略
- **选 hasattr fallback** (TieredRouter OR AIRouter 都支持) — graceful degradation
- **不选强制 TypeError** — 8 外部 production site 全破
- **不选仅支持 TieredRouter** — AgentBase 是通用基类, 留兼容

### 5. MasterController.chat_with_usage scope
- **选 1 new public method** (scenario, prompt → tuple) — minimal 公共 API
- **不选 8 个 _with_usage 变体** (write_chapter_with_usage / audit_chapter_with_usage / polish_*_with_usage) — 8 method 模板化, scope 爆炸, 留 followup
- **不选 0 新方法** (改 AgentComputeFn 单独 bypass) — 缺复用入口, Phase 8.7+ dashboard 重复实现

### 6. AgentComputeFn 改不改
- **选 0 改** — 保持 Phase 8.5 行为, cost_tracker 继续 len()//4 估算
- **不选 1 行加 output.get("usage")** — 本 phase 0 handler 返回 "usage", 新路径 0 触发, 等于 dead code
- **不选完整重写** — 跟 Phase 8.5 重复算, 跟本 phase "0 production wire" 决策冲突

### 7. 8 外部 production site 0 改
- **选 0 改** (用户决策 "GoT path only") — consistency checkers / llm_service / character_repairer / agents/base.py 保持现状
- **不选改全 8 site** — 范围爆炸, 各有 bespoke 错误处理, 大概率 2-3 fixup commits
- **不选渐进迁移** — YAGNI 阶段切分, 留 followup
