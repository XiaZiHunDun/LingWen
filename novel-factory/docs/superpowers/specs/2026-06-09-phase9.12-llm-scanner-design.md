**关键设计原则**:
- 0 改 Phase 9.10/9.11 现有 source API (Q1 Opt-in 保 baseline)
- LLMScanner / EdgeInferrer 是 stateless function class, 只读 chapter, 输出 nodes/edges

### 4.2 Components (5 module 详细 API)

#### Component 1: `LLMScanner` (infra/cross_volume/llm_scanner.py, ~220L)

```python
class LLMScanner:
    """Phase 9.12: 4 维 LLM scanner, 从单章节抽 ReferenceNode 候选.

    - Opt-in: 只在 --use-llm flag 触发
    - 串行调 LLM (Q4), 单次失败 retry 2 次 + 指数退避 (Q4)
    - Cache 命中跳 LLM (Q5)
    - 最后失败回落 Phase 9.11 4 Extractor (Q4)
    """
    def __init__(
        self,
        router: TieredRouter,        # Phase 8.5 真实 router
        cache: LLMCache,             # 复用 cache
        fallback_backfiller: Phase911Backfiller,  # 失败回落
        cost_tracker: CostTracker,   # Phase 8.5+ 真实 cost
        model_tier: ModelTier = ModelTier.SONNET,  # default
    ): ...

    def scan_chapter(
        self,
        chapter_id: int,
        chapter_content: str,
        context: dict,               # {volume_id, characters, prev_chapter_summary, ...}
    ) -> list[ReferenceNode]:       # 4 dim nodes (含 confidence 1-5)
        # 1. 4 dim 串行调 LLM (Q4)
        # 2. cache check (Q5)
        # 3. 解析 JSON → ReferenceNode (confidence 字段从 LLM 1-5 透传)
        # 4. 失败 retry + 回落 Phase 9.11 Extractor (Q4)
        ...
```

**输入**: `(chapter_id, content, context)`
**输出**: `list[ReferenceNode]` — 4 维各 dim 的 node, 字段含 `confidence: int` (1-5)
**依赖**: `TieredRouter`, `LLMCache`, `Phase911Backfiller` (回落), 4 prompt files

#### Component 3: `LLMCache` (infra/cross_volume/llm_cache.py, ~80L)

```python
class LLMCache:
    """Phase 9.12: 内存 + JSON 落盘 cache.

    - key = SHA1(chapter_content + prompt_id + model_id) (Q5)
    - value = JSON {"input_tokens": int, "output_tokens": int, "parsed": dict}
    - 内存 dict (L1) + JSON file (L2 持久化, 默认 ~/.cache/lingwen/llm_cache.json)
    - 命中直接返, miss 调 LLM 后 write-through
    """
    def __init__(self, cache_path: Path = None):  # None → default
        self._mem: dict[str, dict] = {}
        self._path = cache_path or Path.home() / ".cache/lingwen/llm_cache.json"
        self._load()  # 启动时 load

    def get(self, key: str) -> dict | None: ...
    def put(self, key: str, value: dict) -> None: ...  # write-through
    @staticmethod
    def make_key(chapter_content: str, prompt_id: str, model_id: str) -> str: ...

    def _load(self) -> None: ...
    def _save(self) -> None: ...
```

**输入**: `(chapter_content, prompt_id, model_id)`
**输出**: 命中 `dict` / miss `None`
**依赖**: stdlib `hashlib` + `json` + `pathlib` (无新依赖)

### 4.3 Data Flow (端到端 run-time flow)

#### 主流程: `lingwen.py ripple backfill --use-llm --apply`

```
[CLI] lingwen.py ripple backfill --use-llm --apply --cache-path /tmp/llm.json
   │
   ▼
[Phase 9.11 Backfiller.run()] — 现有入口, 加 3 行 opt-in 分支
   │
   ├── if not --use-llm: 走 rule-based (Phase 9.11 path, 0 改)
   │     └→ [4 Extractor] → atomic_batch commit → 返 Done
   │
   └── if --use-llm: 进入 LLM 路径
         │
         ▼
   [NEW] LLMScannerOrchestrator.run() (写进 backfill orchestrator, 1 个新 method)
         │
         │ for chapter in chapters: (串行, Q4 守 RPS)
         ▼
   ┌─────────────────────────────────────────────────┐
   │ 1. cache_key = SHA1(content + prompt_id + model)│
   │ 2. if cache.get(key) → 解析 nodes, 跳 step 3-7 │
   │ 3. retry 2 times:                                │
   │      LLM call (LLMScanner.scan_chapter)           │
   │      if success → cache.put, parse → step 4     │
   │      if fail → wait 2^n seconds, retry           │
   │ 4. if all retry fail → 调 Phase 9.11 Extractor   │
   │      (4 个 dim 各跑 1 次 rule-based 补缺)         │
   │ 5. parse JSON → list[ReferenceNode]              │
   │    (含 confidence: int 1-5 字段, 跟 Q3 一致)       │
   │ 6. 喂 EdgeInferrer.infer_edges(chapter, nodes)    │
   │    filter nodes by confidence ≥ 3 (Q3)            │
   │ 7. retry 2 + cache 同样模式                      │
   │    失败 → 跳 edge, 仅写 nodes                    │
   │ 8. 累积 nodes + edges, 全部章节跑完统一 commit   │
   └─────────────────────────────────────────────────┘
         │
         ▼
   [if --dry-run]: 打印 N nodes / M edges / 成本 $X / 采样 3 例 → Done

```

### 4.4 Error Handling (失败场景 + 降级 + 边界)

#### 4.4.1 错误分类 + 处理策略

| # | 失败场景 | 检测点 | 1st 反应 | 重试 | 最终降级 |
|---|----------|--------|---------|------|---------|
| 1 | LLM API 网络超时 (Timeout) | `httpx.ReadTimeout` | 退避 1s | retry 1 → 2s | Phase 9.11 规则补缺 |
| 2 | LLM API 5xx (Server Error) | `httpx.HTTPStatusError` | 退避 1s | retry 1 → 2s | Phase 9.11 规则补缺 |
| 3 | LLM API 4xx (Client Error: 401/403/429) | `httpx.HTTPStatusError` | **不重试** (4xx 重试无意义) | 0 | Phase 9.11 规则 + log error |
| 4 | LLM 输出非 JSON | `JSONDecodeError` | log warn, 尝试 `parse_response(format_type="json_strict")` | 0 | 跳过该 dim, 其他 dim 继续 |
| 5 | LLM 输出 JSON schema 错 (字段缺失/类型错) | `pydantic.ValidationError` | log warn, 丢弃该 node | 0 | 跳过该 node, 其他 node 继续 |
| 6 | Chapter 内容空 / 超长 (>50K 字) | input validation | 跳过整个 chapter | 0 | log warn, continue |
| 7 | Cache file JSON 损坏 | `json.JSONDecodeError` on load | 删 cache, 重建 | 0 | 重跑全 LLM, 不破流程 |
| 8 | 4 维 prompt 全部失败 (极罕见) | retry 2 后仍 fail | 调 Phase 9.11 4 Extractor 全跑 1 遍 | 0 | 同 Phase 9.11 行为 |
| 9 | Edge 推断失败 | retry 2 后仍 fail | 跳过 edge, 仅写 nodes | 0 | log warn, nodes 仍写 |
| 10 | 写入 RippleStorage 失败 (DB locked/IO err) | `sqlite3.OperationalError` | `atomic_batch` 自动 rollback | 0 | log error, 整批回滚 |
| 11 | Cost tracker DB 写失败 | `sqlite3.OperationalError` | log warn, 不阻流程 | 0 | 静默继续, 不阻 backfill |
| 12 | `--apply` 但无 `--use-llm` | CLI validation | click 提示冲突 | 0 | exit 1, 友好错误信息 |

#### 4.4.3 降级路径 (Defensive Coding)

```python
# LLMScanner.scan_chapter 内部伪码
def scan_chapter(self, chapter_id, content, context):
    nodes: list[ReferenceNode] = []

    for dim in ["character", "foreshadow", "setting", "plot_point"]:
        try:
            dim_nodes = self._scan_one_dim(chapter_id, content, context, dim)
            nodes.extend(dim_nodes)
        except LLMRetryExhausted:
            # 降级到 Phase 9.11 Extractor 单 dim
            self.logger.warning(
                "LLM scan %s ch%d failed, fallback to rule",
                dim, chapter_id,
            )
            fallback_nodes = self.fallback_backfiller.extractors[dim].extract(
                chapter_id, content, context
            )
            nodes.extend(fallback_nodes)

    return nodes
```

**关键**: LLM path 失败不破坏 rule path (Q1 Opt-in 保 baseline, 双路径独立)
