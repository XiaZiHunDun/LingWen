## 问题概述

### 问题1: quality vs quality_tools 双目录（P1）

**现状**：
```
infra/quality/           - 新质量体系（35 py）
├── coordinator.py
├── interfaces.py
├── adapters.py
├── llm/scorers/        # 10个评分器
└── rule_based/validators/ # 6个验证器

infra/quality_tools/    - 旧质量工具（23 py）⚠️ 已标记DEPRECATED
├── soft_scorers/        # 与 quality/llm/scorers 重复
└── hard_validators/     # 与 quality/rule_based/validators 重复
```

**问题**：
- 导入路径混乱（新旧并存）
- 功能重复维护
- Deprecation警告持续出现

### 问题2: consistency vs quality 检测器未统一（P1）

**现状**：
```
infra/consistency/      - 28个checkers（timeline, character, pacing等）
└── checkers/

infra/quality/          - 2个checkers（worldview, ai_trace）
└── checkers/
```

**问题**：
- 两套检测器体系无法互通
- `QualityCoordinator` 无法协调 `consistency/` 的28个checker
- `RuleBasedAdapter` 需要适配两种格式

### 问题3: 工具入口散落三处（P2）

**现状**：
```
novel-factory/tools/    - 主要工具（verify_quality.py等）
infra/tools/            - 辅助工具（workflow/, content/, publish/）
infra/quality_tools/    - 质量工具
```

**问题**：
- 新开发者不知道去哪找工具
- 功能重复（batch_repair.py vs llm_quality_deep_check.py）

### 问题4: state/backends 未完成集成（P2）

**现状**：
- 新增了 `infra/state/backends/` 接口（SQLiteBackend, JSONBackend）
- `UpdateStateAction` 仍直接使用 sqlite3，未通过后端抽象

## 改进方案

### Phase A: 合并 quality_tools → quality/

#### A.1 更新导入路径

将 `infra/quality_tools/soft_scorers/` 的文件内容合并到 `infra/quality/llm/scorers/`，然后更新所有旧导入。

```python
# 旧导入（将移除）
from quality_tools.soft_scorers.tension import TensionScorer

# 新导入（统一）
from infra.quality.llm.scorers.tension import TensionScorer
```

#### A.2 更新所有引用 quality_tools 的文件

```bash
# 查找所有引用
grep -r "quality_tools" infra/ --include="*.py"
```

需要更新的文件类型：
- `infra/quality_tools/soft_scorers/*.py` - 迁移到 `infra/quality/llm/scorers/`
- `infra/quality_tools/hard_validators/*.py` - 迁移到 `infra/quality/rule_based/validators/`
- 其他引用 `quality_tools` 的文件 - 更新导入路径

#### A.3 删除旧目录

迁移完成后，删除 `infra/quality_tools/` 目录。

### Phase B: 统一检测器接口

#### B.1 定义通用 QualityChecker 接口

# infra/quality/interfaces.py
class QualityChecker(Protocol):
    """所有检测器实现的统一接口"""

        """检测维度（S1-S11）"""

    @property
    def name(self) -> str:
        """检测器名称"""

    def check(self, chapter_num: int, content: str) -> List[UnifiedIssue]:
        """检测章节，返回问题列表"""

### Phase D: 完成 state/backends 集成

#### D.1 修改 UpdateStateAction 使用 StateBackend

```python
# infra/hooks/actions/update_state.py
class UpdateStateAction(BaseAction):
    def __init__(self, backend: StateBackend = None):
        self._backend = backend or self._create_default_backend()

    def _create_default_backend(self) -> StateBackend:
        target = os.getenv("STATE_BACKEND", "sqlite://.state/workflow.db")
        if target.startswith("sqlite://"):
            return SQLiteBackend(target.replace("sqlite://", ""))
        return JSONBackend(target)
```

#### D.2 废弃直接 sqlite3 调用

将 `UpdateStateAction._update_sqlite()` 改为使用 `SQLiteBackend`。
