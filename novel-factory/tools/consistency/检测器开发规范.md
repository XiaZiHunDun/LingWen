# 检测器开发规范

> 版本: 1.0 | 更新: 2026-05-19
> 来源: 基于 CCGS 验证驱动开发原则优化

---

## 一、核心原则

```
验证驱动开发 = 先定义"通过标准"，再实现检测逻辑
```

**核心思想**: 在编写检测器之前，先明确这个检测器要发现什么问题、用什么标准判断问题存在或不存在。

---

## 二、开发流程

### 阶段1：定义验收标准

**必须回答的问题**：
1. 这个检测器要发现什么类型的问题？
2. 什么是"通过"（无问题）？
3. 什么是"失败"（有问题）？
4. 边界情况如何处理？

**输出**: 编写 `检测器名称_验收标准.md`

```markdown
# {检测器名称} 验收标准

## 问题类型
{描述检测器要发现的问题}

## 通过标准（无问题）
- 条件1
- 条件2
- ...

## 失败标准（有bug）
- 条件1
- 条件2
- ...

## 测试用例

### 正例（有问题的章节 - 必须能发现）
| 章节 | 问题描述 | 预期检测结果 |
|------|----------|--------------|
| chXXX | {问题描述} | P0/P1/P2 |

### 负例（正常的章节 - 不能误报）
| 章节 | 描述 | 预期检测结果 |
|------|------|--------------|
| chXXX | 正常章节 | 通过 |
```

### 阶段2：实现检测逻辑

**实现优先级**：
1. 规则型检测（正则、字符串匹配）— 快速、准确
2. 结构型检测（文件存在性、格式校验）— 基础
3. LLM辅助检测（语义判断）— 最后考虑

**代码规范**：
```python
# 良好的检测器结构
class DetectorName:
    def __init__(self):
        self.name = "{检测器名称}"
        self.version = "v1.0"

    def detect(self, chapter_content: str, context: dict) -> list:
        """
        检测章节内容
        返回: [{"type": "issue", "severity": "P0", "description": "...", "location": "..."}]
        """
        issues = []
        # 检测逻辑...
        return issues

    def get_name(self) -> str:
        return self.name

    def get_version(self) -> str:
        return self.version
```

### 阶段3：验证有效性

**必须执行的验证**：

1. **用已知问题测试**（正例）
   - 运行检测器
   - 必须能发现所有准备的问题
   - 如有遗漏，分析原因

2. **用正常章节测试**（负例）
   - 运行检测器
   - 不能误报
   - 如有误报，调整阈值

3. **记录测试结果**

```markdown
## 验证结果

### 正例测试
| 测试用例 | 问题描述 | 检测结果 | 状态 |
|----------|----------|----------|------|
| chXXX | 问题A | ✅ 发现 | 通过 |
| chYYY | 问题B | ❌ 未发现 | 失败 |

### 负例测试
| 测试用例 | 描述 | 检测结果 | 状态 |
|----------|------|----------|------|
| chZZZ | 正常章节 | ✅ 通过 | 通过 |
```

### 阶段4：建立回归测试集

**回归测试集结构**：
```
tests/
├── detectors/
│   ├── {detector_name}/
│   │   ├── test_positive_cases/
│   │   │   ├── chXXX.md （有问题）
│   │   │   └── expected.json （期望检测到的问题）
│   │   ├── test_negative_cases/
│   │   │   ├── chYYY.md （正常）
│   │   │   └── expected.json （期望无问题）
│   │   └── test_runner.py
```

**回归测试执行**：
```bash
# 每次代码变更后
python -m pytest tests/detectors/{detector_name}/

# 每次修复问题后
python -m pytest tests/detectors/{detector_name}/ -v
```

## 三、检测器类型

### 3.1 规则型检测器

**适用**: 明确的规则判断

```python
# 示例：章节重复检测
def detect_repetition(chapter_content: str) -> list:
    issues = []
    # 按段落分割
    paragraphs = chapter_content.split('\n\n')
    # 检测连续相似段落
    for i in range(len(paragraphs) - 1):
        similarity = calculate_similarity(paragraphs[i], paragraphs[i+1])
        if similarity > 0.8:
            issues.append({
                "type": "段落重复",
                "severity": "P1",
                "location": f"第{i+1}段"
            })
    return issues
```

### 3.2 结构型检测器

**适用**: 文件格式、存在性检查

```python
# 示例：章节完整性检测
def detect_completeness(chapter_file: str) -> list:
    issues = []
    with open(chapter_file, 'r') as f:
        content = f.read()

    if not content.strip().endswith('本章完'):
        issues.append({
            "type": "缺少本章完标记",
            "severity": "P1"
        })

    word_count = len(content)
    if word_count < 500:
        issues.append({
            "type": f"字数不足({word_count}字)",
            "severity": "P2"
        })

    return issues
```
